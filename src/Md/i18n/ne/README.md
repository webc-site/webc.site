# मार्कडाउन पाठलाई HTML मा सुरक्षित रूपमा पार्स र रेन्डर गर्नुहोस्।

- **सुरक्षा सेनिटाइजेशन**: नेटिभ `setHTML` API वा `DOMParser` क्लीनरहरू प्रयोग गरेर XSS आक्रमणहरू विरुद्ध रक्षा गर्नुहोस्
- **सामग्री निकासी**: प्रारम्भिक समयमा पूर्वनिर्धारित स्लटको पाठ सामग्री स्वचालित रूपमा पढ्नुहोस् र यसलाई रेन्डर गर्नुहोस्।
- **बिल्ट-इन टाइपसेटिङ**: पूर्व निर्धारित सूची, तालिका, कोड ब्लक, उद्धरण र चेतावनी बाकस शैलीहरू

## डेमो प्रयोग गर्नुहोस्

### स्थिर प्रतिपादन

```html
<c-md>
# शीर्षक
यो मार्कडाउन सामग्रीको एक टुक्रा हो
</c-md>
```

### गतिशील असाइनमेन्ट

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.value = "#नयाँ शीर्षक\nनयाँ सामग्री";
</script>
```

## इन्टरफेस प्यारामिटरहरू

मार्कडाउन पाठ सिधै ट्याग भित्र पास गर्नुहोस् (पूर्वनिर्धारित स्लटको रूपमा), वा `value` मार्फत गतिशील रूपमा अपडेट गर्नुहोस्।

## शैली विवरण

`.Md` शैली वर्ग स्वतः थपिएको छ जब कम्पोनेन्ट सुरु हुन्छ। निम्न मार्कडाउन विस्तारित चेतावनी बाकस लेआउट समर्थन गर्दछ (सन्दर्भ वाक्य रचना द्वारा ट्रिगर):

- `[!NOTE]`：प्रम्प्ट जानकारी
- `[!TIP]`: सुझाव/सुझावहरू
- `[!IMPORTANT]`: महत्त्वपूर्ण ध्यान
- `[!WARNING]`: जोखिम चेतावनी
- `[!CAUTION]`: अत्यधिक उच्च जोखिम चेतावनी

## सुरक्षा संयन्त्र

रेन्डरिङ गर्दा, HTML लाई सुरक्षित रूपमा इन्जेक्ट गर्न पहिले नेटिभ `setHTML` API लाई कल गर्नुहोस्। यदि ब्राउजरले यसलाई समर्थन गर्दैन भने, यसले स्वचालित रूपमा `DOMParser` को आधारमा पुनरावर्ती क्लिनरमा डाउनग्रेड गर्नेछ र निम्न सुरक्षा नीति लागू गर्नेछ:
- गैर-सुरक्षित ट्यागहरू हटाउनुहोस् (जस्तै `script`, `iframe`, आदि।)
- `on*` घटना विशेषताहरू खाली गर्नुहोस्
- खतरनाक प्रोटोकलहरू जस्तै `javascript:`, `vbscript:` र `data:` (`data:image/` बाहेक) फिल्टर गरेर XSS आक्रमणहरू विरुद्ध रक्षा गर्नुहोस्।
