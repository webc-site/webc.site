# Markdown 텍스트를 안전하게 구문 분석하고 HTML로 렌더링합니다.

- **보안 삭제**: 기본 `setHTML` API 또는 `DOMParser` 클리너를 사용하여 XSS 공격으로부터 방어합니다.
- **콘텐츠 추출**: 초기화 시 기본 슬롯의 텍스트 콘텐츠를 자동으로 읽고 렌더링합니다.
- **내장 조판**: 사전 설정 목록, 표, 코드 블록, 인용문 및 경고 상자 스타일

## 데모 사용

### 정적 렌더링

```html
<c-md>
# 제목
이것은 Markdown 콘텐츠의 일부입니다.
</c-md>
```

### 동적 할당

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.value = "#새 제목\n새 콘텐츠";
</script>
```

## 인터페이스 매개변수

태그 내에서 직접 Markdown 텍스트를 전달하거나(기본 슬롯으로) `value`을 통해 동적으로 업데이트하세요.

## 스타일 설명

`.Md` 스타일 클래스는 구성 요소가 초기화될 때 자동으로 추가됩니다. 다음 Markdown 확장 경고 상자 레이아웃을 지원합니다(참조 구문에 의해 트리거됨).

- `[!NOTE]`: 프롬프트 정보
- `[!TIP]`: 제안/팁
- `[!IMPORTANT]`: 중요한 주의 사항
- `[!WARNING]`: 위험 경고
- `[!CAUTION]`: 매우 높은 위험 경고

## 보안 메커니즘

렌더링할 때 HTML을 안전하게 삽입하려면 먼저 기본 `setHTML` API를 호출하세요. 브라우저가 이를 지원하지 않는 경우 `DOMParser`를 기반으로 하는 재귀 클리너로 자동으로 다운그레이드되고 다음 보안 정책을 구현합니다.
- 안전하지 않은 태그(예: `script`, `iframe` 등)를 제거하세요.
- `on*` 이벤트 속성 지우기
- `javascript:`, `vbscript:` 및 `data:`(`data:image/` 제외)와 같은 위험한 프로토콜로 시작하는 속성을 필터링하여 XSS 공격으로부터 방어합니다.
