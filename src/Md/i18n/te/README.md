# మార్క్‌డౌన్ వచనాన్ని HTMLకి సురక్షితంగా అన్వయించండి మరియు రెండర్ చేయండి.

- **సెక్యూరిటీ శానిటైజేషన్**: స్థానిక `setHTML` API లేదా `DOMParser` క్లీనర్‌లను ఉపయోగించి XSS దాడులకు వ్యతిరేకంగా రక్షించండి
- **కంటెంట్ ఎక్స్‌ట్రాక్షన్**: ప్రారంభించే సమయంలో డిఫాల్ట్ స్లాట్‌లోని టెక్స్ట్ కంటెంట్‌ని ఆటోమేటిక్‌గా రీడ్ చేయండి మరియు దానిని రెండర్ చేయండి
- **అంతర్నిర్మిత టైప్ సెట్టింగ్**: ప్రీసెట్ లిస్ట్, టేబుల్, కోడ్ బ్లాక్, కోట్ మరియు వార్నింగ్ బాక్స్ స్టైల్స్

## డెమో ఉపయోగించండి

### స్టాటిక్ రెండరింగ్

```html
<c-md>
# శీర్షిక
ఇది మార్క్‌డౌన్ కంటెంట్ యొక్క భాగం
</c-md>
```

### డైనమిక్ అసైన్‌మెంట్

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.textContent = "#కొత్త శీర్షిక\nకొత్త కంటెంట్";
</script>
```

## ఇంటర్ఫేస్ పారామితులు

మార్క్‌డౌన్ టెక్స్ట్‌లో నేరుగా ట్యాగ్‌లో (డిఫాల్ట్ స్లాట్‌గా) పాస్ చేయండి లేదా `textContent` ద్వారా డైనమిక్‌గా అప్‌డేట్ చేయండి.

## శైలి వివరణ

భాగం ప్రారంభించబడినప్పుడు `.Md` శైలి తరగతి స్వయంచాలకంగా జోడించబడుతుంది. కింది మార్క్‌డౌన్ పొడిగించిన హెచ్చరిక పెట్టె లేఅవుట్‌కు మద్దతు ఇస్తుంది (రిఫరెన్స్ సింటాక్స్ ద్వారా ప్రేరేపించబడింది):

- `[!NOTE]`:సత్వర సమాచారం
- `[!TIP]`: సూచనలు/చిట్కాలు
- `[!IMPORTANT]`: ముఖ్యమైన శ్రద్ధ
- `[!WARNING]`: ప్రమాద హెచ్చరిక
- `[!CAUTION]`: అత్యంత ప్రమాదకరమైన హెచ్చరిక

## భద్రతా యంత్రాంగం

రెండరింగ్ చేస్తున్నప్పుడు, HTMLని సురక్షితంగా ఇంజెక్ట్ చేయడానికి ముందుగా స్థానిక `setHTML` APIకి కాల్ చేయండి. బ్రౌజర్ దీనికి మద్దతు ఇవ్వకపోతే, అది స్వయంచాలకంగా `DOMParser` ఆధారంగా రికర్సివ్ క్లీనర్‌కి డౌన్‌గ్రేడ్ చేయబడుతుంది మరియు క్రింది భద్రతా విధానాన్ని అమలు చేస్తుంది:
- సురక్షితం కాని ట్యాగ్‌లను తీసివేయండి (ఉదా. `script`, `iframe`, మొదలైనవి)
- `on*` ఈవెంట్ లక్షణాలను క్లియర్ చేయండి
- `javascript:`, `vbscript:` మరియు `data:` (`data:image/` మినహా) వంటి ప్రమాదకరమైన ప్రోటోకాల్‌లతో ప్రారంభమయ్యే లక్షణాలను ఫిల్టర్ చేయడం ద్వారా XSS దాడులకు వ్యతిరేకంగా రక్షించండి.
