# متن Markdown را با خیال راحت تجزیه و رندر کنید.

- **Security Sanitization**: در برابر حملات XSS با استفاده از `setHTML` API یا `DOMParser` پاک کننده های بومی دفاع کنید
- ** استخراج محتوا**: به طور خودکار محتوای متن شکاف پیش فرض را در حین مقداردهی اولیه خوانده و آن را رندر کنید.
- ** حروفچینی داخلی**: فهرست از پیش تعیین شده، جدول، بلوک کد، نقل قول و سبک های جعبه هشدار

## از دمو استفاده کنید

### رندر استاتیک

```html
<c-md>
# عنوان
این بخشی از محتوای Markdown است
</c-md>
```

### تکلیف پویا

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.value = "#عنوان جدید\nمحتوای جدید";
</script>
```

## پارامترهای رابط

متن Markdown را مستقیماً در برچسب ارسال کنید (به عنوان شکاف پیش‌فرض)، یا به‌صورت پویا از طریق `value` به‌روزرسانی کنید.

## توضیحات سبک

کلاس سبک `.Md` به طور خودکار هنگامی که کامپوننت مقدار دهی اولیه می شود، اضافه می شود. از طرح بندی جعبه هشدار توسعه یافته Markdown زیر پشتیبانی می کند (که توسط نحو مرجع ایجاد می شود):

- `[!NOTE]`：اطلاعات سریع
- `[!TIP]`: پیشنهادات/نکات
- `[!IMPORTANT]`: توجه مهم
- `[!WARNING]`: هشدار خطر
- `[!CAUTION]`: هشدار خطر بسیار بالا

## مکانیزم امنیتی

هنگام رندر، ابتدا با API بومی `setHTML` تماس بگیرید تا با خیال راحت HTML را تزریق کنید. اگر مرورگر از آن پشتیبانی نکند، به طور خودکار به یک پاک کننده بازگشتی بر اساس `DOMParser` تنزل داده و خط مشی امنیتی زیر را اجرا می کند:
- برچسب‌های غیر ایمن را حذف کنید (مانند `script`، `iframe` و غیره)
- صفات رویداد `on*` را پاک کنید
- با فیلتر کردن ویژگی‌هایی که با پروتکل‌های خطرناکی مانند `javascript:`، `vbscript:` و `data:` شروع می‌شوند، در برابر حملات XSS دفاع کنید (به‌جز `data:image/`).
