# Бяспечны аналіз і візуалізацыя тэксту Markdown у HTML.

- **Ачыстка бяспекі**: абарона ад нападаў XSS з выкарыстаннем уласных `setHTML` API або `DOMParser` ачышчальнікаў
- **Выманне змесціва**: аўтаматычнае чытанне тэкставага змесціва слота па змаўчанні падчас ініцыялізацыі і візуалізацыя
- **Убудаваная вёрстка**: прадусталяваны спіс, табліца, блок кода, стылі цытат і папярэджанняў

## Выкарыстоўвайце дэма

### статычны рэндэрынг

```html
<c-md>
# назва
Гэта частка кантэнту Markdown
</c-md>
```

### дынамічнае прызначэнне

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.value = "#Новы загаловак\nНовы змест";
</script>
```

## Параметры інтэрфейсу

Перадайце тэкст Markdown непасрэдна ў тэг (у якасці слота па змаўчанні) або дынамічна абнаўляйце праз `value`.

## Апісанне стылю

Клас стылю `.Md` аўтаматычна дадаецца пры ініцыялізацыі кампанента. Падтрымлівае наступны макет пашыранага поля папярэджання Markdown (выклікаецца сінтаксісам спасылкі):

- `[!NOTE]`：Аператыўная інфармацыя
- `[!TIP]`: Прапановы/парады
- `[!IMPORTANT]`: важная ўвага
- `[!WARNING]`: Папярэджанне аб рызыцы
- `[!CAUTION]`: Папярэджанне аб вельмі высокай рызыцы

## механізм бяспекі

Пры візуалізацыі спачатку выклічце ўласны `setHTML` API, каб бяспечна ўвесці HTML. Калі браўзер яго не падтрымлівае, ён аўтаматычна вернецца да рэкурсіўнага ачышчальніка на аснове `DOMParser` і прыме наступную палітыку бяспекі:
- Выдаліце ​​небяспечныя тэгі (напрыклад, `script`, `iframe` і г.д.)
- Ачысціць `on*` атрыбуты падзеі
- Абараніцеся ад нападаў XSS шляхам фільтрацыі атрыбутаў, якія пачынаюцца з небяспечных пратаколаў, такіх як `javascript:`, `vbscript:` і `data:` (акрамя `data:image/`).
