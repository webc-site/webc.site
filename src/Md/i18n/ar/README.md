# تحليل نص Markdown وتقديمه بأمان إلى HTML.

- **التطهير الأمني**: الدفاع ضد هجمات XSS باستخدام منظفات `setHTML` الأصلية أو `DOMParser`
- **استخراج المحتوى**: قراءة محتوى النص في الفتحة الافتراضية تلقائيًا أثناء التهيئة وعرضه
- ** التنضيد المدمج **: قائمة محددة مسبقًا، والجدول، وكتلة التعليمات البرمجية، والاقتباسات، وأنماط مربع التحذير

## استخدم العرض التوضيحي

### تقديم ثابت

```html
<c-md>
# عنوان
هذا جزء من محتوى Markdown
</c-md>
```

### مهمة ديناميكية

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.value = "#عنوان جديد\nمحتوى جديد";
</script>
```

## معلمات الواجهة

قم بتمرير نص Markdown مباشرة داخل العلامة (باعتبارها الفتحة الافتراضية)، أو قم بالتحديث ديناميكيًا عبر `value`.

## وصف النمط

تتم إضافة فئة النمط `.Md` تلقائيًا عند تهيئة المكون. يدعم تخطيط مربع التحذير الممتد Markdown التالي (يتم تشغيله بواسطة بناء الجملة المرجعي):

- `[!NOTE]`: معلومات سريعة
- `[!TIP]`: اقتراحات/نصائح
- `[!IMPORTANT]`: اهتمام مهم
- `[!WARNING]`: تحذير من المخاطر
- `[!CAUTION]`: تحذير من مخاطر عالية للغاية

## آلية أمنية

عند العرض، اتصل بواجهة برمجة التطبيقات `setHTML` الأصلية أولاً لإدخال HTML بأمان. إذا كان المتصفح لا يدعمه، فسيتم الرجوع تلقائيًا إلى منظف متكرر استنادًا إلى `DOMParser` وتنفيذ سياسة الأمان التالية:
- إزالة العلامات غير الآمنة (على سبيل المثال، `script`، `iframe`، وما إلى ذلك)
- امسح `on*` سمات الحدث
- يمكنك الدفاع ضد هجمات XSS عن طريق تصفية السمات التي تبدأ بالبروتوكولات الخطيرة مثل `javascript:` و`vbscript:` و`data:` (باستثناء `data:image/`).
