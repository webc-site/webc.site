# Markdown テキストを安全に解析して HTML にレンダリングします。

- **セキュリティ サニタイズ**: ネイティブの `setHTML` API または `DOMParser` クリーナーを使用して XSS 攻撃を防御します。
- **コンテンツ抽出**: 初期化中にデフォルト スロットのテキスト コンテンツを自動的に読み取り、レンダリングします。
- **組み込みタイプセット**: プリセット リスト、テーブル、コード ブロック、引用および警告ボックスのスタイル

## デモを使用する

### 静的レンダリング

```html
<c-md>
# タイトル
これは Markdown コンテンツの一部です
</c-md>
```

### 動的割り当て

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.value = "#新しいタイトル\n新しいコンテンツ";
</script>
```

## インターフェースパラメータ

Markdown テキストをタグ内で直接渡すか (デフォルトのスロットとして)、`value` を介して動的に更新します。

## スタイルの説明

`.Md` スタ​​イル クラスは、コンポーネントが初期化されるときに自動的に追加されます。次の Markdown 拡張警告ボックス レイアウトをサポートします (参照構文によってトリガーされます)。

- `[!NOTE]`：プロンプト情報
- `[!TIP]`: 提案/ヒント
- `[!IMPORTANT]`: 重要な注意事項
- `[!WARNING]`: リスク警告
- `[!CAUTION]`: 非常に高いリスクの警告

## セキュリティメカニズム

レンダリングするときは、まずネイティブ `setHTML` API を呼び出して、HTML を安全に挿入します。ブラウザがこれをサポートしていない場合は、`DOMParser`に基づいて再帰的クリーナーに自動的にダウングレードし、次のセキュリティ ポリシーを実装します。
- 安全でないタグ (例: `script`、`iframe` など) を削除します。
- `on*` イベント属性をクリアします
- `javascript:`、`vbscript:`、`data:` (`data:image/` を除く) などの危険なプロトコルで始まる属性をフィルタリングすることで、XSS 攻撃を防御します。
