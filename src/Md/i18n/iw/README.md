# ניתוח ועיבוד בטוח של טקסט Markdown ל-HTML.

- **חיטוי אבטחה**: הגן מפני התקפות XSS באמצעות `setHTML` API או `DOMParser` חומרי ניקוי מקוריים
- **חילוץ תוכן**: קרא באופן אוטומטי את תוכן הטקסט של משבצת ברירת המחדל במהלך האתחול ועיבודו
- **הגדרת כתיבה מובנית**: רשימה מוגדרת מראש, טבלה, בלוק קוד, סגנונות ציטוט ותיבות אזהרה

## השתמש בהדגמה

### עיבוד סטטי

```html
<c-md>
# כּוֹתֶרֶת
זהו חלק מתוכן Markdown
</c-md>
```

### משימה דינמית

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.textContent = "#כותרת חדשה\nתוכן חדש";
</script>
```

## פרמטרים של ממשק

העבר טקסט Markdown ישירות בתוך התג (כמשבצת ברירת המחדל), או עדכן באופן דינמי באמצעות `textContent`.

## תיאור סגנון

מחלקת הסגנון `.Md` מתווספת אוטומטית עם אתחול הרכיב. תומך בפריסת תיבת האזהרה המורחבת של Markdown הבאה (מופעל על ידי תחביר התייחסות):

- `[!NOTE]`：מידע הנחיה
- `[!TIP]`: הצעות/טיפים
- `[!IMPORTANT]`: תשומת לב חשובה
- `[!WARNING]`: אזהרת סיכון
- `[!CAUTION]`: אזהרת סיכון גבוה במיוחד

## מנגנון אבטחה

בעת עיבוד, קרא תחילה ל-API המקורי `setHTML` כדי להחדיר בבטחה HTML. אם הדפדפן אינו תומך בו, הוא ישדרג אוטומטית לאחור למנקה רקורסיבי בהתבסס על `DOMParser` ויישם את מדיניות האבטחה הבאה:
- הסר תגים לא בטוחים (למשל `script`, `iframe` וכו')
- נקה `on*` מאפייני אירוע
- הגן מפני התקפות XSS על ידי סינון תכונות המתחילות בפרוטוקולים מסוכנים כגון `javascript:`, `vbscript:` ו-`data:` (למעט `data:image/`).
