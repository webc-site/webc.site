# ಮಾರ್ಕ್‌ಡೌನ್ ಪಠ್ಯವನ್ನು HTML ಗೆ ಸುರಕ್ಷಿತವಾಗಿ ಪಾರ್ಸ್ ಮಾಡಿ ಮತ್ತು ರೆಂಡರ್ ಮಾಡಿ.

- **ಸೆಕ್ಯುರಿಟಿ ಸ್ಯಾನಿಟೈಸೇಶನ್**: ಸ್ಥಳೀಯ `setHTML` API ಅಥವಾ `DOMParser` ಕ್ಲೀನರ್‌ಗಳನ್ನು ಬಳಸಿಕೊಂಡು XSS ದಾಳಿಗಳ ವಿರುದ್ಧ ರಕ್ಷಿಸಿ
- **ವಿಷಯ ಹೊರತೆಗೆಯುವಿಕೆ**: ಪ್ರಾರಂಭದ ಸಮಯದಲ್ಲಿ ಡೀಫಾಲ್ಟ್ ಸ್ಲಾಟ್‌ನ ಪಠ್ಯ ವಿಷಯವನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಓದಿ ಮತ್ತು ಅದನ್ನು ನಿರೂಪಿಸಿ
- **ಅಂತರ್ನಿರ್ಮಿತ ಟೈಪ್‌ಸೆಟ್ಟಿಂಗ್**: ಮೊದಲೇ ಪಟ್ಟಿ, ಟೇಬಲ್, ಕೋಡ್ ಬ್ಲಾಕ್, ಉಲ್ಲೇಖ ಮತ್ತು ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್ ಶೈಲಿಗಳು

## ಡೆಮೊ ಬಳಸಿ

### ಸ್ಥಿರ ರೆಂಡರಿಂಗ್

```html
<c-md>
# ಶೀರ್ಷಿಕೆ
ಇದು ಮಾರ್ಕ್‌ಡೌನ್ ವಿಷಯದ ಒಂದು ಭಾಗವಾಗಿದೆ
</c-md>
```

### ಕ್ರಿಯಾತ್ಮಕ ನಿಯೋಜನೆ

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.textContent = "#ಹೊಸ ಶೀರ್ಷಿಕೆ\nಹೊಸ ವಿಷಯ";
</script>
```

## ಇಂಟರ್ಫೇಸ್ ನಿಯತಾಂಕಗಳು

ಮಾರ್ಕ್‌ಡೌನ್ ಪಠ್ಯವನ್ನು ನೇರವಾಗಿ ಟ್ಯಾಗ್‌ನಲ್ಲಿ (ಡೀಫಾಲ್ಟ್ ಸ್ಲಾಟ್ ಆಗಿ) ಪಾಸ್ ಮಾಡಿ ಅಥವಾ `textContent` ಮೂಲಕ ಕ್ರಿಯಾತ್ಮಕವಾಗಿ ನವೀಕರಿಸಿ.

## ಶೈಲಿಯ ವಿವರಣೆ

ಘಟಕವನ್ನು ಪ್ರಾರಂಭಿಸಿದಾಗ `.Md` ಶೈಲಿಯ ವರ್ಗವನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸೇರಿಸಲಾಗುತ್ತದೆ. ಕೆಳಗಿನ ಮಾರ್ಕ್‌ಡೌನ್ ವಿಸ್ತೃತ ಎಚ್ಚರಿಕೆ ಬಾಕ್ಸ್ ವಿನ್ಯಾಸವನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ (ಉಲ್ಲೇಖ ಸಿಂಟ್ಯಾಕ್ಸ್‌ನಿಂದ ಪ್ರಚೋದಿಸಲ್ಪಟ್ಟಿದೆ):

- `[!NOTE]`:ಪ್ರಾಂಪ್ಟ್ ಮಾಹಿತಿ
- `[!TIP]`: ಸಲಹೆಗಳು/ಸಲಹೆಗಳು
- `[!IMPORTANT]`: ಪ್ರಮುಖ ಗಮನ
- `[!WARNING]`: ಅಪಾಯದ ಎಚ್ಚರಿಕೆ
- `[!CAUTION]`: ಅತ್ಯಂತ ಹೆಚ್ಚಿನ ಅಪಾಯದ ಎಚ್ಚರಿಕೆ

## ಭದ್ರತಾ ಕಾರ್ಯವಿಧಾನ

ರೆಂಡರಿಂಗ್ ಮಾಡುವಾಗ, HTML ಅನ್ನು ಸುರಕ್ಷಿತವಾಗಿ ಇಂಜೆಕ್ಟ್ ಮಾಡಲು ಮೊದಲು ಸ್ಥಳೀಯ `setHTML` API ಗೆ ಕರೆ ಮಾಡಿ. ಬ್ರೌಸರ್ ಅದನ್ನು ಬೆಂಬಲಿಸದಿದ್ದರೆ, ಅದು ಸ್ವಯಂಚಾಲಿತವಾಗಿ `DOMParser` ಅನ್ನು ಆಧರಿಸಿ ಪುನರಾವರ್ತಿತ ಕ್ಲೀನರ್‌ಗೆ ಡೌನ್‌ಗ್ರೇಡ್ ಮಾಡುತ್ತದೆ ಮತ್ತು ಕೆಳಗಿನ ಭದ್ರತಾ ನೀತಿಯನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸುತ್ತದೆ:
- ಸುರಕ್ಷಿತವಲ್ಲದ ಟ್ಯಾಗ್‌ಗಳನ್ನು ತೆಗೆದುಹಾಕಿ (ಉದಾ. `script`, `iframe`, ಇತ್ಯಾದಿ.)
- `on*` ಈವೆಂಟ್ ಗುಣಲಕ್ಷಣಗಳನ್ನು ತೆರವುಗೊಳಿಸಿ
- `javascript:`, `vbscript:`, ಮತ್ತು `data:` (`data:image/` ಹೊರತುಪಡಿಸಿ) ನಂತಹ ಅಪಾಯಕಾರಿ ಪ್ರೋಟೋಕಾಲ್‌ಗಳೊಂದಿಗೆ ಪ್ರಾರಂಭವಾಗುವ ಗುಣಲಕ್ಷಣಗಳನ್ನು ಫಿಲ್ಟರ್ ಮಾಡುವ ಮೂಲಕ XSS ದಾಳಿಗಳ ವಿರುದ್ಧ ರಕ್ಷಿಸಿ.
