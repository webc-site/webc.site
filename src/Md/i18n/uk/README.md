# Безпечний аналіз і рендеринг тексту Markdown у HTML.

- **Дезінфікація безпеки**: захист від XSS-атак за допомогою рідного `setHTML` API або `DOMParser` засобів очищення
- **Вилучення вмісту**: автоматично зчитувати текстовий вміст слота за замовчуванням під час ініціалізації та відтворювати його
- **Вбудована верстка**: попередньо встановлений список, таблиця, блок коду, цитати та стилі поля попередження

## Використовуйте демо

### статичний рендеринг

```html
<c-md>
# назва
Це частина вмісту Markdown
</c-md>
```

### динамічне призначення

```html
<c-md id="md"></c-md>

<script type="module">
import "webc.site/Md.js";
document.querySelector("#md").value = "#Новий заголовок\nНовий вміст";
</script>
```

## Параметри інтерфейсу

Передайте текст Markdown безпосередньо в тег (як слот за замовчуванням) або динамічно оновлюйте через `value`.

## Опис стилю

Клас стилю `.Md` автоматично додається під час ініціалізації компонента. Підтримує такий макет розширеного вікна попередження Markdown (ініціюється синтаксисом посилання):

- `[!NOTE]`：Швидка інформація
- `[!TIP]`: пропозиції/підказки
- `[!IMPORTANT]`: Важлива увага
- `[!WARNING]`: попередження про ризик
- `[!CAUTION]`: Попередження про надзвичайно високий ризик

## механізм безпеки

Під час візуалізації спочатку викликайте власний `setHTML` API, щоб безпечно вставити HTML. Якщо веб-переглядач не підтримує його, він автоматично повернеться до рекурсивного очищувача на основі `DOMParser` і запровадить таку політику безпеки:
- Видаліть небезпечні теги (наприклад, `script`, `iframe` тощо)
- Очистити атрибути події `on*`
- Захист від атак XSS шляхом фільтрації атрибутів, починаючи з небезпечних протоколів, таких як `javascript:`, `vbscript:` і `data:` (окрім `data:image/`).
