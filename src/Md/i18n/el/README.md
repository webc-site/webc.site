# Αναλύστε με ασφάλεια και αποδώστε το κείμενο Markdown σε HTML.

- **Ασφάλεια Εξυγίανση**: Προστατευτείτε από επιθέσεις XSS χρησιμοποιώντας εγγενή `setHTML` API ή `DOMParser` καθαριστικά
- **Εξαγωγή περιεχομένου**: Αυτόματη ανάγνωση του περιεχομένου κειμένου της προεπιλεγμένης υποδοχής κατά την προετοιμασία και απόδοση του
- **Ενσωματωμένη στοιχειοθεσία**: προκαθορισμένη λίστα, πίνακας, μπλοκ κωδικών, εισαγωγικά και στυλ πλαισίου προειδοποίησης

## Χρησιμοποιήστε το demo

### στατική απόδοση

```html
<c-md>
# τίτλος
Αυτό είναι ένα κομμάτι περιεχομένου Markdown
</c-md>
```

### δυναμική ανάθεση

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.textContent = "#Νέος τίτλος\nΝέο περιεχόμενο";
</script>
```

## Παράμετροι διεπαφής

Περάστε κείμενο Markdown απευθείας μέσα στην ετικέτα (ως προεπιλεγμένη υποδοχή) ή ενημερώστε δυναμικά μέσω `textContent`.

## Περιγραφή στυλ

Η κλάση στυλ `.Md` προστίθεται αυτόματα όταν αρχικοποιείται το στοιχείο. Υποστηρίζει την ακόλουθη διάταξη εκτεταμένου πλαισίου προειδοποίησης Markdown (ενεργοποιείται από τη σύνταξη αναφοράς):

- `[!NOTE]`：Πληροφορίες προτροπής
- `[!TIP]`: Προτάσεις/Συμβουλές
- `[!IMPORTANT]`: Σημαντική προσοχή
- `[!WARNING]`: Προειδοποίηση κινδύνου
- `[!CAUTION]`: Προειδοποίηση εξαιρετικά υψηλού κινδύνου

## μηχανισμός ασφαλείας

Κατά την απόδοση, καλέστε πρώτα το εγγενές `setHTML` API για ασφαλή εισαγωγή HTML. Εάν το πρόγραμμα περιήγησης δεν το υποστηρίζει, θα υποβαθμιστεί αυτόματα σε ένα αναδρομικό καθαριστικό με βάση το `DOMParser` και θα εφαρμόσει την ακόλουθη πολιτική ασφαλείας:
- Κατάργηση μη ασφαλών ετικετών (π.χ. `script`, `iframe` κ.λπ.)
- Διαγραφή `on*` χαρακτηριστικών συμβάντος
- Προστατευτείτε από επιθέσεις XSS φιλτράροντας χαρακτηριστικά που ξεκινούν με επικίνδυνα πρωτόκολλα όπως `javascript:`, `vbscript:` και `data:` (εκτός από το `data:image/`).
