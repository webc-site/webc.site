# Ապահով վերլուծեք և Markdown տեքստը HTML-ի վերածեք:

- **Անվտանգության սանիտարականացում**. Պաշտպանեք XSS հարձակումներից՝ օգտագործելով բնիկ `setHTML` API կամ `DOMParser` մաքրող սարքեր
- **Բովանդակության արդյունահանում**. սկզբնավորման ընթացքում ինքնաբերաբար կարդալ լռելյայն անցքի տեքստային բովանդակությունը և արտապատկերել այն
- ** Ներկառուցված տպագրություն**. նախադրված ցուցակ, աղյուսակ, կոդի բլոկ, մեջբերում և նախազգուշացման տուփի ոճեր

## Օգտագործեք ցուցադրությունը

### ստատիկ մատուցում

```html
<c-md>
# վերնագիր
Սա Markdown-ի բովանդակության մի մասն է
</c-md>
```

### դինամիկ հանձնարարություն

```html
<c-md id="md"></c-md>

<script type="module">
import "webc.site/Md.js";
document.querySelector("#md").value = "#Նոր վերնագիր\nՆոր բովանդակություն";
</script>
```

## Ոճի նկարագրություն

`.Md` ոճի դասը ավտոմատ կերպով ավելացվում է բաղադրիչի սկզբնավորման ժամանակ: Աջակցում է հետևյալ Markdown-ի ընդլայնված նախազգուշացման վանդակի դասավորությունը (գործարկվում է հղման շարահյուսությամբ).

- `[!NOTE]`：Հրաժեշտ տեղեկատվություն
- `[!TIP]`: Առաջարկություններ/Խորհուրդներ
- `[!IMPORTANT]`: Կարևոր ուշադրություն
- `[!WARNING]`: Ռիսկի նախազգուշացում
- `[!CAUTION]`. չափազանց բարձր ռիսկի նախազգուշացում

## անվտանգության մեխանիզմ

Արտադրելիս նախ զանգահարեք բնիկ `setHTML` API-ին, որպեսզի ապահով կերպով ներարկեք HTML: Եթե ​​զննարկիչը չի աջակցում այն, այն ավտոմատ կերպով կնվազեցվի `DOMParser`-ի վրա հիմնված ռեկուրսիվ մաքրիչի և կիրականացնի հետևյալ անվտանգության քաղաքականությունը.
- Հեռացրեք ոչ անվտանգ պիտակները (օրինակ՝ `script`, `iframe` և այլն)
- Մաքրել `on*` իրադարձության հատկանիշները
- Պաշտպանեք XSS հարձակումներից՝ զտելով այնպիսի ատրիբուտներ, որոնք սկսվում են վտանգավոր արձանագրություններից, ինչպիսիք են `javascript:`, `vbscript:` և `data:` (բացառությամբ `data:image/`-ի):
