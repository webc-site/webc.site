# แยกวิเคราะห์และแสดงข้อความ Markdown เป็น HTML ได้อย่างปลอดภัย

- **การฆ่าเชื้อด้านความปลอดภัย**: ป้องกันการโจมตี XSS โดยใช้ตัวล้าง `setHTML` API หรือ `DOMParser` ดั้งเดิม
- **การแยกเนื้อหา**: อ่านเนื้อหาข้อความของช่องเริ่มต้นโดยอัตโนมัติระหว่างการเริ่มต้นและแสดงผล
- **การเรียงพิมพ์ในตัว**: รายการที่กำหนดไว้ล่วงหน้า ตาราง บล็อกโค้ด เครื่องหมายคำพูด และรูปแบบกล่องคำเตือน

## ใช้การสาธิต

### การแสดงผลแบบคงที่

```html
<c-md>
# ชื่อ
นี่คือเนื้อหาส่วนหนึ่งของ Markdown
</c-md>
```

### การมอบหมายแบบไดนามิก

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.textContent = "#ชื่อใหม่\nเนื้อหาใหม่";
</script>
```

## พารามิเตอร์อินเทอร์เฟซ

ส่งผ่านข้อความ Markdown ภายในแท็กโดยตรง (เป็นช่องเริ่มต้น) หรืออัปเดตแบบไดนามิกผ่าน `textContent`

## คำอธิบายสไตล์

`.Md` คลาสสไตล์จะถูกเพิ่มโดยอัตโนมัติเมื่อมีการเตรียมใช้งานส่วนประกอบ รองรับรูปแบบกล่องคำเตือนแบบขยาย Markdown ต่อไปนี้ (กระตุ้นโดยไวยากรณ์อ้างอิง):

- `[!NOTE]`：ข้อมูลพร้อมท์
- `[!TIP]`: คำแนะนำ/เคล็ดลับ
- `[!IMPORTANT]`: ความสนใจที่สำคัญ
- `[!WARNING]`: คำเตือนความเสี่ยง
- `[!CAUTION]`: คำเตือนความเสี่ยงสูงมาก

## กลไกการรักษาความปลอดภัย

เมื่อเรนเดอร์ ให้เรียก `setHTML` API ดั้งเดิมก่อนเพื่อแทรก HTML อย่างปลอดภัย หากเบราว์เซอร์ไม่รองรับ มันจะปรับลดรุ่นเป็นตัวล้างแบบเรียกซ้ำโดยอัตโนมัติตาม `DOMParser` และใช้นโยบายความปลอดภัยต่อไปนี้:
- ลบแท็กที่ไม่ปลอดภัย (เช่น `script`, `iframe` ฯลฯ)
- ล้างแอตทริบิวต์เหตุการณ์ `on*`
- ป้องกันการโจมตี XSS โดยการกรองคุณลักษณะที่เริ่มต้นด้วยโปรโตคอลที่เป็นอันตราย เช่น `javascript:`, `vbscript:` และ `data:` (ยกเว้น `data:image/`)
