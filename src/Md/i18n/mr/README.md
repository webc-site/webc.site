# मार्कडाउन मजकूर HTML वर सुरक्षितपणे पार्स करा आणि रेंडर करा.

- **सुरक्षा स्वच्छता**: नेटिव्ह `setHTML` API किंवा `DOMParser` क्लीनर वापरून XSS हल्ल्यांपासून बचाव करा
- **कंटेंट एक्सट्रॅक्शन**: इनिशिएलायझेशन दरम्यान डीफॉल्ट स्लॉटची मजकूर सामग्री स्वयंचलितपणे वाचा आणि ती रेंडर करा
- **बिल्ट-इन टाइपसेटिंग**: प्रीसेट सूची, टेबल, कोड ब्लॉक, कोट आणि चेतावणी बॉक्स शैली

## डेमो वापरा

### स्थिर प्रस्तुतीकरण

```html
<c-md>
# शीर्षक
हा मार्कडाउन सामग्रीचा एक भाग आहे
</c-md>
```

### डायनॅमिक असाइनमेंट

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.textContent = "#नवीन शीर्षक\nनवीन सामग्री";
</script>
```

## इंटरफेस पॅरामीटर्स

मार्कडाउन मजकूर थेट टॅगमध्ये पास करा (डीफॉल्ट स्लॉट म्हणून), किंवा `textContent` द्वारे डायनॅमिकरित्या अपडेट करा.

## शैलीचे वर्णन

घटक सुरू केल्यावर `.Md` शैली वर्ग आपोआप जोडला जातो. खालील मार्कडाउन विस्तारित चेतावणी बॉक्स लेआउटचे समर्थन करते (संदर्भ वाक्यरचनाद्वारे ट्रिगर):

- `[!NOTE]`: त्वरित माहिती
- `[!TIP]`: सूचना/टिपा
- `[!IMPORTANT]`: महत्त्वाचे लक्ष
- `[!WARNING]`: धोक्याची चेतावणी
- `[!CAUTION]`: अत्यंत उच्च जोखमीची चेतावणी

## सुरक्षा यंत्रणा

प्रस्तुत करताना, HTML सुरक्षितपणे इंजेक्ट करण्यासाठी प्रथम नेटिव्ह `setHTML` API ला कॉल करा. जर ब्राउझर त्यास समर्थन देत नसेल, तर ते `DOMParser` वर आधारित रिकर्सिव क्लीनरवर आपोआप डाउनग्रेड करेल आणि खालील सुरक्षा धोरण लागू करेल:
- गैर-सुरक्षित टॅग काढा (उदा. `script`, `iframe` इ.)
- `on*` इव्हेंट विशेषता साफ करा
- `javascript:`, `vbscript:` आणि `data:` (`data:image/` वगळता) सारख्या धोकादायक प्रोटोकॉलपासून सुरू होणाऱ्या विशेषता फिल्टर करून XSS हल्ल्यांपासून बचाव करा.
