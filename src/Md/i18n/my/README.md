# လုံခြုံစွာခွဲခြမ်းစိပ်ဖြာပြီး Markdown စာသားကို HTML သို့ ပြန်ပေးပါ။

- **Security Sanitization**- မူရင်း `setHTML` API သို့မဟုတ် `DOMParser` သန့်ရှင်းရေးများကို အသုံးပြု၍ XSS တိုက်ခိုက်မှုများကို ခုခံကာကွယ်ပါ
- **အကြောင်းအရာ ထုတ်ယူခြင်း**- အစပြုစဉ်အတွင်း မူရင်းအထိုင်၏ စာသားအကြောင်းအရာကို အလိုအလျောက်ဖတ်ပြီး ၎င်းကို တင်ဆက်ခြင်း
- ** Built-in အမျိုးအစားသတ်မှတ်ခြင်း**- ကြိုတင်သတ်မှတ်စာရင်း၊ ဇယား၊ ကုဒ်ပိတ်ဆို့ခြင်း၊ ကိုးကားချက်နှင့် သတိပေးသေတ္တာပုံစံများ

## ဒီမိုကိုသုံးပါ။

### static rendering

```html
<c-md>
# ခေါင်းစဉ်
ဤသည်မှာ Markdown အကြောင်းအရာအပိုင်းအစဖြစ်သည်။
</c-md>
```

### ရွေ့လျားမှုတာဝန်

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.textContent = "#ခေါင်းစဉ်အသစ်\nအကြောင်းအရာအသစ်";
</script>
```

## မျက်နှာပြင်ဘောင်များ

Markdown စာသားကို တဂ်အတွင်း တိုက်ရိုက်ဖြတ်သန်းပါ (မူလအထိုင်အဖြစ်) သို့မဟုတ် `textContent`မှတစ်ဆင့် ဒိုင်းနမစ်ဖြင့် အပ်ဒိတ်လုပ်ပါ။

## ပုံစံဖော်ပြချက်

အစိတ်အပိုင်းကို စတင်လုပ်ဆောင်သည့်အခါ `.Md`စတိုင်အတန်းသည် အလိုအလျောက် ပေါင်းထည့်သည်။ အောက်ဖော်ပြပါ Markdown တိုးချဲ့သတိပေးချက်သေတ္တာအပြင်အဆင်ကို ပံ့ပိုးပေးသည် (ကိုးကားချက်အထားအသိုမှ အစပျိုးသည်-

- `[!NOTE]`: အချက်ပေးအချက်အလက်
- `[!TIP]`- အကြံပြုချက်များ/အကြံပြုချက်များ
- `[!IMPORTANT]`- အရေးကြီးသောအာရုံ
- `[!WARNING]`- အန္တရာယ် သတိပေးချက်
- `[!CAUTION]`- အလွန်အန္တရာယ်များသော သတိပေးချက်

## လုံခြုံရေးယန္တရား

တင်ဆက်သည့်အခါ HTML ကို အန္တရာယ်ကင်းစွာ ထည့်သွင်းရန် ဇာတိ `setHTML` API ကို ဦးစွာခေါ်ဆိုပါ။ ဘရောင်ဇာက ၎င်းကို မပံ့ပိုးပါက၊ ၎င်းသည် `DOMParser`အပေါ် အခြေခံ၍ ထပ်တလဲလဲ သန့်စင်သည့် စက်သို့ အလိုအလျောက် အဆင့်နှိမ့်ပေးပြီး အောက်ပါ လုံခြုံရေးမူဝါဒကို အကောင်အထည်ဖော်ပါ-
- မလုံခြုံသော တဂ်များကို ဖယ်ရှားပါ (ဥပမာ `script`၊ `iframe`၊ စသည်ဖြင့်)
- `on*`ဖြစ်ရပ် ရည်ညွှန်းချက်များကို ရှင်းလင်းပါ။
- `javascript:`၊ `vbscript:`၊ နှင့် `data:` (`data:image/` မှလွဲ၍) ကဲ့သို့သော အန္တရာယ်ရှိသော ပရိုတိုကောများဖြင့် စတင်သည့် အရည်အချင်းများကို စစ်ထုတ်ခြင်းဖြင့် XSS တိုက်ခိုက်မှုများကို ခုခံကာကွယ်ပါ။
