# மார்க் டவுன் உரையை HTMLக்கு பாதுகாப்பாக அலசவும்.

- **பாதுகாப்பு சுத்திகரிப்பு**: சொந்த `setHTML` API அல்லது `DOMParser` கிளீனர்களைப் பயன்படுத்தி XSS தாக்குதல்களுக்கு எதிராக பாதுகாக்கவும்
- **உள்ளடக்க பிரித்தெடுத்தல்**: துவக்கத்தின் போது இயல்புநிலை ஸ்லாட்டின் உரை உள்ளடக்கத்தை தானாகப் படித்து அதை ரெண்டர் செய்யவும்
- **உள்ளமைக்கப்பட்ட தட்டச்சு அமைப்பு**: முன்னமைக்கப்பட்ட பட்டியல், அட்டவணை, குறியீடு தொகுதி, மேற்கோள் மற்றும் எச்சரிக்கை பெட்டி பாணிகள்

## டெமோவைப் பயன்படுத்தவும்

### நிலையான ரெண்டரிங்

```html
<c-md>
# தலைப்பு
இது மார்க் டவுன் உள்ளடக்கத்தின் ஒரு பகுதி
</c-md>
```

### மாறும் பணி

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.value = "#புதிய தலைப்பு\nபுதிய உள்ளடக்கம்";
</script>
```

## இடைமுக அளவுருக்கள்

குறிச்சொல்லுக்குள் நேரடியாக மார்க் டவுன் உரையை அனுப்பவும் (இயல்புநிலை ஸ்லாட்டாக), அல்லது `value` வழியாக மாறும் வகையில் புதுப்பிக்கவும்.

## உடை விளக்கம்

கூறு துவக்கப்படும் போது `.Md` பாணி வகுப்பு தானாகவே சேர்க்கப்படும். பின்வரும் மார்க் டவுன் நீட்டிக்கப்பட்ட எச்சரிக்கை பெட்டி தளவமைப்பை ஆதரிக்கிறது (குறிப்பு தொடரியல் மூலம் தூண்டப்பட்டது):

- `[!NOTE]`: உடனடி தகவல்
- `[!TIP]`: பரிந்துரைகள்/உதவிக்குறிப்புகள்
- `[!IMPORTANT]`: முக்கிய கவனம்
- `[!WARNING]`: ஆபத்து எச்சரிக்கை
- `[!CAUTION]`: மிக அதிக ஆபத்து எச்சரிக்கை

## பாதுகாப்பு பொறிமுறை

ரெண்டரிங் செய்யும் போது, ​​HTML ஐ பாதுகாப்பாக உட்செலுத்த, சொந்த `setHTML` API ஐ அழைக்கவும். உலாவி அதை ஆதரிக்கவில்லை எனில், அது தானாகவே `DOMParser` அடிப்படையில் ஒரு சுழல்நிலை கிளீனருக்கு தரமிறக்கி பின்வரும் பாதுகாப்புக் கொள்கையைச் செயல்படுத்தும்:
- பாதுகாப்பற்ற குறிச்சொற்களை அகற்று (எ.கா. `script`, `iframe`, முதலியன)
- `on*` நிகழ்வு பண்புகளை அழிக்கவும்
- `javascript:`, `vbscript:`, மற்றும் `data:` (`data:image/` தவிர) போன்ற ஆபத்தான நெறிமுறைகளுடன் தொடங்கும் பண்புகளை வடிகட்டுவதன் மூலம் XSS தாக்குதல்களுக்கு எதிராகப் பாதுகாக்கவும்.
