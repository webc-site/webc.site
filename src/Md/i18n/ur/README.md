# مارک ڈاؤن ٹیکسٹ کو HTML میں محفوظ طریقے سے پارس اور رینڈر کریں۔

- **سیکیورٹی سینیٹائزیشن**: مقامی `setHTML` API یا `DOMParser` کلینر کا استعمال کرتے ہوئے XSS حملوں کے خلاف دفاع
- **مواد نکالنا**: ابتدا کے دوران ڈیفالٹ سلاٹ کے متنی مواد کو خود بخود پڑھیں اور اسے رینڈر کریں
- **بلٹ ان ٹائپ سیٹنگ**: پیش سیٹ لسٹ، ٹیبل، کوڈ بلاک، اقتباس اور وارننگ باکس اسٹائل

## ڈیمو استعمال کریں۔

### جامد رینڈرنگ

```html
<c-md>
# عنوان
یہ مارک ڈاؤن مواد کا ایک ٹکڑا ہے۔
</c-md>
```

### متحرک تفویض

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.textContent = "#نیا عنوان\nنیا مواد"؛
</script>
```

## انٹرفیس کے پیرامیٹرز

مارک ڈاؤن ٹیکسٹ کو براہ راست ٹیگ کے اندر (بطور ڈیفالٹ سلاٹ) پاس کریں، یا `textContent` کے ذریعے متحرک طور پر اپ ڈیٹ کریں۔

## انداز کی تفصیل

`.Md` اسٹائل کلاس خود بخود شامل ہو جاتی ہے جب جزو شروع ہوتا ہے۔ مندرجہ ذیل مارک ڈاون توسیعی وارننگ باکس لے آؤٹ کو سپورٹ کرتا ہے (حوالہ نحو سے متحرک):

- `[!NOTE]`： فوری معلومات
- `[!TIP]`: تجاویز/تجاویز
- `[!IMPORTANT]`: اہم توجہ
- `[!WARNING]`: خطرے کی وارننگ
- `[!CAUTION]`: انتہائی زیادہ خطرے کی وارننگ

## سیکورٹی میکانزم

رینڈرنگ کرتے وقت، HTML کو محفوظ طریقے سے انجیکشن کرنے کے لیے پہلے مقامی `setHTML` API کو کال کریں۔ اگر براؤزر اس کو سپورٹ نہیں کرتا ہے، تو یہ `DOMParser` کی بنیاد پر خود بخود ایک ریکرسیو کلینر پر ڈاون گریڈ ہو جائے گا اور درج ذیل سیکیورٹی پالیسی کو نافذ کرے گا:
- غیر محفوظ ٹیگز کو ہٹا دیں (جیسے `script`، `iframe`، وغیرہ)
- `on*` ایونٹ کی خصوصیات کو صاف کریں۔
- خطرناک پروٹوکولز جیسے `javascript:`، `vbscript:`، اور `data:` (سوائے `data:image/`) سے شروع ہونے والی خصوصیات کو فلٹر کرکے XSS حملوں کے خلاف دفاع کریں۔
