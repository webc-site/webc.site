# უსაფრთხოდ გააანალიზეთ და გადაიტანეთ Markdown ტექსტი HTML-ში.

- **უსაფრთხოების გაჯანსაღება**: დაიცავი თავი XSS შეტევებისგან მშობლიური `setHTML` API ან `DOMParser` გამწმენდების გამოყენებით
- **შინაარსის ამოღება**: ავტომატურად წაიკითხეთ ნაგულისხმევი სლოტის ტექსტის შინაარსი ინიციალიზაციის დროს და გადაიყვანეთ იგი
- ** ჩაშენებული ტიპაჟი **: წინასწარ დაყენებული სია, ცხრილი, კოდის ბლოკი, ციტატები და გამაფრთხილებელი ველის სტილები

## გამოიყენეთ დემო ვერსია

### სტატიკური რენდერირება

```html
<c-md>
# სათაური
ეს არის Markdown შინაარსის ნაწილი
</c-md>
```

### დინამიური დავალება

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.value = "#ახალი სათაური\nახალი კონტენტი";
</script>
```

## ინტერფეისის პარამეტრები

ჩაატარეთ Markdown ტექსტი პირდაპირ ტეგში (როგორც ნაგულისხმევი სლოტი), ან განაახლეთ დინამიურად `value`-ის საშუალებით.

## სტილის აღწერა

`.Md` სტილის კლასი ავტომატურად ემატება კომპონენტის ინიციალიზაციისას. მხარს უჭერს Markdown-ის გაფართოებული გამაფრთხილებელი ყუთის განლაგებას (გამოწვეული მითითების სინტაქსით):

- `[!NOTE]`：მოთხოვნის ინფორმაცია
- `[!TIP]`: წინადადებები/რჩევები
- `[!IMPORTANT]`: მნიშვნელოვანი ყურადღება
- `[!WARNING]`: რისკის გაფრთხილება
- `[!CAUTION]`: უკიდურესად მაღალი რისკის გაფრთხილება

## უსაფრთხოების მექანიზმი

რენდერის დროს, ჯერ დაურეკეთ მშობლიურ `setHTML` API-ს, რათა უსაფრთხოდ შეიყვანოთ HTML. თუ ბრაუზერი არ უჭერს მხარს მას, ის ავტომატურად გადავა რეკურსიულ გამწმენდზე `DOMParser`-ის საფუძველზე და განახორციელებს უსაფრთხოების შემდეგ პოლიტიკას:
- წაშალეთ არაუსაფრთხო თეგები (მაგ. `script`, `iframe` და ა.შ.)
- `on*` მოვლენის ატრიბუტების გასუფთავება
- დაიცავით თავი XSS შეტევებისგან სახიფათო პროტოკოლებით დაწყებული ატრიბუტების გაფილტვრით, როგორიცაა `javascript:`, `vbscript:` და `data:` (გარდა `data:image/`-ისა).
