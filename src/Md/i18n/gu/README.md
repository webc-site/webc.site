# માર્કડાઉન ટેક્સ્ટને HTML પર સુરક્ષિત રીતે પાર્સ અને રેન્ડર કરો.

- **સુરક્ષા સેનિટાઇઝેશન**: મૂળ `setHTML` API અથવા `DOMParser` ક્લીનર્સનો ઉપયોગ કરીને XSS હુમલાઓ સામે બચાવ
- **સામગ્રી નિષ્કર્ષણ**: આરંભ દરમિયાન ડિફૉલ્ટ સ્લોટની ટેક્સ્ટ સામગ્રીને આપમેળે વાંચો અને તેને રેન્ડર કરો
- **બિલ્ટ-ઇન ટાઇપસેટિંગ**: પ્રીસેટ સૂચિ, ટેબલ, કોડ બ્લોક, અવતરણ અને ચેતવણી બોક્સ શૈલીઓ

## ડેમોનો ઉપયોગ કરો

### સ્થિર રેન્ડરીંગ

```html
<c-md>
# શીર્ષક
આ માર્કડાઉન સામગ્રીનો એક ભાગ છે
</c-md>
```

### ગતિશીલ સોંપણી

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.textContent = "#નવું શીર્ષક\nનવી સામગ્રી";
</script>
```

## ઇન્ટરફેસ પરિમાણો

સીધા જ ટેગની અંદર માર્કડાઉન ટેક્સ્ટ પાસ કરો (ડિફૉલ્ટ સ્લોટ તરીકે), અથવા `textContent` દ્વારા ગતિશીલ રીતે અપડેટ કરો.

## શૈલી વર્ણન

જ્યારે ઘટક શરૂ થાય છે ત્યારે `.Md` શૈલી વર્ગ આપમેળે ઉમેરવામાં આવે છે. નીચેના માર્કડાઉન વિસ્તૃત ચેતવણી બોક્સ લેઆઉટને સપોર્ટ કરે છે (સંદર્ભ વાક્યરચના દ્વારા ટ્રિગર):

- `[!NOTE]`: પ્રોમ્પ્ટ માહિતી
- `[!TIP]`: સૂચનો/ટિપ્સ
- `[!IMPORTANT]`: મહત્વપૂર્ણ ધ્યાન
- `[!WARNING]`: જોખમની ચેતવણી
- `[!CAUTION]`: અત્યંત ઉચ્ચ જોખમની ચેતવણી

## સુરક્ષા મિકેનિઝમ

રેન્ડર કરતી વખતે, HTML ને સુરક્ષિત રીતે ઇન્જેક્ટ કરવા માટે પહેલા નેટિવ `setHTML` API ને કૉલ કરો. જો બ્રાઉઝર તેને સપોર્ટ કરતું નથી, તો તે `DOMParser`ના આધારે આપમેળે પુનરાવર્તિત ક્લીનર પર ડાઉનગ્રેડ કરશે અને નીચેની સુરક્ષા નીતિનો અમલ કરશે:
- બિન-સુરક્ષિત ટૅગ્સ દૂર કરો (દા.ત. `script`, `iframe`, વગેરે.)
- `on*` ઇવેન્ટ લક્ષણો સાફ કરો
- `javascript:`, `vbscript:` અને `data:` (`data:image/` સિવાય).
