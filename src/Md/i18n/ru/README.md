# Безопасно анализируйте и преобразуйте текст Markdown в HTML.

- **Очистка безопасности**: защита от XSS-атак с помощью встроенного API `setHTML` или средств очистки `DOMParser`.
- **Извлечение содержимого**: автоматическое считывание текстового содержимого слота по умолчанию во время инициализации и его визуализация.
- **Встроенная верстка**: стили списка предустановок, таблицы, блока кода, цитат и предупреждений.

## Используйте демо-версию

### статический рендеринг

```html
<c-md>
# заголовок
Это часть контента Markdown
</c-md>
```

### динамическое назначение

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.textContent = "#Новый заголовок\nНовый контент";
</script>
```

## Параметры интерфейса

Передавайте текст Markdown непосредственно в теге (в качестве слота по умолчанию) или обновляйте динамически с помощью `textContent`.

## Описание стиля

Класс стиля `.Md` автоматически добавляется при инициализации компонента. Поддерживает следующий макет расширенного окна предупреждения Markdown (активируется ссылочным синтаксисом):

- `[!NOTE]`：Быстрая информация
- `[!TIP]`: предложения/советы
- `[!IMPORTANT]`: важное внимание
- `[!WARNING]`: предупреждение о риске
- `[!CAUTION]`: предупреждение об очень высоком риске.

## механизм безопасности

При рендеринге сначала вызовите собственный API `setHTML`, чтобы безопасно внедрить HTML. Если браузер не поддерживает это, он автоматически перейдет на рекурсивную очистку на основе `DOMParser` и реализует следующую политику безопасности:
- Удалите небезопасные теги (например, `script`, `iframe` и т. д.).
- Очистить атрибуты события `on*`
- Защититесь от XSS-атак, фильтруя атрибуты, начинающиеся с опасных протоколов, таких как `javascript:`, `vbscript:` и `data:` (кроме `data:image/`).
