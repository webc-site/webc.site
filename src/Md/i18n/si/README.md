# ආරක්ෂිතව විග්‍රහ කර Markdown පෙළ HTML වෙත ලබා දෙන්න.

- **ආරක්ෂක සනීපාරක්ෂාව**: දේශීය `setHTML` API හෝ `DOMParser` ක්ලීනර් භාවිතයෙන් XSS ප්‍රහාරවලින් ආරක්ෂා කරන්න
- **අන්තර්ගත උපුටා ගැනීම**: ආරම්භයේදී පෙරනිමි ස්ලොට් එකේ පාඨ අන්තර්ගතය ස්වයංක්‍රීයව කියවා එය ලබා දෙන්න
- **බිල්ට් අකුරු සැකසීම**: පෙරසිටුවීම් ලැයිස්තුව, වගුව, කේත වාරණ, උද්ධෘත සහ අනතුරු ඇඟවීමේ පෙට්ටි මෝස්තර

## Demo භාවිතා කරන්න

### ස්ථිතික විදැහුම්කරණය

```html
<c-md>
# මාතෘකාව
මෙය Markdown අන්තර්ගතයේ කොටසකි
</c-md>
```

### ගතික පැවරුම

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.value = "#නව මාතෘකාව\nනව අන්තර්ගතය";
</script>
```

## අතුරුමුහුණත් පරාමිතීන්

මාර්ක්ඩවුන් පෙළ කෙලින්ම ටැගය තුළට යන්න (පෙරනිමි ස්ලට් ලෙස), නැතහොත් `value` හරහා ගතිකව යාවත්කාලීන කරන්න.

## විලාස විස්තරය

සංරචකය ආරම්භ කරන විට `.Md` මෝස්තර පන්තිය ස්වයංක්‍රීයව එකතු වේ. පහත දැක්වෙන Markdown විස්තීරණ අනතුරු ඇඟවීමේ පෙට්ටි පිරිසැලසුම සඳහා සහය දක්වයි (යොමු වාක්‍ය ඛණ්ඩය මගින් අවුලුවන):

- `[!NOTE]`: ඉක්මන් තොරතුරු
- `[!TIP]`: යෝජනා/ඉඟි
- `[!IMPORTANT]`: වැදගත් අවධානය
- `[!WARNING]`: අවදානම් අනතුරු ඇඟවීම
- `[!CAUTION]`: අතිශයින් ඉහළ අවදානම් අනතුරු ඇඟවීමක්

## ආරක්ෂක යාන්ත්රණය

විදැහුම්කරණයේදී, ආරක්ෂිතව HTML එන්නත් කිරීමට පළමුව ස්වදේශික `setHTML` API අමතන්න. බ්‍රවුසරය එයට සහය නොදක්වන්නේ නම්, එය `DOMParser` මත පදනම්ව ස්වයංක්‍රීයව ප්‍රත්‍යාවර්තී පිරිසිදුකාරකයකට ශ්‍රේණිගත කර පහත ආරක්ෂක ප්‍රතිපත්තිය ක්‍රියාත්මක කරයි:
- අනාරක්ෂිත ටැග් ඉවත් කරන්න (උදා. `script`, `iframe`, ආදිය)
- `on*` සිදුවීම් ගුණාංග හිස් කරන්න
- `javascript:`, `vbscript:`, සහ `data:` (`data:image/` හැර) වැනි භයානක ප්‍රොටෝකෝල වලින් ආරම්භ වන ගුණාංග පෙරීම මගින් XSS ප්‍රහාර වලින් ආරක්ෂා වන්න.
