# Безбедно анализирајте и прикажувајте Markdown текст во HTML.

- **Безбедносна санација**: Заштитете се од нападите на XSS користејќи мајчин `setHTML` API или `DOMParser` средства за чистење
- **Екстракција на содржина**: Автоматски читајте ја содржината на текстот на стандардниот слот за време на иницијализацијата и прикажувајте ја
- **Вградено наборување**: претходно поставена листа, табела, блок код, стилови на цитати и поле за предупредување

## Користете го демо

### статичко рендерирање

```html
<c-md>
# наслов
Ова е дел од содржината на Markdown
</c-md>
```

### динамична задача

```html
<c-md id="md-el"></c-md>

<script type="module">
  const el = document.querySelector("#md-el");
  el.value = "#Нов наслов\nНова содржина";
</script>
```

## Параметри на интерфејсот

Пренесете го текстот Markdown директно во ознаката (како стандарден слот) или ажурирајте динамично преку `value`.

## Опис на стилот

Класата за стилови `.Md` автоматски се додава кога компонентата е иницијализирана. Го поддржува следниов распоред на проширено предупредувачко поле Markdown (активиран од референтна синтакса):

- `[!NOTE]`：Прави информации
- `[!TIP]`: Предлози/Совети
- `[!IMPORTANT]`: Важно внимание
- `[!WARNING]`: предупредување за ризик
- `[!CAUTION]`: Предупредување со екстремно висок ризик

## безбедносен механизам

При рендерирање, прво повикајте го мајчин `setHTML` API за безбедно да внесете HTML. Ако прелистувачот не го поддржува, тој автоматски ќе се деградира на рекурзивен чистач заснован на `DOMParser` и ќе ја спроведе следната безбедносна политика:
- Отстранете ги небезбедните ознаки (на пр. `script`, `iframe` итн.)
- Исчистете ги `on*` атрибутите на настанот
- Заштитете се од нападите на XSS со филтрирање на атрибути кои започнуваат со опасни протоколи како што се `javascript:`, `vbscript:` и `data:` (освен `data:image/`).
