# Пераключальныя ўкладкі

- Абраны стан цэтліка цалкам інтэграваны ў вобласць змесціва ніжэй.
- Структурныя спецыфікацыі: даччыныя элементы фіксуюцца для выкарыстання `nav > a` у якасці навігацыі па цэтліках, а прамы даччыны элемент `b` выкарыстоўваецца ў якасці вобласці змесціва.
- Аўтаматычна звязваць тэг `a[value]` з вузлом змесціва `b[slot]`.
- Падтрымлівае аўтаматычнае захаванне бягучай актыўнай старонкі ў `localStorage` праз атрыбут `key`.
- Падтрымліваецца ўказанне актываванай старонкі ўкладкі праз атрыбут `value` (калі няма кэша і `value`, па змаўчанні будзе абраны тэг з `class="A"` або першы тэг).
- Лёгкая рэалізацыя Light DOM, адсутнасць перашкод Shadow DOM, высокая ступень свабоды ў наладзе стылю.

## Выкарыстоўвайце дэма

```html
<c-tab key="my-tab-key" value="banana">
  <nav>
    <a value="apple">яблык</a>
    <a value="banana">банан</a>
    <a value="orange">мандарын</a>
  </nav>
  <b slot="apple">Яблык - пажыўны фрукт</b>
  <b slot="banana">Банан - гэта трапічны фрукт</b>
  <b slot="orange">Апельсіны багатыя вітамінам С</b>
</c-tab>
```

## уласнасць

- `key`: імя ключа, якое выкарыстоўваецца для захоўвання localStorage
- `value`: выбранае значэнне

## Карыстальніцкія падзеі

- `change`: Спрацоўвае пры пераключэнні ўкладкі, `e.value` - гэта значэнне `value` бягучага актываванага тэга.
