# ແຍກວິເຄາະຢ່າງປອດໄພ ແລະສະແດງຂໍ້ຄວາມ Markdown ເປັນ HTML.

- **Security Sanitization**: ປ້ອງກັນການໂຈມຕີ XSS ໂດຍໃຊ້ຕົວທຳຄວາມສະອາດເດີມ `setHTML` API ຫຼື `DOMParser`
- ** ການ​ສະ​ກັດ​ເນື້ອ​ໃນ **​: ອ່ານ​ອັດ​ຕະ​ໂນ​ມັດ​ເນື້ອ​ຫາ​ຂອງ​ຂໍ້​ຄວາມ​ຂອງ​ຊ່ອງ​ໃນ​ລະ​ຫວ່າງ​ການ​ເລີ່ມ​ຕົ້ນ​ແລະ​ເຮັດ​ໃຫ້​ມັນ​.
- **ການ​ຈັດ​ປະ​ເພດ​ໃນ​ຕົວ**​: ບັນ​ຊີ​ລາຍ​ການ​ທີ່​ຕັ້ງ​ໄວ້​ກ່ອນ​, ຕາ​ຕະ​ລາງ​, ຕັນ​ລະ​ຫັດ​, quote ແລະ​ຮູບ​ແບບ​ປ່ອງ​ເຕືອນ​

## ໃຊ້ຕົວຢ່າງ

### ການສະແດງຜົນແບບຄົງທີ່

```html
<c-md>
# ຫົວຂໍ້
ນີ້ແມ່ນເນື້ອໃນຂອງ Markdown
</c-md>
```

### ການມອບໝາຍແບບເຄື່ອນໄຫວ

```html
<c-md id="md"></c-md>

<script type="module">
import "webc.site/Md.js";
document.querySelector("#md").value = "#ຫົວຂໍ້ໃໝ່\nເນື້ອຫາໃໝ່";
</script>
```

## ຕົວກໍານົດການໃນການໂຕ້ຕອບ

ຜ່ານຂໍ້ຄວາມ Markdown ໂດຍກົງພາຍໃນແທັກ (ເປັນຊ່ອງເລີ່ມຕົ້ນ), ຫຼືອັບເດດແບບໄດນາມິກຜ່ານ `value`.

## ລາຍລະອຽດຮູບແບບ

`.Md`ຊັ້ນສະໄຕລ໌ຖືກເພີ່ມໂດຍອັດຕະໂນມັດເມື່ອອົງປະກອບເລີ່ມຕົ້ນ. ສະ​ຫນັບ​ສະ​ຫນູນ​ດັ່ງ​ຕໍ່​ໄປ​ນີ້​ຮູບ​ແບບ​ປ່ອງ​ເຕືອນ​ທີ່​ຂະ​ຫຍາຍ Markdown (ເກີດ​ໂດຍ syntax ກະ​ສານ​ອ້າງ​ອີງ​)​:

- `[!NOTE]`:ຂໍ້ມູນການເຕືອນ
- `[!TIP]`: ຄຳແນະນຳ/ຄຳແນະນຳ
- `[!IMPORTANT]`: ເອົາໃຈໃສ່ສຳຄັນ
- `[!WARNING]`: ການເຕືອນໄພຄວາມສ່ຽງ
- `[!CAUTION]`: ການເຕືອນໄພຄວາມສ່ຽງສູງທີ່ສຸດ

## ກົນ​ໄກ​ຄວາມ​ປອດ​ໄພ​

ໃນ​ເວ​ລາ​ທີ່ rendering, call the native `setHTML` API ທໍາອິດເພື່ອ inject HTML ຢ່າງປອດໄພ. ຖ້າບຣາວເຊີບໍ່ຮອງຮັບມັນ, ມັນຈະດາວເກຣດເປັນຕົວເຮັດຄວາມສະອາດແບບ recursive ໂດຍອັດຕະໂນມັດໂດຍອີງໃສ່ `DOMParser` ແລະປະຕິບັດນະໂຍບາຍຄວາມປອດໄພຕໍ່ໄປນີ້:
- ເອົາແທັກທີ່ບໍ່ປອດໄພອອກ (ເຊັ່ນ: `script`, `iframe`, ແລະອື່ນໆ)
- ລຶບລ້າງ `on*`ຄຸນສົມບັດເຫດການ
- ປ້ອງກັນການໂຈມຕີ XSS ໂດຍການກັ່ນຕອງຄຸນສົມບັດເລີ່ມຕົ້ນດ້ວຍໂປຣໂຕຄໍອັນຕະລາຍເຊັ່ນ `javascript:`, `vbscript:`, ແລະ `data:` (ຍົກເວັ້ນ `data:image/`).
