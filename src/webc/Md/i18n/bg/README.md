# Безопасно анализирайте и рендирайте Markdown текст в HTML.

- **Дезифициране на сигурността**: Защитете се срещу XSS атаки с помощта на собствен `setHTML` API или `DOMParser` почистващи средства
- **Извличане на съдържание**: Автоматично четене на текстовото съдържание на слота по подразбиране по време на инициализация и го изобразява
- **Вградена настройка на текста**: предварително зададени стилове на списък, таблица, кодов блок, котировки и предупредителни кутии

## Използвайте демонстрацията

### статично изобразяване

```html
<c-md>
# заглавие
Това е част от съдържанието на Markdown
</c-md>
```

### динамично задание

```html
<c-md id="md"></c-md>

<script type="module">
import "webc.site/Md.js";
document.querySelector("#md").value = "#Ново заглавие\nНово съдържание";
</script>
```

## Параметри на интерфейса

Предайте Markdown текст директно в тага (като слот по подразбиране) или актуализирайте динамично чрез `value`.

## Описание на стила

Класът стил `.Md` се добавя автоматично, когато компонентът се инициализира. Поддържа следното оформление на разширеното поле за предупреждение Markdown (задействано от референтния синтаксис):

- `[!NOTE]`：Бърза информация
- `[!TIP]`: Предложения/Съвети
- `[!IMPORTANT]`: Важно внимание
- `[!WARNING]`: Предупреждение за риск
- `[!CAUTION]`: Предупреждение за изключително висок риск

## защитен механизъм

При рендиране първо извикайте собствения `setHTML` API, за да инжектирате безопасно HTML. Ако браузърът не го поддържа, той автоматично ще премине към рекурсивна програма за почистване, базирана на `DOMParser` и ще приложи следната политика за сигурност:
- Премахнете небезопасните тагове (напр. `script`, `iframe` и др.)
- Изчистване на `on*` атрибутите на събитието
- Защита срещу XSS атаки чрез филтриране на атрибути, започващи с опасни протоколи като `javascript:`, `vbscript:` и `data:` (с изключение на `data:image/`).
