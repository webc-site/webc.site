# मार्कडाउन टेक्स्ट को HTML में सुरक्षित रूप से पार्स और रेंडर करें।

- **सुरक्षा स्वच्छता**: देशी `setHTML` API या `DOMParser` क्लीनर का उपयोग करके XSS हमलों से बचाव करें
- **सामग्री निष्कर्षण**: आरंभीकरण के दौरान डिफ़ॉल्ट स्लॉट की पाठ्य सामग्री को स्वचालित रूप से पढ़ें और उसे प्रस्तुत करें
- **अंतर्निहित टाइपसेटिंग**: प्रीसेट सूची, तालिका, कोड ब्लॉक, उद्धरण और चेतावनी बॉक्स शैलियाँ

## डेमो का प्रयोग करें

### स्थैतिक प्रतिपादन

```html
<c-md>
# शीर्षक
यह मार्कडाउन सामग्री का एक अंश है
</c-md>
```

### गतिशील असाइनमेंट

```html
<c-md id="md"></c-md>

<script type="module">
import "webc.site/Md.js";
document.querySelector("#md").value = "#नया शीर्षक\nनया सामग्री";
</script>
```

## इंटरफ़ेस पैरामीटर

मार्कडाउन टेक्स्ट को सीधे टैग के भीतर पास करें (डिफ़ॉल्ट स्लॉट के रूप में), या `value` के माध्यम से गतिशील रूप से अपडेट करें।

## शैली विवरण

घटक आरंभ होने पर `.Md` शैली वर्ग स्वचालित रूप से जुड़ जाता है। निम्नलिखित मार्कडाउन विस्तारित चेतावनी बॉक्स लेआउट का समर्थन करता है (संदर्भ सिंटैक्स द्वारा ट्रिगर):

- `[!NOTE]`：शीघ्र जानकारी
- `[!TIP]`: सुझाव/सुझाव
- `[!IMPORTANT]`: महत्वपूर्ण ध्यान
- `[!WARNING]`: जोखिम चेतावनी
- `[!CAUTION]`: अत्यंत उच्च जोखिम चेतावनी

## सुरक्षा तंत्र

रेंडर करते समय, HTML को सुरक्षित रूप से इंजेक्ट करने के लिए पहले मूल `setHTML` API को कॉल करें। यदि ब्राउज़र इसका समर्थन नहीं करता है, तो यह स्वचालित रूप से `DOMParser` के आधार पर एक पुनरावर्ती क्लीनर में डाउनग्रेड हो जाएगा और निम्नलिखित सुरक्षा नीति लागू करेगा:
- गैर-सुरक्षित टैग हटाएं (जैसे `script`, `iframe`, आदि)
- `on*` इवेंट विशेषताएँ साफ़ करें
- `javascript:`, `vbscript:`, और `data:` (`data:image/` को छोड़कर) जैसे खतरनाक प्रोटोकॉल से शुरू होने वाली विशेषताओं को फ़िल्टर करके XSS हमलों से बचाव करें।
