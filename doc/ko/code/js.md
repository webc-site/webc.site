# JS 코드 사양

## 코딩 스타일

- 가장 현대적인 js 구문을 사용하여 간단하고 우아하며 효율적입니다.
- 지나치게 긴 기능을 분할하고 인터페이스 디자인은 결합도는 낮고 응집도는 높아야 합니다.
- 배열 + 루프/forEach/맵을 더 많이 사용하고 덜 반복적인 코드를 작성하세요.
- 과도한 주석을 피하기 위해 함수 주석을 간단한 코드로 작성하지 마세요. 복잡한 논리와 특별한 요구사항을 세련된 중국어로 설명합니다.
- 반복되는 코드는 중복성을 줄이기 위해 함수로 추상화됩니다. 예를 들어 `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })`은 `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()`로 쓸 수 있습니다.
- 변수 선언: 여러 연속 `const` 선언을 하나의 명령문으로 결합합니다. 세 줄로 작성하는 대신 `const a=1, b=2, c=3;`를 작성하세요.
- 비동기 처리: `await` 사용, `.then` 체인 호출 사용 금지
- 예외 처리 코드를 자동으로 생성하지 않음, `try...catch` 자동으로 작성하지 않음(`try ... catch`는 수동으로 유지 관리, 기존 `try catch`은 삭제하면 안 됨)
- 순수 함수 우선: 순수 함수만 작성하고 클래스는 작성하지 않음
- 화살표 기능 `const funcName = () => {}`을 사용하고 `function` 키워드는 사용하지 마세요(생성기 제외). .bind를 사용하여 매개변수를 바인딩할 수 있다면 함수 정의를 피하세요.
- 코드 재사용: 재사용에 주의를 기울이고, 더 많은 작은 함수를 추출하고, 유사하거나 복사하여 붙여넣은 코드 구조를 많이 피하십시오.
- 객체 액세스: 필수 속성을 추출하기 위해 구조 분해 할당을 사용하여 우선 순위를 지정하고, 내부적으로 점을 반복적으로 사용하여 깊고 중첩된 속성에 액세스하지 않고, 반복되는 선택적 체인 판단을 병합합니다.
- 함수 매개변수에 객체를 사용하지 마세요. `{a,b,c}`을 쓰지 말고, `a, b, c`를 쓰세요. 선택적 매개변수가 많은 경우 [[구성 항목 수, 구성 항목], [구성 항목 수, 구성 항목],..]을 사용합니다. 이 패러다임에서는 구성 항목이 숫자 상수로 정의됩니다.
- 다중 값 반환의 경우 `{a,b,c}` 대신 `[a,b,c]` 배열을 사용하세요. 반환 값이 3개 이상인 경우 숫자 상수를 사용하여 위치 의미를 정의합니다.
- 상태를 나타내기 위해 문자열이 필요하지 않습니다. 숫자 상수를 사용하여 상태를 정의합니다.
- 문자열 템플릿(``)이 사용되지 않으며 가져오기를 제외하고 문자열 연결(+)이 사용됩니다(Vite 정적 분석에 편리함)
- 루프의 경우 일련번호가 필요한 경우 `i++` 대신 `++i`을 사용하세요.
- 불필요한`?.` 방지 및 방어적인 프로그래밍 금지
- 변수 이름은 복수형일 필요는 없으며 `_li`로 끝나 목록을 나타냅니다.

## 명명 규칙

- 네이밍은 미니멀리즘을 추구합니다. 짧지만 의미 있는 이름을 사용하세요. 예를 들어 `remove`, `delete`, `del` 대신 `rm`을 사용하세요. 그러나 극단적인 사용도 피해야 합니다. 예를 들어 `map` 대신 `m` 문자를 하나만 사용하지 마세요.
- 함수 이름을 지정할 때는 동사만 사용하세요. 한 단어로 표현할 수 있다면 두 단어를 사용하지 마세요. 명사는 파일 이름에 반영됩니다. 파일명에 동사를 추가해야 하는 경우에는 명사를 먼저 넣고 동사를 맨 뒤에 넣어주세요. 예: `setProfile.js` 대신 `profileSet.js`
- 변수 이름: 밑줄 스타일(snake_case)을 사용합니다(예: `user_auth_token`). 변수가 함수인 경우 소문자 카멜 케이스 이름 지정 스타일을 사용하세요.
- 함수 이름: 소문자 camelCase(camelCase)를 사용하세요(예: `userData`).
- 함수 매개변수: 콜백 함수인 경우 `onChange`과 같이 소문자 카멜 케이스로 이름을 지정합니다.
- 모듈 수준 상수 정의는 대문자 밑줄 스타일 `UPPER_SNAKE_CASE`을 사용합니다.
- 다음과 같이 의미 없는 접두어인 `get`을 쓰지 마십시오. `getCookie` 대신 `cookieByHeader`를 쓰십시오.
- 전역/모듈 수준 상수: 대문자 밑줄 스타일(UPPER_SNAKE_CASE)을 사용합니다. 예: `CODE_TO_ID`, `ID_TO_LANG`

## 모듈식 메커니즘

- 가져오기: 요청 시 정확하게 기능을 가져오고, 전체 모듈을 직접 가져오는 것을 금지합니다(`import * as x` 방지 또는 큰 개체 가져오기).
- 내보내기: 개체 내보내기를 비활성화합니다. 함수와 변수를 `export const X=1, abc=()=>{};`과 같은 단위로 내보냅니다. 내보낸 콘텐츠를 선언하려면 const + 쉼표와 결합해 보세요. 파일에 하나의 기능만 있는 경우 `export default`를 사용하세요.
- 내부적으로 기본 내보내기 함수를 호출해야 하는 경우가 아니면 먼저 상수를 선언한 다음 파일 끝에서 내보내는 것을 피하세요.
- 경로 확인: 현재 디렉터리 경로를 얻을 때 `import.meta.dirname`을 사용해야 합니다.

## 실수

- 문자열 오류를 방지하고 const를 사용하여 지속적인 오류 코드를 선언하세요.
- 잘못된 데이터 정보를 반환해야 하는 경우 [오류 코드, 오류 메시지...]를 사용하세요. 이렇게 하면 오류 메시지가 텍스트 설명이 아닌 숫자 값(예: [FILE_OVERSIZE, file_size, max_size])이 됩니다.
