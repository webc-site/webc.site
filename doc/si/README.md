[වෙබ් සංරචක](https://developer.mozilla.org/docs/Web/API/Web_components) මත පදනම්ව සංවර්ධනය කරන ලද, එය ධාවන කාලය ස්වාධීන වන අතර සියලු රාමු සඳහා සුදුසු වේ.

ප්‍රමාණය සඳහා අතිශයින් ප්‍රශස්ත කර ඇත (සමාන ජනප්‍රිය පුස්තකාලවල ප්‍රමාණයෙන් දහයෙන් එකක්).

භාෂා 75 කට සහය දක්වයි. ඔබට සබැඳි භාවිතය සඳහා js සහ css සෘජුවම ආයාත කළ හැකිය, නැතහොත් ද්විතීයික සංවර්ධනය සඳහා අවශ්‍ය පරිදි ඔබේ ව්‍යාපෘතියට සංරචක එක් කළ හැකිය.

මෙය [කතන්දර පොත](https://storybook.js.org) හි ස්ථිතික බ්ලොග් අනුවාදයයි.

##### Shadcn සමඟ සංසන්දනය කිරීම

සහ `webc.site` හි සංරචකය:

## ස්වයංක්‍රීය සංවර්ධනය සහ දෝෂහරණය

### දේශීය සංවර්ධනය සහ දෝෂහරණය

සෑම වෙබ් පිටු සංරචකයක්ම සංවර්ධන සේවාදායකය ස්වාධීනව ආරම්භ කිරීමට සහය දක්වයි, අවම සංවර්ධනයක් සහ දෝශ නිරාකරණ පරිසරයක් සපයයි:

- **නව සංරචකයක් සාදන්න**: සංරචක සංවර්ධන ඇටසැකිල්ලක් ඉක්මනින් ජනනය කිරීමට `./sh/new.js [css|js] [සංරචක නාමය]` ධාවනය කරන්න (ආදර්ශන පිටුව සහ ජාත්‍යන්තරකරණ වින්‍යාසය ඇතුළුව).
- **ස්වාධීන නිදොස්කරණය**: Vite සංවර්ධන සේවාදායකය ආරම්භ කිරීමට සහ බ්‍රවුසරයේ සංරචකයේ ආදර්ශන පිටුව ස්වයංක්‍රීයව විවෘත කිරීමට `./dev.js [සංරචක නාමය]` (හෝ `./dev.js` සෘජුවම ධාවනය කරන්න) ධාවනය කරන්න. සංරචක උණුසුම් යාවත්කාලීන (HMR) සඳහා සහය දක්වයි. සංවර්ධන සහ දත්ත සබැඳි විසංයෝජනය කර ඇති බැවින්, ශුන්‍ය සංවර්ධන ඉදිරිපස සබැඳියක් ඇත, එය AI සඳහා ඉතා යෝග්‍ය වේ (Claude Code, Google Antigravity, ආදිය) සම්පූර්ණයෙන්ම ස්වයංක්‍රීය සංවර්ධනයට සහ බ්‍රවුසර නිදොස්කරණයට සහය වීමට.

### සංරචක පුස්තකාලය ප්‍රකාශනය කරන්න

#### webc dist

ඇසුරුම්කරණය සහ ප්‍රකාශන ක්‍රියාවලිය ඉතා ස්වයංක්‍රීය වේ:
- **ස්ථිතික සම්පත් ඇසුරුම්**: `./sh/dist.js` ධාවනය කිරීම (එනම්, `webc dist` කාර්යය ක්‍රියාත්මක කිරීම) `src/` යටතේ ඇති සියලුම වෙබ් පිටු සංරචක ස්වයංක්‍රීයව පරිලෝකනය කර, ඒවා `dist/` බහලුම වෙත සම්පාදනය, සම්පීඩනය සහ ප්‍රතිදානය කළ හැක. මෙම ඩිරෙක්ටරියේ නිෂ්පාදන බෙදා හැරීම සඳහා JS/CSS ගොනු, GitHub පිටු හෝ Cloudflare පිටු වෙත යෙදවීමට සූදානම් ආදර්ශන අඩවියක් සහ ස්වයංක්‍රීයව `_redirects` සහ `404.html` මාර්ගගත ගොනු ජනනය කර ඇත.
- **NPM ප්‍රකාශනය**: `./sh/npmDist.js` ධාවනය කරන්න, ස්ක්‍රිප්ට් ස්වයංක්‍රීයව සංරචක පරායත්තයන් හසුරුවනු ඇත, පැකේජ අනුවාද අංකය වැඩි කරයි, සහ සංරචක පුස්තකාලය NPM වෙත ප්‍රකාශනය කිරීමට පැකේජය අවසන් වූ පසු එක් ක්ලික් කිරීමකින් `npm publish` ක්‍රියාත්මක කරයි.

### පරිවර්තනය සඳහා වේදිකාව වගකිව යුතුය

රාමුව තුළ ස්වයංක්‍රීය පරිවර්තන ක්‍රියාවලි ඇත:
1. ව්‍යාපෘති මූල නාමාවලියෙහි `tran.yml` හි පරිවර්තන රීති (මූලාශ්‍ර භාෂාව, ඉලක්ක භාෂාව සහ පරිවර්තනය කළ යුතු නාමාවලිය වැනි) වින්‍යාස කරන්න.
2. ඔබේ කේතයේ බහු භාෂා දෘඪ කේතයකින් තොරව සංරචක හෝ ලේඛනවල ස්ථාන දරන්නන් භාවිතා කරන්න.
3. `./sh/tran.js` ධාවනය කරන්න, එවිට පරිවර්තන එන්ජිම ස්වයංක්‍රීයව පෙළ උකහා ගෙන සැකසීම සඳහා WebC පරිවර්තන සේවාව අමතනු ඇත, ස්වයංක්‍රීයව YAML පරිවර්තන ඇතුළත් කිරීම් සහ විවිධ භාෂාවලින් README ලේඛන ජනනය කරයි.

### පාරිසරික විචල්යයන්

ස්වයංක්‍රීය පරිවර්තන සිදු කරන විට, පරිවර්තන API වෙත ප්‍රවේශය අවසර දීමට ඔබ පහත පරිසර විචල්‍යයන් වින්‍යාස කිරීම අවශ්‍ය වේ:

- **`WEBC_SITE_TOKEN`** (හෝ **`WEBC_TOKEN`**): `webc.site` ස්වයංක්‍රීය පරිවර්තන API වෙත ප්‍රවේශ වීමට භාවිතා කරන සත්‍යාපන යතුර. පරිසර විචල්‍යයක් ලෙස එන්නත් කළ හැක හෝ දේශීය පරිශීලකයාගේ මුල් නාමාවලියෙහි ඇති `~/.config/webc.site.yml` වින්‍යාස ගොනුවේ `token` ලෙස ලිවිය හැක.
