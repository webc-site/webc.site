# JS කේත පිරිවිතර

## කේතීකරණ විලාසය

- නවීනතම js සින්ටැක්ස් භාවිතා කරමින් සරල, අලංකාර සහ කාර්යක්ෂම
- අධික ලෙස දිගු කාර්යයන් බෙදන්න, සහ අතුරු මුහුණත නිර්මාණය අඩු සම්බන්ධ කිරීම සහ ඉහළ එකමුතුකම විය යුතුය.
- arrays + loops/forEach/map වැඩිපුර භාවිතා කර අඩු පුනරාවර්තන කේතයක් ලියන්න
- අධික අදහස් දැක්වීම් වලක්වා ගැනීම සඳහා සරල කේතයකින් කාර්යය අදහස් ලියන්න එපා. සංකීර්ණ තර්කනය සහ විශේෂ අවශ්‍යතා පිරිපහදු කළ චීන භාෂාවෙන් පැහැදිලි කර ඇත.
- පුනරාවර්තන කේතය අතිරික්තය අඩු කිරීම සඳහා ශ්‍රිතවලට වියුක්ත කර ඇත. උදාහරණයක් ලෙස, `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()` ලෙස ලිවිය හැක
- විචල්‍ය ප්‍රකාශනය: එක ප්‍රකාශයකට අඛණ්ඩව `const` ප්‍රකාශන කිහිපයක් ඒකාබද්ධ කරන්න. පේළි තුනකින් ලියනවා වෙනුවට `const a=1, b=2, c=3;` ලියන්න
- අසමමුහුර්ත සැකසුම්: `await` භාවිතා කරන්න, `.then` දාම ඇමතුම් භාවිතය තහනම්
- ව්‍යතිරේක හැසිරවීමේ කේතය ස්වයංක්‍රීයව උත්පාදනය නොකරන්න, `try...catch` ස්වයංක්‍රීයව ලියන්න එපා (`try ... catch` අතින් නඩත්තු වේ, පවතින `try catch` මකා නොදැමිය යුතුය)
- පළමුව පිරිසිදු ශ්‍රිත: පිරිසිදු ශ්‍රිත පමණක් ලියන්න, කිසිවිටක පන්ති නොවේ
- ඊතල ශ්‍රිත `const funcName = () => {}` භාවිතා කරන්න සහ `function` මූල පදය භාවිතා නොකරන්න (ජනක යන්ත්‍ර හැර); ඔබට පරාමිති බැඳීමට .bind භාවිතා කළ හැකි නම්, ශ්‍රිත නිර්වචනය කිරීමෙන් වළකින්න
- කේත නැවත භාවිතය: නැවත භාවිතා කිරීමට අවධානය යොමු කරන්න, තවත් කුඩා කාර්යයන් උකහා ගන්න, සහ සමාන හෝ පිටපත්-අලවන ලද කේත ව්‍යුහයන් විශාල සංඛ්‍යාවක් අධිෂ්ඨානශීලීව වළක්වා ගන්න.
- වස්තු ප්‍රවේශය: අවශ්‍ය ගුණාංග උකහා ගැනීම සඳහා විනාශ කිරීමේ පැවරුම භාවිතා කිරීමට ප්‍රමුඛත්වය දෙන්න, ගැඹුරු සහ කැදලි ගතිලක්ෂණ වෙත ප්‍රවේශ වීමට අභ්‍යන්තරව තිත් නැවත නැවත භාවිතා කිරීමෙන් වළකින්න, සහ නැවත නැවතත් විකල්ප දාම විනිශ්චයන් ඒකාබද්ධ කරන්න
- ශ්‍රිත පරාමිති සඳහා වස්තු භාවිතා නොකරන්න, `{a,b,c}` ලියන්න එපා, `a, b, c` ලියන්න; බොහෝ විකල්ප පරාමිතීන් තිබේ නම්, භාවිතා කරන්න [[වින්‍යාස අයිතම ගණන, වින්‍යාස අයිතම], [වින්‍යාස අයිතම ගණන, වින්‍යාස අයිතම],..] මෙම සුසමාදර්ශයේ, වින්‍යාස අයිතම සංඛ්‍යාත්මක නියතයන් සමඟ අර්ථ දක්වා ඇත.
- බහු අගය ප්‍රතිලාභ සඳහා, `{a,b,c}` වෙනුවට `[a,b,c]` අරාව භාවිතා කරන්න. ප්‍රතිලාභ අගයන් 3කට වඩා තිබේ නම්, ස්ථානීය අර්ථය අර්ථ දැක්වීමට සංඛ්‍යාත්මක නියතයන් භාවිතා කරන්න.
- රාජ්‍යය නියෝජනය කිරීමට තන්තුවක් අවශ්‍ය නොවේ, රාජ්‍යය නිර්වචනය කිරීමට සංඛ්‍යාත්මක නියතයන් භාවිතා කරන්න
- ආයාත කිරීම හැර තන්තු අච්චුවක් (``) භාවිතා නොවේ, තන්තු සංයුක්ත කිරීම (+) භාවිතා වේ (Vite ස්ථිතික විශ්ලේෂණය සඳහා පහසු)
- loop සඳහා, ඔබට අනුක්‍රමික අංකයක් අවශ්‍ය නම්, `i++` වෙනුවට `++i` භාවිතා කරන්න.
- අනවශ්‍ය`?.` වළකින්න සහ ආරක්‍ෂක වැඩසටහන් තහනම් කරන්න
- විචල්‍ය නාමය බහු වචන වීමට අවශ්‍ය නොවන අතර ලැයිස්තුවක් දැක්වීමට `_li` සමඟ අවසන් වේ.

## නම් කිරීමේ සම්මුතිය

- නම් කිරීම අවමවාදය අනුගමනය කරයි. කෙටි නමුත් අර්ථවත් නම් භාවිතා කරන්න, උදාහරණයක් ලෙස: `remove` වෙනුවට `rm`, `delete`, `del` භාවිතා කරන්න. කෙසේ වෙතත්, ඔබ අන්තයට යාමෙන් වැළකිය යුතුය, උදාහරණයක් ලෙස: `map` වෙනුවට `m` එක අකුරක් භාවිතා නොකරන්න.
- ශ්‍රිත නාමකරණයේදී ක්‍රියාපද පමණක් භාවිතා කිරීමට උත්සාහ කරන්න. ඔබට එය ප්රකාශ කිරීමට එක් වචනයක් භාවිතා කළ හැකි නම්, වචන දෙකක් භාවිතා නොකරන්න. නාම පද ගොනු නාම වලින් පිළිබිඹු වේ. ඔබට ගොනු නාමයට ක්‍රියාපදයක් එක් කිරීමට අවශ්‍ය නම්, කරුණාකර නාම පදය පළමුව සහ ක්‍රියා පදය අවසන් කරන්න. උදාහරණයක් ලෙස: `setProfile.js` වෙනුවට `profileSet.js`
- විචල්‍ය නම්: යටි ලකුණු විලාසය (snake_case) භාවිතා කරන්න, උදාහරණයක් ලෙස `user_auth_token`
- කාර්යයේ නම: කුඩා අකුරු කැමල්කේස් (camelCase) භාවිතා කරන්න, උදාහරණයක් ලෙස `userData`
- ශ්‍රිත පරාමිති: එය ආපසු කැඳවීමේ ශ්‍රිතයක් නම්, `onChange` වැනි කුඩා ඔටුවන්ගේ අකුරින් එය නම් කරන්න
- මොඩියුල මට්ටමේ නියත අර්ථ දැක්වීම් `UPPER_SNAKE_CASE` ලොකු අකුරු යටි ඉරි විලාසය භාවිතා කරයි
- `get` වැනි අර්ථ විරහිත උපසර්ග ලියන්න එපා: `getCookie` වෙනුවට `cookieByHeader` ලියන්න
- ගෝලීය/මොඩියුල මට්ටමේ නියතයන්: ලොකු අකුරු යටි ඉරි විලාසය භාවිතා කරන්න (UPPER_SNAKE_CASE), උදා. `CODE_TO_ID`, `ID_TO_LANG`

## මොඩියුලර් යාන්ත්රණය

- ආයාත කිරීම: ඉල්ලුම මත නිවැරදිව ක්‍රියා කරයි, සම්පූර්ණ මොඩියුලයම සෘජුවම ආනයනය කිරීම තහනම් කරයි (`import * as x` හෝ විශාල වස්තු ආයාත කිරීමෙන් වළකින්න)
- අපනයනය: වස්තු අපනයනය අක්රිය කරන්න. `export const X=1, abc=()=>{};` වැනි ශ්‍රිත සහ විචල්‍ය ඒකක ලෙස අපනයනය කරන්න. අපනයනය කළ අන්තර්ගතය ප්‍රකාශ කිරීමට ඒවා const + කොමාවක් සමඟ ඒකාබද්ධ කිරීමට උත්සාහ කරන්න. ගොනුවකට ඇත්තේ එක් කාර්යයක් නම්, `export default` භාවිතා කරන්න
- ඔබට නිර්යාත පෙරනිමි ශ්‍රිතය අභ්‍යන්තරව ඇමතීමට අවශ්‍ය නම් මිස, ප්‍රථමයෙන් නියතයන් ප්‍රකාශ කිරීමෙන් සහ ගොනුව අවසානයේ ඒවා අපනයනය කිරීමෙන් වළකින්න.
- මාර්ග විභේදනය: වත්මන් බහලුම මාර්ගය ලබා ගැනීමේදී, `import.meta.dirname` භාවිතා කළ යුතුය
