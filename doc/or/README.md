- ** ଏକ ନୂତନ ଉପାଦାନ ସୃଷ୍ଟି କରନ୍ତୁ **: ଏକ ଉପାଦାନ ବିକାଶ କଙ୍କାଳ (ଡେମୋ ପୃଷ୍ଠା ଏବଂ ଆନ୍ତର୍ଜାତୀୟକରଣ ବିନ୍ୟାସକୁ ଅନ୍ତର୍ଭୂକ୍ତ କରି) ଶୀଘ୍ର ସୃଷ୍ଟି କରିବାକୁ <b ଶ୍ରେଣୀ = "କୋଡ୍"> ./ sh / new.js [css | js] [ଉପାଦାନ ନାମ] ` ଚଲାନ୍ତୁ |
- ** ସ୍ ent ାଧୀନ ତ୍ରୁଟି ନିବାରଣ **: ଭିଟ୍ ବିକାଶ ସର୍ଭର ଆରମ୍ଭ କରିବା ପାଇଁ ଏବଂ ସ୍ୱୟଂଚାଳିତ ଭାବରେ ବ୍ରାଉଜରରେ ଉପାଦାନର ଡେମୋ ପୃଷ୍ଠା ଖୋଲିବା ପାଇଁ <b ଶ୍ରେଣୀ = "କୋଡ୍"> ./ dev.js [ଉପାଦାନ ନାମ] ` (କିମ୍ବା `./dev.js` ଚଲାନ୍ତୁ) ଚଲାନ୍ତୁ | ଉପାଦାନ ହଟ ଅପଡେଟ୍ (HMR) କୁ ସମର୍ଥନ କରେ | କାରଣ ବିକାଶ ଏବଂ ଡାଟା ଲିଙ୍କଗୁଡିକ ଦ୍ୱିଗୁଣିତ ହୋଇଛି, ସେଠାରେ ଶୂନ ବିକାଶ ଫ୍ରଣ୍ଟ-ଏଣ୍ଡ ଲିଙ୍କ ଅଛି, ଯାହା ସମ୍ପୂର୍ଣ୍ଣ ସ୍ୱୟଂଚାଳିତ ବିକାଶ ଏବଂ ବ୍ରାଉଜର ତ୍ରୁଟି ନିବାରଣରେ ସାହାଯ୍ୟ କରିବା ପାଇଁ AI (ଯେପରିକି କ୍ଲୋଡ୍ କୋଡ୍, ଗୁଗୁଲ୍ ଆଣ୍ଟିଗ୍ରାଭିଟି ଇତ୍ୟାଦି) ପାଇଁ ଅତ୍ୟନ୍ତ ଉପଯୁକ୍ତ |

### ଉପାଦାନ ଲାଇବ୍ରେରୀ ପ୍ରକାଶ କରନ୍ତୁ |

#### webc dist

ପ୍ୟାକେଜିଂ ଏବଂ ପ୍ରକାଶନ ପ୍ରକ୍ରିୟା ଅତ୍ୟଧିକ ସ୍ୱୟଂଚାଳିତ:
- ** ଷ୍ଟାଟିକ୍ ରିସୋର୍ସ ପ୍ୟାକେଜିଂ **: `./sh/dist.js` (ଅର୍ଥାତ୍ `webc dist` ଟାସ୍କକୁ ଚଲାଇବା) ସ୍ୱୟଂଚାଳିତ ଭାବରେ ସମସ୍ତ ୱେବ୍ ପୃଷ୍ଠା ଉପାଦାନଗୁଡ଼ିକୁ `src/` ଅଧୀନରେ ସ୍କାନ୍ କରିପାରିବ, ସଂକଳନ, ସଙ୍କୋଚନ ଏବଂ `dist/` ଡିରେକ୍ଟୋରୀକୁ ବାହାର କରିପାରିବ | ଏହି ଡିରେକ୍ଟୋରୀରେ ଉତ୍ପାଦନ ବଣ୍ଟନ ପାଇଁ JS / CSS ଫାଇଲଗୁଡ଼ିକ, GitHub ପୃଷ୍ଠାଗୁଡ଼ିକ କିମ୍ବା କ୍ଲାଉଡଫ୍ଲେର୍ ପୃଷ୍ଠାଗୁଡ଼ିକରେ ନିୟୋଜନ ପାଇଁ ପ୍ରସ୍ତୁତ ଏକ ଡେମୋ ସାଇଟ୍ ରହିଥାଏ ଏବଂ ସ୍ୱୟଂଚାଳିତ ଭାବରେ `_redirects` ଏବଂ `404.html` ରାଉଟିଙ୍ଗ୍ ଫାଇଲଗୁଡିକ ସୃଷ୍ଟି କରିଥାଏ |
- ** NPM ପ୍ରକାଶନ **: `./sh/npmDist.js` ଚଲାନ୍ତୁ, ସ୍କ୍ରିପ୍ଟ ସ୍ୱୟଂଚାଳିତ ଭାବରେ ଉପାଦାନ ନିର୍ଭରଶୀଳତା ପରିଚାଳନା କରିବ, ପ୍ୟାକେଜ୍ ସଂସ୍କରଣ ସଂଖ୍ୟାକୁ ବୃଦ୍ଧି କରିବ ଏବଂ NPM ରେ ଉପାଦାନ ଲାଇବ୍ରେରୀ ପ୍ରକାଶ କରିବା ପାଇଁ ପ୍ୟାକେଜିଂ ସମାପ୍ତ ହେବା ପରେ ଗୋଟିଏ କ୍ଲିକ୍ ସହିତ `npm publish` ଏକଜେକ୍ୟୁଟ୍ କରିବ |

### ପ୍ଲାଟଫର୍ମ ଅନୁବାଦ ପାଇଁ ଦାୟୀ |

The ାଞ୍ଚାରେ ବିଲ୍ଟ-ଇନ୍ ସ୍ୱୟଂଚାଳିତ ଅନୁବାଦ ପ୍ରକ୍ରିୟା ଅଛି:
1. ପ୍ରୋଜେକ୍ଟ ରୁଟ୍ ଡିରେକ୍ଟୋରୀରେ `tran.yml` ରେ ଅନୁବାଦ ନିୟମଗୁଡିକ (ଯେପରିକି ଉତ୍ସ ଭାଷା, ଲକ୍ଷ୍ୟ ଭାଷା ଏବଂ ଅନୁବାଦ ହେବାକୁ ଥିବା ଡିରେକ୍ଟୋରୀ) ବିନ୍ୟାସ କରନ୍ତୁ |

3. `./sh/tran.js` ଚଲାନ୍ତୁ, ଏବଂ ଅନୁବାଦ ଇଞ୍ଜିନ୍ ସ୍ୱୟଂଚାଳିତ ଭାବରେ ପାଠ୍ୟ ବାହାର କରିବ ଏବଂ ପ୍ରକ୍ରିୟାକରଣ ପାଇଁ ୱେବ୍ ଅନୁବାଦ ସେବାକୁ ଡାକିବ, ସ୍ୱୟଂଚାଳିତ ଭାବରେ ବିଭିନ୍ନ ଭାଷାରେ YAML ଅନୁବାଦ ଏଣ୍ଟ୍ରି ଏବଂ README ଡକ୍ୟୁମେଣ୍ଟେସନ୍ ସୃଷ୍ଟି କରିବ |

### ପରିବେଶ ଭେରିଏବଲ୍ |

ସ୍ୱୟଂଚାଳିତ ଅନୁବାଦ କରିବାବେଳେ, ଅନୁବାଦ API କୁ ପ୍ରବେଶକୁ ଅନୁମତି ଦେବା ପାଇଁ ଆପଣଙ୍କୁ ନିମ୍ନଲିଖିତ ପରିବେଶ ଭେରିଏବଲ୍ ବିନ୍ୟାସ କରିବାକୁ ପଡିବ:

- ** `WEBC_SITE_TOKEN` ** (କିମ୍ବା ** `WEBC_TOKEN` **): `webc.site` ସ୍ୱୟଂଚାଳିତ ଅନୁବାଦ API କୁ ପ୍ରବେଶ କରିବାକୁ ପ୍ରାମାଣିକିକରଣ ଚାବି | ଏକ ପରିବେଶ ଭେରିଏବଲ୍ ଭାବରେ ଇଞ୍ଜେକ୍ସନ ଦିଆଯାଇପାରେ କିମ୍ବା ସ୍ଥାନୀୟ ବ୍ୟବହାରକାରୀଙ୍କ ହୋମ ଡିରେକ୍ଟୋରୀରେ `~/.config/webc.site.yml` ବିନ୍ୟାସ ଫାଇଲରେ `token` ଲେଖାଯାଇପାରିବ |
