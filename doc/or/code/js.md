# JS କୋଡ୍ ନିର୍ଦ୍ଦିଷ୍ଟକରଣ |

## କୋଡିଂ ଶ style ଳୀ

- ଅତି ଆଧୁନିକ js ବାକ୍ୟବିନ୍ୟାସ ବ୍ୟବହାର କରି ସରଳ, ଚମତ୍କାର ଏବଂ ଦକ୍ଷ |
- ଅତ୍ୟଧିକ ଲମ୍ବା କାର୍ଯ୍ୟଗୁଡ଼ିକୁ ବିଭାଜନ କର, ଏବଂ ଇଣ୍ଟରଫେସ୍ ଡିଜାଇନ୍ କମ୍ କପଲିଙ୍ଗ୍ ଏବଂ ଉଚ୍ଚ ସମନ୍ୱୟ ହେବା ଉଚିତ୍ |
- ଆରେ + ଲୁପ୍ / forEach / ମାନଚିତ୍ରକୁ ଅଧିକ ବ୍ୟବହାର କରନ୍ତୁ ଏବଂ କମ୍ ପୁନରାବୃତ୍ତି କୋଡ୍ ଲେଖନ୍ତୁ |
- ଅତ୍ୟଧିକ ମନ୍ତବ୍ୟକୁ ଏଡାଇବା ପାଇଁ ସରଳ କୋଡ଼ରେ ଫଙ୍କସନ୍ କମେଣ୍ଟ ଲେଖନ୍ତୁ ନାହିଁ | ବିଶୋଧିତ ଚାଇନିଜରେ ଜଟିଳ ତର୍କ ଏବଂ ବିଶେଷ ଆବଶ୍ୟକତା ବ୍ୟାଖ୍ୟା କରାଯାଇଛି |
- ଅନାବଶ୍ୟକତା ହ୍ରାସ କରିବା ପାଇଁ ବାରମ୍ବାର କୋଡ୍ କାର୍ଯ୍ୟରେ ଆବୃତ ହୋଇଛି | ଉଦାହରଣ ସ୍ୱରୂପ, `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()` ଭାବରେ ଲେଖାଯାଇପାରିବ |
- ଭେରିଏବଲ୍ ଘୋଷଣାନାମା: କ୍ରମାଗତ ଏକାଧିକ `const` ଘୋଷଣାନାମାକୁ ଏକ ଷ୍ଟେଟମେଣ୍ଟରେ ମିଶାନ୍ତୁ | ଏହାକୁ ତିନୋଟି ଧାଡିରେ ଲେଖିବା ପରିବର୍ତ୍ତେ `const a=1, b=2, c=3;` ଲେଖନ୍ତୁ |
- ଅସନ୍ତୁଳିତ ପ୍ରକ୍ରିୟାକରଣ: `await` ବ୍ୟବହାର କରନ୍ତୁ, `.then` ଶୃଙ୍ଖଳା କଲ୍ ବ୍ୟବହାରକୁ ବାରଣ କରନ୍ତୁ |
- ସ୍ୱୟଂଚାଳିତ ଭାବରେ ବ୍ୟତିକ୍ରମ ନିୟନ୍ତ୍ରଣ କୋଡ୍ ସୃଷ୍ଟି କରନ୍ତୁ ନାହିଁ, ସ୍ୱୟଂଚାଳିତ ଭାବରେ `try...catch` ଲେଖନ୍ତୁ ନାହିଁ (`try ... catch` ମାନୁଆଲୀ ଭାବରେ ପରିଚାଳିତ ହେଉଛି, ବିଦ୍ୟମାନ `try catch` ବିଲୋପ ହେବା ଉଚିତ ନୁହେଁ)
- ପ୍ରଥମେ ଶୁଦ୍ଧ କାର୍ଯ୍ୟଗୁଡ଼ିକ: କେବଳ ଶୁଦ୍ଧ କାର୍ଯ୍ୟଗୁଡ଼ିକ ଲେଖ, କେବେ କ୍ଲାସ୍ କର ନାହିଁ |
- ତୀର କାର୍ଯ୍ୟଗୁଡିକ `const funcName = () => {}` ବ୍ୟବହାର କରନ୍ତୁ ଏବଂ `function` କୀୱାର୍ଡ ବ୍ୟବହାର କରନ୍ତୁ ନାହିଁ (ଜେନେରେଟର ବ୍ୟତୀତ); ଯଦି ଆପଣ ପାରାମିଟରଗୁଡିକ ବାନ୍ଧିବା ପାଇଁ .bind ବ୍ୟବହାର କରିପାରିବେ, କାର୍ଯ୍ୟଗୁଡ଼ିକୁ ବ୍ୟାଖ୍ୟା କରନ୍ତୁ ନାହିଁ |
- କୋଡ୍ ପୁନ use ବ୍ୟବହାର: ପୁନ use ବ୍ୟବହାର ଉପରେ ଧ୍ୟାନ ଦିଅନ୍ତୁ, ଅଧିକ ଛୋଟ କାର୍ଯ୍ୟ ବାହାର କରନ୍ତୁ, ଏବଂ ବହୁ ସଂଖ୍ୟକ ସମାନ କିମ୍ବା କପି-ପେଷ୍ଟ କୋଡ୍ ସଂରଚନାକୁ ଦୃ resol ଭାବରେ ଏଡାନ୍ତୁ |
- ଅବଜେକ୍ଟ ଆକ୍ସେସ୍: ଆବଶ୍ୟକ ଗୁଣଗୁଡିକ ବାହାର କରିବା ପାଇଁ ବିନାଶକାରୀ ଆସାଇନମେଣ୍ଟ ବ୍ୟବହାରକୁ ପ୍ରାଥମିକତା ଦିଅନ୍ତୁ, ଗଭୀର ଏବଂ ନେଷ୍ଟେଡ୍ ଆଟ୍ରିବ୍ୟୁଟ୍ ଗୁଡିକୁ ପ୍ରବେଶ କରିବାକୁ ବାରମ୍ବାର ବିନ୍ଦୁ ବ୍ୟବହାର କରିବା ଠାରୁ ଦୂରେଇ ରୁହନ୍ତୁ ଏବଂ ବାରମ୍ବାର ଇଚ୍ଛାଧୀନ ଶୃଙ୍ଖଳା ବିଚାରକୁ ମିଶ୍ରଣ କରନ୍ତୁ |
- ଫଙ୍କସନ୍ ପାରାମିଟର ପାଇଁ ବସ୍ତୁ ବ୍ୟବହାର କରନ୍ତୁ ନାହିଁ, `{a,b,c}` ଲେଖନ୍ତୁ ନାହିଁ, `a, b, c` ଲେଖନ୍ତୁ; ଯଦି ସେଠାରେ ଅନେକ ବ al କଳ୍ପିକ ପାରାମିଟର ଅଛି, [[ବିନ୍ୟାସ ଆଇଟମ୍ ସଂଖ୍ୟା, ବିନ୍ୟାସ ଆଇଟମ୍], [ବିନ୍ୟାସ ଆଇଟମ୍ ସଂଖ୍ୟା, ବିନ୍ୟାସ ଆଇଟମ୍] ବ୍ୟବହାର କରନ୍ତୁ, ..] ଏହି ପାରାଦୀପରେ, ବିନ୍ୟାସ ଆଇଟମ୍ ଗୁଡିକ ସାଂଖ୍ୟିକ ସ୍ଥିର ସହିତ ବ୍ୟାଖ୍ୟା କରାଯାଇଛି |
- ମଲ୍ଟି ଭାଲ୍ୟୁ ରିଟର୍ନ ପାଇଁ, `{a,b,c}` ପରିବର୍ତ୍ତେ ଆରେ `[a,b,c]` ବ୍ୟବହାର କରନ୍ତୁ | ଯଦି 3 ରୁ ଅଧିକ ରିଟର୍ନ ଭାଲ୍ୟୁ ଅଛି, ସ୍ଥିତିର ଅର୍ଥ ବ୍ୟାଖ୍ୟା କରିବାକୁ ସାଂଖ୍ୟିକ ସ୍ଥିର ବ୍ୟବହାର କରନ୍ତୁ |
- ରାଜ୍ୟକୁ ପ୍ରତିନିଧିତ୍ୱ କରିବା, ରାଜ୍ୟକୁ ପରିଭାଷିତ କରିବା ପାଇଁ ସାଂଖ୍ୟିକ କନଷ୍ଟାଣ୍ଟ ବ୍ୟବହାର କରିବା ପାଇଁ କ string ଣସି ଷ୍ଟ୍ରିଙ୍ଗର ଆବଶ୍ୟକତା ନାହିଁ |
- କ import ଣସି ଷ୍ଟ୍ରିଙ୍ଗ୍ ଟେମ୍ପଲେଟ୍ (``) ବ୍ୟବହୃତ ହୁଏ ନାହିଁ, ଆମଦାନୀ ବ୍ୟତୀତ ଷ୍ଟ୍ରିଙ୍ଗ୍ କନକାଟେନେସନ୍ (+) ବ୍ୟବହୃତ ହୁଏ ନାହିଁ (ଭିଟ୍ ଷ୍ଟାଟିକ୍ ଆନାଲିସିସ୍ ପାଇଁ ସୁବିଧା)
- ଲୁପ୍ ପାଇଁ, ଯଦି ଆପଣ ଏକ କ୍ରମିକ ସଂଖ୍ୟା ଆବଶ୍ୟକ କରନ୍ତି, ତେବେ `i++` ପରିବର୍ତ୍ତେ `++i` ବ୍ୟବହାର କରନ୍ତୁ |
- ଅନାବଶ୍ୟକ `?.` ଠାରୁ ଦୂରେଇ ରୁହନ୍ତୁ ଏବଂ ପ୍ରତିରକ୍ଷା ପ୍ରୋଗ୍ରାମିଂକୁ ବାରଣ କରନ୍ତୁ |
- ଭେରିଏବଲ୍ ନାମ ବହୁବଚନ ହେବା ଆବଶ୍ୟକ କରେ ନାହିଁ, ଏବଂ ଏକ ତାଲିକା ସୂଚାଇବା ପାଇଁ `_li` ସହିତ ସମାପ୍ତ ହୁଏ |

## ନାମକରଣ ସମ୍ମିଳନୀ |

- ନାମକରଣ ସର୍ବନିମ୍ନତା ଅନୁସରଣ କରେ | କ୍ଷୁଦ୍ର କିନ୍ତୁ ଅର୍ଥପୂର୍ଣ୍ଣ ନାମ ବ୍ୟବହାର କରନ୍ତୁ, ଉଦାହରଣ ସ୍ୱରୂପ: `remove`, `delete`, `del` ପରିବର୍ତ୍ତେ `rm` ବ୍ୟବହାର କରନ୍ତୁ | ତଥାପି, ଆପଣ ମଧ୍ୟ ଅତ୍ୟଧିକ ପରିସ୍ଥିତିକୁ ଏଡ଼ାଇବା ଉଚିତ୍, ଉଦାହରଣ ସ୍ୱରୂପ: `map` ପରିବର୍ତ୍ତେ ଗୋଟିଏ ଅକ୍ଷର `m` ବ୍ୟବହାର କରନ୍ତୁ ନାହିଁ |
- ଫଙ୍କସନ୍ ନାମକରଣରେ କେବଳ କ୍ରିୟା ବ୍ୟବହାର କରିବାକୁ ଚେଷ୍ଟା କରନ୍ତୁ | ଯଦି ଆପଣ ଏହାକୁ ପ୍ରକାଶ କରିବାକୁ ଗୋଟିଏ ଶବ୍ଦ ବ୍ୟବହାର କରିପାରିବେ, ତେବେ ଦୁଇଟି ଶବ୍ଦ ବ୍ୟବହାର କରନ୍ତୁ ନାହିଁ | ବିଶେଷ୍ୟଗୁଡ଼ିକ ଫାଇଲ ନାମରେ ପ୍ରତିଫଳିତ ହୁଏ | ଯଦି ଆପଣ ଫାଇଲ୍ ନାମରେ ଏକ କ୍ରିୟା ଯୋଡିବାକୁ ଆବଶ୍ୟକ କରନ୍ତି, ଦୟାକରି ବିଶେଷ୍ୟକୁ ପ୍ରଥମେ ଏବଂ କ୍ରିୟାକୁ ଶେଷରେ ରଖନ୍ତୁ | ଉଦାହରଣ ସ୍ୱରୂପ: `setProfile.js` ପରିବର୍ତ୍ତେ `profileSet.js` |
- ପରିବର୍ତ୍ତନଶୀଳ ନାମ: ଅଣ୍ଡରସ୍କୋର ଶ style ଳୀ (ସାପ_କେସ୍) ବ୍ୟବହାର କରନ୍ତୁ, ଉଦାହରଣ ସ୍ୱରୂପ `user_auth_token` |
- କାର୍ଯ୍ୟ ନାମ: ଲୋୟର କେସ୍ ଓଟକେସ୍ (ଓଟ କେସ୍) ବ୍ୟବହାର କରନ୍ତୁ, ଉଦାହରଣ ସ୍ୱରୂପ `userData`
- ଫଙ୍କସନ୍ ପାରାମିଟରଗୁଡିକ: ଯଦି ଏହା ଏକ କଲବ୍ୟାକ୍ ଫଙ୍କସନ୍, ତେବେ ଏହାକୁ ଲୋୟର କେସ୍ ଓଟ କେସରେ ନାମ ଦିଅ, ଯେପରିକି `onChange`
- ମଡ୍ୟୁଲ୍ ସ୍ତରର ସ୍ଥିର ପରିଭାଷା ଅକ୍ଷର ଅକ୍ଷର ଅଣ୍ଡରଲାଇନ୍ ଶ style ଳୀ `UPPER_SNAKE_CASE` ବ୍ୟବହାର କରେ |
- `get` ଲେଖନ୍ତୁ ନାହିଁ, ଯେପରି ଅର୍ଥହୀନ ଉପସର୍ଗ, ଯେପରିକି: `getCookie` ପରିବର୍ତ୍ତେ `cookieByHeader` ଲେଖନ୍ତୁ |
- ଗ୍ଲୋବାଲ୍ / ମଡ୍ୟୁଲ୍-ଲେଭଲ୍ କନଷ୍ଟାଣ୍ଟସ୍: ବଡ ଅକ୍ଷର ଅଣ୍ଡରଲାଇନ୍ ଶ style ଳୀ (UPPER_SNAKE_CASE) ବ୍ୟବହାର କରନ୍ତୁ, ଯଥା `CODE_TO_ID`, `ID_TO_LANG` |

## ମଡ୍ୟୁଲାର୍ ମେକାନିଜିମ୍ |

- ଆମଦାନୀ: ଚାହିଦା ଅନୁଯାୟୀ କାର୍ଯ୍ୟଗୁଡିକ ସଠିକ୍ ଭାବରେ ଆମଦାନୀ କରନ୍ତୁ, ସମଗ୍ର ମଡ୍ୟୁଲ୍ ର ପ୍ରତ୍ୟକ୍ଷ ଆମଦାନୀକୁ ବାରଣ କରନ୍ତୁ (`import * as x` କିମ୍ବା ବଡ଼ ବସ୍ତୁ ଆମଦାନୀ କରନ୍ତୁ)
- ରପ୍ତାନି: ବସ୍ତୁର ରପ୍ତାନି ଅକ୍ଷମ କରନ୍ତୁ | ୟୁନିଟ୍ ଭାବରେ ଫଙ୍କସନ୍ ଏବଂ ଭେରିଏବଲ୍ ରପ୍ତାନି କରନ୍ତୁ, ଯେପରିକି `export const X=1, abc=()=>{};` | ରପ୍ତାନି ହୋଇଥିବା ବିଷୟବସ୍ତୁ ଘୋଷଣା କରିବାକୁ ସେମାନଙ୍କୁ ଏକ କନଷ୍ଟ + କମା ସହିତ ମିଶାଇବାକୁ ଚେଷ୍ଟା କରନ୍ତୁ | ଯଦି ଏକ ଫାଇଲ୍ ର କେବଳ ଗୋଟିଏ କାର୍ଯ୍ୟ ଅଛି, `export default` ବ୍ୟବହାର କରନ୍ତୁ |
- ଯଦି ଆପଣ ଆଭ୍ୟନ୍ତରୀଣ ରପ୍ତାନି ଡିଫଲ୍ଟ ଫଙ୍କସନ୍ କୁ ଡାକିବା ଆବଶ୍ୟକ କରନ୍ତି ନାହିଁ, ପ୍ରଥମେ କନଷ୍ଟାଣ୍ଟ ଘୋଷଣା କରିବା ଠାରୁ ଦୂରେଇ ରୁହନ୍ତୁ ଏବଂ ତାପରେ ଫାଇଲ୍ ଶେଷରେ ରପ୍ତାନି କରନ୍ତୁ |
- ପଥ ରେଜୋଲୁସନ: ସାମ୍ପ୍ରତିକ ଡିରେକ୍ଟୋରୀ ପଥ ପାଇବାବେଳେ, `import.meta.dirname` ବ୍ୟବହାର ହେବା ଜରୁରୀ |

## ଭୁଲ

- ଷ୍ଟ୍ରିଙ୍ଗ୍ ତ୍ରୁଟିରୁ ଦୂରେଇ ରୁହନ୍ତୁ ଏବଂ କ୍ରମାଗତ ତ୍ରୁଟି ସଂକେତ ଘୋଷଣା କରିବାକୁ const ବ୍ୟବହାର କରନ୍ତୁ |
- ଯଦି ତୁମେ ଭୁଲ ତଥ୍ୟ ସୂଚନା ଫେରସ୍ତ କରିବାକୁ ପଡିବ, [ତ୍ରୁଟି ସଂକେତ, ତ୍ରୁଟି ବାର୍ତ୍ତା, ...] ବ୍ୟବହାର କର, ଏହି ଉପାୟରେ, ତ୍ରୁଟି ସନ୍ଦେଶ ଏକ ପାଠ୍ୟ ବର୍ଣ୍ଣନା ନୁହେଁ, କିନ୍ତୁ ଏକ ସାଂଖ୍ୟିକ ମୂଲ୍ୟ (ଯେପରିକି [FILE_OVERSIZE, file_size, max_size])
