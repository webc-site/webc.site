# WebC.site

## ອັນນີ້ແມ່ນຫຍັງ?

### ນີ້ແມ່ນຫ້ອງສະໝຸດອົງປະກອບເວັບພື້ນເມືອງ

ພັດທະນາໂດຍອີງໃສ່ [ອົງປະກອບເວັບ](https://developer.mozilla.org/docs/Web/API/Web_components), ມັນເປັນ runtime ເອກະລາດແລະເຫມາະສົມສໍາລັບກອບທັງຫມົດ.

ປັບປຸງໃຫ້ເໝາະສົມທີ່ສຸດສຳລັບຂະໜາດ (ຂະໜາດໜຶ່ງສ່ວນສິບຂອງຫ້ອງສະໝຸດທີ່ມີຄວາມນິຍົມຄ້າຍຄືກັນ).

ຮອງຮັບ 75 ພາສາ. ທ່ານສາມາດນໍາເຂົ້າໂດຍກົງ js ແລະ css ສໍາລັບການນໍາໃຊ້ອອນໄລນ໌, ຫຼືເພີ່ມອົງປະກອບໃນໂຄງການຂອງທ່ານຕາມຄວາມຕ້ອງການສໍາລັບການພັດທະນາຂັ້ນສອງ.

<img src="https://i-01.eu.org/20260604114250265.avif" width="400">

### ນີ້​ແມ່ນ​ໂຄງ​ຮ່າງ​ການ​ພັດ​ທະ​ນາ​ຫ້ອງ​ສະ​ຫມຸດ​ອົງ​ປະ​ກອບ​

ນີ້ແມ່ນສະບັບບລັອກຄົງທີ່ຂອງ [storybook](https://storybook.js.org).

ການກໍ່ສ້າງຫ້ອງສະຫມຸດອົງປະກອບສາມາດປັບປຸງປະສິດທິພາບການພັດທະນາພາຍໃນຂອງບໍລິສັດແລະສ້າງຮູບພາບຂອງຍີ່ຫໍ້ໃນວົງການດ້ານວິຊາການ.

ອີງໃສ່ [webc.site](https://github.com/webc-site/webc.site), ທີມງານສາມາດສ້າງຫ້ອງສະໝຸດອົງປະກອບຂອງຕົນເອງໄດ້ຢ່າງງ່າຍດາຍ ແລະ ເຜີຍແຜ່ພວກມັນໄປໃສ່ [github page](https://pages.github.com), [cloudflare page](https://pages.cloudflare.com), ແລະອື່ນໆ. ອົງປະກອບທີ່ເຜີຍແຜ່ສາມາດຖືກແກ້ໄຂໄດ້ທາງອອນລາຍໃນ [codepen.io](https://codepen.io) ແລະ [jsfiddle](https://jsfiddle.net).

ໃນເວລາດຽວກັນ, ກອບການແປພາສາທີ່ສ້າງຂຶ້ນສາມາດຮັບຮູ້ການເປັນສາກົນຂອງອົງປະກອບແລະເອກະສານຢ່າງໄວວາ, ດັ່ງນັ້ນຜູ້ຊົມຜະລິດຕະພັນຂອງທ່ານບໍ່ຖືກຜູກມັດດ້ວຍພາສາ.

### ນີ້​ແມ່ນ​ຮູບ​ແບບ​ໃໝ່​ຂອງ​ການ​ພັດ​ທະ​ນາ​ດ້ານ​ໜ້າ​ສຳ​ລັບ​ຍຸກ​ສະ​ໄໝ​ປັນຍາ​ປະດິດ

ກອບແມ່ນສ້າງຂຶ້ນໃນ `.agents/skills`, ອະນຸຍາດໃຫ້ປັນຍາປະດິດຊ່ວຍໃຫ້ທ່ານພັດທະນາອົງປະກອບໃຫມ່ດ້ວຍການຄລິກດຽວຕາມການປະຕິບັດທີ່ດີທີ່ສຸດ.

<img src="https://i-01.eu.org/20260604121408079.avif" width="400">

ໃຫ້ລະຫັດດ້ານຫນ້າຂອງການປ່ຽນແປງຈາກພູເຂົາຂີ້ເຫຍື້ອທີ່ບໍ່ຍືນຍົງຂອງການຂຽນໂປລແກລມບັນຍາກາດເຂົ້າໄປໃນພາກສ່ວນມາດຕະຖານທີ່ສາມາດຮັກສາໄດ້ແລະພັດທະນາຢ່າງຕໍ່ເນື່ອງໃນໄລຍະຍາວ.

ໃນທັດສະນະຂອງຄຸນລັກສະນະຂອງການພັດທະນາການຊ່ວຍເຫຼືອທາງປັນຍາທຽມ, ພວກເຮົາໄດ້ສະເຫນີແລະອອກແບບຮູບແບບການພັດທະນາດ້ານຫນ້າໃຫມ່:

1. ** ທຸກ​ສິ່ງ​ທຸກ​ຢ່າງ​ແມ່ນ​ອົງ​ປະ​ກອບ​ຂອງ​ເວັບ **
2. **ອົງປະກອບຕ້ອງເປັນເອກະລາດຂອງການເຊື່ອມຕໍ່ຂໍ້ມູນ back-end, ແລະສ່ວນຕິດຕໍ່ຂໍ້ມູນຖືກເປີດເຜີຍເປັນຟັງຊັນໂທກັບ**

ດ້ວຍວິທີນີ້, ປັນຍາປະດິດສາມາດສ້າງຂໍ້ມູນປອມແລະສະແດງສະຖານະທີ່ແຕກຕ່າງກັນຂອງອົງປະກອບໃນຫນ້າຕົວຢ່າງ.

ການ​ແບ່ງ​ແຍກ​ການ​ພັດ​ທະ​ນາ​ອົງ​ປະ​ກອບ​ແລະ​ການ​ໂຕ້​ຕອບ​ຂໍ້​ມູນ​, ລວມ​ກັບ [webc.site ອົງ​ປະ​ກອບ​ກອບ​](/0​) ຄຸນ​ນະ​ສົມ​ບັດ​ທີ່​ສໍາ​ຄັນ​ອື່ນ​:

**ແຕ່ລະອົງປະກອບສາມາດເລີ່ມເຊີບເວີການພັດທະນາຢ່າງເປັນອິດສະຫຼະ ແລະພັດທະນາ ແລະດີບັກໃນໜ້າສາທິດ**.

ອັນນີ້ຊ່ວຍຫຼຸດການເຊື່ອມຕໍ່ດ້ານໜ້າສຳລັບການພັດທະນາ ແລະ ການດີບັກຂອງແຕ່ລະອົງປະກອບເປັນ `0`.

ດ້ວຍວິທີນີ້, ເຄື່ອງມືພັດທະນາທາງດ້ານປັນຍາປະດິດ ([Claude Code](https://code.claude.com), [OpenAI Codex](https://openai.com), [Google Antigravity](https://antigravity.google), [Cursor](https://cursor.com), [Windsurf](https://windsurf.com), [Devin](https://devin.ai), [OpenHands](https://github.com/All-Hands-AI/OpenHands) ແລະອື່ນໆ), ທ່ານສາມາດເພີດເພີນໄປກັບການພັດທະນາແບບອັດຕະໂນມັດເຕັມທີ່, ບໍ່ມີການປິດບຼາວເຊີ (Browser) ອີກຕໍ່ໄປ, ບໍ່ມີການປິດກັ້ນຂໍ້ມູນໃດໆ ອີກຕໍ່ໄປ. ສະຖານະເຂົ້າສູ່ລະບົບ, ເສັ້ນທາງເສັ້ນທາງ, ແລະອື່ນໆ).

ຖ້າທ່ານຕ້ອງການຄວບຄຸມປັນຍາປະດິດແລະຫຼຸດຜ່ອນຄ່າໃຊ້ຈ່າຍແລະເພີ່ມປະສິດທິພາບ, ທ່ານຕ້ອງການຕົວແບບໃຫມ່ທີ່ຫຼຸດຜ່ອນການພັດທະນາແລະການແກ້ໄຂສະພາບແວດລ້ອມຂອງແຕ່ລະອົງປະກອບ.

# ຂ້ອຍຍັງບໍ່ໄດ້ເລີ່ມຂຽນ TODO ເທື່ອ

### ຫ້ອງສະຫມຸດອົງປະກອບເວັບ

#### ການປັບແຕ່ງຂະໜາດສູງສຸດ

ຕົວຢ່າງ, ສໍາລັບອົງປະກອບແຖບເລື່ອນສະເໝືອນ, ໃຫ້ປຽບທຽບຂະໜາດຂອງສະຄຣິບ [OverlayScrollbars](https://github.com/KingSora/OverlayScrollbars) ທີ່ນິຍົມ ແລະຮູບແບບຫຼັງ `gz`, ດັ່ງທີ່ສະແດງຢູ່ໃນຕາຕະລາງຕໍ່ໄປນີ້:

|                   |   js    |  css   |     total      |
| :---------------- | :-----: | :----: | :------------: |
| overlayscrollbars | 14520 B | 2584 B | 17104 B (9.1X) |
| Scroll            | 1159 B  | 721 B  |   ⭐ 1880 ບ    |

ອົງປະກອບແຖບເລື່ອນສະເໝືອນຈິງຂອງ `webc.site`ແມ່ນພຽງແຕ່ `OverlayScrollbars`ຂອງ `10%`.

#### Runtime ເອກະລາດ

ສ້າງຂຶ້ນຕາມມາດຕະຖານ [Web Components](https://developer.mozilla.org/docs/Web/API/Web_components), ຫ້ອງສະໝຸດອົງປະກອບຂອງ webc.site ແມ່ນ runtime-agnostic ຕາມທໍາມະຊາດ. ບໍ່ວ່າທ່ານກໍາລັງໃຊ້ React, Vue, Angular, Svelte, ຫຼືໂຄງການ HTML / JS ບໍລິສຸດແບບດັ້ງເດີມ, ອົງປະກອບທັງຫມົດສາມາດດໍາເນີນການໂດຍກົງໃນຕົວທ່ອງເວັບເຊັ່ນແທັກ HTML ພື້ນເມືອງ, ບໍ່ຈໍາເປັນຕ້ອງມີການຫຸ້ມຫໍ່ຮອງສໍາລັບກອບສະເພາະ.

##### ການປຽບທຽບກັບ shadcn

[shadcn/ui](https://ui.shadcn.com) ຮັບຮອງເອົາຮູບແບບການແຈກຢາຍລະຫັດ, ເຊິ່ງຮຽກຮ້ອງໃຫ້ທ່ານຄັດລອກລະຫັດອົງປະກອບໂດຍກົງເຂົ້າໃນໂຄງການຂອງທ່ານເອງ. ມັນໄດ້ຖືກຜູກມັດຢ່າງແຫນ້ນຫນາກັບລະບົບນິເວດ React ແລະຕ່ອງໂສ້ເຄື່ອງມືການກໍ່ສ້າງສະເພາະ (ເຊັ່ນ: Tailwind CSS), ແລະບໍ່ສາມາດຖືກນໍາໃຊ້ໃນທົ່ວກອບ. ມັນຍັງມີຄວາມຕ້ອງການສະເພາະໃດຫນຶ່ງສໍາລັບການກໍາຫນົດຄ່າວິສະວະກໍາໂຄງການ.

ແລະອົງປະກອບຂອງ `webc.site`:
- **ງານຂ້າມຂອບແບບທົ່ວໆໄປ**: ຂຽນຄັ້ງດຽວ ແລະສາມາດແນະນຳໄດ້ໂດຍກົງ ແລະເຮັດວຽກໃນກອບໜ້າວຽກໃດໆກໍຕາມ (ແມ້ແຕ່ສະພາບແວດລ້ອມທີ່ບໍ່ມີກອບ).
- ** Environment zero dependency**: ບໍ່ຈຳເປັນຕ້ອງມີເຄື່ອງມືການຫຸ້ມຫໍ່ສະເພາະ ຫຼື CSS preprocessors, ພຽງແຕ່ນຳເຂົ້າໄຟລ໌ຄົງທີ່ທີ່ລວບລວມແລ້ວກໍສາມາດໃຊ້ອອກຈາກກ່ອງໄດ້.

## ການພັດທະນາອັດຕະໂນມັດແລະການດີບັກ

### ການພັດທະນາທ້ອງຖິ່ນແລະການດີບັກ

ແຕ່ລະອົງປະກອບຂອງຫນ້າເວັບສະຫນັບສະຫນູນການເລີ່ມຕົ້ນເຄື່ອງແມ່ຂ່າຍການພັດທະນາເປັນເອກະລາດ, ສະຫນອງການພັດທະນາຫນ້ອຍທີ່ສຸດແລະສະພາບແວດລ້ອມດີບັກ:

- **ສ້າງອົງປະກອບໃໝ່**: ແລ່ນ `./sh/new.js [css|js] [component name]` ເພື່ອສ້າງໂຄງກະດູກການພັດທະນາອົງປະກອບ (ລວມທັງໜ້າຕົວຢ່າງ ແລະການຕັ້ງຄ່າສາກົນ).
- **ການດີບັ໊ກແບບອິດສະລະ**: ແລ່ນ `./dev.js [ຊື່ອົງປະກອບ]` (ຫຼືແລ່ນ `./dev.js`ໂດຍກົງ) ເພື່ອເລີ່ມຕົ້ນເຊີບເວີພັດທະນາ Vite ແລະເປີດໜ້າສາທິດຂອງອົງປະກອບໃນບຣາວເຊີໂດຍອັດຕະໂນມັດ. ຮອງຮັບການອັບເດດຮ້ອນອົງປະກອບ (HMR). ເນື່ອງຈາກວ່າການພັດທະນາແລະການເຊື່ອມໂຍງຂໍ້ມູນຖືກ decoupled, ມີສູນການພັດທະນາເຊື່ອມຕໍ່ທາງຫນ້າ, ເຊິ່ງເຫມາະສົມຫຼາຍສໍາລັບ AI (ເຊັ່ນ: Claude Code, Google Antigravity, ແລະອື່ນໆ) ເພື່ອຊ່ວຍໃນການພັດທະນາອັດຕະໂນມັດຢ່າງເຕັມສ່ວນແລະການດີບັກຂອງຕົວທ່ອງເວັບ.

### ເຜີຍແຜ່ຫ້ອງສະໝຸດອົງປະກອບ

#### webc dist

ຂະ​ບວນ​ການ​ຫຸ້ມ​ຫໍ່​ແລະ​ການ​ພິມ​ເຜີຍ​ແຜ່​ແມ່ນ​ອັດ​ຕະ​ໂນ​ມັດ​ສູງ​:
- **ການຫຸ້ມຫໍ່ຊັບພະຍາກອນແບບຄົງທີ່**: ແລ່ນ `./sh/dist.js` (ນັ້ນຄືການດໍາເນີນການ `webc dist`ໜ້າວຽກ) ສາມາດສະແກນທຸກອົງປະກອບຂອງໜ້າເວັບພາຍໃຕ້ `src/`, ລວບລວມ, ບີບອັດ ແລະສົ່ງພວກມັນອອກໄປຫາໄດເລກະທໍລີ `dist/`. ໄດເຣັກທໍຣີນີ້ມີໄຟລ໌ JS/CSS ສຳລັບການແຈກຢາຍການຜະລິດ, ເວັບໄຊສາທິດພ້ອມນຳໃຊ້ໃຫ້ກັບ GitHub Pages ຫຼື Cloudflare Pages, ແລະສ້າງໄຟລ໌ `_redirects` ແລະ `404.html` ກຳນົດເສັ້ນທາງໂດຍອັດຕະໂນມັດ.
- ** NPM Publish**: ແລ່ນ `./sh/npmDist.js` , ສະຄຣິບຈະຈັດການການຂຶ້ນກັບອົງປະກອບອັດຕະໂນມັດ, ເພີ່ມເລກເວີຊັ່ນຂອງແພັກເກັດ, ແລະດໍາເນີນການ `npm publish`ດ້ວຍການຄລິກດຽວຫຼັງຈາກການຫຸ້ມຫໍ່ສໍາເລັດເພື່ອເຜີຍແຜ່ຫ້ອງສະໝຸດອົງປະກອບໄປຍັງ NPM.

### ເວທີແມ່ນຮັບຜິດຊອບສໍາລັບການແປພາສາ

ກອບມີຂະບວນການແປພາສາອັດຕະໂນມັດໃນຕົວ:
1. ຕັ້ງຄ່າກົດລະບຽບການແປ (ເຊັ່ນ: ພາສາຕົ້ນສະບັບ, ພາສາເປົ້າໝາຍ ແລະໄດເຣັກທໍຣີທີ່ຈະແປ) ໃນ `tran.yml`ໃນລະບົບຮາກໂຄງການ.
2. ໃຊ້ຕົວຍຶດໃນອົງປະກອບຫຼືເອກະສານໂດຍບໍ່ຈໍາເປັນຕ້ອງ hardcode ຫຼາຍພາສາໃນລະຫັດຂອງທ່ານ.
3. ດໍາເນີນການ `./sh/tran.js`, ແລະເຄື່ອງຈັກການແປພາສາອັດຕະໂນມັດຈະສະກັດຂໍ້ຄວາມແລະໂທຫາບໍລິການແປພາສາ WebC ສໍາລັບການປະມວນຜົນ, ອັດຕະໂນມັດສ້າງລາຍການແປພາສາ YAML ແລະເອກະສານ README ໃນພາສາຕ່າງໆ.

### ຕົວແປສະພາບແວດລ້ອມ

ເມື່ອດໍາເນີນການແປພາສາອັດຕະໂນມັດ, ທ່ານຈໍາເປັນຕ້ອງກໍານົດຕົວແປສະພາບແວດລ້ອມຕໍ່ໄປນີ້ເພື່ອອະນຸຍາດໃຫ້ເຂົ້າເຖິງ API ການແປພາສາ:

- **`WEBC_SITE_TOKEN`** (ຫຼື **`WEBC_TOKEN`**): ກະແຈການພິສູດຢືນຢັນທີ່ໃຊ້ເພື່ອເຂົ້າເຖິງ `webc.site` API ການແປອັດຕະໂນມັດ. ສາມາດຖືກສີດເປັນຕົວແປສະພາບແວດລ້ອມ ຫຼືຂຽນ `token`ໃນ `~/.config/webc.site.yml`ໄຟລ໌ການຕັ້ງຄ່າຢູ່ໃນໄດເລກະທໍລີເຮືອນຂອງຜູ້ໃຊ້ໃນທ້ອງຖິ່ນ.
