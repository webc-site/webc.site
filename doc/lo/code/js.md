# ຂໍ້ກໍາຫນົດລະຫັດ JS

## ຮູບແບບການຂຽນລະຫັດ

- ງ່າຍດາຍ, elegant ແລະປະສິດທິພາບ, ການນໍາໃຊ້ syntax js ທີ່ທັນສະໄຫມທີ່ສຸດ
- ແບ່ງ​ປະ​ຕິ​ບັດ​ຫນ້າ​ທີ່​ຍາວ​ເກີນ​ໄປ​, ແລະ​ການ​ອອກ​ແບບ​ການ​ໂຕ້​ຕອບ​ຄວນ​ຈະ​ເປັນ​ການ​ເຊື່ອມ​ຕໍ່​ຕ​່​ໍ​າ​ແລະ​ຄວາມ​ເປັນ​ກັນ​ສູງ​.
- ໃຊ້ arrays + loops/forEach/map ຫຼາຍຂຶ້ນ ແລະຂຽນລະຫັດຊໍ້າກັນໜ້ອຍລົງ
- ຢ່າຂຽນຄໍາຄິດເຫັນຫນ້າທີ່ໃນລະຫັດງ່າຍໆເພື່ອຫຼີກເວັ້ນການສະແດງຄວາມຄິດເຫັນຫຼາຍເກີນໄປ. ເຫດຜົນທີ່ສັບສົນ ແລະຄວາມຕ້ອງການພິເສດແມ່ນໄດ້ຖືກອະທິບາຍເປັນພາສາຈີນທີ່ຫລອມໂລຫະ.
- ລະຫັດຊ້ໍາຊ້ອນແມ່ນບໍ່ມີຕົວຕົນເຂົ້າໄປໃນຫນ້າທີ່ເພື່ອຫຼຸດຜ່ອນການຊ້ໍາຊ້ອນ. ຕົວຢ່າງ, `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })`ສາມາດຂຽນເປັນ `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()`
- ການປະກາດຕົວແປ: ສົມທົບການປະກາດຫຼາຍອັນຕິດຕໍ່ກັນ `const` ເຂົ້າໄປໃນຄໍາຖະແຫຼງດຽວ. ແທນທີ່ຈະຂຽນເປັນສາມແຖວ, ຂຽນ `const a=1, b=2, c=3;`
- ການປະມວນຜົນ Asynchronous: ໃຊ້ `await`, ຫ້າມການໃຊ້ `.then`ການໂທລະບົບຕ່ອງໂສ້
- ຢ່າສ້າງລະຫັດການຈັດການຂໍ້ຍົກເວັ້ນໂດຍອັດຕະໂນມັດ, ຢ່າຂຽນອັດຕະໂນມັດ `try...catch` (`try ... catch`ຖືກຮັກສາໄວ້ດ້ວຍຕົນເອງ, ທີ່ມີຢູ່ແລ້ວ `try catch`ບໍ່ຄວນຖືກລຶບ)
- ຫນ້າທີ່ບໍລິສຸດທໍາອິດ: ພຽງແຕ່ຂຽນຫນ້າທີ່ບໍລິສຸດ, ບໍ່ເຄີຍຫ້ອງຮຽນ
- ໃຊ້ຟັງຊັນລູກສອນ `const funcName = () => {}` ແລະບໍ່ໃຊ້ຄໍາສໍາຄັນ `function` (ຍົກເວັ້ນເຄື່ອງສ້າງ); ຖ້າທ່ານສາມາດນໍາໃຊ້ .bind ເພື່ອຜູກມັດພາລາມິເຕີ, ຫຼີກເວັ້ນການກໍານົດຫນ້າທີ່
- ການໃຊ້ລະຫັດຄືນໃຫມ່: ເອົາໃຈໃສ່ກັບການນໍາໃຊ້ຄືນໃຫມ່, ສະກັດເອົາຫນ້າທີ່ຂະຫນາດນ້ອຍຫຼາຍ, ແລະແກ້ໄຂຢ່າງແນ່ນອນເພື່ອຫຼີກເວັ້ນການຈໍານວນຂະຫນາດໃຫຍ່ຂອງໂຄງສ້າງລະຫັດທີ່ຄ້າຍຄືກັນຫຼືຄັດລອກ.
- ການເຂົ້າເຖິງວັດຖຸ: ຈັດລຳດັບຄວາມສຳຄັນໂດຍນຳໃຊ້ການກຳນົດໂຄງສ້າງເພື່ອແຍກຄຸນລັກສະນະທີ່ຕ້ອງການ, ຫຼີກເວັ້ນການໃຊ້ຈຸດພາຍໃນຊ້ຳໆເພື່ອເຂົ້າເຖິງຄຸນລັກສະນະທີ່ເລິກເຊິ່ງ ແລະ ຊ້ອນກັນ, ແລະຮວມການຕັດສິນລະບົບຕ່ອງໂສ້ທາງເລືອກຊ້ຳໆ.
- ຢ່າໃຊ້ວັດຖຸສໍາລັບພາລາມິເຕີຟັງຊັນ, ຢ່າຂຽນ `{a,b,c}`, ຂຽນ `a, b, c`; ຖ້າມີຕົວກໍານົດການທາງເລືອກຫຼາຍ, ໃຫ້ໃຊ້ [[ຈໍານວນລາຍການການຕັ້ງຄ່າ, ລາຍການການຕັ້ງຄ່າ], [ຈໍານວນລາຍການການຕັ້ງຄ່າ, ລາຍການການຕັ້ງຄ່າ], ..] ໃນພາລາມິເຕີນີ້, ລາຍການການຕັ້ງຄ່າແມ່ນຖືກກໍານົດດ້ວຍຄ່າຄົງທີ່ຕົວເລກ.
- ສຳລັບຜົນຕອບແທນຫຼາຍຄ່າ, ໃຫ້ໃຊ້ array `[a,b,c]`ແທນ `{a,b,c}`. ຖ້າມີຄ່າຕອບແທນຫຼາຍກວ່າ 3, ໃຫ້ໃຊ້ຄ່າຄົງທີ່ຕົວເລກເພື່ອກໍານົດຄວາມໝາຍຂອງຕຳແໜ່ງ.
- ບໍ່ຈໍາເປັນຕ້ອງມີສະຕຣິງເພື່ອເປັນຕົວແທນຂອງລັດ, ໃຊ້ຄ່າຄົງທີ່ຕົວເລກເພື່ອກໍານົດລັດ
- ບໍ່ມີແມ່ແບບສະຕຣິງ (``) ຖືກໃຊ້, ການເຊື່ອມສະຕຣິງ (+) ຖືກໃຊ້, ຍົກເວັ້ນການນໍາເຂົ້າ (ສະດວກສໍາລັບການວິເຄາະສະຖິດ Vite)
- ສໍາລັບ loop, ຖ້າທ່ານຕ້ອງການເລກລໍາດັບ, ໃຊ້ `++i` ແທນ `i++`
- ຫຼີກເວັ້ນການທີ່ບໍ່ຈໍາເປັນ`?.` ແລະຫ້າມການຂຽນໂປລແກລມປ້ອງກັນ
- ຊື່ຕົວແປບໍ່ຈໍາເປັນຕ້ອງເປັນພະຫຸພົດ, ແລະລົງທ້າຍດ້ວຍ `_li`ເພື່ອຊີ້ບອກລາຍຊື່.

## ສົນທິສັນຍາການຕັ້ງຊື່

- ການ​ຕັ້ງ​ຊື່​ດໍາ​ເນີນ​ການ​ຫນ້ອຍ​. ໃຊ້ຊື່ສັ້ນແຕ່ມີຄວາມໝາຍ, ຕົວຢ່າງ: ໃຊ້ `rm`ແທນ `remove`, `delete`, `del`. ແນວໃດກໍ່ຕາມ, ເຈົ້າຄວນຫຼີກລ່ຽງການໄປຈົນເກີນໄປ, ຕົວຢ່າງ: ຢ່າໃຊ້ຕົວອັກສອນດຽວ `m`ແທນ `map`
- ພະຍາຍາມໃຊ້ພຽງແຕ່ຄໍາກິລິຍາໃນການຕັ້ງຊື່ຟັງຊັນ. ຖ້າທ່ານສາມາດນໍາໃຊ້ຄໍາຫນຶ່ງເພື່ອສະແດງອອກ, ຢ່າໃຊ້ສອງຄໍາ. ຄໍານາມແມ່ນສະທ້ອນໃຫ້ເຫັນໃນຊື່ໄຟລ໌. ຖ້າທ່ານຕ້ອງການເພີ່ມ verb ໃສ່ຊື່ໄຟລ໌, ກະລຸນາໃສ່ຄໍານາມກ່ອນແລະຄໍານາມສຸດທ້າຍ. ຕົວຢ່າງ: `profileSet.js` ແທນ `setProfile.js`
- ຊື່ຕົວແປ: ໃຊ້ແບບຂີດກ້ອງ (snake_case), ຕົວຢ່າງ `user_auth_token`; ຖ້າຕົວແປເປັນຟັງຊັນ, ໃຊ້ຮູບແບບການຕັ້ງຊື່ຕົວພິມນ້ອຍ
- ຊື່ຟັງຊັນ: ໃຊ້ຕົວພິມນ້ອຍ camelCase (camelCase), ຕົວຢ່າງ `userData`
- ພາລາມິເຕີຟັງຊັນ: ຖ້າມັນເປັນການເອີ້ນຄືນຟັງຊັນ, ຕັ້ງຊື່ມັນເປັນຕົວພິມນ້ອຍ, ເຊັ່ນ: `onChange`
- ຄໍານິຍາມຄົງທີ່ລະດັບໂມດູນໃຊ້ຮູບແບບການຂີດກ້ອງຕົວພິມໃຫຍ່ `UPPER_SNAKE_CASE`
- ຢ່າຂຽນ `get`, ຄໍານໍາໜ້າທີ່ບໍ່ມີຄວາມຫມາຍເຊັ່ນ: ຂຽນ `cookieByHeader`ແທນ `getCookie`
- ຄົງທີ່ລະດັບໂລກ/ໂມດູນ: ໃຊ້ຮູບແບບການຂີດກ້ອງຕົວພິມໃຫຍ່ (UPPER_SNAKE_CASE), ເຊັ່ນ. `CODE_TO_ID`, `ID_TO_LANG`

## ກົນໄກໂມດູນ

- ການນໍາເຂົ້າ: ຫນ້າທີ່ນໍາເຂົ້າຢ່າງຖືກຕ້ອງຕາມຄວາມຕ້ອງການ, ຫ້າມການນໍາເຂົ້າໂດຍກົງຂອງໂມດູນທັງຫມົດ (ຫຼີກເວັ້ນການ `import * as x` ຫຼືນໍາເຂົ້າວັດຖຸຂະຫນາດໃຫຍ່)
- ສົ່ງອອກ: ປິດການສົ່ງອອກວັດຖຸ. ສົ່ງອອກຟັງຊັນ ແລະຕົວແປເປັນຫົວໜ່ວຍ, ເຊັ່ນ `export const X=1, abc=()=>{};`. ພະຍາຍາມສົມທົບພວກມັນດ້ວຍ const + comma ເພື່ອປະກາດເນື້ອຫາທີ່ສົ່ງອອກ. ຖ້າໄຟລ໌ມີຟັງຊັນດຽວ, ໃຫ້ໃຊ້ `export default`
- ເວັ້ນເສຍແຕ່ວ່າທ່ານຕ້ອງການເອີ້ນຟັງຊັນມາດຕະຖານການສົ່ງອອກພາຍໃນ, ຫຼີກເວັ້ນການປະກາດຄ່າຄົງທີ່ກ່ອນແລະຫຼັງຈາກນັ້ນສົ່ງອອກພວກມັນໃນຕອນທ້າຍຂອງໄຟລ໌.
- ການແກ້ໄຂເສັ້ນທາງ: ເມື່ອໄດ້ຮັບເສັ້ນທາງໄດເລກະທໍລີປະຈຸບັນ, `import.meta.dirname`ຕ້ອງຖືກນໍາໃຊ້
