# JSコード仕様

## コーディングスタイル

- 最新の js 構文を使用した、シンプル、エレガント、効率的
- 過度に長い機能を分割し、インターフェイス設計は結合度が低く、凝集度が高いものにする必要があります。
- 配列 + ループ/forEach/map をより多く使用し、繰り返しの少ないコードを作成する
- 過剰なコメントを避けるために、単純なコード内に関数コメントを記述しないでください。複雑なロジックや特殊なニーズを洗練された中国語で説明します。
- 繰り返されるコードは関数に抽象化され、冗長性が削減されます。たとえば、`cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` は `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()` と書くことができます。
- 変数宣言: 複数の連続する`const`宣言を 1 つのステートメントに結合します。 3 行で書く代わりに`const a=1, b=2, c=3;`と書きます。
- 非同期処理: `await` を使用し、`.then` チェーン呼び出しの使用を禁止します。
- 例外処理コードを自動的に生成せず、`try...catch` を自動的に記述しません (`try ... catch` は手動で維持され、既存の `try catch` は削除しないでください)。
- 純粋な関数を最初に作成します。純粋な関数のみを記述し、クラスは作成しません。
- アロー関数 `const funcName = () => {}` を使用し、 `function` キーワードは使用しないでください (ジェネレータを除く)。 .bind を使用してパラメータをバインドできる場合は、関数の定義を避けてください
- コードの再利用: 再利用に注意し、より小さな機能を抽出し、多数の類似したコード構造やコピー＆ペーストされたコード構造を断固として避けます。
- オブジェクト アクセス: 必要な属性を抽出するための分割代入の使用を優先し、深くネストされた属性にアクセスするために内部でドットを繰り返し使用することを回避し、繰り返されるオプションのチェーン判断をマージします。
- 関数パラメータにはオブジェクトを使用しないでください。`{a,b,c}` と書かず、`a, b, c` と書きます。オプションのパラメーターが多数ある場合は、[[構成アイテムの数, 構成アイテム], [構成アイテムの数, 構成アイテム],..] を使用します。 このパラダイムでは、構成アイテムは数値定数で定義されます。
- 複数の値を返す場合は、`{a,b,c}` の代わりに配列 `[a,b,c]` を使用します。戻り値が 3 つを超える場合は、数値定数を使用して位置の意味を定義します。
- 状態を表す文字列は必要ありません。状態を定義するには数値定数を使用してください。
- インポートを除き、文字列テンプレート (``) は使用されず、文字列連結 (+) が使用されます (Vite 静的分析に便利)
- ループの場合、シリアル番号が必要な場合は、`i++` の代わりに `++i` を使用します。
- 不必要な`?.`を避け、防御的なプログラミングを禁止します。
- 変数名は複数である必要はなく、リストを示すために `_li` で終わります。

## 命名規則

- ミニマリズムを追求したネーミングです。短くても意味のある名前を使用します。たとえば、`remove`、`delete`、`del` の代わりに `rm` を使用します。ただし、極端なことは避けてください。たとえば、`map` の代わりに単一の文字 `m` を使用しないでください。
- 関数の名前付けには動詞のみを使用するようにしてください。 1 つの単語で表現できる場合は、2 つの単語を使用しないでください。名詞はファイル名に反映されます。ファイル名に動詞を追加する必要がある場合は、名詞を最初に、動詞を最後に入力してください。例: `setProfile.js` ではなく `profileSet.js`
- 変数名: 下線スタイル (snake_case) を使用します (例: `user_auth_token`)。変数が関数の場合は、小文字のキャメルケースの命名スタイルを使用します
- 関数名: 小文字のキャメルケース (camelCase) を使用します。例: `userData`
- 関数パラメータ: コールバック関数の場合は、`onChange` のように小文字のキャメルケースで名前を付けます。
- モジュールレベルの定数定義では大文字の下線スタイルを使用します`UPPER_SNAKE_CASE`
- `getCookie` の代わりに `cookieByHeader` と書くなど、無意味なプレフィックス `get` は書かないでください。
- グローバル/モジュールレベルの定数: 大文字の下線スタイル (UPPER_SNAKE_CASE) を使用します。 `CODE_TO_ID`、`ID_TO_LANG`

## モジュラー機構

- インポート: オンデマンドで関数を正確にインポートし、モジュール全体の直接インポートを禁止します (`import * as x` または大きなオブジェクトのインポートを避ける)
- エクスポート: オブジェクトのエクスポートを無効にします。関数と変数を`export const X=1, abc=()=>{};`などのユニットとしてエクスポートします。これらを const + カンマと組み合わせて、エクスポートされたコンテンツを宣言してみてください。ファイルに機能が 1 つしかない場合は、 `export default` を使用します。
- 内部的にエクスポート デフォルト関数を呼び出す必要がない限り、最初に定数を宣言してからファイルの最後でエクスポートすることは避けてください。
- パス解決: 現在のディレクトリ パスを取得するときは、`import.meta.dirname` を使用する必要があります。

## 間違い

- 文字列エラーを回避し、const を使用して定数エラー コードを宣言します。
- 間違ったデータ情報を返す必要がある場合は、[エラー コード、エラー メッセージ...] を使用します。この方法では、エラー メッセージはテキストの説明ではなく、数値 ([FILE_OVERSIZE、file_size、max_size] など) になります。

## ブラウザ互換の API を使用してみる

- 暗号化と復号化: ネイティブ Web Crypto API の使用を強制します
- バイナリ データ: バイナリを扱うときは、 `Uint8Array` を統一して使用するようにしてください。
