# WebC.site

## ეს რა არის?

### ეს არის მშობლიური ვებ კომპონენტის ბიბლიოთეკა

შემუშავებული [ვებ კომპონენტების] (/0) საფუძველზე, ის არის დამოუკიდებელი მუშაობის დრო და შესაფერისია ყველა ფრეიმიკისთვის.

უკიდურესად ოპტიმიზირებულია ზომისთვის (მსგავსი პოპულარული ბიბლიოთეკების ზომის მეათედი).

მხარს უჭერს 75 ენას. თქვენ შეგიძლიათ პირდაპირ შემოიტანოთ js და css ონლაინ გამოყენებისთვის, ან დაამატოთ კომპონენტები თქვენს პროექტში, როგორც საჭიროა მეორადი განვითარებისთვის.

### ეს არის კომპონენტი ბიბლიოთეკის განვითარების ჩარჩო

ეს არის [storybook](https://storybook.js.org) ბლოგის სტატიკური ვერსია.

კომპონენტის ბიბლიოთეკის აშენებამ შეიძლება გააუმჯობესოს კომპანიის შიდა განვითარების ეფექტურობა და შექმნას ბრენდის იმიჯი ტექნიკურ წრეში.

[webc.site](https://github.com/webc-site/webc.site) საფუძველზე, გუნდებს შეუძლიათ ადვილად შექმნან საკუთარი კომპონენტების ბიბლიოთეკები და გამოაქვეყნონ ისინი [github page](https://pages.github.com), [cloudflare page](https://pages.cloudflare.com) და ა.შ. გამოქვეყნებული კომპონენტების გამართვა შესაძლებელია ონლაინ რეჟიმში [codepen.io](https://codepen.io) და [jsfiddle](https://jsfiddle.net).

ამავდროულად, ჩაშენებული თარგმანის ჩარჩოს შეუძლია სწრაფად გააცნობიეროს კომპონენტებისა და დოკუმენტების ინტერნაციონალიზაცია, ისე, რომ თქვენი პროდუქტის აუდიტორია აღარ იყოს შეზღუდული ენით.

### ეს არის ფრონტის განვითარების ახალი პარადიგმა ხელოვნური ინტელექტის ეპოქისთვის

1. ** ყველაფერი არის ვებ კომპონენტი **

ამ გზით, ხელოვნურ ინტელექტს შეუძლია ყალბი მონაცემების გენერირება და დემო-გვერდზე კომპონენტების სხვადასხვა მდგომარეობის ჩვენება.

კომპონენტის განვითარებისა და მონაცემთა ინტერფეისის გამიჯვნა, კომბინირებული [webc.site კომპონენტის ჩარჩო](https://github.com/webc-site/webc.site) კიდევ ერთი ძირითადი ფუნქცია:

### ლოკალური განვითარება და გამართვა

ვებ გვერდის თითოეული კომპონენტი მხარს უჭერს განვითარების სერვერის დამოუკიდებლად დაწყებას, რაც უზრუნველყოფს მინიმალურ განვითარებისა და გამართვის გარემოს:

- **შექმენით ახალი კომპონენტი**: გაუშვით `./sh/new.js [css|js] [კომპონენტის სახელი]`, რათა სწრაფად შექმნათ კომპონენტის განვითარების ჩონჩხი (მათ შორის, დემო გვერდი და ინტერნაციონალიზაციის კონფიგურაცია).
- **დამოუკიდებელი გამართვა**: გაუშვით `./dev.js [კომპონენტის სახელი]` (ან გაუშვით `./dev.js` პირდაპირ), რათა ჩართოთ Vite განვითარების სერვერი და ავტომატურად გახსნათ კომპონენტის დემო გვერდი ბრაუზერში. მხარს უჭერს კომპონენტის ცხელ განახლებას (HMR). იმის გამო, რომ განვითარებისა და მონაცემთა ბმულები გათიშულია, არსებობს ნულოვანი განვითარების წინა ნაწილის ბმული, რომელიც ძალიან შესაფერისია AI-სთვის (როგორიცაა Claude Code, Google Antigravity და ა.შ.), რათა დაეხმაროს სრულად ავტომატურ განვითარებას და ბრაუზერის გამართვას.
