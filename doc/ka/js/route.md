# route.js

ერთი გვერდის წინა ბოლო მარშრუტიზაციის მართვა

## nowUrl()

მიიღეთ მიმდინარე URL ბილიკი, წაშლილი წამყვანი ზოლით

დაბრუნების მნიშვნელობა: მიმდინარე ბილიკის სტრიქონი

## route(hook)

დაარეგისტრირეთ მოსასმენი გამოძახება მარშრუტის ცვლილებებისთვის. რეგისტრაციისას ის შესრულდება ერთხელ მიმდინარე ბილიკით.

პარამეტრი:
- `hook` : მარშრუტის მონიტორინგის გამოძახების ფუნქცია
  - პარამეტრი
    - `url` : მიმდინარე გზა
    - `pre` : ბოლო გზა

დაბრუნების მნიშვნელობა: არარეგისტრირებული გაწმენდის ფუნქცია

## setPre(url)

შეინახეთ ბოლო ჩაწერილი URL

პარამეტრი:
- `url` : ბილიკის სტრიქონი

## preUrl()

მიიღეთ ბოლო ჩანაწერის URL

დაბრუნების მნიშვნელობა: ბოლო ბილიკის სტრიქონი

## refresh(url)

აიძულეთ ყველა მარშრუტის მოსმენის გამოძახება გამოძახდეს მითითებულ გზაზე და განაახლეთ წინასწარ

პარამეტრი:
- `url` : სამიზნე გზა, ნაგულისხმევი მიმდინარე გზა

## removeSlash(url)

წაშალეთ ზოლები ბილიკის სათაურებიდან

პარამეტრი:
- `url` : ბილიკის სტრიქონი

დაბრუნების მნიშვნელობა: გზა წამყვანი ზოლის ამოღების შემდეგ

## setUrl(url)

განაახლეთ ბრაუზერის მისამართის ზოლის URL-ის სტატუსი

პარამეტრი:
- `url` : სამიზნე გზა

დაბრუნების მნიშვნელობა: 1, თუ გზა რეალურად იცვლება

## goto(url)

განაახლეთ ბრაუზერის მისამართი და ჩართეთ მარშრუტის განახლება

პარამეტრი:
- `url` : სამიზნე გზა
