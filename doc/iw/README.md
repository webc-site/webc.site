# WebC.site

## מה זה?

### זוהי ספריית רכיבי אינטרנט מקורית

פותח על בסיס [רכיבי אינטרנט](https://developer.mozilla.org/docs/Web/API/Web_components), הוא בלתי תלוי בזמן ריצה ומתאים לכל המסגרות.

מותאם במיוחד לגודל (עשירית מהגודל של ספריות פופולריות דומות).

תומך ב-75 שפות. אתה יכול לייבא ישירות js ו-css לשימוש מקוון, או להוסיף רכיבים לפרויקט שלך לפי הצורך לפיתוח משני.

<img src="https://i-01.eu.org/20260604114250265.avif" width="400">

### זוהי מסגרת הפיתוח של ספריית הרכיבים

זוהי גרסת הבלוג הסטטית של [ספר סיפורים](https://storybook.js.org).

בניית ספריית רכיבים יכולה לשפר את יעילות הפיתוח הפנימית של החברה ולבנות תדמית מותגית במעגל הטכני.

בהתבסס על [webc.site](https://github.com/webc-site/webc.site), צוותים יכולים ליצור בקלות ספריות רכיבים משלהם ולפרסם אותם ב-[github page](https://pages.github.com), [cloudflare page](https://pages.cloudflare.com) וכו'. ניתן לנפות באגים ברכיבים שפורסמו באופן מקוון ב-[codepen.io](https://codepen.io) ו-[jsfiddle](https://jsfiddle.net).

יחד עם זאת, מסגרת התרגום המובנית יכולה לממש במהירות את האינטרנציונליזציה של רכיבים ומסמכים, כך שקהל המוצרים שלך כבר לא מחויב לשפה.

### זוהי פרדיגמה חדשה של פיתוח חזיתי לעידן הבינה המלאכותית

המסגרת מובנית ב`.agents/skills`, ומאפשרת לבינה מלאכותית לעזור לך לפתח רכיבים חדשים בלחיצה אחת בהתאם לשיטות העבודה המומלצות.

<img src="https://i-01.eu.org/20260604121408079.avif" width="400">

תן לקוד החזיתי להשתנות מהר זבל לא בר קיימא של תכנות אווירה לחלק סטנדרטי שניתן לתחזק ולהתפתח באופן איטרטיבי לאורך הטווח הארוך.

לאור המאפיינים של פיתוח בעזרת בינה מלאכותית, הצענו ותכננו פרדיגמת פיתוח חזיתית חדשה:

1. **הכל הוא רכיב אינטרנט**
2. **הרכיב חייב להיות בלתי תלוי בקישור הנתונים האחורי, וממשק הנתונים חשוף כפונקציית התקשרות חוזרת**

בדרך זו, בינה מלאכותית יכולה ליצור נתונים מזויפים ולהציג מצבים שונים של רכיבים בדף ההדגמה.

הפרדת פיתוח רכיבים וממשק נתונים, בשילוב עם [webc.site component framework](https://github.com/webc-site/webc.site) תכונה מרכזית נוספת:

**כל רכיב יכול להפעיל את שרת הפיתוח באופן עצמאי ולפתח ולבצע ניפוי באגים בדף ההדגמה**.

זה מפחית את הקישור החזיתי לפיתוח וניפוי באגים של כל רכיב ל`0`.

בדרך זו, כלי פיתוח הנעזרים בבינה מלאכותית ([Claude Code](https://code.claude.com), [OpenAI Codex](https://openai.com), [Google Antigravity](https://antigravity.google), [Cursor](https://cursor.com), [Windsurf](https://windsurf.com), [Devin](https://devin.ai), [OpenHands](https://github.com/All-Hands-AI/OpenHands) וכו'), אתה יכול ליהנות יותר מפיתוח אוטומטי של דפדפן פתוח ולא ניתוק מלא של בלגן, ולא ניתן יותר לחסום דפדפן בלאגן. אם אתה רוצה לשלוט בבינה מלאכותית ולהפחית עלויות ולהגדיל את היעילות, אתה צריך פרדיגמה חדשה שממזערת את סביבת הפיתוח והניפוי של כל רכיב.

#### זמן ריצה עצמאי

ספריית הרכיבים webc.site, בנויה על סטנדרטים [רכיבי אינטרנט](https://developer.mozilla.org/docs/Web/API/Web_components), היא באופן טבעי אגנוסטית בזמן ריצה. בין אם אתה משתמש ב-React, Vue, Angular, Svelte או בפרויקט HTML/JS טהור מסורתי, כל הרכיבים יכולים לפעול ישירות בדפדפן כמו תגי HTML מקוריים, ללא צורך באריזה משנית למסגרת ספציפית.

##### השוואה עם shadcn

[shadcn/ui](https://ui.shadcn.com) מאמץ את מצב הפצת הקוד, המחייב אותך להעתיק את קוד הרכיב ישירות לפרויקט שלך. הוא קשור באופן הדוק למערכת האקולוגית של React ולשרשרות כלי בנייה ספציפיות (כגון Tailwind CSS), ולא ניתן להשתמש בו בין מסגרות. יש לו גם דרישות מסוימות לתצורת הנדסת פרויקט.

והרכיב של `webc.site`:
- ** חוצה מסגרות אוניברסלי**: כתוב פעם אחת וניתן להציג ישירות ולהפעיל בכל מסגרת חזיתית (אפילו בסביבה נטולת מסגרת).
- **אפס תלות בסביבה**: אין צורך בכלי אריזה ספציפיים או במעבדי קדם CSS, רק ייבא את הקבצים הסטטיים שהורכבו וניתן להשתמש בהם מחוץ לקופסה.

## פיתוח אוטומטי ואיתור באגים

### פיתוח מקומי ואיתור באגים

כל רכיב של דף אינטרנט תומך בהפעלת שרת הפיתוח באופן עצמאי, ומספק סביבת פיתוח ואיתור באגים מינימלית:

- **צור רכיב חדש**: הפעל את `./sh/new.js [css|js] [שם הרכיב]` כדי ליצור במהירות שלד פיתוח של רכיבים (כולל דף הדגמה ותצורת בינאום).
- **ניפוי באגים עצמאי**: הפעל את `./dev.js [שם הרכיב]` (או הפעל את `./dev.js` ישירות) כדי להפעיל את שרת הפיתוח של Vite ולפתוח אוטומטית את דף ההדגמה של הרכיב בדפדפן. תומך בעדכון חם של רכיבים (HMR). מכיוון שקישורי פיתוח ונתונים מנותקים, יש אפס קישור חזיתי לפיתוח, שמתאים מאוד ל-AI (כגון Claude Code, Google Antigravity וכו') כדי לסייע בפיתוח אוטומטי לחלוטין ובניפוי באגים בדפדפן.

### פרסם את ספריית הרכיבים

#### webc dist

תהליך האריזה והפרסום הוא אוטומטי ביותר:
- **אריזה של משאבים סטטיים**: הפעלת `./sh/dist.js` (כלומר, ביצוע המשימה `webc dist`) יכולה לסרוק אוטומטית את כל רכיבי דף האינטרנט תחת `src/`, להדר, לדחוס ולהוציא אותם לספרייה `dist/`. ספרייה זו מכילה קבצי JS/CSS להפצת ייצור, אתר הדגמה מוכן לפריסה לדפי GitHub או דפי Cloudflare, וקובצי `_redirects` ו`404.html` ניתוב שנוצרו באופן אוטומטי.
- **NPM Publish**: הפעל את `./sh/npmDist.js`, הסקריפט יטפל אוטומטית בתלות של רכיבים, יגדיל את מספר גרסת החבילה ויבצע `npm publish` בלחיצה אחת לאחר השלמת האריזה כדי לפרסם את ספריית הרכיבים ל-NPM.

### הפלטפורמה אחראית על התרגום

למסגרת יש תהליכי תרגום אוטומטיים מובנים:
1. הגדר כללי תרגום (כגון שפת מקור, שפת יעד וספרייה לתרגום) ב-`tran.yml` בספריית השורש של הפרויקט.
2. השתמש במצייני מיקום ברכיבים או במסמכים ללא צורך בקוד קשיח של מספר שפות בקוד שלך.
3. הפעל את `./sh/tran.js`, ומנוע התרגום יחלץ אוטומטית את הטקסט ויתקשר לשירות התרגום של WebC לעיבוד, וייצור אוטומטית ערכי תרגום YAML ותיעוד README בשפות שונות.

### משתני סביבה

בעת ביצוע תרגום אוטומטי, עליך להגדיר את משתני הסביבה הבאים כדי לאשר גישה ל-API של התרגום:

- **`WEBC_SITE_TOKEN`** (או **`WEBC_TOKEN`**): מפתח אימות המשמש לגישה ל-`webc.site` API לתרגום אוטומטי. ניתן להחדיר כמשתנה סביבה או לכתוב `token` בקובץ התצורה `~/.config/webc.site.yml` בספריית הבית של המשתמש המקומי.
