# જેએસ કોડ સ્પષ્ટીકરણ

## કોડિંગ શૈલી

- સૌથી આધુનિક જેએસ સિન્ટેક્સનો ઉપયોગ કરીને સરળ, ભવ્ય અને કાર્યક્ષમ
- વધુ પડતા લાંબા કાર્યોને વિભાજિત કરો, અને ઇન્ટરફેસ ડિઝાઇન ઓછી જોડાણ અને ઉચ્ચ સુસંગતતા હોવી જોઈએ.
- દરેક/નકશા માટે એરે + લૂપ્સનો વધુ ઉપયોગ કરો અને ઓછા પુનરાવર્તિત કોડ લખો
- વધુ પડતી ટિપ્પણીઓ ટાળવા માટે સરળ કોડમાં કાર્ય ટિપ્પણીઓ લખશો નહીં. જટિલ તર્ક અને વિશેષ જરૂરિયાતો શુદ્ધ ચાઇનીઝમાં સમજાવવામાં આવી છે.
- રીડન્ડન્સી ઘટાડવા માટે પુનરાવર્તિત કોડને કાર્યોમાં અમૂર્ત કરવામાં આવે છે. ઉદાહરણ તરીકે, `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` ને `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()` તરીકે લખી શકાય છે
- ચલ ઘોષણા: એક નિવેદનમાં એકથી વધુ સળંગ `const` ઘોષણાઓને જોડો. તેને ત્રણ લીટીમાં લખવાને બદલે `const a=1, b=2, c=3;` લખો
- અસુમેળ પ્રક્રિયા: `await` નો ઉપયોગ કરો, `.then` ચેઇન કૉલ્સનો ઉપયોગ પ્રતિબંધિત કરો
- અપવાદ હેન્ડલિંગ કોડ આપમેળે જનરેટ કરશો નહીં, આપમેળે `try...catch` લખશો નહીં (`try ... catch` મેન્યુઅલી જાળવવામાં આવે છે, હાલનું `try catch` કાઢી નાખવું જોઈએ નહીં)
- શુદ્ધ કાર્યો પ્રથમ: ફક્ત શુદ્ધ કાર્યો લખો, ક્યારેય વર્ગો નહીં
- એરો ફંક્શન્સનો ઉપયોગ કરો `const funcName = () => {}` અને `function` કીવર્ડનો ઉપયોગ કરશો નહીં (જનરેટર સિવાય); જો તમે પરિમાણોને બાંધવા માટે .bind નો ઉપયોગ કરી શકો છો, તો કાર્યોને વ્યાખ્યાયિત કરવાનું ટાળો
- કોડ પુનઃઉપયોગ: પુનઃઉપયોગ પર ધ્યાન આપો, વધુ નાના કાર્યોને બહાર કાઢો અને મોટી સંખ્યામાં સમાન અથવા કૉપિ-પેસ્ટ કરેલા કોડ સ્ટ્રક્ચર્સને નિશ્ચિતપણે ટાળો.
- ઑબ્જેક્ટ ઍક્સેસ: જરૂરી વિશેષતાઓ કાઢવા માટે ડિસ્ટ્રક્ચરિંગ અસાઇનમેન્ટનો ઉપયોગ કરીને પ્રાથમિકતા આપો, ડીપ અને નેસ્ટેડ એટ્રિબ્યુટ્સને ઍક્સેસ કરવા માટે આંતરિક રીતે ડોટ્સનો વારંવાર ઉપયોગ કરવાનું ટાળો અને વારંવાર વૈકલ્પિક સાંકળના નિર્ણયોને મર્જ કરો
- ફંક્શન પેરામીટર્સ માટે ઑબ્જેક્ટ્સનો ઉપયોગ કરશો નહીં, `{a,b,c}` લખશો નહીં, `a, b, c` લખો; જો ત્યાં ઘણા વૈકલ્પિક પરિમાણો હોય, તો [[રૂપરેખાંકન વસ્તુઓની સંખ્યા, રૂપરેખાંકન વસ્તુઓ], [રૂપરેખાંકન વસ્તુઓની સંખ્યા, રૂપરેખાંકન વસ્તુઓ],..] નો ઉપયોગ કરો
- બહુ-મૂલ્ય વળતર માટે, `{a,b,c}`ને બદલે એરે `[a,b,c]` નો ઉપયોગ કરો. જો ત્યાં 3 કરતાં વધુ વળતર મૂલ્યો હોય, તો સ્થિતિકીય અર્થને વ્યાખ્યાયિત કરવા માટે સંખ્યાત્મક સ્થિરાંકોનો ઉપયોગ કરો.
- રાજ્યનું પ્રતિનિધિત્વ કરવા માટે કોઈ શબ્દમાળાની જરૂર નથી, રાજ્યને વ્યાખ્યાયિત કરવા માટે આંકડાકીય સ્થિરાંકોનો ઉપયોગ કરો
- કોઈ સ્ટ્રિંગ ટેમ્પલેટ (``) નો ઉપયોગ થતો નથી, આયાત સિવાય (Vite સ્ટેટિક પૃથ્થકરણ માટે અનુકૂળ) સ્ટ્રિંગ કન્કેટનેશન (+) નો ઉપયોગ થાય છે.
- લૂપ માટે, જો તમને સીરીયલ નંબરની જરૂર હોય, તો `i++` ને બદલે `++i` નો ઉપયોગ કરો
- બિનજરૂરી`?.` ટાળો અને રક્ષણાત્મક પ્રોગ્રામિંગને પ્રતિબંધિત કરો
- ચલ નામ બહુવચન હોવું જરૂરી નથી, અને સૂચિ દર્શાવવા માટે `_li` સાથે સમાપ્ત થાય છે.

## નામકરણ સંમેલન

- નામકરણ લઘુત્તમવાદને અનુસરે છે. ટૂંકા પરંતુ અર્થપૂર્ણ નામોનો ઉપયોગ કરો, ઉદાહરણ તરીકે: `remove`, `delete`, `del`ને બદલે `rm` નો ઉપયોગ કરો. જો કે, તમારે ચરમસીમા પર જવાનું પણ ટાળવું જોઈએ, ઉદાહરણ તરીકે: `map` ને બદલે એક અક્ષર `m` નો ઉપયોગ કરશો નહીં
- ફંક્શન નામકરણમાં માત્ર ક્રિયાપદોનો ઉપયોગ કરવાનો પ્રયાસ કરો. જો તમે તેને વ્યક્ત કરવા માટે એક શબ્દનો ઉપયોગ કરી શકો છો, તો બે શબ્દોનો ઉપયોગ કરશો નહીં. સંજ્ઞાઓ ફાઇલના નામોમાં પ્રતિબિંબિત થાય છે. જો તમારે ફાઇલના નામમાં ક્રિયાપદ ઉમેરવાની જરૂર હોય, તો કૃપા કરીને સંજ્ઞાને પહેલા અને ક્રિયાપદને છેલ્લે મૂકો. ઉદાહરણ તરીકે: `setProfile.js`ને બદલે `profileSet.js`
- ચલ નામ: અન્ડરલાઇન શૈલીનો ઉપયોગ કરો (snake_case), ઉદાહરણ તરીકે `user_auth_token`; જો વેરીએબલ ફંક્શન છે, તો લોઅરકેસ કેમલ કેસ નામકરણ શૈલીનો ઉપયોગ કરો
- કાર્યનું નામ: લોઅરકેસ કેમલકેસ (કેમલકેસ) નો ઉપયોગ કરો, ઉદાહરણ તરીકે `userData`
- ફંક્શન પેરામીટર્સ: જો તે કૉલબેક ફંક્શન છે, તો તેને લોઅરકેસ કેમલ કેસમાં નામ આપો, જેમ કે `onChange`
- મોડ્યુલ-સ્તરની સ્થિર વ્યાખ્યાઓ અપરકેસ અન્ડરલાઇન શૈલીનો ઉપયોગ કરે છે `UPPER_SNAKE_CASE`
- `get` લખશો નહીં, આવા અર્થહીન ઉપસર્ગો, જેમ કે: `getCookie`ને બદલે `cookieByHeader` લખો
- વૈશ્વિક/મોડ્યુલ-સ્તર સ્થિરાંકો: અપરકેસ અન્ડરલાઇન શૈલીનો ઉપયોગ કરો (UPPER_SNAKE_CASE), દા.ત. `CODE_TO_ID`, `ID_TO_LANG`

## મોડ્યુલર મિકેનિઝમ

- આયાત કરો: માંગ પર સચોટ રીતે આયાત કાર્યો, સમગ્ર મોડ્યુલની સીધી આયાતને પ્રતિબંધિત કરો (`import * as x` ટાળો અથવા મોટા પદાર્થોની આયાત કરો)
- નિકાસ: ઑબ્જેક્ટ્સની નિકાસને અક્ષમ કરો. ફંક્શન્સ અને ચલોને યુનિટ તરીકે નિકાસ કરો, જેમ કે `export const X=1, abc=()=>{};`. નિકાસ કરેલ સામગ્રી જાહેર કરવા માટે તેમને કોન્સ્ટ + અલ્પવિરામ સાથે જોડવાનો પ્રયાસ કરો. જો ફાઇલમાં માત્ર એક જ કાર્ય હોય, તો `export default` નો ઉપયોગ કરો
- જ્યાં સુધી તમારે નિકાસ ડિફૉલ્ટ ફંક્શનને આંતરિક રીતે કૉલ કરવાની જરૂર નથી, ત્યાં સુધી પહેલા સ્થિરાંકો જાહેર કરવાનું ટાળો અને પછી ફાઇલના અંતે તેમને નિકાસ કરવાનું ટાળો.
- પાથ રિઝોલ્યુશન: વર્તમાન ડિરેક્ટરી પાથ મેળવતી વખતે, `import.meta.dirname` નો ઉપયોગ કરવો આવશ્યક છે

## ભૂલ

- સ્ટ્રિંગ ભૂલો ટાળો અને સતત ભૂલ કોડ્સ જાહેર કરવા માટે const નો ઉપયોગ કરો.
- જો તમારે ખોટી ડેટા માહિતી પરત કરવાની જરૂર હોય, તો [ભૂલ કોડ, ભૂલ સંદેશ,...] નો ઉપયોગ કરો આ રીતે, ભૂલ સંદેશ એ ટેક્સ્ટ વર્ણન નથી, પરંતુ સંખ્યાત્મક મૂલ્ય છે (જેમ કે [FILE_OVERSIZE, file_size, max_size])
