# WebC.site

## આ શું છે?

### આ મૂળ વેબ ઘટક પુસ્તકાલય છે

[વેબ ઘટકો](https://developer.mozilla.org/docs/Web/API/Web_components) ના આધારે વિકસિત, તે રનટાઇમ સ્વતંત્ર છે અને તમામ ફ્રેમવર્ક માટે યોગ્ય છે.

કદ માટે અત્યંત ઑપ્ટિમાઇઝ (સમાન લોકપ્રિય પુસ્તકાલયોના દસમા ભાગના કદ).

75 ભાષાઓને સપોર્ટ કરે છે. તમે ઓનલાઈન ઉપયોગ માટે સીધા જ js અને css આયાત કરી શકો છો અથવા ગૌણ વિકાસ માટે તમારા પ્રોજેક્ટમાં ઘટકો ઉમેરી શકો છો.

<img src="https://i-01.eu.org/20260604114250265.avif" width="400">

### આ ઘટક પુસ્તકાલય વિકાસ માળખું છે

આ [સ્ટોરીબુક](https://storybook.js.org) નું સ્થિર બ્લોગ સંસ્કરણ છે.

એક ઘટક પુસ્તકાલયનું નિર્માણ કંપનીની આંતરિક વિકાસ કાર્યક્ષમતામાં સુધારો કરી શકે છે અને તકનીકી વર્તુળમાં બ્રાન્ડની છબી બનાવી શકે છે.

[webc.site](https://github.com/webc-site/webc.site) ના આધારે, ટીમો સરળતાથી તેમની પોતાની ઘટક લાઇબ્રેરીઓ બનાવી શકે છે અને તેને [ગીથબ પૃષ્ઠ](https://pages.github.com), [ક્લાઉડફ્લેર પૃષ્ઠ](https://pages.cloudflare.com), વગેરે પર પ્રકાશિત કરી શકે છે. પ્રકાશિત ઘટકોને [codepen.io](https://codepen.io) અને [jsfiddle](https://jsfiddle.net) પર ઑનલાઇન ડિબગ કરી શકાય છે.

તે જ સમયે, બિલ્ટ-ઇન અનુવાદ ફ્રેમવર્ક ઘટકો અને દસ્તાવેજોના આંતરરાષ્ટ્રીયકરણને ઝડપથી અનુભવી શકે છે, જેથી તમારા ઉત્પાદન પ્રેક્ષકો હવે ભાષા દ્વારા બંધાયેલા નથી.

### કૃત્રિમ બુદ્ધિમત્તાના યુગ માટે આ ફ્રન્ટ એન્ડ ડેવલપમેન્ટનો નવો દાખલો છે

ફ્રેમવર્ક બિલ્ટ-ઇન છે `.agents/skills`, આર્ટિફિશિયલ ઇન્ટેલિજન્સ તમને શ્રેષ્ઠ પ્રથાઓ અનુસાર એક ક્લિક સાથે નવા ઘટકો વિકસાવવામાં મદદ કરવા માટે પરવાનગી આપે છે.

<img src="https://i-01.eu.org/20260604121408079.avif" width="400">

ફ્રન્ટ-એન્ડ કોડને વાતાવરણ પ્રોગ્રામિંગના બિનટકાઉ કચરાના પર્વતમાંથી એક પ્રમાણભૂત ભાગમાં બદલવા દો કે જે લાંબા ગાળા માટે જાળવી શકાય અને પુનરાવર્તિત રીતે વિકસિત થઈ શકે.

આર્ટિફિશિયલ ઇન્ટેલિજન્સ-આસિસ્ટેડ ડેવલપમેન્ટની લાક્ષણિકતાઓને ધ્યાનમાં રાખીને, અમે એક નવો ફ્રન્ટ-એન્ડ ડેવલપમેન્ટ પેરાડાઈમ પ્રસ્તાવિત અને ડિઝાઇન કર્યો છે:

1. **બધું એક વેબ ઘટક છે**
2. **કમ્પોનન્ટ બેક-એન્ડ ડેટા લિંકથી સ્વતંત્ર હોવું જોઈએ, અને ડેટા ઈન્ટરફેસ કૉલબેક ફંક્શન તરીકે ખુલ્લું છે**

આ રીતે, આર્ટિફિશિયલ ઇન્ટેલિજન્સ નકલી ડેટા જનરેટ કરી શકે છે અને ડેમો પેજ પર ઘટકોની વિવિધ સ્થિતિઓ પ્રદર્શિત કરી શકે છે.

ઘટક વિકાસ અને ડેટા ઈન્ટરફેસનું વિભાજન, [webc.site ઘટક ફ્રેમવર્ક](https://github.com/webc-site/webc.site) સાથે જોડાયેલું અન્ય મુખ્ય લક્ષણ:

**દરેક ઘટક વિકાસ સર્વરને સ્વતંત્ર રીતે શરૂ કરી શકે છે અને ડેમો પૃષ્ઠ પર વિકાસ અને ડીબગ કરી શકે છે**.

<img src="https://i-01.eu.org/webcSiteDemo.svg" width="400">
<img src="https://i-01.eu.org/20260604112614499.avif" width="400">

આ દરેક ઘટકના વિકાસ અને ડીબગીંગ માટે ફ્રન્ટ-એન્ડ લિંકને `0` સુધી ઘટાડે છે.

આ રીતે, આર્ટિફિશિયલ ઈન્ટેલિજન્સ-આસિસ્ટેડ ડેવલપમેન્ટ ટૂલ્સ ([ક્લોડ કોડ](https://code.claude.com), [ઓપનએઆઈ કોડેક્સ](https://openai.com), [ગૂગલ એન્ટિગ્રેવિટી](https://antigravity.google), [કર્સર](https://cursor.com), [વિન્ડસર્ફ](https://windsurf.com), [ડેવિન](https://devin.ai), [ઓપનહેન્ડ્સ](https://github.com/All-Hands-AI/OpenHands) વગેરે, તમે બ્રાઉઝરને પૂર્ણપણે ખોલી શકતા નથી અને વિકાસનો આનંદ માણી શકતા નથી. વિવિધ અવ્યવસ્થિત પરિબળો (બેકએન્ડ ડેટા, લોગિન સ્થિતિ, પાથ રૂટીંગ, વગેરે) દ્વારા અવરોધિત.

જો તમે કૃત્રિમ બુદ્ધિમત્તાને નિયંત્રિત કરવા અને ખર્ચ ઘટાડવા અને કાર્યક્ષમતામાં વધારો કરવા માંગતા હો, તો તમારે એક નવા દાખલાની જરૂર છે જે દરેક ઘટકના વિકાસ અને ડીબગિંગ વાતાવરણને ઓછું કરે.

#### રનટાઇમ સ્વતંત્ર

ધોરણો [વેબ ઘટકો](https://developer.mozilla.org/docs/Web/API/Web_components) પર બનેલ, webc.site ઘટક લાઇબ્રેરી કુદરતી રીતે રનટાઇમ-અજ્ઞેયવાદી છે. ભલે તમે React, Vue, Angular, Svelte, અથવા પરંપરાગત શુદ્ધ HTML/JS પ્રોજેક્ટનો ઉપયોગ કરી રહ્યાં હોવ, બધા ઘટકો ચોક્કસ ફ્રેમવર્ક માટે ગૌણ પેકેજિંગની જરૂરિયાત વિના, મૂળ HTML ટૅગ્સ જેવા બ્રાઉઝરમાં સીધા જ ચાલી શકે છે.

##### shadcn સાથે સરખામણી

[shadcn/ui](https://ui.shadcn.com) કોડ ડિસ્ટ્રિબ્યુશન મોડને અપનાવે છે, જેના માટે તમારે ઘટક કોડની સીધી તમારા પોતાના પ્રોજેક્ટમાં કૉપિ કરવાની જરૂર છે. તે રિએક્ટ ઇકોસિસ્ટમ અને ચોક્કસ કન્સ્ટ્રક્શન ટૂલ ચેઇન્સ (જેમ કે ટેઇલવિન્ડ CSS) સાથે ચુસ્તપણે બંધાયેલું છે અને સમગ્ર ફ્રેમવર્કમાં તેનો ઉપયોગ કરી શકાતો નથી. તે પ્રોજેક્ટ એન્જિનિયરિંગ ગોઠવણી માટે ચોક્કસ જરૂરિયાતો પણ ધરાવે છે.

અને `webc.site` ના ઘટક:
- **ક્રોસ-ફ્રેમવર્ક યુનિવર્સલ**: એકવાર લખો અને સીધા જ રજૂ કરી શકાય છે અને કોઈપણ ફ્રન્ટ-એન્ડ ફ્રેમવર્ક (ફ્રેમવર્ક-મુક્ત વાતાવરણ પણ) માં ચલાવી શકાય છે.
- **પર્યાવરણ શૂન્ય નિર્ભરતા**: કોઈ વિશિષ્ટ પેકેજિંગ ટૂલ્સ અથવા CSS પ્રીપ્રોસેસરની જરૂર નથી, ફક્ત સંકલિત સ્ટેટિક ફાઇલોને આયાત કરો અને તેનો ઉપયોગ બૉક્સની બહાર થઈ શકે છે.

## સ્વચાલિત વિકાસ અને ડિબગીંગ

### સ્થાનિક વિકાસ અને ડિબગીંગ

દરેક વેબ પેજ કમ્પોનન્ટ ન્યૂનતમ વિકાસ અને ડીબગીંગ વાતાવરણ પૂરું પાડીને સ્વતંત્ર રીતે ડેવલપમેન્ટ સર્વરને શરૂ કરવાનું સમર્થન કરે છે:

- **નવું ઘટક બનાવો**: એક ઘટક વિકાસ હાડપિંજર (ડેમો પૃષ્ઠ અને આંતરરાષ્ટ્રીયકરણ ગોઠવણી સહિત) ઝડપથી જનરેટ કરવા માટે `./sh/new.js [css|js] [ઘટક નામ]` ચલાવો.
- **સ્વતંત્ર ડીબગીંગ**: Vite ડેવલપમેન્ટ સર્વર શરૂ કરવા માટે `./dev.js [ઘટક નામ]` ચલાવો (અથવા `./dev.js` સીધું ચલાવો) અને બ્રાઉઝરમાં ઘટકનું ડેમો પેજ આપમેળે ખોલો. કમ્પોનન્ટ હોટ અપડેટ (HMR) ને સપોર્ટ કરે છે. કારણ કે ડેવલપમેન્ટ અને ડેટા લિંક્સ ડીકપ્લ્ડ છે, ત્યાં શૂન્ય ડેવલપમેન્ટ ફ્રન્ટ-એન્ડ લિંક છે, જે સંપૂર્ણપણે સ્વચાલિત વિકાસ અને બ્રાઉઝર ડિબગિંગમાં સહાય કરવા માટે AI (જેમ કે ક્લાઉડ કોડ, Google એન્ટિગ્રેવિટી, વગેરે) માટે ખૂબ જ યોગ્ય છે.

### ઘટક પુસ્તકાલય પ્રકાશિત કરો

#### webc dist

પેકેજિંગ અને પ્રકાશન પ્રક્રિયા અત્યંત સ્વચાલિત છે:
- **સ્ટેટિક રિસોર્સ પેકેજીંગ**: `./sh/dist.js` (એટલે ​​​​કે, `webc dist` કાર્ય ચલાવવું) `src/` હેઠળના તમામ વેબ પૃષ્ઠ ઘટકોને આપમેળે સ્કેન કરી શકે છે, કમ્પાઇલ, સંકુચિત અને `dist/` ડિરેક્ટરીમાં આઉટપુટ કરી શકે છે. આ નિર્દેશિકામાં ઉત્પાદન વિતરણ માટે JS/CSS ફાઇલો છે, જે GitHub પૃષ્ઠો અથવા Cloudflare પૃષ્ઠો પર જમાવટ માટે તૈયાર ડેમો સાઇટ છે, અને `_redirects` અને `404.html` રાઉટીંગ ફાઇલો આપમેળે જનરેટ થાય છે.
- **NPM પબ્લિશ**: `./sh/npmDist.js` ચલાવો, સ્ક્રિપ્ટ આપમેળે ઘટક અવલંબનને હેન્ડલ કરશે, પેકેજ વર્ઝન નંબરમાં વધારો કરશે અને NPM પર ઘટક લાઇબ્રેરીને પ્રકાશિત કરવા માટે પેકેજિંગ પૂર્ણ થયા પછી એક ક્લિક સાથે `npm publish` એક્ઝિક્યુટ કરશે.

### પ્લેટફોર્મ અનુવાદ માટે જવાબદાર છે

ફ્રેમવર્કમાં બિલ્ટ-ઇન સ્વચાલિત અનુવાદ પ્રક્રિયાઓ છે:
1. પ્રોજેક્ટ રૂટ ડિરેક્ટરીમાં `tran.yml`માં અનુવાદ નિયમો (જેમ કે સ્રોત ભાષા, લક્ષ્ય ભાષા અને નિર્દેશિકા) ગોઠવો.
2. તમારા કોડમાં બહુવિધ ભાષાઓને હાર્ડકોડ કર્યા વિના ઘટકો અથવા દસ્તાવેજોમાં પ્લેસહોલ્ડર્સનો ઉપયોગ કરો.
3. `./sh/tran.js` ચલાવો, અને અનુવાદ એન્જિન આપમેળે ટેક્સ્ટને બહાર કાઢશે અને પ્રક્રિયા માટે વેબસી અનુવાદ સેવાને કૉલ કરશે, વિવિધ ભાષાઓમાં આપમેળે YAML અનુવાદ એન્ટ્રીઓ અને README દસ્તાવેજીકરણ જનરેટ કરશે.

### પર્યાવરણ ચલો

સ્વચાલિત અનુવાદ કરતી વખતે, તમારે અનુવાદ API ની ઍક્સેસને અધિકૃત કરવા માટે નીચેના પર્યાવરણ ચલોને ગોઠવવાની જરૂર છે:

- **`WEBC_SITE_TOKEN`** (અથવા **`WEBC_TOKEN`**): ઓથેન્ટિકેશન કીનો ઉપયોગ `webc.site` સ્વચાલિત અનુવાદ API ને ઍક્સેસ કરવા માટે થાય છે. પર્યાવરણ ચલ તરીકે ઇન્જેક્ટ કરી શકાય છે અથવા સ્થાનિક વપરાશકર્તાની હોમ ડિરેક્ટરીમાં `~/.config/webc.site.yml` કન્ફિગરેશન ફાઇલમાં `token` લખી શકાય છે.
