# route.js

စာမျက်နှာတစ်ခုတည်း ရှေ့ဆုံးလမ်းကြောင်း စီမံခန့်ခွဲမှု

## nowUrl()

ဦးဆောင်မျဥ်းစောင်းများဖယ်ရှားခြင်းဖြင့် လက်ရှိ URL လမ်းကြောင်းကို ရယူပါ။

ပြန်ပို့တန်ဖိုး- လက်ရှိလမ်းကြောင်း string

## route(hook)

လမ်းကြောင်းပြောင်းလဲမှုများအတွက် နားထောင်ခြင်းခေါ်ဆိုမှုကို မှတ်ပုံတင်ပါ။ မှတ်ပုံတင်သည့်အခါ၊ ၎င်းကို လက်ရှိလမ်းကြောင်းဖြင့် တစ်ကြိမ်လုပ်ဆောင်မည်ဖြစ်သည်။

ကန့်သတ်ချက်-
- `hook` : လမ်းကြောင်းပြန်ခေါ်ခြင်းကို စောင့်ကြည့်ခြင်း လုပ်ဆောင်ချက်
  - ကန့်သတ်ချက်
    - `url` : လက်ရှိလမ်းကြောင်း
    - `pre` : နောက်ဆုံးလမ်းကြောင်း

ပြန်ပို့သည့်တန်ဖိုး- စာရင်းသွင်းမထားသည့် သန့်ရှင်းရေးလုပ်ဆောင်မှု

## setPre(url)

နောက်ဆုံးမှတ်တမ်းတင်ထားသော URL ကိုသိမ်းဆည်းပါ။

ကန့်သတ်ချက်-
- `url` : လမ်းကြောင်း စာကြောင်း

## preUrl()

နောက်ဆုံးမှတ်တမ်း၏ URL ကိုရယူပါ။

ပြန်ပို့တန်ဖိုး- နောက်ဆုံးလမ်းကြောင်း စာကြောင်း

## refresh(url)

သတ်မှတ်ထားသောလမ်းကြောင်းတွင် နားထောင်သည့် လမ်းကြောင်းအားလုံးကို ပြန်ခေါ်ခိုင်းပြီး ကြိုတင်မွမ်းမံပါ။

ကန့်သတ်ချက်-
- `url` : ပစ်မှတ်လမ်းကြောင်း၊ မူရင်းလက်ရှိလမ်းကြောင်း

## removeSlash(url)

လမ်းကြောင်းခေါင်းစီးများမှ မျဉ်းစောင်းများကို ဖယ်ရှားပါ။

ကန့်သတ်ချက်-
- `url` : လမ်းကြောင်း စာကြောင်း

တန်ဖိုးကို ပြန်ပို့သည်- ဦးဆောင်မျဉ်းစောင်းကို ဖယ်ရှားပြီးနောက် လမ်းကြောင်း

## setUrl(url)

ဘရောင်ဇာလိပ်စာဘား URL အခြေအနေကို အပ်ဒိတ်လုပ်ပါ။

ကန့်သတ်ချက်-
- `url` : ပစ်မှတ်လမ်းကြောင်း

လမ်းကြောင်းအမှန်တကယ်ပြောင်းလဲပါက ပြန်တန်ဖိုး- 1

## goto(url)

ဘရောင်ဇာလိပ်စာကို အပ်ဒိတ်လုပ်ပြီး လမ်းကြောင်းပြန်လည်ဆန်းသစ်မှုတစ်ခု အစပျိုးပါ။

ကန့်သတ်ချက်-
- `url` : ပစ်မှတ်လမ်းကြောင်း
