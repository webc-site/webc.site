# WebC.site

## ဒါဘာလဲ?

### ၎င်းသည် မူရင်းဝဘ် အစိတ်အပိုင်း စာကြည့်တိုက်တစ်ခု ဖြစ်သည်။

[ဝဘ်အစိတ်အပိုင်းများ](https://developer.mozilla.org/docs/Web/API/Web_components) ကို အခြေခံ၍ တီထွင်ထားခြင်းဖြစ်ပြီး ၎င်းသည် runtime သီးခြားဖြစ်ပြီး framework အားလုံးအတွက် သင့်လျော်သည်။

အရွယ်အစား (အလားတူ လူကြိုက်များသော စာကြည့်တိုက်များ၏ ဆယ်ပုံတစ်ပုံ အရွယ်အစား) အတွက် အလွန်ကောင်းမွန်အောင် ပြုလုပ်ထားသည်။

ဘာသာစကား 75 မျိုးကိုထောက်ပံ့ပေးသည်။ အွန်လိုင်းအသုံးပြုမှုအတွက် js နှင့် css ကို တိုက်ရိုက်တင်သွင်းနိုင်သည် သို့မဟုတ် ဆင့်ပွားဖွံ့ဖြိုးတိုးတက်မှုအတွက် လိုအပ်သလို သင့်ပရောဂျက်သို့ အစိတ်အပိုင်းများကို ထည့်နိုင်သည်။

<img src="https://i-01.eu.org/20260604114250265.avif" width="400">

### ဤအရာသည် စာကြည့်တိုက် ဖွံ့ဖြိုးတိုးတက်ရေး မူဘောင်၏ အစိတ်အပိုင်းဖြစ်သည်။

ဤသည်မှာ [ပုံပြင်စာအုပ်](https://storybook.js.org) ၏ တည်ငြိမ်သောဘလော့ဂ်ဗားရှင်းဖြစ်သည်။

အစိတ်အပိုင်း စာကြည့်တိုက်တစ်ခု တည်ဆောက်ခြင်းသည် ကုမ္ပဏီတွင်း ဖွံ့ဖြိုးတိုးတက်မှု စွမ်းဆောင်ရည်ကို မြှင့်တင်နိုင်ပြီး နည်းပညာဆိုင်ရာ စက်ဝိုင်းတွင် အမှတ်တံဆိပ်ပုံသဏ္ဍာန်ကို တည်ဆောက်နိုင်သည်။

[webc.site](https://github.com/webc-site/webc.site) ကို အခြေခံ၍ အဖွဲ့များသည် ၎င်းတို့၏ကိုယ်ပိုင် အစိတ်အပိုင်း စာကြည့်တိုက်များကို အလွယ်တကူ ဖန်တီးနိုင်ပြီး ၎င်းတို့ကို [github စာမျက်နှာ](https://pages.github.com), [cloudflare စာမျက်နှာ](https://pages.cloudflare.com)၊ စသည်တို့တွင် ထုတ်ဝေနိုင်ပါသည်။ ထုတ်ဝေထားသော အစိတ်အပိုင်းများကို [codepen.io](https://codepen.io) နှင့် [jsfiddle](https://jsfiddle.net) တွင် အွန်လိုင်းတွင် အမှားရှာတွေ့နိုင်ပါသည်။

တစ်ချိန်တည်းမှာပင်၊ built-in ဘာသာပြန်မူဘောင်သည် အစိတ်အပိုင်းများနှင့် စာရွက်စာတမ်းများ၏ နိုင်ငံတကာသို့ ကူးပြောင်းမှုကို လျင်မြန်စွာ သိရှိနိုင်ပြီး၊ ထို့ကြောင့် သင်၏ထုတ်ကုန်ပရိသတ်ကို ဘာသာစကားဖြင့် ချည်နှောင်ခြင်းမပြုတော့ပါ။

### ၎င်းသည် ဉာဏ်ရည်တုခေတ်အတွက် ရှေ့ဆုံးဖွံ့ဖြိုးတိုးတက်မှု၏ ပါရာဒိုင်းအသစ်ဖြစ်သည်။

မူဘောင်သည် `.agents/skills`ထည့်သွင်းထားပြီး၊ အကောင်းဆုံးအလေ့အကျင့်များနှင့်အညီ ကလစ်တစ်ချက်နှိပ်ရုံဖြင့် အစိတ်အပိုင်းအသစ်များကို တီထွင်နိုင်စေရန် ဉာဏ်ရည်တုနည်းပညာကို ပံ့ပိုးပေးသည်။

<img src="https://i-01.eu.org/20260604121408079.avif" width="400">

ရှေ့ဆုံးကုဒ်ကို လေထု၏ ရေရှည်တည်တံ့ခိုင်မြဲသော အမှိုက်တောင်ပေါ်မှ ပရိုဂရမ်ရေးဆွဲခြင်းမှ ရေရှည်တွင် ထိန်းသိမ်းနိုင်ပြီး ထပ်ခါတလဲလဲ ပြောင်းလဲနိုင်သည့် စံအပိုင်းတစ်ခုအဖြစ် ပြောင်းလဲပါစေ။

Artificial Intelligence-Assisted Development ၏ ဝိသေသလက္ခဏာများအားဖြင့် ကျွန်ုပ်တို့သည် ရှေ့ဆုံးအဆင့် ဖွံ့ဖြိုးတိုးတက်မှုဆိုင်ရာ ပါရာဒိုင်းအသစ်ကို အဆိုပြုပြီး ဒီဇိုင်းထုတ်ခဲ့သည်-

1. **အရာအားလုံးသည် ဝဘ်အစိတ်အပိုင်းတစ်ခုဖြစ်သည်**
2. ** အစိတ်အပိုင်းသည် back-end ဒေတာလင့်ခ်မှ သီးခြားဖြစ်ရမည်၊ ဒေတာအင်တာဖေ့စ်ကို ပြန်ခေါ်သည့်လုပ်ဆောင်ချက်အဖြစ် ထင်ရှားစေသည်**

ဤနည်းအားဖြင့်၊ ဉာဏ်ရည်တုသည် ဒေတာအတုများကို ထုတ်လုပ်နိုင်ပြီး သရုပ်ပြစာမျက်နှာတွင် အစိတ်အပိုင်းများ၏ မတူညီသောအခြေအနေများကို ပြသနိုင်သည်။

[webc.site အစိတ်အပိုင်းဘောင်](https://github.com/webc-site/webc.site) အခြားအဓိကအင်္ဂါရပ်နှင့် ပေါင်းစပ်ထားသော အစိတ်အပိုင်းဖွံ့ဖြိုးတိုးတက်မှုနှင့် ဒေတာကြားခံအား ပိုင်းခြားခြင်း-

** အစိတ်အပိုင်းတစ်ခုစီသည် ဖွံ့ဖြိုးတိုးတက်မှုဆာဗာကို လွတ်လပ်စွာစတင်နိုင်ပြီး သရုပ်ပြစာမျက်နှာတွင် အမှားရှာဖန်တီးနိုင်သည်**။

၎င်းသည် အစိတ်အပိုင်းတစ်ခုစီ၏ ဖွံ့ဖြိုးတိုးတက်မှုနှင့် အမှားရှာပြင်ခြင်းအတွက် ရှေ့ဆုံးလင့်ခ်ကို `0`သို့ လျှော့ချပေးသည်။

ဤနည်းဖြင့်၊ ဉာဏ်ရည်တု-အထောက်အကူပြု ဖွံ့ဖြိုးတိုးတက်ရေးကိရိယာများ ([Claude Code](https://code.claude.com), [OpenAI Codex](https://openai.com), [Google Antigravity](https://antigravity.google), [Cursor](https://cursor.com), [Windsurf](https://windsurf.com), [Devin](https://devin.ai), [OpenHands](https://github.com/All-Hands-AI/OpenHands) စသည်ဖြင့်)၊ သင်သည် အပြည့်အဝ အလိုအလျောက် ပြန်လည်ပိတ်ဆို့ခြင်း နှင့် ဒေတာဖွင့်ခြင်း မရှိတော့ဘဲ ဘရောက်ဇာကို အမျိုးမျိုးသော အတားအဆီးများဖြင့် အပြည့်အဝခံစားနိုင်သည် လော့ဂ်အင်အခြေအနေ၊ လမ်းကြောင်းလမ်းကြောင်း စသည်ဖြင့်)။

အကယ်၍ သင်သည် ဉာဏ်ရည်တုကို ထိန်းချုပ်ပြီး ကုန်ကျစရိတ်များကို လျှော့ချရန်နှင့် ထိရောက်မှုကို တိုးမြှင့်လိုပါက၊ အစိတ်အပိုင်းတစ်ခုစီ၏ ဖွံ့ဖြိုးတိုးတက်မှုနှင့် အမှားရှာပြင်ခြင်းပတ်ဝန်းကျင်ကို အနည်းဆုံးဖြစ်စေမည့် ပါရာဒိုင်းအသစ်တစ်ခု လိုအပ်ပါသည်။

# ကျွန်တော် TODO ကို မရေးရသေးဘူး။

### ဝဘ် အစိတ်အပိုင်း စာကြည့်တိုက်

#### Ultimate size optimization

ဥပမာအားဖြင့်၊ virtual scroll bar အစိတ်အပိုင်းအတွက်၊ အောက်ပါဇယားတွင်ပြထားသည့်အတိုင်း `gz`ပြီးနောက် လူကြိုက်များသော [OverlayScrollbars](https://github.com/KingSora/OverlayScrollbars) script နှင့် style ကို နှိုင်းယှဉ်ကြည့်ပါ-

|                   |   js    |  css   |     total      |
| :---------------- | :-----: | :----: | :------------: |
| overlayscrollbars | 14520 B | 2584 B | 17104 B (9.1X) |
| Scroll            | 1159 B  | 721 B  |   ⭐ 1880 B    |

`webc.site` ၏ ပကတိ ရွေ့ပြောင်းဘား အစိတ်အပိုင်းသည် `OverlayScrollbars` ၏ `10%` သာဖြစ်သည်။

#### Runtime သည် သီးခြားဖြစ်သည်။

[Web Components](https://developer.mozilla.org/docs/Web/API/Web_components)၊ webc.site အစိတ်အပိုင်းစာကြည့်တိုက်သည် သဘာဝအတိုင်း runtime-agnostic ဖြစ်သည်။ သင်သည် React၊ Vue၊ Angular၊ Svelte သို့မဟုတ် သမားရိုးကျ သန့်စင်သော HTML/JS ပရောဂျက်ကို အသုံးပြုနေသည်ဖြစ်စေ အစိတ်အပိုင်းအားလုံးသည် သီးခြားဘောင်တစ်ခုအတွက် ဆင့်ပွားထုပ်ပိုးမှုမလိုအပ်ဘဲ မူရင်း HTML တဂ်များကဲ့သို့ ဘရောက်ဆာတွင် တိုက်ရိုက်လည်ပတ်နိုင်သည်။

##### shadcn နှင့် နှိုင်းယှဉ်ခြင်း။

[shadcn/ui](https://ui.shadcn.com) သည် သင့်ကိုယ်ပိုင်ပရောဂျက်သို့ အစိတ်အပိုင်းကုဒ်ကို တိုက်ရိုက်ကူးယူရန် လိုအပ်သည့် ကုဒ်ဖြန့်ဝေမှုမုဒ်ကို လက်ခံသည်။ ၎င်းသည် React ဂေဟစနစ်နှင့် တိကျသော ဆောက်လုပ်ရေးကိရိယာ ကွင်းဆက်များ (Tailwind CSS ကဲ့သို့) နှင့် တင်းတင်းကျပ်ကျပ် ချည်နှောင်ထားပြီး မူဘောင်များတစ်လျှောက်တွင် အသုံးမပြုနိုင်ပါ။ ၎င်းတွင် ပရောဂျက်အင်ဂျင်နီယာဖွဲ့စည်းမှုပုံစံအတွက် အချို့သော လိုအပ်ချက်များလည်း ရှိသည်။

နှင့် `webc.site` ၏ အစိတ်အပိုင်း-
- **Cross-framework universal**- တစ်ကြိမ်ရေးပြီး တိုက်ရိုက်မိတ်ဆက်ပြီး မည်သည့် front-end framework (မူဘောင်ကင်းစင်သော ပတ်ဝန်းကျင်တွင်ပင်) လုပ်ဆောင်နိုင်ပါသည်။
- **ပတ်ဝန်းကျင် လုံးဝမှီခိုမှု**- သီးခြားထုပ်ပိုးမှုကိရိယာများ သို့မဟုတ် CSS ကြိုတင်ပရိုဆက်ဆာများ မလိုအပ်ပါ၊ စုစည်းထားသော static ဖိုင်များကို တင်သွင်းရုံဖြင့် ၎င်းကို ဘောက်စ်အတွင်းမှ အသုံးပြုနိုင်ပါသည်။

## အလိုအလျောက် ဖွံ့ဖြိုးတိုးတက်မှုနှင့် အမှားရှာပြင်ခြင်း။

### ဒေသဖွံ့ဖြိုးရေးနှင့် အမှားရှာပြင်ခြင်း။

ဝဘ်စာမျက်နှာ အစိတ်အပိုင်းတစ်ခုစီသည် ဖွံ့ဖြိုးတိုးတက်မှုဆာဗာကို သီးခြားလွတ်လပ်စွာ စတင်ခြင်းဖြင့် ပံ့ပိုးပေးကာ အနည်းငယ်မျှသာ ဖွံ့ဖြိုးတိုးတက်မှုနှင့် အမှားရှာပြင်သည့် ပတ်ဝန်းကျင်ကို ပံ့ပိုးပေးသည်-

- **အစိတ်အပိုင်းအသစ်တစ်ခုဖန်တီးပါ**- (သရုပ်ပြစာမျက်နှာနှင့် နိုင်ငံတကာပုံစံဖွဲ့စည်းမှုအပါအဝင်) အစိတ်အပိုင်းတစ်ခုအား လျင်မြန်စွာထုတ်လုပ်ရန် **အစိတ်အပိုင်းအသစ်တစ်ခုကို ဖန်တီးပါ**./sh/new.js [css|js] [component name]` ကိုဖွင့်ပါ။
- **လွတ်လပ်သော အမှားရှာပြင်ခြင်း**- လည်ပတ်ရန် `./dev.js [အစိတ်အပိုင်းအမည်]` (သို့မဟုတ် `./dev.js` တိုက်ရိုက်) Vite ဖွံ့ဖြိုးတိုးတက်မှုဆာဗာကို စတင်ရန်နှင့် ဘရောင်ဇာရှိ အစိတ်အပိုင်း၏သရုပ်ပြစာမျက်နှာကို အလိုအလျောက်ဖွင့်ပါ။ အစိတ်အပိုင်း hot update (HMR) ကို ပံ့ပိုးပေးသည်။ ဖွံ့ဖြိုးတိုးတက်မှုနှင့် ဒေတာလင့်ခ်များကို ဖြတ်တောက်ထားသောကြောင့်၊ AI (ဥပမာ Claude Code၊ Google Antigravity စသည်ဖြင့်) အတွက် အလွန်သင့်လျော်သော ဖွံ့ဖြိုးတိုးတက်မှုနှင့် ဘရောက်ဆာ အမှားရှာပြင်ခြင်းအတွက် လုံးဝသင့်လျော်သော သုညဖွံ့ဖြိုးမှု ရှေ့တန်းလင့်ခ် ရှိပါသည်။

### အစိတ်အပိုင်း စာကြည့်တိုက်ကို ထုတ်ဝေပါ။

#### webc dist

ထုပ်ပိုးခြင်းနှင့် ထုတ်ဝေခြင်းလုပ်ငန်းစဉ်သည် အလွန်အလိုအလျောက်လုပ်ဆောင်သည်-
- **Static resource packaging**: Running `./sh/dist.js` (ဆိုလိုတာက၊ `webc dist` လုပ်ငန်းဆောင်တာကို လုပ်ဆောင်ခြင်း) သည် `src/` အောက်ရှိ ဝဘ်စာမျက်နှာ အစိတ်အပိုင်းအားလုံးကို အလိုအလျောက် စကင်န်ဖတ်နိုင်သည်၊ စုစည်း၊ ချုံ့ပြီး ၎င်းတို့ကို `dist/` လမ်းညွှန်သို့ ထုတ်ပေးနိုင်သည်။ ဤလမ်းညွှန်တွင် ထုတ်လုပ်ဖြန့်ချီရန်အတွက် JS/CSS ဖိုင်များ၊ GitHub စာမျက်နှာများ သို့မဟုတ် Cloudflare Pages များသို့ ဖြန့်ကျက်ရန် အသင့်ရှိသော သရုပ်ပြဆိုက်တစ်ခုနှင့် `_redirects` နှင့် `404.html` လမ်းပြခြင်းဖိုင်များကို အလိုအလျောက်ထုတ်ပေးပါသည်။
- **NPM Publish**: Run `./sh/npmDist.js`၊ script သည် အစိတ်အပိုင်း မှီခိုမှုများအား အလိုအလျောက် ကိုင်တွယ်ပေးမည်ဖြစ်ပြီး၊ ပက်ကေ့ဂျ်ဗားရှင်းနံပါတ်ကို တိုးမြှင့်ကာ ထုပ်ပိုးမှုပြီးစီးပြီးနောက် NPM သို့ အစိတ်အပိုင်းစာကြည့်တိုက်ကို ထုတ်ဝေရန် ကလစ်တစ်ချက်နှိပ်ရုံဖြင့် လုပ်ဆောင်မည်ဖြစ်သည်။

### ဘာသာပြန်ခြင်းအတွက် ပလက်ဖောင်းတွင် တာဝန်ရှိသည်။

မူဘောင်တွင် အလိုအလျောက်ဘာသာပြန်ခြင်း လုပ်ငန်းစဉ်များ ပါ၀င်သည်-
1. `tran.yml` ပရောဂျက် အမြစ်လမ်းညွှန်တွင် ဘာသာပြန်ခြင်းဆိုင်ရာ စည်းမျဉ်းများ (အရင်းအမြစ်ဘာသာစကား၊ ပစ်မှတ်ဘာသာစကားနှင့် ဘာသာပြန်ရမည့် လမ်းညွှန်ချက်ကဲ့သို့သော) ဘာသာပြန်စည်းမျဉ်းများကို သတ်မှတ်ပါ။
2. သင့်ကုဒ်တွင် ဘာသာစကားများစွာကို hardcode လုပ်စရာမလိုဘဲ အစိတ်အပိုင်းများ သို့မဟုတ် စာရွက်စာတမ်းများတွင် နေရာကိုင်ဆောင်ထားသူများကို အသုံးပြုပါ။
3. `./sh/tran.js`ကို run ပါ၊ ထို့နောက် ဘာသာပြန်အင်ဂျင်သည် စာသားကို အလိုအလျောက်ထုတ်ယူပြီး လုပ်ဆောင်ရန်အတွက် WebC ဘာသာပြန်ဝန်ဆောင်မှုကို ခေါ်ဆိုမည်ဖြစ်ပြီး၊ YAML ဘာသာပြန်ချက်များနှင့် README စာရွက်စာတမ်းများကို ဘာသာစကားအမျိုးမျိုးဖြင့် အလိုအလျောက်ထုတ်ပေးမည်ဖြစ်သည်။

### ပတ်ဝန်းကျင်ပြောင်းလဲမှုများ

အလိုအလျောက်ဘာသာပြန်ဆိုခြင်းကို လုပ်ဆောင်သည့်အခါ၊ ဘာသာပြန် API ကို ဝင်ရောက်ခွင့်ခွင့်ပြုရန် အောက်ပါပတ်ဝန်းကျင် variable များကို သတ်မှတ်ရန် လိုအပ်သည်-

- **`WEBC_SITE_TOKEN`** (သို့မဟုတ် **`WEBC_TOKEN`**): `webc.site` အလိုအလျောက် ဘာသာပြန်ခြင်း API ကို ဝင်ရောက်ကြည့်ရှုရန် အသုံးပြုသော စစ်မှန်ကြောင်းအထောက်အထားပြသော့။ ပတ်ဝန်းကျင် ပြောင်းလဲနိုင်သော ပုံစံတစ်ခုအဖြစ် ထိုးသွင်းနိုင်သည် သို့မဟုတ် `token` စက်တွင်းအသုံးပြုသူ၏ ပင်မလမ်းညွှန်ရှိ ဖွဲ့စည်းမှုဖိုင်တွင် `~/.config/webc.site.yml` ဖွဲ့စည်းမှုဖိုင်တွင် ရေးသားနိုင်သည်။
