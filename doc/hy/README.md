#### Runtime անկախ

Կառուցված ստանդարտների վրա [Web Components](https://developer.mozilla.org/docs/Web/API/Web_components)՝ webc.site բաղադրիչի գրադարանը, բնականաբար, գործարկման ժամանակի ագնոստիկ է: Անկախ նրանից, թե դուք օգտագործում եք React, Vue, Angular, Svelte կամ ավանդական մաքուր HTML/JS նախագիծ, բոլոր բաղադրիչները կարող են ուղղակիորեն գործարկվել բրաուզերում, ինչպես հայրենի HTML թեգերը, առանց հատուկ շրջանակի համար երկրորդական փաթեթավորման անհրաժեշտության:

##### Համեմատություն shadcn-ի հետ

[shadcn/ui](https://ui.shadcn.com) ընդունում է կոդի բաշխման ռեժիմը, որը պահանջում է, որ դուք պատճենեք բաղադրիչի կոդը անմիջապես ձեր սեփական նախագծի մեջ: Այն սերտորեն կապված է React էկոհամակարգի և հատուկ շինարարական գործիքների շղթաների հետ (օրինակ՝ Tailwind CSS) և չի կարող օգտագործվել շրջանակների միջև: Այն նաև ունի որոշակի պահանջներ նախագծի ինժեներական կազմաձևման համար:

Իսկ `webc.site` բաղադրիչը.
- **Cross-framework universal**. գրել մեկ անգամ և կարող է ուղղակիորեն ներմուծվել և գործարկվել ցանկացած ճակատային շրջանակում (նույնիսկ առանց շրջանակների միջավայրում):
- **Շրջակա միջավայրի զրոյական կախվածություն**. Փաթեթավորման հատուկ գործիքներ կամ CSS նախապրոցեսորներ չեն պահանջվում, պարզապես ներմուծեք կազմված ստատիկ ֆայլերը և այն կարող է օգտագործվել առանց տուփի:

## Ավտոմատ մշակում և վրիպազերծում

### Տեղական զարգացում և վրիպազերծում

Վեբ էջի յուրաքանչյուր բաղադրիչ աջակցում է զարգացման սերվերի ինքնուրույն գործարկումը՝ ապահովելով նվազագույն զարգացման և վրիպազերծման միջավայր.

- **Ստեղծեք նոր բաղադրիչ**. Գործարկեք `./sh/new.js [css|js] [բաղադրիչի անունը]`՝ արագ ստեղծելու բաղադրիչի մշակման կմախք (ներառյալ ցուցադրական էջը և միջազգայնացման կոնֆիգուրացիան):
- **Անկախ վրիպազերծում**. Գործարկեք `./dev.js [բաղադրիչի անունը]` (կամ գործարկեք `./dev.js` ուղղակիորեն) Vite-ի մշակման սերվերը գործարկելու և բրաուզերում բաղադրիչի ցուցադրական էջը ավտոմատ կերպով բացելու համար: Աջակցում է բաղադրիչի տաք թարմացմանը (HMR): Քանի որ մշակման և տվյալների հղումներն անջատված են, գոյություն ունի զրոյական զարգացման ճակատային հղում, որը շատ հարմար է AI-ի համար (օրինակ՝ Claude Code, Google Antigravity և այլն), որպեսզի օգնի լիովին ավտոմատ մշակմանը և դիտարկիչի վրիպազերծմանը:

### Հրապարակեք բաղադրիչի գրադարանը

#### webc dist

Փաթեթավորման և հրապարակման գործընթացը խիստ ավտոմատացված է.
- **Ստատիկ ռեսուրսների փաթեթավորում**. `./sh/dist.js`-ի գործարկումը (այսինքն՝ կատարելով `webc dist` առաջադրանքը) կարող է ավտոմատ կերպով սկանավորել վեբ էջի բոլոր բաղադրիչները `src/`-ում, կազմել, սեղմել և դուրս բերել դրանք `dist/` գրացուցակում: Այս գրացուցակը պարունակում է JS/CSS ֆայլեր՝ արտադրական բաշխման համար, ցուցադրական կայք, որը պատրաստ է տեղակայման GitHub էջերին կամ Cloudflare էջերին, և ավտոմատ ստեղծվող `_redirects` և `404.html` երթուղային ֆայլեր:
- **NPM Publish**. Գործարկեք `./sh/npmDist.js`-ը, սկրիպտը ավտոմատ կերպով կկարգավորի բաղադրիչի կախվածությունները, կավելացնի փաթեթի տարբերակի համարը և մեկ սեղմումով կկատարի `npm publish` փաթեթավորումն ավարտելուց հետո՝ բաղադրիչի գրադարանը NPM-ում հրապարակելու համար:

### Հարթակը պատասխանատու է թարգմանության համար

2. Օգտագործեք տեղապահեր բաղադրիչներում կամ փաստաթղթերում՝ առանց ձեր կոդի մի քանի լեզուների կոշտ կոդավորման:
3. Գործարկեք `./sh/tran.js`-ը, և թարգմանչական շարժիչը ավտոմատ կերպով կհանի տեքստը և կկանչի WebC թարգմանչական ծառայությունը մշակման համար՝ ավտոմատ կերպով ստեղծելով YAML թարգմանության գրառումներ և README փաստաթղթեր տարբեր լեզուներով:

### շրջակա միջավայրի փոփոխականներ

Ավտոմատ թարգմանություն կատարելիս դուք պետք է կարգավորեք հետևյալ միջավայրի փոփոխականները՝ թարգմանության API մուտք գործելու համար.

- **`WEBC_SITE_TOKEN`** (կամ **`WEBC_TOKEN`**). Նույնականացման բանալի, որն օգտագործվում է `webc.site` Ավտոմատացված թարգմանության API մուտք գործելու համար: Կարող է ներարկվել որպես շրջակա միջավայրի փոփոխական կամ գրել `token` `~/.config/webc.site.yml` կազմաձևման ֆայլում տեղական օգտագործողի հիմնական գրացուցակում:
