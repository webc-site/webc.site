# WebC.site

## هي ڇا آهي؟

### هي هڪ مقامي ويب جزو لائبريري آهي

[ويب اجزاء] (/0) جي بنياد تي ترقي ڪئي وئي، اھو رن ٽائم آزاد آھي ۽ سڀني فريم ورڪ لاءِ موزون آھي.

سائيز لاءِ انتهائي بهتر ڪيل (ساڳئي مشهور لائبريرين جو هڪ ڏهين نمبر).

75 ٻولين کي سپورٽ ڪري ٿو. توھان سڌو سنئون درآمد ڪري سگھو ٿا js ۽ css آن لائن استعمال لاءِ، يا پنھنجي پروجيڪٽ ۾ اجزاء شامل ڪريو جيئن ثانوي ترقي لاءِ گهربل هجي.

### هي آهي جزو لائبريري ڊولپمينٽ فريم ورڪ

هي [ڪهاڻي ڪتاب](https://storybook.js.org) جو جامد بلاگ ورزن آهي.

جزوي لائبريري جي تعمير ڪمپني جي اندروني ترقي جي ڪارڪردگي کي بهتر بڻائي سگهي ٿي ۽ ٽيڪنيڪل دائري ۾ هڪ برانڊ تصوير ٺاهي سگھي ٿي.

جي بنياد تي [webc.site](https://github.com/webc-site/webc.site)، ٽيمون آساني سان پنهنجون جزوي لائبريريون ٺاهي سگهن ٿيون ۽ انهن کي [github page](https://pages.github.com)، [cloudflare page](https://pages.cloudflare.com) وغيره تي شايع ڪري سگهن ٿيون. شايع ٿيل جزن کي آن لائن ڊيبگ ڪري سگهجي ٿو [codepen.io](https://codepen.io) ۽ [jsfiddle](https://jsfiddle.net).

ساڳئي وقت، تعمير ٿيل ترجمي واري فريم ورڪ کي جلدي اجزاء ۽ دستاويزن جي بين الاقواميت کي محسوس ڪري سگهي ٿو، انهي ڪري ته توهان جي پيداوار جا سامعين هاڻي ٻولي جي پابند نه آهن.

### هي مصنوعي ذهانت جي دور لاءِ فرنٽ اينڊ ڊولپمينٽ جو هڪ نئون نمونو آهي

فريم ورڪ تعمير ٿيل آهي `.agents/skills`، مصنوعي ذهانت کي توهان جي مدد ڪرڻ جي اجازت ڏئي ٿي نئين اجزاء کي هڪ ڪلڪ سان بهترين عملن جي مطابق.

مصنوعي ذهانت جي مدد سان ترقي جي خاصيتن کي نظر ۾ رکندي، اسان هڪ نئون فرنٽ اينڊ ڊولپمينٽ پيراڊم تجويز ڪيو ۽ ڊزائين ڪيو:

2. ** جزو لازمي طور تي پٺتي پيل ڊيٽا لنڪ کان آزاد هجڻ گهرجي، ۽ ڊيٽا انٽرفيس کي ڪال بيڪ فنڪشن جي طور تي ظاهر ڪيو ويو آهي**

هن طريقي سان، مصنوعي ذهانت جعلي ڊيٽا ٺاهي سگهي ٿي ۽ ڊيمو پيج تي اجزاء جي مختلف رياستن کي ڏيکاري ٿي.

جزو جي ترقي ۽ ڊيٽا انٽرفيس جي علحدگي، [webc.site جزو فريم ورڪ] (/0) سان گڏ هڪ ٻي وڏي خصوصيت:

### ويب اجزاء لائبريري

#### حتمي سائيز جي اصلاح

مثال طور، ورچوئل اسڪرول بار جي جزءَ لاءِ، مشهور [OverlayScrollbars](https://github.com/KingSora/OverlayScrollbars) اسڪرپٽ ۽ انداز جي `gz` کان پوءِ سائيز جو موازنہ ڪريو، جيئن هيٺ ڏنل جدول ۾ ڏيکاريل آهي:

|                   |   js    |  css   |     total      |
| :---------------- | :-----: | :----: | :------------: |
| overlayscrollbars | 14520 B | 2584 B | 17104 B (9.1X) |
| Scroll            | 1159 B  | 721 B  |   ⭐ 1880ع    |

`webc.site` جو ورچوئل اسڪرول بار جو حصو صرف `OverlayScrollbars` جي `10%` آهي.

#### هلندڙ وقت آزاد

معيارن تي ٺهيل [Web Components](https://developer.mozilla.org/docs/Web/API/Web_components)، webc.site جزوي لائبريري قدرتي طور رن ٽائم-اجناسٽڪ آهي. ڇا توهان React، Vue، Angular، Svelte، يا هڪ روايتي خالص HTML/JS پروجيڪٽ استعمال ڪري رهيا آهيو، سڀئي جزا سڌو سنئون برائوزر ۾ هلائي سگهن ٿا جهڙوڪ اصلي HTML ٽيگ، ڪنهن مخصوص فريم ورڪ لاءِ ثانوي پيڪنگنگ جي ضرورت کان سواءِ.

##### shadcn سان ڀيٽ ڪريو

[shadcn/ui](https://ui.shadcn.com) ڪوڊ ڊسٽريبيوشن موڊ کي اختيار ڪري ٿو، جنهن لاءِ توهان کي ضرورت آهي ته جزوي ڪوڊ کي نقل ڪريو سڌو پنهنجي پروجيڪٽ ۾. اهو مضبوطي سان React ecosystem ۽ مخصوص تعميراتي ٽول زنجيرن سان جڙيل آهي (جهڙوڪ Tailwind CSS)، ۽ فريم ورڪ ۾ استعمال نٿو ڪري سگهجي. اهو پڻ آهي ڪجهه گهرجون پروجيڪٽ انجنيئرنگ جي ترتيب لاءِ.

۽ جو جزو `webc.site`:
- **ڪراس فريم ورڪ آفاقي**: هڪ ڀيرو لکو ۽ سڌو سنئون متعارف ڪرائي سگھجي ٿو ۽ ڪنهن به فرنٽ-اينڊ فريم ورڪ ۾ (جيتوڻيڪ فريم ورڪ کان آزاد ماحول) ۾.
- **ماحول صفر انحصار**: ڪو خاص پيڪيجنگ اوزار يا CSS پري پروسيسرز جي ضرورت ناهي، صرف مرتب ٿيل جامد فائلن کي درآمد ڪريو ۽ ان کي دٻي کان ٻاهر استعمال ڪري سگهجي ٿو.

## خودڪار ترقي ۽ ڊيبنگ

### مقامي ترقي ۽ ڊيبنگ

هر ويب پيج جو حصو ڊولپمينٽ سرور کي آزاديءَ سان شروع ڪرڻ جي حمايت ڪري ٿو، گهٽ ۾ گهٽ ترقي ۽ ڊيبگنگ ماحول مهيا ڪري ٿو:

- **نئون جز ٺاهيو**: هلو `./sh/new.js [css|js] [جزو جو نالو]` جلدي ٺاهڻ لاءِ هڪ جزو جي ترقيءَ جو ڍانچو (بشمول ڊيمو پيج ۽ بين الاقوامي ترتيب جي ترتيب).
- **آزاد ڊيبگنگ**: هلايو `./dev.js [جزو جو نالو]` (يا ھلايو `./dev.js` سڌو) Vite ڊولپمينٽ سرور کي شروع ڪرڻ لاءِ ۽ خودڪار طريقي سان برائوزر ۾ جزو جي ڊيمو صفحي کي کوليو. جزو گرم اپڊيٽ (HMR) کي سپورٽ ڪري ٿو. ڇو ته ڊولپمينٽ ۽ ڊيٽا جون لنڪس ڊيڪپل ٿيل آهن، اتي صفر ڊولپمينٽ فرنٽ-اينڊ لنڪ آهي، جيڪو مڪمل طور تي خودڪار ڊولپمينٽ ۽ برائوزر ڊيبگنگ ۾ مدد ڪرڻ لاءِ AI (جهڙوڪ Claude Code، Google Antigravity، وغيره) لاءِ بلڪل موزون آهي.

### اجزاء لائبريري شايع ڪريو

#### webc dist

پيڪنگنگ ۽ پبلشنگ عمل انتهائي خودڪار آهي:
- ** جامد وسيلن جي پيڪنگنگ**: هلائڻ `./sh/dist.js` (يعني `webc dist` ٽاسڪ کي هلائڻ) خودڪار طريقي سان سڀني ويب پيج جي اجزاء کي اسڪين ڪري سگھي ٿو `src/` تحت، مرتب ڪري، ڪمپريس ڪري ۽ ٻاھر ڪڍو انھن کي `dist/` ڊاريڪٽري ۾. ھن ڊاريڪٽري ۾ پيداوار جي ورڇ لاءِ JS/CSS فائلون شامل آھن، ھڪ ڊيمو سائيٽ GitHub Pages يا Cloudflare صفحن تي لڳائڻ لاءِ تيار آھي، ۽ پاڻمرادو ٺاھيل `_redirects` ۽ `404.html` روٽنگ فائلون.
- **NPM Publish**: هلايو `./sh/npmDist.js`، اسڪرپٽ خود بخود اجزاء جي انحصار کي سنڀاليندو، پيڪيج جو ورجن نمبر وڌائيندو، ۽ پيڪنگ مڪمل ٿيڻ کان پوءِ هڪ ڪلڪ سان عمل ڪيو `npm publish` جزو لائبريري کي NPM ۾ شايع ڪرڻ لاءِ.

### پليٽ فارم ترجمي لاء ذميوار آهي

فريم ورڪ ۾ تعمير ٿيل خودڪار ترجمي جا عمل آهن:
1. پروجيڪٽ روٽ ڊاريڪٽري ۾ `tran.yml` ۾ ترجمي جا ضابطا ترتيب ڏيو (جهڙوڪ ماخذ جي ٻولي، ٽارگيٽ ٻولي ۽ ڊاريڪٽري جو ترجمو ڪيو وڃي).
2. توهان جي ڪوڊ ۾ ڪيترن ئي ٻولين کي هارڊ ڪوڊ ڪرڻ جي بغير اجزاء يا دستاويزن ۾ جڳهه هولڊر استعمال ڪريو.
3. هلايو `./sh/tran.js`، ۽ ترجمي واري انجڻ خودڪار طريقي سان ٽيڪسٽ ڪڍندو ۽ پروسيسنگ لاءِ ويب سي ترجمي جي خدمت کي ڪال ڪندو، خودڪار طريقي سان YAML ترجمي جون داخلائون ۽ README دستاويز مختلف ٻولين ۾ ٺاهيندي.

### ماحولياتي متغير

خودڪار ترجمي کي انجام ڏيڻ دوران، توهان کي ترجمي جي API تائين رسائي کي اختيار ڏيڻ لاءِ هيٺين ماحوليات کي ترتيب ڏيڻ جي ضرورت آهي:

- **`WEBC_SITE_TOKEN`** (يا **`WEBC_TOKEN`**): تصديق واري چيڪ استعمال ڪئي وئي `webc.site` خودڪار ترجمو API تائين رسائي لاءِ. انجيڪشن ٿي سگھي ٿو ماحوليات جي متغير جي طور تي يا لکي سگھجي ٿو `token` مقامي صارف جي گهر ڊاريڪٽري ۾ `~/.config/webc.site.yml` ترتيب واري فائل ۾.
