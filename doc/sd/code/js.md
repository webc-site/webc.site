# JS ڪوڊ جي وضاحت

## ڪوڊنگ جو انداز

- سادو، خوبصورت ۽ ڪارائتو، جديد js نحو استعمال ڪندي
- ورهايو گھڻو ڊگھو ڪم، ۽ انٽرفيس ڊيزائن کي گھٽ ملائڻ ۽ اعلي همٿ هجڻ گھرجي.
- arrays + loops/forEach/نقشي وڌيڪ استعمال ڪريو ۽ گھٽ بار بار ڪوڊ لکو
- وڌيڪ تبصرن کان بچڻ لاءِ سادي ڪوڊ ۾ فنڪشنل تبصرا نه لکو. پيچيده منطق ۽ خاص ضرورتن کي سڌريل چيني ۾ بيان ڪيو ويو آهي.
- بار بار ڪوڊ کي ختم ڪيو ويو آهي ڪمن ۾ بيڪارگي کي گهٽائڻ لاءِ. مثال طور، `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` لکي سگھجي ٿو `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()`
- متغير اعلان: هڪ بيان ۾ ڪيترن ئي مسلسل `const` بيانن کي گڏ ڪريو. ان کي ٽن لائينن ۾ لکڻ بدران `const a=1, b=2, c=3;` لکو
- هم وقت سازي: استعمال ڪريو `await`، استعمال کي روڪيو `.then` زنجير ڪالون
- خود بخود استثنيٰ هينڊلنگ ڪوڊ پيدا نه ڪريو، پاڻمرادو نه لکو `try...catch` (`try ... catch` دستي طور تي برقرار رکيو وڃي ٿو، موجود `try catch` کي ختم نه ڪيو وڃي)
- خالص افعال پهريون: صرف خالص افعال لکو، ڪڏهن به ڪلاس
- استعمال ڪريو تير فنڪشن `const funcName = () => {}` ۽ استعمال نه ڪريو `function` لفظ (سواءِ جنريٽر)؛ جيڪڏھن توھان استعمال ڪري سگھوٿا .bind پيرا ميٽرن کي پابند ڪرڻ لاءِ، افعال جي وضاحت ڪرڻ کان پاسو ڪريو
- ڪوڊ ٻيهر استعمال ڪريو: ٻيهر استعمال ڪرڻ تي ڌيان ڏيو، وڌيڪ ننڍڙا ڪم ڪڍو، ۽ وڏي تعداد ۾ ساڳي يا ڪاپي پيسٽ ٿيل ڪوڊ جي جوڙجڪ کان پاسو ڪريو.
- اعتراض جي رسائي: گهربل خاصيتون ڪڍڻ لاءِ تباهي واري اسائنمنٽ کي استعمال ڪرڻ کي ترجيح ڏيو، ڊگھي ۽ نيسٽ ٿيل خاصيتن تائين رسائي حاصل ڪرڻ لاءِ اندروني طور تي ڊاٽن کي بار بار استعمال ڪرڻ کان پاسو ڪريو، ۽ بار بار اختياري زنجير جي فيصلي کي ضم ڪريو
- شيون استعمال نه ڪريو فنڪشن پيٽرولر لاءِ، نه لکو `{a,b,c}`، لکو `a, b, c`؛ جيڪڏهن اتي ڪيترائي اختياري پيرا ميٽرز آهن، استعمال ڪريو [[ترتيب واري شين جو تعداد، ترتيب واري شيون]، [ترتيب واري شين جو تعداد، ترتيب واري شيون]، ..] هن مثال ۾، ترتيب واري شيون عددي تسلسل سان بيان ڪيون ويون آهن.
- گھڻن قدر واپسي لاءِ، استعمال ڪريو صف `[a,b,c]` بدران `{a,b,c}`. جيڪڏهن 3 کان وڌيڪ موٽڻ وارا قدر آهن، عددي مستقل استعمال ڪريو پوزيشن جي معني کي بيان ڪرڻ لاء.
- رياست جي نمائندگي ڪرڻ لاءِ اسٽرنگ جي ڪا ضرورت ناهي، رياست کي بيان ڪرڻ لاءِ عددي مستقل استعمال ڪريو
- ڪابه اسٽرنگ ٽيمپليٽ (``) استعمال ٿيل نه آهي، اسٽرنگ ڪنٽينشن (+) استعمال ٿيل آهي، سواء درآمد جي (Vite جامد تجزيي لاءِ آسان)
- لوپ لاءِ، جيڪڏھن توھان کي سيريل نمبر جي ضرورت آھي، استعمال ڪريو `++i` بدران `i++`
- غير ضروري`?.` کان پاسو ڪريو ۽ دفاعي پروگرامنگ کي روڪيو
- متغير جو نالو جمع ٿيڻ جي ضرورت نه آهي، ۽ `_li` سان ختم ٿئي ٿو هڪ فهرست ظاهر ڪرڻ لاء.

## نالو ڏيڻ جو رواج

- نالو minimalism جي پيروي ڪري ٿو. مختصر پر معنيٰ وارا نالا استعمال ڪريو، مثال طور: استعمال ڪريو `rm` بدران `remove`، `delete`، `del`. تنهن هوندي، توهان کي انتها تي وڃڻ کان به پاسو ڪرڻ گهرجي، مثال طور: هڪ اکر استعمال نه ڪريو `m` بدران `map`
- فعل جي نالي ۾ صرف فعل استعمال ڪرڻ جي ڪوشش ڪريو. جيڪڏهن توهان هڪ لفظ استعمال ڪري سگهو ٿا ان کي ظاهر ڪرڻ لاء، ٻه لفظ استعمال نه ڪريو. اسم فائل جي نالن ۾ ظاهر ٿيندا آهن. جيڪڏھن توھان کي فائل جي نالي ۾ ھڪڙو فعل شامل ڪرڻ جي ضرورت آھي، مھرباني ڪري اسم کي پھريون ۽ فعل کي آخري ڪريو. مثال طور: `profileSet.js` بدران `setProfile.js`
- متغير جا نالا: استعمال ڪريو انڊر اسڪور انداز (snake_case)، مثال طور `user_auth_token`
- فنڪشن جو نالو: استعمال ڪريو لوئر ڪيس ڪيمل ڪيس (ڪيمل ڪيس)، مثال طور `userData`
- فنڪشن پيرا ميٽرز: جيڪڏهن اهو هڪ ڪال بيڪ فنڪشن آهي، ان کي نالو ڏيو ننڍي ڪيمل ڪيس ۾، جهڙوڪ `onChange`
- ماڊل-سطح مسلسل وصفون استعمال ڪن ٿيون اپر ڪيز انڊر لائن اسلوب `UPPER_SNAKE_CASE`
- نه لکو `get`، اهڙيون بي معنيٰ اڳڪٿيون، جهڙوڪ: `getCookie` جي بدران `cookieByHeader` لکو.
- گلوبل/ماڊيول-سطح مستقل: استعمال ڪريو اپر ڪيز انڊر لائن اسلوب (UPPER_SNAKE_CASE)، مثال طور `CODE_TO_ID`، `ID_TO_LANG`

## ماڊلر ميڪانيزم

- درآمد ڪريو: مطالبن تي صحيح طور تي افعال درآمد ڪريو، سڄي ماڊل جي سڌي طرح درآمد کي روڪيو (بچايو `import * as x` يا وڏيون شيون درآمد ڪريو)
- برآمد ڪريو: شين جي برآمد کي بند ڪريو. ايڪسپورٽ افعال ۽ متغيرن کي يونٽ طور، جيئن ته `export const X=1, abc=()=>{};`. برآمد ڪيل مواد کي بيان ڪرڻ لاءِ انھن کي ڪانسٽ + ڪاما سان گڏ ڪرڻ جي ڪوشش ڪريو. جيڪڏهن هڪ فائل ۾ صرف هڪ فنڪشن آهي، استعمال ڪريو `export default`
- جيستائين توهان کي ڪال ڪرڻ جي ضرورت نه آهي ايڪسپورٽ ڊفالٽ فنڪشن اندروني طور تي، پهرين بيان ڪرڻ کان پاسو ڪريو ۽ پوءِ فائل جي آخر ۾ برآمد ڪريو.
- رستو حل: موجوده ڊاريڪٽري جو رستو حاصل ڪرڻ وقت، `import.meta.dirname` استعمال ڪيو وڃي

## غلطي

- اسٽرنگ جي غلطين کان پاسو ڪريو ۽ استعمال ڪريو const استعمال ڪرڻ لاءِ مسلسل نقص ڪوڊ بيان ڪرڻ لاءِ.
- جيڪڏهن توهان کي غلط ڊيٽا جي معلومات واپس ڪرڻ جي ضرورت آهي، استعمال ڪريو [غلطي ڪوڊ، غلطي پيغام،...] هن طريقي سان، غلطي پيغام متن جي وضاحت نه آهي، پر هڪ عددي قدر (جهڙوڪ [FILE_OVERSIZE، file_size، max_size])
