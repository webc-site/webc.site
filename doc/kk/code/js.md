# JS кодының сипаттамасы

## кодтау стилі

- Ең заманауи JS синтаксисін қолдана отырып, қарапайым, талғампаз және тиімді
- Тым ұзақ функцияларды бөліңіз және интерфейс дизайны төмен байланыс және жоғары үйлесімділік болуы керек.
- Массивтер + циклдар/forEach/map көбірек пайдаланыңыз және қайталанбайтын кодты азырақ жазыңыз
- Шамадан тыс түсініктемелерді болдырмау үшін функциялық түсініктемелерді қарапайым кодпен жазбаңыз. Күрделі логика мен арнайы қажеттіліктер таза қытай тілінде түсіндіріледі.
- Қайталанатын код артықшылықты азайту үшін функцияларға абстракцияланады. Мысалы, `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()` түрінде жазылуы мүмкін
- Айнымалылар туралы мәлімдеме: бірнеше дәйекті `const` мәлімдемелерді бір мәлімдемеге біріктіріңіз. Оны үш жолға жазудың орнына `const a=1, b=2, c=3;` деп жазыңыз
- Асинхронды өңдеу: `await` пайдалану, `.then` тізбекті қоңырауларды пайдалануға тыйым салу
- Ерекшелік өңдеу кодын автоматты түрде жасамаңыз, автоматты түрде `try...catch` жазбаңыз (`try ... catch` қолмен сақталады, бар `try catch` жойылмауы керек)
- Алдымен таза функциялар: тек таза функцияларды жазыңыз, ешқашан сыныптар
- `const funcName = () => {}` көрсеткі функцияларын пайдаланыңыз және `function` кілт сөзін пайдаланбаңыз (генераторлардан басқа); параметрлерді байланыстыру үшін .bind пайдалана алсаңыз, функцияларды анықтаудан аулақ болыңыз
- Кодты қайта пайдалану: Қайта пайдалануға назар аударыңыз, көбірек кішігірім функцияларды шығарып алыңыз және ұқсас немесе көшіру-қойылған код құрылымдарының көп санын батыл түрде болдырмаңыз.
- Нысанға қол жеткізу: қажетті атрибуттарды шығару үшін құрылымдық тағайындауды пайдалануды басымдықпен белгілеңіз, терең және кірістірілген атрибуттарға қол жеткізу үшін нүктелерді іштей қайталап пайдаланбаңыз және қайталанатын қосымша тізбекті пайымдауларды біріктіріңіз.
- Функция параметрлері үшін объектілерді пайдаланбаңыз, `{a,b,c}` жазбаңыз, `a, b, c` жазбаңыз; егер көптеген қосымша параметрлер болса, [[конфигурация элементтерінің саны, конфигурация элементтері], [конфигурация элементтерінің саны, конфигурация элементтері],..] пайдаланыңыз.] Бұл парадигмада конфигурация элементтері сандық тұрақтылармен анықталады.
- Көп мәнді қайтарулар үшін `{a,b,c}` орнына `[a,b,c]` массивін пайдаланыңыз. Егер 3-тен көп қайтарылатын мән болса, позициялық мағынаны анықтау үшін сандық тұрақтыларды пайдаланыңыз.
- Күйді көрсету үшін жолдың қажеті жоқ, күйді анықтау үшін сандық тұрақтыларды пайдаланыңыз
- Ешбір жол үлгісі (``) пайдаланылмайды, импорттан басқа жолды біріктіру (+) пайдаланылады (Vite статикалық талдауы үшін ыңғайлы)
- Цикл үшін сериялық нөмір қажет болса, `i++` орнына `++i` пайдаланыңыз.
- Қажетсіз`?.` қолданбаңыз және қорғаныстық бағдарламалауға тыйым салыңыз
- Айнымалы атау көпше болуы қажет емес және тізімді көрсету үшін `_li` белгісімен аяқталады.

## Атау шарты

- Атау минимализмді көздейді. Қысқа, бірақ мағыналы атауларды пайдаланыңыз, мысалы: `remove`, `delete`, `del` орнына `rm` пайдаланыңыз. Дегенмен, шектен шығудан да аулақ болу керек, мысалы: `map` орнына бір әріп `m` қолданбаңыз.
- Функция атауында тек етістіктерді қолдануға тырысыңыз. Егер сіз оны білдіру үшін бір сөзді қолдансаңыз, екі сөзді қолданбаңыз. Зат есімдер файл атауларында көрсетіледі. Егер файл атауына етістік қосу керек болса, алдымен зат есімді, ал соңғы етістікті қойыңыз. Мысалы: `setProfile.js` орнына `profileSet.js`
- Айнымалы атауы: астын сызу мәнерін пайдаланыңыз (snake_case), мысалы `user_auth_token`; айнымалы функция болса, түйе регистрін атау мәнерін пайдаланыңыз
- Функция атауы: кіші әріпті camelCase (camelCase) пайдаланыңыз, мысалы `userData`
- Функция параметрлері: егер бұл кері шақыру функциясы болса, оны `onChange` сияқты кіші әріптермен атаңыз.
- Модуль деңгейіндегі тұрақты анықтамалар `UPPER_SNAKE_CASE` бас әріптің астын сызу мәнерін пайдаланады
- `get`, мағынасыз префикстерді жазбаңыз, мысалы: `getCookie` орнына `cookieByHeader` жазыңыз.
- Жаһандық/модуль деңгейіндегі тұрақтылар: бас әріппен астын сызу стилін пайдаланыңыз (UPPER_SNAKE_CASE), мысалы: `CODE_TO_ID`, `ID_TO_LANG`

## Модульдік механизм

- Импорттау: Функцияларды сұраныс бойынша дәл импорттау, бүкіл модульді тікелей импорттауға тыйым салу (`import * as x` болдырмау немесе үлкен нысандарды импорттау)
- Экспорттау: нысандарды экспорттауды өшіру. Функциялар мен айнымалы мәндерді `export const X=1, abc=()=>{};` сияқты бірлік ретінде экспорттаңыз. Экспортталған мазмұнды жариялау үшін оларды const + үтірмен біріктіріп көріңіз. Егер файлда бір ғана функция болса, `export default` пайдаланыңыз
- Әдепкі экспорттау функциясын ішкі шақыру қажет болмаса, алдымен тұрақты мәндерді жариялаудан, содан кейін оларды файлдың соңында экспорттаудан аулақ болыңыз.
- Жол рұқсаты: ағымдағы каталог жолын алу кезінде `import.meta.dirname` пайдалану керек

## қате

- Жол қателерінен аулақ болыңыз және тұрақты қате кодтарын жариялау үшін const пайдаланыңыз.
- Қате деректер ақпаратын қайтару қажет болса, [қате коды, қате туралы хабар,...] пайдаланыңыз Осылайша, қате туралы хабар мәтіндік сипаттама емес, сандық мән болады (мысалы, [FILE_OVERSIZE, файл_өлшемі, ең үлкен_өлшем])
