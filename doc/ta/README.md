# WebC.site

## என்ன இது?

### இது ஒரு சொந்த வலை கூறு நூலகம்

[இணைய கூறுகள்](https://developer.mozilla.org/docs/Web/API/Web_components) அடிப்படையில் உருவாக்கப்பட்டது, இது இயங்கும் நேரச் சார்பற்றது மற்றும் அனைத்து கட்டமைப்புகளுக்கும் ஏற்றது.

அளவுக்கு மிகவும் உகந்ததாக உள்ளது (ஒத்த பிரபலமான நூலகங்களின் பத்தில் ஒரு பங்கு அளவு).

75 மொழிகளை ஆதரிக்கிறது. ஆன்லைன் பயன்பாட்டிற்காக நீங்கள் நேரடியாக js மற்றும் css ஐ இறக்குமதி செய்யலாம் அல்லது இரண்டாம் நிலை மேம்பாட்டிற்கு தேவையான கூறுகளை உங்கள் திட்டத்தில் சேர்க்கலாம்.

<img src="https://i-01.eu.org/20260604114250265.avif" width="400">

### இது கூறு நூலக மேம்பாட்டு கட்டமைப்பாகும்

இது [கதை புத்தகம்](https://storybook.js.org) இன் நிலையான வலைப்பதிவு பதிப்பு.

ஒரு கூறு நூலகத்தை உருவாக்குவது நிறுவனத்தின் உள் வளர்ச்சி செயல்திறனை மேம்படுத்துவதோடு தொழில்நுட்ப வட்டத்தில் ஒரு பிராண்ட் படத்தை உருவாக்க முடியும்.

[webc.site](https://github.com/webc-site/webc.site) அடிப்படையில், குழுக்கள் தங்களுடைய சொந்த கூறு நூலகங்களை உருவாக்கி அவற்றை [github page](https://pages.github.com), [cloudflare page](https://pages.cloudflare.com) போன்றவற்றில் வெளியிடலாம். வெளியிடப்பட்ட கூறுகளை ஆன்லைனில் [codepen.io](https://codepen.io) மற்றும் [jsfiddle](https://jsfiddle.net) இல் பிழைத்திருத்தம் செய்யலாம்.

அதே நேரத்தில், உள்ளமைக்கப்பட்ட மொழிபெயர்ப்பு கட்டமைப்பானது கூறுகள் மற்றும் ஆவணங்களின் சர்வதேசமயமாக்கலை விரைவாக உணர முடியும், இதனால் உங்கள் தயாரிப்பு பார்வையாளர்கள் இனி மொழிக்கு கட்டுப்பட மாட்டார்கள்.

### இது செயற்கை நுண்ணறிவு சகாப்தத்திற்கான முன்-இறுதி வளர்ச்சியின் புதிய முன்னுதாரணமாகும்

கட்டமைப்பானது `.agents/skills` உள்ளமைந்துள்ளது, சிறந்த நடைமுறைகளின்படி ஒரே கிளிக்கில் புதிய கூறுகளை உருவாக்க உங்களுக்கு உதவ செயற்கை நுண்ணறிவை அனுமதிக்கிறது.

<img src="https://i-01.eu.org/20260604121408079.avif" width="400">

முன்-இறுதி குறியீடு, வளிமண்டல நிரலாக்கத்தின் நீடித்த குப்பை மலையிலிருந்து நிலையான பகுதியாக மாறட்டும், இது நீண்ட காலத்திற்கு பராமரிக்கப்பட்டு மீண்டும் மீண்டும் உருவாக்கப்படும்.

செயற்கை நுண்ணறிவு-உதவி வளர்ச்சியின் சிறப்பியல்புகளின் பார்வையில், நாங்கள் ஒரு புதிய முன்-இறுதி மேம்பாட்டு முன்னுதாரணத்தை முன்மொழிந்து வடிவமைத்தோம்:

1. **எல்லாமே இணைய கூறுகள்**
2. **கூறு பின்-இறுதி தரவு இணைப்பிலிருந்து சுயாதீனமாக இருக்க வேண்டும், மேலும் தரவு இடைமுகம் ஒரு கால்பேக் செயல்பாடாக வெளிப்படும்**

இந்த வழியில், செயற்கை நுண்ணறிவு போலியான தரவை உருவாக்கலாம் மற்றும் டெமோ பக்கத்தில் வெவ்வேறு நிலைகளின் கூறுகளைக் காட்டலாம்.

கூறு மேம்பாடு மற்றும் தரவு இடைமுகத்தை பிரித்தல், [webc.site கூறு கட்டமைப்புடன்](https://github.com/webc-site/webc.site) மற்றொரு முக்கிய அம்சம்:

** ஒவ்வொரு கூறுகளும் டெமோ பக்கத்தில் டெவலப்மெண்ட் சர்வரைத் தொடங்கலாம் மற்றும் டெமோ பக்கத்தில் டெவலப் செய்து பிழைத்திருத்தம் செய்யலாம்**.

இது ஒவ்வொரு கூறுகளின் மேம்பாடு மற்றும் பிழைத்திருத்தத்திற்கான முன்-இறுதி இணைப்பை `0`க்கு குறைக்கிறது.

இந்த வழியில், செயற்கை நுண்ணறிவு-உதவி மேம்பாடு கருவிகள் ([Claude Code](https://code.claude.com), [OpenAI Codex](https://openai.com), [Google Antigravity](https://antigravity.google), [Cursor](https://cursor.com), [Windsurf](https://windsurf.com), [Devin](https://devin.ai), [OpenHands](https://github.com/All-Hands-AI/OpenHands) முதலியன), நீங்கள் டெவலப் மெம்ப் பிளாக் பலவகையான காரணிகளால், டெவலப் மெஸ்ஸி பிளாக் ப்ளௌஸர், இனி தானாக இயங்காது, ரசிக்கலாம் (பின்னணி தரவு, உள்நுழைவு நிலை, பாதை ரூட்டிங், முதலியன).

நீங்கள் செயற்கை நுண்ணறிவைக் கட்டுப்படுத்தவும், செலவுகளைக் குறைக்கவும், செயல்திறனை அதிகரிக்கவும் விரும்பினால், ஒவ்வொரு கூறுகளின் வளர்ச்சி மற்றும் பிழைத்திருத்த சூழலைக் குறைக்கும் புதிய முன்னுதாரணம் உங்களுக்குத் தேவை.

# நான் இன்னும் TODO எழுத ஆரம்பிக்கவில்லை

### வலை கூறு நூலகம்

#### இறுதி அளவு தேர்வுமுறை

எடுத்துக்காட்டாக, விர்ச்சுவல் ஸ்க்ரோல் பார் கூறுகளுக்கு, பின்வரும் அட்டவணையில் காட்டப்பட்டுள்ளபடி, `gz`க்குப் பிறகு பிரபலமான [OverlayScrollbars](https://github.com/KingSora/OverlayScrollbars) ஸ்கிரிப்ட் மற்றும் ஸ்டைலின் அளவை ஒப்பிடவும்:

|                   |   js    |  css   |     total      |
| :---------------- | :-----: | :----: | :------------: |
| overlayscrollbars | 14520 B | 2584 B | 17104 B (9.1X) |
| Scroll            | 1159 B  | 721 B  |   ⭐ 1880 பி    |

`webc.site`ன் மெய்நிகர் ஸ்க்ரோல் பார் கூறு `OverlayScrollbars`ன் `10%` மட்டுமே.

#### இயக்க நேரம் சுயாதீனமானது

தரநிலைகளில் [இணைய கூறுகள்](https://developer.mozilla.org/docs/Web/API/Web_components) கட்டமைக்கப்பட்டுள்ளது, webc.site கூறு நூலகம் இயற்கையாகவே இயக்க நேரம்-அஞ்ஞானமானது. நீங்கள் React, Vue, Angular, Svelte அல்லது பாரம்பரிய தூய HTML/JS திட்டத்தைப் பயன்படுத்தினாலும், ஒரு குறிப்பிட்ட கட்டமைப்பிற்கான இரண்டாம் நிலை பேக்கேஜிங் தேவையில்லாமல், அனைத்து கூறுகளும் சொந்த HTML குறிச்சொற்கள் போன்ற உலாவியில் நேரடியாக இயங்க முடியும்.

##### shadcn உடன் ஒப்பீடு

[shadcn/ui](https://ui.shadcn.com) குறியீடு விநியோக பயன்முறையை ஏற்றுக்கொள்கிறது, இதற்கு நீங்கள் கூறு குறியீட்டை நேரடியாக உங்கள் சொந்த திட்டத்தில் நகலெடுக்க வேண்டும். இது ரியாக்ட் சுற்றுச்சூழல் அமைப்பு மற்றும் குறிப்பிட்ட கட்டுமான கருவி சங்கிலிகளுடன் (டெயில்விண்ட் சிஎஸ்எஸ் போன்றவை) இறுக்கமாக பிணைக்கப்பட்டுள்ளது, மேலும் கட்டமைப்புகள் முழுவதும் பயன்படுத்த முடியாது. திட்டப் பொறியியல் கட்டமைப்பிற்கான சில தேவைகளையும் கொண்டுள்ளது.

மற்றும் `webc.site`ன் கூறு:
- **கிராஸ்-ஃப்ரேம்வொர்க் யுனிவர்சல்**: ஒருமுறை எழுதுங்கள் மற்றும் நேரடியாக அறிமுகப்படுத்தப்பட்டு எந்த முன்-இறுதி கட்டமைப்பிலும் இயக்கலாம் (கட்டமைப்பு இல்லாத சூழல் கூட).
- **சுற்றுச்சூழல் பூஜ்ஜிய சார்பு**: குறிப்பிட்ட பேக்கேஜிங் கருவிகள் அல்லது CSS முன்செயலிகள் தேவையில்லை, தொகுக்கப்பட்ட நிலையான கோப்புகளை இறக்குமதி செய்து அதை பெட்டிக்கு வெளியே பயன்படுத்தலாம்.

## தானியங்கு மேம்பாடு மற்றும் பிழைத்திருத்தம்

### உள்ளூர் மேம்பாடு மற்றும் பிழைத்திருத்தம்

ஒவ்வொரு இணையப் பக்க கூறுகளும் வளர்ச்சி சேவையகத்தை சுயாதீனமாக தொடங்குவதை ஆதரிக்கிறது, இது ஒரு குறைந்தபட்ச வளர்ச்சி மற்றும் பிழைத்திருத்த சூழலை வழங்குகிறது:

- **புதிய கூறுகளை உருவாக்கவும்**: ஒரு கூறு மேம்பாட்டு எலும்புக்கூட்டை (டெமோ பக்கம் மற்றும் சர்வதேசமயமாக்கல் உள்ளமைவு உட்பட) விரைவாக உருவாக்க `./sh/new.js [css|js] [கூறு பெயர்]` ஐ இயக்கவும்.
- **சுதந்திர பிழைத்திருத்தம்**: வைட் டெவலப்மெண்ட் சர்வரைத் தொடங்க `./dev.js [கூறு பெயர்]` (அல்லது நேரடியாக `./dev.js` இயக்கவும்) இயக்கவும் மற்றும் தானாக உலாவியில் கூறுகளின் டெமோ பக்கத்தைத் திறக்கவும். கூறு சூடான புதுப்பிப்பை (HMR) ஆதரிக்கிறது. மேம்பாடு மற்றும் தரவு இணைப்புகள் துண்டிக்கப்பட்டதால், பூஜ்ஜிய வளர்ச்சி முன்-இறுதி இணைப்பு உள்ளது, இது AI க்கு மிகவும் பொருத்தமானது (கிளாட் குறியீடு, கூகுள் ஆண்டிகிராவிட்டி போன்றவை) முழு தானியங்கி மேம்பாடு மற்றும் உலாவி பிழைத்திருத்தத்திற்கு உதவுகின்றன.

### கூறு நூலகத்தை வெளியிடவும்

#### webc dist

பேக்கேஜிங் மற்றும் வெளியீட்டு செயல்முறை மிகவும் தானியங்கி முறையில் உள்ளது:
- **நிலையான ஆதார பேக்கேஜிங்**: `./sh/dist.js`ஐ இயக்குவது (அதாவது, `webc dist` பணியை செயல்படுத்துவது) `src/`ன் கீழ் உள்ள அனைத்து வலைப்பக்க கூறுகளையும் தானாகவே ஸ்கேன் செய்து, அவற்றை `dist/` கோப்பகத்தில் தொகுத்து, சுருக்கி வெளியிடலாம். இந்த கோப்பகத்தில் உற்பத்தி விநியோகத்திற்கான JS/CSS கோப்புகள் உள்ளன, இது GitHub பக்கங்கள் அல்லது Cloudflare பக்கங்களுக்குப் பயன்படுத்துவதற்குத் தயாராக இருக்கும் டெமோ தளம், மேலும் தானாகவே `_redirects` மற்றும் `404.html` ரூட்டிங் கோப்புகளை உருவாக்குகிறது.
- **NPM வெளியீடு**: `./sh/npmDist.js`ஐ இயக்கவும், ஸ்கிரிப்ட் தானாகவே கூறு சார்புகளைக் கையாளும், தொகுப்பு பதிப்பு எண்ணை அதிகரிக்கும், மேலும் NPM இல் கூறு நூலகத்தை வெளியிட பேக்கேஜிங் முடிந்ததும் ஒரே கிளிக்கில் `npm publish`ஐ இயக்கும்.

### மொழிபெயர்ப்பிற்கு தளம் பொறுப்பாகும்

கட்டமைப்பில் உள்ளமைக்கப்பட்ட தானியங்கு மொழிபெயர்ப்பு செயல்முறைகள் உள்ளன:
1. திட்ட ரூட் கோப்பகத்தில் `tran.yml` இல் மொழிபெயர்ப்பு விதிகளை (மூல மொழி, இலக்கு மொழி மற்றும் மொழிபெயர்க்க வேண்டிய அடைவு போன்றவை) உள்ளமைக்கவும்.
2. உங்கள் குறியீட்டில் பல மொழிகளை ஹார்ட்கோட் செய்யாமல் கூறுகள் அல்லது ஆவணங்களில் பிளேஸ்ஹோல்டர்களைப் பயன்படுத்தவும்.
3. `./sh/tran.js`ஐ இயக்கவும், மொழிபெயர்ப்பு இயந்திரம் தானாகவே உரையைப் பிரித்தெடுத்து, செயலாக்கத்திற்காக WebC மொழிபெயர்ப்புச் சேவையை அழைக்கும், பல்வேறு மொழிகளில் YAML மொழிபெயர்ப்பு உள்ளீடுகள் மற்றும் README ஆவணங்களை தானாக உருவாக்கும்.

### சுற்றுச்சூழல் மாறிகள்

தானியங்கு மொழிபெயர்ப்பைச் செய்யும்போது, ​​மொழிபெயர்ப்பு APIக்கான அணுகலை அங்கீகரிக்க பின்வரும் சூழல் மாறிகளை நீங்கள் கட்டமைக்க வேண்டும்:

- **`WEBC_SITE_TOKEN`** (அல்லது **`WEBC_TOKEN`**): `webc.site` தானியங்கு மொழிபெயர்ப்பு API ஐ அணுகுவதற்கு அங்கீகார விசை பயன்படுத்தப்படுகிறது. சூழல் மாறியாக உட்செலுத்தப்படலாம் அல்லது உள்ளூர் பயனரின் முகப்பு கோப்பகத்தில் உள்ள `~/.config/webc.site.yml` உள்ளமைவு கோப்பில் `token` என்று எழுதலாம்.
