# JS குறியீடு விவரக்குறிப்பு

## குறியீட்டு பாணி

- எளிமையான, நேர்த்தியான மற்றும் திறமையான, நவீன js தொடரியல் பயன்படுத்தி
- மிக நீண்ட செயல்பாடுகளை பிரித்து, இடைமுக வடிவமைப்பு குறைந்த இணைப்பு மற்றும் அதிக ஒருங்கிணைப்பு இருக்க வேண்டும்.
- வரிசைகள் + லூப்கள்/ஒவ்வொருவருக்கும்/வரைபடம் அதிகம் பயன்படுத்தவும் மற்றும் குறைவான மீண்டும் மீண்டும் வரும் குறியீட்டை எழுதவும்
- அதிகப்படியான கருத்துகளைத் தவிர்க்க எளிய குறியீட்டில் செயல்பாடு கருத்துகளை எழுத வேண்டாம். சிக்கலான தர்க்கம் மற்றும் சிறப்புத் தேவைகள் சுத்திகரிக்கப்பட்ட சீன மொழியில் விளக்கப்பட்டுள்ளன.
- பணிநீக்கத்தைக் குறைப்பதற்காக மீண்டும் மீண்டும் குறியீடு செயல்பாடுகளில் சுருக்கப்படுகிறது. எடுத்துக்காட்டாக, `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` என்பதை `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()` என்று எழுதலாம்
- மாறி அறிவிப்பு: பல தொடர்ச்சியான `const` அறிவிப்புகளை ஒரு அறிக்கையாக இணைக்கவும். மூன்று வரிகளில் எழுதுவதற்குப் பதிலாக, `const a=1, b=2, c=3;` என்று எழுதவும்
- ஒத்திசைவற்ற செயலாக்கம்: `await`ஐப் பயன்படுத்தவும், `.then` தொடர் அழைப்புகளைப் பயன்படுத்துவதைத் தடை செய்யவும்
- விதிவிலக்கு கையாளுதல் குறியீட்டை தானாக உருவாக்க வேண்டாம், தானாகவே `try...catch` எழுத வேண்டாம் (`try ... catch` கைமுறையாக பராமரிக்கப்படுகிறது, ஏற்கனவே உள்ள `try catch` நீக்கப்படக்கூடாது)
- முதலில் தூய செயல்பாடுகள்: தூய்மையான செயல்பாடுகளை மட்டும் எழுதுங்கள், வகுப்புகள் இல்லை
- அம்பு செயல்பாடுகளை `const funcName = () => {}` பயன்படுத்தவும் மற்றும் `function` முக்கிய சொல்லைப் பயன்படுத்த வேண்டாம் (ஜெனரேட்டர்கள் தவிர); நீங்கள் அளவுருக்களை பிணைக்க .bind ஐப் பயன்படுத்தினால், செயல்பாடுகளை வரையறுப்பதைத் தவிர்க்கவும்
- குறியீடு மறுபயன்பாடு: மீண்டும் பயன்படுத்துவதில் கவனம் செலுத்துங்கள், அதிக சிறிய செயல்பாடுகளைப் பிரித்தெடுக்கவும், மேலும் அதிக எண்ணிக்கையிலான ஒத்த அல்லது நகலெடுத்து ஒட்டப்பட்ட குறியீடு கட்டமைப்புகளைத் தவிர்க்கவும்.
- பொருள் அணுகல்: தேவையான பண்புக்கூறுகளைப் பிரித்தெடுக்க, டிஸ்ட்ரக்ச்சரிங் அசைன்மென்ட்டைப் பயன்படுத்துவதற்கு முன்னுரிமை கொடுங்கள், ஆழமான மற்றும் உள்ளமைக்கப்பட்ட பண்புக்கூறுகளை அணுக, மீண்டும் மீண்டும் புள்ளிகளைப் பயன்படுத்துவதைத் தவிர்க்கவும், மேலும் மீண்டும் மீண்டும் விருப்பத் தொடர் தீர்ப்புகளை ஒன்றிணைக்கவும்.
- செயல்பாடு அளவுருக்களுக்கு பொருட்களைப் பயன்படுத்த வேண்டாம், `{a,b,c}` எழுத வேண்டாம், `a, b, c` எழுதவும்; பல விருப்ப அளவுருக்கள் இருந்தால், [[உள்ளமைவு உருப்படிகளின் எண்ணிக்கை, உள்ளமைவு உருப்படிகள்], [உள்ளமைவு உருப்படிகளின் எண்ணிக்கை, உள்ளமைவு உருப்படிகள்],..] இந்த முன்னுதாரணத்தில், உள்ளமைவு உருப்படிகள் எண் மாறிலிகளுடன் வரையறுக்கப்படுகின்றன.
- பல-மதிப்பு வருமானத்திற்கு, `{a,b,c}`க்குப் பதிலாக `[a,b,c]` வரிசையைப் பயன்படுத்தவும். 3 க்கும் மேற்பட்ட வருவாய் மதிப்புகள் இருந்தால், நிலை அர்த்தத்தை வரையறுக்க எண் மாறிலிகளைப் பயன்படுத்தவும்.
- மாநிலத்தை குறிக்க ஒரு சரம் தேவையில்லை, மாநிலத்தை வரையறுக்க எண் மாறிலிகளைப் பயன்படுத்தவும்
- சரம் டெம்ப்ளேட் (``) பயன்படுத்தப்படவில்லை, சரம் இணைப்பு (+) பயன்படுத்தப்படுகிறது, இறக்குமதி தவிர (வைட் நிலையான பகுப்பாய்விற்கு வசதியானது)
- லூப்பிற்கு, உங்களுக்கு வரிசை எண் தேவைப்பட்டால், `i++`க்குப் பதிலாக `++i` ஐப் பயன்படுத்தவும்
- தேவையற்ற`?.`ஐத் தவிர்க்கவும் மற்றும் தற்காப்பு நிரலாக்கத்தைத் தடை செய்யவும்
- மாறி பெயர் பன்மையாக இருக்க வேண்டிய அவசியமில்லை, பட்டியலைக் குறிக்க `_li` உடன் முடிவடையும்.

## பெயரிடும் மரபு

- பெயரிடுதல் மினிமலிசத்தைப் பின்பற்றுகிறது. குறுகிய ஆனால் அர்த்தமுள்ள பெயர்களைப் பயன்படுத்தவும், எடுத்துக்காட்டாக: `remove`க்குப் பதிலாக `rm`, `delete`, `del` ஐப் பயன்படுத்தவும். இருப்பினும், நீங்கள் உச்சநிலைக்குச் செல்வதைத் தவிர்க்க வேண்டும், எடுத்துக்காட்டாக: `map` என்பதற்குப் பதிலாக `m` என்ற ஒற்றை எழுத்தைப் பயன்படுத்த வேண்டாம்.
- செயல்பாட்டு பெயரிடலில் வினைச்சொற்களை மட்டுமே பயன்படுத்த முயற்சிக்கவும். அதை வெளிப்படுத்த நீங்கள் ஒரு வார்த்தையைப் பயன்படுத்தினால், இரண்டு வார்த்தைகளைப் பயன்படுத்த வேண்டாம். பெயர்ச்சொற்கள் கோப்பு பெயர்களில் பிரதிபலிக்கின்றன. கோப்பு பெயரில் ஒரு வினைச்சொல் சேர்க்க வேண்டும் என்றால், முதலில் பெயர்ச்சொல்லையும் கடைசியாக வினைச்சொல்லையும் வைக்கவும். எடுத்துக்காட்டாக: `setProfile.js`க்கு பதிலாக `profileSet.js`
- மாறி பெயர்: அடிக்கோடிட்டு நடை (snake_case) பயன்படுத்தவும், எடுத்துக்காட்டாக `user_auth_token`; மாறி ஒரு செயல்பாடாக இருந்தால், சிற்றெழுத்து ஒட்டக வழக்கு பெயரிடும் பாணியைப் பயன்படுத்தவும்
- செயல்பாட்டின் பெயர்: சிற்றெழுத்து ஒட்டகத்தை (camelCase) பயன்படுத்தவும், எடுத்துக்காட்டாக `userData`
- செயல்பாட்டு அளவுருக்கள்: இது ஒரு திரும்ப அழைக்கும் செயல்பாடாக இருந்தால், `onChange` போன்ற சிறிய எழுத்து ஒட்டகத்தில் பெயரிடவும்
- தொகுதி-நிலை நிலையான வரையறைகள் பெரிய எழுத்து அடிக்கோடி நடை `UPPER_SNAKE_CASE`ஐப் பயன்படுத்துகின்றன
- `get` போன்ற அர்த்தமற்ற முன்னொட்டுகளை எழுத வேண்டாம்: `getCookie` என்பதற்கு பதிலாக `cookieByHeader` என்று எழுதவும்
- உலகளாவிய/தொகுதி-நிலை மாறிலிகள்: பெரிய எழுத்து நடையை (UPPER_SNAKE_CASE) பயன்படுத்தவும், எ.கா. `CODE_TO_ID`, `ID_TO_LANG`

## மாடுலர் மெக்கானிசம்

- இறக்குமதி: தேவைக்கேற்ப துல்லியமாக செயல்பாடுகளை இறக்குமதி செய்யவும், முழு தொகுதியின் நேரடி இறக்குமதியை தடை செய்யவும் (`import * as x` அல்லது பெரிய பொருட்களை இறக்குமதி செய்வதைத் தவிர்க்கவும்)
- ஏற்றுமதி: பொருட்களின் ஏற்றுமதியை முடக்கு. `export const X=1, abc=()=>{};` போன்ற செயல்பாடுகள் மற்றும் மாறிகளை அலகுகளாக ஏற்றுமதி செய்யவும். ஏற்றுமதி செய்யப்பட்ட உள்ளடக்கத்தை அறிவிக்க, அவற்றை const + கமாவுடன் இணைக்க முயற்சிக்கவும். ஒரு கோப்பில் ஒரே ஒரு செயல்பாடு இருந்தால், `export default`ஐப் பயன்படுத்தவும்
- நீங்கள் ஏற்றுமதி இயல்புநிலை செயல்பாட்டை உள்நாட்டில் அழைக்க வேண்டும் எனில், முதலில் மாறிலிகளை அறிவித்து பின்னர் கோப்பின் முடிவில் அவற்றை ஏற்றுமதி செய்வதைத் தவிர்க்கவும்.
- பாதைத் தீர்மானம்: தற்போதைய அடைவுப் பாதையைப் பெறும்போது, ​​`import.meta.dirname` பயன்படுத்தப்பட வேண்டும்
