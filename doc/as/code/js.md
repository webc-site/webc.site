# JS ক'ড স্পেচিফিকেশন

## ক'ডিং শৈলী

- সৰল, মাৰ্জিত আৰু দক্ষ, অতি আধুনিক js বাক্যবিন্যাস ব্যৱহাৰ কৰি
- অতি দীঘল ফলনসমূহ বিভাজিত কৰক, আৰু আন্তঃপৃষ্ঠৰ ডিজাইন কম সংযোজন আৰু উচ্চ সংহতি হ'ব লাগে।
- এৰে + লুপ/forEach/মেপ অধিক ব্যৱহাৰ কৰক আৰু কম পুনৰাবৃত্তিমূলক ক'ড লিখক
- অত্যধিক মন্তব্যৰ পৰা হাত সাৰিবলৈ ফাংচনৰ মন্তব্য সহজ ক'ডত লিখিব নালাগে। জটিল যুক্তি আৰু বিশেষ প্ৰয়োজনীয়তাসমূহ পৰিশোধিত চীনা ভাষাত ব্যাখ্যা কৰা হৈছে।
- পুনৰাবৃত্তিমূলক ক'ডক ফাংচনত বিমূৰ্ত কৰা হয় যাতে অতিৰিক্ততা হ্ৰাস পায়। উদাহৰণস্বৰূপে, `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })`ক `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()` হিচাপে লিখিব পাৰি
- চলক ঘোষণা: একাধিক একেৰাহে `const` ঘোষণাক এটা বিবৃতিত একত্ৰিত কৰক। তিনিটা শাৰীত লিখাৰ পৰিৱৰ্তে `const a=1, b=2, c=3;` লিখক
- এচিনক্ৰ'নাছ প্ৰচেছিং: `await` ব্যৱহাৰ কৰক, `.then` শৃংখল কলৰ ব্যৱহাৰ নিষিদ্ধ কৰক
- স্বয়ংক্ৰিয়ভাৱে ব্যতিক্ৰম নিয়ন্ত্ৰণ ক'ড সৃষ্টি নকৰিব, স্বয়ংক্ৰিয়ভাৱে `try...catch` লিখিব নালাগে (`try ... catch` হস্তচালিতভাৱে ৰক্ষণাবেক্ষণ কৰা হয়, বৰ্তমানৰ `try catch` মচি পেলোৱা উচিত নহয়)
- প্ৰথমে বিশুদ্ধ ফাংচন: কেৱল বিশুদ্ধ ফাংচন লিখক, কেতিয়াও ক্লাছ লিখক
- কাঁড় চিহ্নসমূহ `const funcName = () => {}` ব্যৱহাৰ কৰক আৰু `function` চাবিশব্দ ব্যৱহাৰ নকৰিব (জেনেৰেটৰৰ বাহিৰে); যদি আপুনি প্ৰাচলসমূহ বান্ধিবলে .bind ব্যৱহাৰ কৰিব পাৰে, ফলনসমূহ সংজ্ঞায়িত কৰা এৰক
- ক'ড পুনৰ ব্যৱহাৰ: পুনৰ ব্যৱহাৰত মনোযোগ দিয়ক, অধিক সৰু ফাংচনসমূহ এক্সট্ৰেক্ট কৰক, আৰু বৃহৎ সংখ্যক একেধৰণৰ বা কপি-পেষ্ট কৰা ক'ড গঠনসমূহ দৃঢ়ভাৱে এৰাই চলক।
- বস্তু অভিগম: প্ৰয়োজনীয় বৈশিষ্ট্যসমূহ এক্সট্ৰেক্ট কৰিবলে ডিষ্ট্ৰাকচাৰিং এচাইনমেণ্ট ব্যৱহাৰ কৰাটো অগ্ৰাধিকাৰ দিয়ক, গভীৰ আৰু নেষ্টেড বৈশিষ্ট্যসমূহ অভিগম কৰিবলে বিন্দুসমূহ আভ্যন্তৰীণভাৱে বাৰে বাৰে ব্যৱহাৰ কৰা এৰক, আৰু পুনৰাবৃত্তিমূলক বৈকল্পিক শৃংখল বিচাৰসমূহ একত্ৰিত কৰক
- ফাংচন প্ৰাচলৰ বাবে বস্তু ব্যৱহাৰ নকৰিব, `{a,b,c}` লিখক, `a, b, c` লিখক; যদি বহুতো বৈকল্পিক প্ৰাচল আছে, [[বিন্যাস বস্তুৰ সংখ্যা, বিন্যাস বস্তু], [বিন্যাস বস্তুৰ সংখ্যা, বিন্যাস বস্তু],..] ব্যৱহাৰ কৰক এই আৰ্হিত, বিন্যাস বস্তুসমূহক সংখ্যাগত ধ্ৰুৱকসমূহৰ সৈতে সংজ্ঞায়িত কৰা হয়
- বহু-মান ঘূৰাই দিয়াৰ বাবে, `{a,b,c}` ৰ পৰিবৰ্তে `[a,b,c]` এৰে ব্যৱহাৰ কৰক। যদি ৩টাতকৈ অধিক ৰিটাৰ্ণ মান আছে, তেন্তে অৱস্থান অৰ্থ সংজ্ঞায়িত কৰিবলৈ সংখ্যাগত ধ্ৰুৱক ব্যৱহাৰ কৰক।
- অৱস্থাটোক প্ৰতিনিধিত্ব কৰিবলৈ এটা ষ্ট্ৰিঙৰ প্ৰয়োজন নাই, অৱস্থাটো সংজ্ঞায়িত কৰিবলৈ সংখ্যাগত ধ্ৰুৱক ব্যৱহাৰ কৰক
- কোনো ষ্ট্ৰিং টেমপ্লেট (``) ব্যৱহাৰ কৰা হোৱা নাই, ষ্ট্ৰিং সংযোগ (+) ব্যৱহাৰ কৰা হয়, আমদানিৰ বাহিৰে (Vite স্থিতিশীল বিশ্লেষণৰ বাবে সুবিধাজনক)
- লুপৰ বাবে, যদি আপুনি এটা ক্ৰমিক নম্বৰৰ প্ৰয়োজন, `i++` ৰ পৰিবৰ্তে `++i` ব্যৱহাৰ কৰক
- অপ্ৰয়োজনীয়`?.` এৰক আৰু প্ৰতিৰক্ষামূলক প্ৰগ্ৰেমিং নিষিদ্ধ কৰক
- চলকৰ নাম বহুবচন হোৱাৰ প্ৰয়োজন নাই, আৰু এটা তালিকা সূচাবলৈ `_li` ৰে শেষ হয়।

## নামকৰণৰ নিয়ম

- নামকৰণে নূন্যতমতাক অনুসৰণ কৰে। চুটি কিন্তু অৰ্থপূৰ্ণ নাম ব্যৱহাৰ কৰক, যেনে: `remove`, `delete`, `del`ৰ পৰিবৰ্তে `rm` ব্যৱহাৰ কৰক। কিন্তু, আপুনি চৰম পৰ্যায়লৈ যোৱাটোও এৰাই চলিব লাগে, যেনে: `map`ৰ পৰিবৰ্তে এটা আখৰ `m` ব্যৱহাৰ নকৰিব
- ফাংচন নামকৰণত কেৱল ক্ৰিয়া ব্যৱহাৰ কৰিবলৈ চেষ্টা কৰক। প্ৰকাশ কৰিবলৈ যদি এটা শব্দ ব্যৱহাৰ কৰিব পাৰে তেন্তে দুটা শব্দ ব্যৱহাৰ নকৰিব। বিশেষ্য ফাইলৰ নামত প্ৰতিফলিত হয়। যদি আপুনি ফাইলৰ নামত কোনো ক্ৰিয়া যোগ কৰিব লাগে তেন্তে অনুগ্ৰহ কৰি বিশেষ্যটো প্ৰথম আৰু ক্ৰিয়াটো শেষত ৰাখক। উদাহৰণস্বৰূপ: `setProfile.js` ৰ পৰিবৰ্তে `profileSet.js`
- চলকৰ নাম: আণ্ডাৰস্কোৰ শৈলী (snake_case) ব্যৱহাৰ কৰক, উদাহৰণস্বৰূপ `user_auth_token`
- কাৰ্য্যৰ নাম: সৰু আখৰ camelCase (camelCase) ব্যৱহাৰ কৰক, উদাহৰণস্বৰূপ `userData`
- ফাংচন প্ৰাচলসমূহ: যদি ই এটা কলবেক ফাংচন, ইয়াক সৰু আখৰ উটৰ আখৰেৰে নাম দিয়ক, যেনে `onChange`
- মডিউল-স্তৰৰ ধ্ৰুৱক সংজ্ঞাসমূহে ডাঙৰ আখৰৰ আণ্ডাৰলাইন শৈলী `UPPER_SNAKE_CASE` ব্যৱহাৰ কৰে
- `get`, এনে অৰ্থহীন উপসৰ্গ, যেনে: `getCookie`ৰ পৰিৱৰ্তে `cookieByHeader` লিখক
- গ্লোবেল/মডিউল-স্তৰৰ ধ্ৰুৱকসমূহ: ডাঙৰ আখৰৰ আণ্ডাৰলাইন শৈলী (UPPER_SNAKE_CASE) ব্যৱহাৰ কৰক, যেনে। `CODE_TO_ID`, `ID_TO_LANG`

## মডুলাৰ ব্যৱস্থা

- আমদানি: চাহিদা অনুসৰি সঠিকভাৱে কাৰ্য্যসমূহ আমদানি কৰক, সম্পূৰ্ণ মডিউলৰ প্ৰত্যক্ষ আমদানি নিষিদ্ধ কৰক (`import * as x` এৰাই চলিব বা ডাঙৰ বস্তু আমদানি কৰক)
- ৰপ্তানি: বস্তুৰ ৰপ্তানি নিষ্ক্ৰিয় কৰক। ফলন আৰু চলকসমূহক একক হিচাপে ৰপ্তানি কৰক, যেনে `export const X=1, abc=()=>{};`। ৰপ্তানি কৰা বিষয়বস্তু ঘোষণা কৰিবলৈ এটা const + কমাৰ সৈতে সিহতক একত্ৰিত কৰিবলৈ চেষ্টা কৰক। যদি এটা ফাইলৰ মাত্ৰ এটা ফলন আছে, `export default` ব্যৱহাৰ কৰক
- যেতিয়ালৈকে আপুনি ৰপ্তানি অবিকল্পিত ফলনক আভ্যন্তৰীণভাৱে কল কৰিব নালাগে, প্ৰথমে ধ্ৰুৱকসমূহ ঘোষণা কৰা এৰক আৰু তাৰ পিছত সিহতক নথিপত্ৰৰ শেষত ৰপ্তানি কৰা।
- পথ সমাধান: বৰ্তমান পঞ্জিকা পথ লাভ কৰাৰ সময়ত, `import.meta.dirname` ব্যৱহাৰ কৰিব লাগিব

## ভুল

- ষ্ট্ৰিং ভুল এৰক আৰু স্থিৰ ভুল ক'ড ঘোষণা কৰিবলে const ব্যৱহাৰ কৰক।
- যদি আপুনি ভুল তথ্য তথ্য ঘূৰাই দিব লাগে, [ভুল ক'ড, ভুল বাৰ্তা,...] ব্যৱহাৰ কৰক এইদৰে, ভুল বাৰ্তাটো এটা লিখনী বিৱৰণ নহয়, কিন্তু এটা সংখ্যাগত মান (যেনে [FILE_OVERSIZE, file_size, max_size])
