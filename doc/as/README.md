# WebC.site

## এইটো কি?

### এইটো এটা স্থানীয় ৱেব উপাদান লাইব্ৰেৰী

[web components](https://developer.mozilla.org/docs/Web/API/Web_components) ৰ ওপৰত ভিত্তি কৰি বিকশিত কৰা হৈছে, ই চলনসময় স্বাধীন আৰু সকলো কাঠামোৰ বাবে উপযুক্ত।

আকাৰৰ বাবে অতি অনুকূলিত (একেধৰণৰ জনপ্ৰিয় লাইব্ৰেৰীসমূহৰ আকাৰৰ এক দশম ভাগ)।

৭৫টা ভাষা সমৰ্থন কৰে। আপুনি অনলাইন ব্যৱহাৰৰ বাবে js আৰু css প্ৰত্যক্ষভাৱে আমদানি কৰিব পাৰে, বা গৌণ বিকাশৰ বাবে প্ৰয়োজন অনুসৰি আপোনাৰ প্ৰকল্পত উপাদান যোগ কৰিব পাৰে।

### এইটোৱেই হৈছে উপাদান লাইব্ৰেৰী বিকাশৰ কাঠামো

এইটো [storybook](https://storybook.js.org) ৰ ষ্টেটিক ব্লগ সংস্কৰণ।

এটা কম্পোনেণ্ট লাইব্ৰেৰী নিৰ্মাণে কোম্পানীটোৰ আভ্যন্তৰীণ বিকাশৰ কাৰ্যক্ষমতা উন্নত কৰিব পাৰে আৰু কাৰিকৰী বৃত্তত ব্ৰেণ্ড ইমেজ গঢ়ি তুলিব পাৰে।

[webc.site](https://github.com/webc-site/webc.site) ৰ ওপৰত ভিত্তি কৰি, দলসমূহে সহজে নিজৰ উপাদান লাইব্ৰেৰীসমূহ সৃষ্টি কৰিব পাৰে আৰু সিহতক [github page](https://pages.github.com), [cloudflare page](https://pages.cloudflare.com), ইত্যাদিত প্ৰকাশ কৰিব পাৰে। প্ৰকাশিত উপাদানসমূহক [codepen.io](https://codepen.io) আৰু [jsfiddle](https://jsfiddle.net) ত অনলাইন ডিবাগ কৰিব পাৰি।

একে সময়তে, অন্তৰ্নিৰ্মিত অনুবাদ কাঠামোৱে উপাদান আৰু দস্তাবেজসমূহৰ আন্তৰ্জাতিককৰণ দ্ৰুতভাৱে উপলব্ধি কৰিব পাৰে, যাতে আপোনাৰ পণ্যৰ দৰ্শকসকল ভাষাৰ দ্বাৰা আৰু বান্ধ খাই নাথাকে।

### কৃত্ৰিম বুদ্ধিমত্তাৰ যুগৰ বাবে এইটো হৈছে ফ্ৰন্ট এণ্ড ডেভেলপমেণ্টৰ এক নতুন পেৰাডাইম

কাঠামোটো অন্তৰ্নিৰ্মিত `.agents/skills`, কৃত্ৰিম বুদ্ধিমত্তাই আপোনাক উত্তম পদ্ধতি অনুসৰি এটা ক্লিকৰ সৈতে নতুন উপাদানসমূহ বিকাশ কৰাত সহায় কৰাৰ অনুমতি দিয়ে।

<img src="https://i-01.eu.org/20260604121408079.avif" width="400">

ফ্ৰন্ট-এণ্ড ক'ডক বায়ুমণ্ডল প্ৰগ্ৰেমিংৰ এটা অস্থায়ী জাবৰৰ পাহাৰৰ পৰা এটা মানক অংশলৈ সলনি হওক যিটো দীৰ্ঘম্যাদীভাৱে ৰক্ষণাবেক্ষণ আৰু পুনৰাবৃত্তিমূলকভাৱে বিকশিত কৰিব পৰা যায়।

কৃত্ৰিম বুদ্ধিমত্তা-সহায়ক বিকাশৰ বৈশিষ্ট্যসমূহৰ প্ৰতি লক্ষ্য ৰাখি আমি এটা নতুন ফ্ৰন্ট-এণ্ড বিকাশৰ আৰ্হি প্ৰস্তাৱ আৰু ডিজাইন কৰিলোঁ:

1. **সকলো এটা ৱেব উপাদান**
2. **উপাদানটো বেক-এণ্ড ডাটা লিংকৰ পৰা স্বাধীন হ'ব লাগিব, আৰু ডাটা আন্তঃপৃষ্ঠক এটা কলবেক ফাংচন হিচাপে উন্মোচিত কৰা হয়**

এইদৰে কৃত্ৰিম বুদ্ধিমত্তাই ভুৱা তথ্য সৃষ্টি কৰিব পাৰে আৰু ডেমো পৃষ্ঠাত উপাদানসমূহৰ বিভিন্ন অৱস্থা প্ৰদৰ্শন কৰিব পাৰে।

উপাদান বিকাশ আৰু তথ্য আন্তঃপৃষ্ঠৰ পৃথকীকৰণ, [webc.site component framework](https://github.com/webc-site/webc.site) ৰ সৈতে সংযুক্ত কৰা অন্য এটা প্ৰধান বৈশিষ্ট্য:

**প্ৰতিটো উপাদানে বিকাশ চাৰ্ভাৰ স্বতন্ত্ৰভাৱে আৰম্ভ কৰিব পাৰে আৰু ডেমো পৃষ্ঠাত বিকাশ আৰু ডিবাগ কৰিব পাৰে** ।

<img src="https://i-01.eu.org/webcSiteDemo.svg" width="400">
<img src="https://i-01.eu.org/20260604112614499.avif" width="400">

ই প্ৰতিটো উপাদানৰ বিকাশ আৰু ডিবাগিঙৰ বাবে ফ্ৰন্ট-এণ্ড সংযোগক `0` লৈ হ্ৰাস কৰে ।

এইদৰে, কৃত্ৰিম বুদ্ধিমত্তা-সহায়ক বিকাশ সঁজুলি ([ক্ল'ড ক'ড](https://code.claude.com), [OpenAI কোডেক্স](https://openai.com), [গুগল এন্টিগ্ৰেভিটি](https://antigravity.google), [কাৰ্চাৰ](https://cursor.com), [উইণ্ডচাৰ্ফ](https://windsurf.com), [ডেভিন](https://devin.ai), [OpenHands](https://github.com/All-Hands-AI/OpenHands) ইত্যাদি), আপুনি সম্পূৰ্ণ স্বয়ংক্ৰিয় বিকাশ উপভোগ কৰিব পাৰে, মুক্ত ব্ৰাউজাৰ ডিবাগিং, আৰু আৰু ব্লক কৰিব নোৱাৰিব বিভিন্ন অগোছাল কাৰক (বেকএণ্ড তথ্য, প্ৰৱেশ অৱস্থা, পথ ৰাউটিং, ইত্যাদি)।

যদি আপুনি কৃত্ৰিম বুদ্ধিমত্তা নিয়ন্ত্ৰণ কৰিব বিচাৰে আৰু খৰচ হ্ৰাস কৰিব বিচাৰে আৰু দক্ষতা বৃদ্ধি কৰিব বিচাৰে, আপুনি এটা নতুন আৰ্হিৰ প্ৰয়োজন যি প্ৰতিটো উপাদানৰ বিকাশ আৰু ডিবাগিং পৰিৱেশ কম কৰে।

# মই এতিয়াও TODO লিখিবলৈ আৰম্ভ কৰা নাই৷

### ৱেব উপাদান লাইব্ৰেৰী

#### চূড়ান্ত আকাৰ অনুকূলন

উদাহৰণস্বৰূপ, ভাৰ্চুৱেল স্ক্ৰল বাৰ উপাদানৰ বাবে, জনপ্ৰিয় [OverlayScrollbars](https://github.com/KingSora/OverlayScrollbars) স্ক্ৰিপ্ট আৰু শৈলীৰ আকাৰ `gz`ৰ পিছত তুলনা কৰক, নিম্নলিখিত টেবুলত দেখুৱাৰ দৰে:

|                   |   js    |  css   |     total      |
| :---------------- | :-----: | :----: | :------------: |
| overlayscrollbars | 14520 B | 2584 B | 17104 B (9.1X) |
| Scroll            | 1159 B  | 721 B  |   ⭐ ১৮৮০ খ    |

`webc.site`ৰ ভাৰ্চুৱেল স্ক্ৰল বাৰ উপাদান কেৱল `OverlayScrollbars`ৰ `10%` ।

#### ৰানটাইম স্বতন্ত্ৰ

প্ৰামাণিক [Web Components](https://developer.mozilla.org/docs/Web/API/Web_components) ৰ ওপৰত নিৰ্মিত, webc.site উপাদান লাইব্ৰেৰী স্বাভাৱিকতে চলনসময়-অজ্ঞান। আপুনি React, Vue, Angular, Svelte, বা এটা পৰম্পৰাগত বিশুদ্ধ HTML/JS প্ৰকল্প ব্যৱহাৰ কৰক, সকলো উপাদান ব্ৰাউজাৰত প্ৰত্যক্ষভাৱে স্থানীয় HTML টেগসমূহৰ দৰে চলাব পাৰে, এটা নিৰ্দিষ্ট কাঠামোৰ বাবে গৌণ পেকেজিঙৰ প্ৰয়োজন নোহোৱাকৈ।

##### shadcn ৰ সৈতে তুলনা কৰা

[shadcn/ui](https://ui.shadcn.com) এ ক'ড বিতৰণ ধৰণ গ্ৰহণ কৰে, যি আপোনাক উপাদান ক'ডক প্ৰত্যক্ষভাৱে আপোনাৰ নিজৰ প্ৰকল্পলৈ কপি কৰাৰ প্ৰয়োজন । ই React পৰিৱেশতন্ত্ৰ আৰু নিৰ্দিষ্ট নিৰ্মাণ সঁজুলি শৃংখলসমূহৰ সৈতে কঠোৰভাৱে বান্ধ খাই আছে (যেনে Tailwind CSS), আৰু কাঠামোসমূহৰ মাজেৰে ব্যৱহাৰ কৰিব নোৱাৰি। ইয়াৰ প্ৰকল্প অভিযান্ত্ৰিক বিন্যাসৰ বাবেও কিছুমান প্ৰয়োজনীয়তা আছে।

আৰু `webc.site`ৰ উপাদানটো:
- **ক্ৰছ-ফ্ৰেমৱৰ্ক ইউনিভাৰ্চেল**: এবাৰ লিখক আৰু প্ৰত্যক্ষভাৱে প্ৰৱৰ্তন কৰিব পাৰি আৰু যিকোনো ফ্ৰন্ট-এণ্ড ফ্ৰেমৱৰ্কত (আনকি এটা ফ্ৰেমৱৰ্ক-মুক্ত পৰিৱেশ) চলাব পাৰি।
- **পৰিৱেশ শূন্য নিৰ্ভৰশীলতা**: কোনো নিৰ্দিষ্ট পেকেজিং সঁজুলি বা CSS প্ৰিপ্ৰচেছৰৰ প্ৰয়োজন নাই, মাত্ৰ কমপাইল কৰা স্থিতিশীল নথিপত্ৰসমূহ আমদানি কৰক আৰু ইয়াক বাকচৰ বাহিৰত ব্যৱহাৰ কৰিব পাৰি।

## স্বয়ংক্ৰিয় বিকাশ আৰু ডিবাগিং

### স্থানীয় বিকাশ আৰু ডিবাগিং

প্ৰতিটো ৱেব পৃষ্ঠা উপাদানে বিকাশ চাৰ্ভাৰক স্বতন্ত্ৰভাৱে আৰম্ভ কৰাটো সমৰ্থন কৰে, এটা নূন্যতম বিকাশ আৰু ডিবাগিং পৰিৱেশ প্ৰদান কৰি:

- **এটা নতুন উপাদান সৃষ্টি কৰক**: এটা উপাদান বিকাশ কংকাল দ্ৰুতভাৱে সৃষ্টি কৰিবলে (ডেমো পৃষ্ঠা আৰু আন্তৰ্জাতিককৰণ বিন্যাস অন্তৰ্ভুক্ত কৰি) `./sh/new.js [css|js] [উপাদানৰ নাম]` চলাওক।
- **স্বাধীন ডিবাগিং**: Vite বিকাশ চাৰ্ভাৰ আৰম্ভ কৰিবলে আৰু ব্ৰাউজাৰত উপাদানৰ ডেমো পৃষ্ঠা স্বয়ংক্ৰিয়ভাৱে খোলিবলৈ `./dev.js [উপাদানৰ নাম]` চলাওক (বা `./dev.js` প্ৰত্যক্ষভাৱে চলাওক)। উপাদান হট আপডেইট (HMR) সমৰ্থন কৰে। যিহেতু ডেভেলপমেণ্ট আৰু ডাটা লিংকসমূহ ডিকাপল কৰা হৈছে, সেয়েহে শূন্য ডেভেলপমেণ্ট ফ্ৰন্ট-এণ্ড লিংক আছে, যিটো সম্পূৰ্ণ স্বয়ংক্ৰিয় ডেভেলপমেণ্ট আৰু ব্ৰাউজাৰ ডিবাগিঙত সহায় কৰিবলৈ এআই (যেনে ক্ল’ড ক’ড, গুগল এন্টিগ্ৰেভিটি আদি)ৰ বাবে অতি উপযোগী।

### উপাদান লাইব্ৰেৰী প্ৰকাশ কৰক

#### webc dist

পেকেজিং আৰু প্ৰকাশন প্ৰক্ৰিয়া অতি স্বয়ংক্ৰিয়:
- **স্থিতিশীল সম্পদ পেকেজিং**: `./sh/dist.js` চলাই (অৰ্থাৎ, `webc dist` কাৰ্য্য এক্সিকিউট কৰি) স্বয়ংক্ৰিয়ভাৱে `src/`ৰ অন্তৰ্গত সকলো ৱেব পৃষ্ঠা উপাদান স্কেন কৰিব পাৰে, কম্পাইল কৰিব পাৰে, সংকোচিত কৰিব পাৰে আৰু সিহতক `dist/` পঞ্জিকালৈ আউটপুট কৰিব পাৰে। এই পঞ্জিকাত উৎপাদন বিতৰণৰ বাবে JS/CSS নথিপত্ৰসমূহ, GitHub পৃষ্ঠাসমূহ বা Cloudflare পৃষ্ঠাসমূহলে মোতায়েনৰ বাবে সাজু এটা ডেমো চাইট, আৰু স্বয়ংক্ৰিয়ভাৱে সৃষ্টি কৰা `_redirects` আৰু `404.html` ৰাউটিং নথিপত্ৰসমূহ।
- **NPM প্ৰকাশ**: `./sh/npmDist.js` চলাওক, স্ক্ৰিপ্ট স্বয়ংক্ৰিয়ভাৱে উপাদান নিৰ্ভৰশীলতাসমূহ নিয়ন্ত্ৰণ কৰিব, পেকেইজ সংস্কৰণ সংখ্যা বৃদ্ধি কৰিব, আৰু উপাদান লাইব্ৰেৰীক NPM লে প্ৰকাশ কৰিবলে পেকেজিং সম্পূৰ্ণ হোৱাৰ পিছত এটা ক্লিকৰ সৈতে `npm publish` এক্সিকিউট কৰিব।

### অনুবাদৰ দায়িত্ব এই মঞ্চৰ

কাঠামোটোত অন্তৰ্নিৰ্মিত স্বয়ংক্ৰিয় অনুবাদ প্ৰক্ৰিয়া আছে:
1. অনুবাদ নিয়মসমূহ (যেনে উৎস ভাষা, লক্ষ্য ভাষা আৰু অনুবাদ কৰিবলগীয়া পঞ্জিকা) প্ৰকল্প ৰূট পঞ্জিকাত `tran.yml`ত বিন্যাস কৰক।
2. আপোনাৰ ক'ডত একাধিক ভাষা হাৰ্ডকোড নকৰাকৈ উপাদান বা দস্তাবেজসমূহত প্লেচহোল্ডাৰসমূহ ব্যৱহাৰ কৰক।
3. `./sh/tran.js` চলাওক, আৰু অনুবাদ ইঞ্জিনে স্বয়ংক্ৰিয়ভাৱে লিখনী এক্সট্ৰেক্ট কৰিব আৰু প্ৰচেছিঙৰ বাবে WebC অনুবাদ সেৱাক কল কৰিব, স্বয়ংক্ৰিয়ভাৱে বিভিন্ন ভাষাত YAML অনুবাদ প্ৰবিষ্ট আৰু README দস্তাবেজ সৃষ্টি কৰিব।

### পৰিৱেশ চলকসমূহ

স্বয়ংক্ৰিয় অনুবাদ সম্পাদন কৰাৰ সময়ত, আপুনি অনুবাদ API লৈ অভিগম অনুমোদন কৰিবলে নিম্নলিখিত পৰিৱেশ চলকসমূহ বিন্যাস কৰিব লাগিব:

- **`WEBC_SITE_TOKEN`** (বা **`WEBC_TOKEN`**): `webc.site` স্বয়ংক্ৰিয় অনুবাদ API অভিগম কৰিবলে ব্যৱহৃত প্ৰমাণীকৰণ চাবি। এটা পৰিৱেশ চলক হিচাপে ইনজেকচন কৰিব পাৰি বা স্থানীয় ব্যৱহাৰকাৰীৰ ঘৰ পঞ্জিকাত `~/.config/webc.site.yml` বিন্যাস নথিপত্ৰত `token` লিখিব পাৰি ।
