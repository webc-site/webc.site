# WebC.site

## Τι είναι αυτό;

### Αυτή είναι μια εγγενής βιβλιοθήκη στοιχείων web

Αναπτύχθηκε με βάση το [web components](https://developer.mozilla.org/docs/Web/API/Web_components), είναι ανεξάρτητο από το χρόνο εκτέλεσης και κατάλληλο για όλα τα πλαίσια.

Εξαιρετικά βελτιστοποιημένο για μέγεθος (το ένα δέκατο του μεγέθους παρόμοιων δημοφιλών βιβλιοθηκών).

Υποστηρίζει 75 γλώσσες. Μπορείτε να εισαγάγετε απευθείας js και css για διαδικτυακή χρήση ή να προσθέσετε στοιχεία στο έργο σας όπως απαιτείται για δευτερεύουσα ανάπτυξη.

### Αυτό είναι το πλαίσιο ανάπτυξης της βιβλιοθήκης συστατικών

Αυτή είναι η στατική έκδοση ιστολογίου του [storybook](https://storybook.js.org).

Η δημιουργία μιας βιβλιοθήκης στοιχείων μπορεί να βελτιώσει την αποτελεσματικότητα της εσωτερικής ανάπτυξης της εταιρείας και να δημιουργήσει μια εικόνα της επωνυμίας στον τεχνικό κύκλο.

Με βάση το [webc.site](https://github.com/webc-site/webc.site), οι ομάδες μπορούν εύκολα να δημιουργήσουν τις δικές τους βιβλιοθήκες στοιχείων και να τις δημοσιεύσουν σε [github page](https://pages.github.com), [cloudflare page](https://pages.cloudflare.com), κ.λπ. Τα δημοσιευμένα στοιχεία μπορούν να εντοπιστούν ηλεκτρονικά σε [codepen.io](https://codepen.io) και [jsfiddle](https://jsfiddle.net).

Ταυτόχρονα, το ενσωματωμένο πλαίσιο μετάφρασης μπορεί να πραγματοποιήσει γρήγορα τη διεθνοποίηση στοιχείων και εγγράφων, έτσι ώστε το κοινό του προϊόντος σας να μην δεσμεύεται πλέον από τη γλώσσα.

### Αυτό είναι ένα νέο παράδειγμα ανάπτυξης front-end για την εποχή της τεχνητής νοημοσύνης

Το πλαίσιο είναι ενσωματωμένο `.agents/skills`, επιτρέποντας την τεχνητή νοημοσύνη να σας βοηθήσει να αναπτύξετε νέα στοιχεία με ένα κλικ σύμφωνα με τις βέλτιστες πρακτικές.

<img src="https://i-01.eu.org/20260604121408079.avif" width="400">

Αφήστε τον κώδικα της διεπαφής να αλλάξει από έναν μη βιώσιμο προγραμματισμό σκουπιδιών ατμόσφαιρας σε ένα τυπικό μέρος που μπορεί να διατηρηθεί και να εξελιχθεί επαναληπτικά μακροπρόθεσμα.

Λαμβάνοντας υπόψη τα χαρακτηριστικά της υποβοηθούμενης από τεχνητή νοημοσύνη ανάπτυξης, προτείναμε και σχεδιάσαμε ένα νέο παράδειγμα ανάπτυξης front-end:

1. **Τα πάντα είναι ένα στοιχείο Ιστού**
2. **Το στοιχείο πρέπει να είναι ανεξάρτητο από τη σύνδεση δεδομένων back-end και η διεπαφή δεδομένων εκτίθεται ως λειτουργία επανάκλησης**

Με αυτόν τον τρόπο, η τεχνητή νοημοσύνη μπορεί να δημιουργήσει πλαστά δεδομένα και να εμφανίσει διαφορετικές καταστάσεις στοιχείων στη σελίδα επίδειξης.

Διαχωρισμός ανάπτυξης στοιχείων και διεπαφής δεδομένων, σε συνδυασμό με [webc.site component framework](https://github.com/webc-site/webc.site) Ένα άλλο σημαντικό χαρακτηριστικό:

**Κάθε στοιχείο μπορεί να ξεκινήσει ανεξάρτητα τον διακομιστή ανάπτυξης και να αναπτύξει και να εντοπίσει σφάλματα στη σελίδα επίδειξης**.

<img src="https://i-01.eu.org/webcSiteDemo.svg" width="400">
<img src="https://i-01.eu.org/20260604112614499.avif" width="400">

Αυτό μειώνει τη σύνδεση διεπαφής για την ανάπτυξη και τον εντοπισμό σφαλμάτων κάθε στοιχείου σε `0`.

Με αυτόν τον τρόπο, τα εργαλεία ανάπτυξης που υποβοηθούνται από τεχνητή νοημοσύνη ([Claude Code](https://code.claude.com), [OpenAI Codex](https://openai.com), [Google Antigravity](https://antigravity.google), [Cursor](https://cursor.com), [Windsurf](https://windsurf.com), [Devin](https://devin.ai), [OpenHands](https://github.com/All-Hands-AI/OpenHands) κ.λπ.), μπορείτε να απολαμβάνετε την ανάπτυξη πλήρως ανοιχτή και αυτόματης απομάκρυνσης του προγράμματος περιήγησης. (δεδομένα backend, κατάσταση σύνδεσης, δρομολόγηση διαδρομής κ.λπ.).

Εάν θέλετε να ελέγξετε την τεχνητή νοημοσύνη και να μειώσετε το κόστος και να αυξήσετε την αποτελεσματικότητα, χρειάζεστε ένα νέο παράδειγμα που ελαχιστοποιεί το περιβάλλον ανάπτυξης και εντοπισμού σφαλμάτων κάθε στοιχείου.

| overlayscrollbars | 14520 B | 2584 B | 17104 B (9.1X) |
| Scroll            | 1159 B  | 721 B  |   ⭐ 1880 π    |

Το στοιχείο εικονικής γραμμής κύλισης του `webc.site` είναι μόνο το `10%` του `OverlayScrollbars`.

#### Ανεξάρτητος χρόνος εκτέλεσης

Χτισμένη σε πρότυπα [Web Components](https://developer.mozilla.org/docs/Web/API/Web_components), η βιβλιοθήκη στοιχείων webc.site είναι φυσικά αγνωστική στο χρόνο εκτέλεσης. Είτε χρησιμοποιείτε React, Vue, Angular, Svelte ή ένα παραδοσιακό έργο καθαρού HTML/JS, όλα τα στοιχεία μπορούν να εκτελούνται απευθείας στο πρόγραμμα περιήγησης όπως οι εγγενείς ετικέτες HTML, χωρίς να απαιτείται δευτερεύουσα συσκευασία για ένα συγκεκριμένο πλαίσιο.

##### Σύγκριση με shadcn

Το [shadcn/ui](https://ui.shadcn.com) υιοθετεί τη λειτουργία διανομής κώδικα, η οποία απαιτεί να αντιγράψετε τον κώδικα του στοιχείου απευθείας στο δικό σας έργο. Είναι στενά συνδεδεμένο με το οικοσύστημα React και συγκεκριμένες αλυσίδες εργαλείων κατασκευής (όπως το Tailwind CSS) και δεν μπορεί να χρησιμοποιηθεί σε διάφορα πλαίσια. Έχει επίσης ορισμένες απαιτήσεις για τη διαμόρφωση μηχανικής έργου.

Και το στοιχείο του `webc.site`:
- **Καθολικά μεταξύ πλαισίων**: Γράψτε μία φορά και μπορεί να εισαχθεί απευθείας και να εκτελεστεί σε οποιοδήποτε πλαίσιο διεπαφής (ακόμα και σε περιβάλλον χωρίς πλαίσιο).
- **Εξάρτηση μηδενικού περιβάλλοντος**: Δεν απαιτούνται συγκεκριμένα εργαλεία συσκευασίας ή προεπεξεργαστές CSS, απλώς εισάγετε τα μεταγλωττισμένα στατικά αρχεία και μπορούν να χρησιμοποιηθούν εκτός συσκευασίας.

## Αυτοματοποιημένη ανάπτυξη και εντοπισμός σφαλμάτων

### Τοπική ανάπτυξη και εντοπισμός σφαλμάτων

Κάθε στοιχείο ιστοσελίδας υποστηρίζει την ανεξάρτητη εκκίνηση του διακομιστή ανάπτυξης, παρέχοντας ένα ελάχιστο περιβάλλον ανάπτυξης και εντοπισμού σφαλμάτων:

- **Δημιουργήστε ένα νέο στοιχείο**: Εκτελέστε το `./sh/new.js [css|js] [όνομα στοιχείου]` για να δημιουργήσετε γρήγορα έναν σκελετό ανάπτυξης στοιχείων (συμπεριλαμβανομένης της σελίδας επίδειξης και της διαμόρφωσης διεθνοποίησης).
- **Ανεξάρτητος εντοπισμός σφαλμάτων**: Εκτελέστε το `./dev.js [όνομα στοιχείου]` (ή εκτελέστε το `./dev.js` απευθείας) για να ξεκινήσετε τον διακομιστή ανάπτυξης Vite και να ανοίξετε αυτόματα τη σελίδα επίδειξης του στοιχείου στο πρόγραμμα περιήγησης. Υποστηρίζει hot update (HMR). Επειδή οι σύνδεσμοι ανάπτυξης και δεδομένων είναι αποσυνδεδεμένοι, δεν υπάρχει μηδενικός σύνδεσμος διεπαφής ανάπτυξης, ο οποίος είναι πολύ κατάλληλος για την τεχνητή νοημοσύνη (όπως ο Claude Code, το Google Antigravity κ.λπ.) που βοηθά στην πλήρως αυτόματη ανάπτυξη και τον εντοπισμό σφαλμάτων του προγράμματος περιήγησης.

### Δημοσίευση βιβλιοθήκης στοιχείων

#### webc dist

Η διαδικασία συσκευασίας και δημοσίευσης είναι εξαιρετικά αυτοματοποιημένη:
- **Στατική συσκευασία πόρων**: Η εκτέλεση του `./sh/dist.js` (δηλαδή, η εκτέλεση της εργασίας `webc dist`) μπορεί να σαρώσει αυτόματα όλα τα στοιχεία της ιστοσελίδας στο `src/`, να μεταγλωττίσει, να συμπιέσει και να τα εξάγει στον κατάλογο `dist/`. Αυτός ο κατάλογος περιέχει αρχεία JS/CSS για διανομή παραγωγής, έναν ιστότοπο επίδειξης έτοιμο για ανάπτυξη σε Σελίδες GitHub ή Σελίδες Cloudflare και αρχεία δρομολόγησης που δημιουργούνται αυτόματα `_redirects` και `404.html`.
- **Δημοσίευση NPM**: Εκτελέστε το `./sh/npmDist.js`, το σενάριο θα χειριστεί αυτόματα τις εξαρτήσεις στοιχείων, θα αυξήσει τον αριθμό έκδοσης του πακέτου και θα εκτελέσει το `npm publish` με ένα κλικ μετά την ολοκλήρωση της συσκευασίας για τη δημοσίευση της βιβλιοθήκης στοιχείων στο NPM.

### Η πλατφόρμα είναι υπεύθυνη για τη μετάφραση

Το πλαίσιο έχει ενσωματωμένες αυτοματοποιημένες διαδικασίες μετάφρασης:
1. Διαμορφώστε τους κανόνες μετάφρασης (όπως η γλώσσα πηγής, η γλώσσα προορισμού και ο κατάλογος προς μετάφραση) στο `tran.yml` στον ριζικό κατάλογο του έργου.
2. Χρησιμοποιήστε σύμβολα κράτησης θέσης σε στοιχεία ή έγγραφα χωρίς να χρειάζεται να κωδικοποιήσετε πολλές γλώσσες στον κώδικά σας.
3. Εκτελέστε το `./sh/tran.js` και η μηχανή μετάφρασης θα εξαγάγει αυτόματα το κείμενο και θα καλέσει την υπηρεσία μετάφρασης WebC για επεξεργασία, δημιουργώντας αυτόματα καταχωρήσεις μετάφρασης YAML και τεκμηρίωση README σε διάφορες γλώσσες.

### μεταβλητές περιβάλλοντος

Κατά την εκτέλεση αυτοματοποιημένης μετάφρασης, πρέπει να διαμορφώσετε τις ακόλουθες μεταβλητές περιβάλλοντος για να εξουσιοδοτήσετε την πρόσβαση στο API μετάφρασης:

- **`WEBC_SITE_TOKEN`** (ή **`WEBC_TOKEN`**): Κλειδί ελέγχου ταυτότητας που χρησιμοποιείται για πρόσβαση στο `webc.site` API αυτόματης μετάφρασης. Μπορεί να εγχυθεί ως μεταβλητή περιβάλλοντος ή να γραφεί `token` στο αρχείο διαμόρφωσης `~/.config/webc.site.yml` στον αρχικό κατάλογο του τοπικού χρήστη.
