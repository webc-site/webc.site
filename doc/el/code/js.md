# Προδιαγραφή κώδικα JS

## στυλ κωδικοποίησης

- Απλό, κομψό και αποτελεσματικό, χρησιμοποιώντας την πιο σύγχρονη σύνταξη js
- Διαχωρίστε τις υπερβολικά μεγάλες λειτουργίες και ο σχεδιασμός της διεπαφής θα πρέπει να είναι χαμηλής ζεύξης και υψηλής συνοχής.
- Χρησιμοποιήστε περισσότερους πίνακες + βρόχους/forEach/map και γράψτε λιγότερο επαναλαμβανόμενο κώδικα
- Μην γράφετε σχόλια συναρτήσεων με απλό κώδικα για να αποφύγετε τα υπερβολικά σχόλια. Η σύνθετη λογική και οι ειδικές ανάγκες εξηγούνται στα εκλεπτυσμένα κινέζικα.
- Ο επαναλαμβανόμενος κώδικας αφαιρείται σε συναρτήσεις για να μειωθεί ο πλεονασμός. Για παράδειγμα, το `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` μπορεί να γραφτεί ως `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()`
- Δήλωση μεταβλητής: Συνδυάστε πολλαπλές διαδοχικές `const` δηλώσεις σε μία δήλωση. Αντί να το γράψετε σε τρεις γραμμές, γράψτε `const a=1, b=2, c=3;`
- Ασύγχρονη επεξεργασία: χρησιμοποιήστε `await`, απαγορεύστε τη χρήση αλυσιδωτών κλήσεων `.then`
- Να μην δημιουργείται αυτόματα κώδικας χειρισμού εξαιρέσεων, να μην γράφεται αυτόματα `try...catch` (το`try ... catch` διατηρείται χειροκίνητα, το υπάρχον `try catch` δεν πρέπει να διαγραφεί)
- Πρώτα καθαρές συναρτήσεις: γράφετε μόνο καθαρές συναρτήσεις, ποτέ κλάσεις
- Χρησιμοποιήστε τις συναρτήσεις βέλους `const funcName = () => {}` και μην χρησιμοποιείτε τη λέξη-κλειδί `function` (εκτός από γεννήτριες). Εάν μπορείτε να χρησιμοποιήσετε παραμέτρους .bind για δέσμευση, αποφύγετε τον ορισμό συναρτήσεων
- Επαναχρησιμοποίηση κώδικα: Δώστε προσοχή στην επαναχρησιμοποίηση, εξάγετε περισσότερες μικρές λειτουργίες και αποφύγετε αποφασιστικά μεγάλο αριθμό παρόμοιων ή επικολλημένων δομών κώδικα.
- Πρόσβαση αντικειμένου: Δώστε προτεραιότητα στη χρήση ανάθεσης αποδομής για την εξαγωγή απαιτούμενων χαρακτηριστικών, αποφύγετε την επανειλημμένη χρήση κουκκίδων εσωτερικά για πρόσβαση σε βαθιά και ένθετα χαρακτηριστικά και συγχώνευση επαναλαμβανόμενων προαιρετικών κρίσεων αλυσίδας
- Μην χρησιμοποιείτε αντικείμενα για παραμέτρους συνάρτησης, μην γράφετε `{a,b,c}`, γράφετε `a, b, c`. εάν υπάρχουν πολλές προαιρετικές παράμετροι, χρησιμοποιήστε [[αριθμός στοιχείων διαμόρφωσης, στοιχεία διαμόρφωσης], [αριθμός στοιχείων διαμόρφωσης, στοιχεία διαμόρφωσης],..] Σε αυτό το παράδειγμα, τα στοιχεία διαμόρφωσης ορίζονται με αριθμητικές σταθερές
- Για επιστροφές πολλών τιμών, χρησιμοποιήστε τον πίνακα `[a,b,c]` αντί του `{a,b,c}`. Εάν υπάρχουν περισσότερες από 3 τιμές επιστροφής, χρησιμοποιήστε αριθμητικές σταθερές για να ορίσετε τη σημασία της θέσης.
- Δεν υπάρχει ανάγκη για συμβολοσειρά για την αναπαράσταση της κατάστασης, χρησιμοποιήστε αριθμητικές σταθερές για να ορίσετε την κατάσταση
- Δεν χρησιμοποιείται πρότυπο συμβολοσειράς (``), χρησιμοποιείται συνένωση συμβολοσειρών (+), εκτός από την εισαγωγή (βολικό για στατική ανάλυση Vite)
- Για βρόχο, εάν χρειάζεστε σειριακό αριθμό, χρησιμοποιήστε `++i` αντί για `i++`
- Αποφύγετε τον περιττό`?.` και απαγορεύστε τον αμυντικό προγραμματισμό
- Το όνομα της μεταβλητής δεν χρειάζεται να είναι πληθυντικός και τελειώνει με `_li` για να υποδεικνύει μια λίστα.

## Σύμβαση ονομασίας

- Η ονομασία επιδιώκει τον μινιμαλισμό. Χρησιμοποιήστε σύντομα αλλά ουσιαστικά ονόματα, για παράδειγμα: χρησιμοποιήστε `rm` αντί για `remove`, `delete`, `del`. Ωστόσο, θα πρέπει επίσης να αποφύγετε τα άκρα, για παράδειγμα: μην χρησιμοποιείτε ούτε ένα γράμμα `m` αντί για `map`
- Προσπαθήστε να χρησιμοποιείτε μόνο ρήματα στην ονομασία συναρτήσεων. Εάν μπορείτε να χρησιμοποιήσετε μία λέξη για να το εκφράσετε, μην χρησιμοποιείτε δύο λέξεις. Τα ουσιαστικά αντικατοπτρίζονται στα ονόματα αρχείων. Εάν χρειάζεται να προσθέσετε ένα ρήμα στο όνομα του αρχείου, βάλτε πρώτα το ουσιαστικό και τελευταίο το ρήμα. Για παράδειγμα: `profileSet.js` αντί για `setProfile.js`
- Όνομα μεταβλητής: χρησιμοποιήστε το στυλ υπογράμμισης (snake_case), για παράδειγμα `user_auth_token`; Εάν η μεταβλητή είναι συνάρτηση, χρησιμοποιήστε στυλ ονομασίας πεζών κεφαλαίων
- Όνομα συνάρτησης: Χρησιμοποιήστε πεζά camelCase (camelCase), για παράδειγμα `userData`
- Παράμετροι συνάρτησης: Εάν πρόκειται για συνάρτηση επανάκλησης, ονομάστε την με πεζά γράμματα camel, όπως `onChange`
- Οι ορισμοί σταθερών σε επίπεδο μονάδας χρησιμοποιούν στυλ υπογράμμισης κεφαλαίων `UPPER_SNAKE_CASE`
- Μην γράφετε `get`, τέτοια προθέματα χωρίς νόημα, όπως: γράψτε `cookieByHeader` αντί για `getCookie`
- Καθολικές σταθερές επιπέδου/μονάδας: χρησιμοποιήστε στυλ υπογράμμισης κεφαλαίων (UPPER_SNAKE_CASE), π.χ. `CODE_TO_ID`, `ID_TO_LANG`

## Αρθρωτός μηχανισμός

- Εισαγωγή: Εισαγάγετε λειτουργίες με ακρίβεια κατόπιν ζήτησης, απαγορεύστε την άμεση εισαγωγή ολόκληρης της μονάδας (αποφύγετε `import * as x` ή εισάγετε μεγάλα αντικείμενα)
- Εξαγωγή: Απενεργοποίηση εξαγωγής αντικειμένων. Εξαγωγή συναρτήσεων και μεταβλητών ως μονάδες, όπως `export const X=1, abc=()=>{};`. Προσπαθήστε να τα συνδυάσετε με ένα const + κόμμα για να δηλώσετε το εξαγόμενο περιεχόμενο. Εάν ένα αρχείο έχει μόνο μία λειτουργία, χρησιμοποιήστε το `export default`
- Εκτός και αν χρειαστεί να καλέσετε την προεπιλεγμένη συνάρτηση εξαγωγής εσωτερικά, αποφύγετε να δηλώνετε σταθερές πρώτα και μετά να τις εξάγετε στο τέλος του αρχείου.
- Ανάλυση διαδρομής: Κατά τη λήψη της τρέχουσας διαδρομής καταλόγου, πρέπει να χρησιμοποιείται `import.meta.dirname`

## λάθος

- Αποφύγετε τα σφάλματα συμβολοσειράς και χρησιμοποιήστε το const για να δηλώσετε σταθερούς κωδικούς σφάλματος.
- Εάν πρέπει να επιστρέψετε λανθασμένες πληροφορίες δεδομένων, χρησιμοποιήστε [κωδικός σφάλματος, μήνυμα σφάλματος,...] Με αυτόν τον τρόπο, το μήνυμα σφάλματος δεν είναι μια περιγραφή κειμένου, αλλά μια αριθμητική τιμή (όπως [FILE_OVERSIZE, file_size, max_size])
