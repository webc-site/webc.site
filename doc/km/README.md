ឧទាហរណ៍ សម្រាប់សមាសភាគរបាររមូរនិម្មិត ប្រៀបធៀបទំហំនៃស្គ្រីប [OverlayScrollbars](https://github.com/KingSora/OverlayScrollbars) ពេញនិយម និងរចនាប័ទ្មបន្ទាប់ពី `gz` ដូចបង្ហាញក្នុងតារាងខាងក្រោម៖

#### ពេលដំណើរការឯករាជ្យ

បង្កើតឡើងតាមស្តង់ដារ [Web Components](https://developer.mozilla.org/docs/Web/API/Web_components) បណ្ណាល័យសមាសធាតុ webc.site មានលក្ខណៈដំណើរការតាមធម្មជាតិ។ មិនថាអ្នកកំពុងប្រើ React, Vue, Angular, Svelte ឬគម្រោង HTML/JS សុទ្ធធម្មតាទេ សមាសធាតុទាំងអស់អាចដំណើរការដោយផ្ទាល់នៅក្នុងកម្មវិធីរុករកតាមអ៊ីនធឺណិតដូចជាស្លាក HTML ដើម ដោយមិនចាំបាច់ត្រូវការវេចខ្ចប់បន្ទាប់បន្សំសម្រាប់ក្របខ័ណ្ឌជាក់លាក់នោះទេ។

សមាសធាតុនៃគេហទំព័រនីមួយៗគាំទ្រការចាប់ផ្តើមម៉ាស៊ីនមេអភិវឌ្ឍន៍ដោយឯករាជ្យ ដោយផ្តល់នូវការអភិវឌ្ឍន៍តិចតួចបំផុត និងបរិស្ថានបំបាត់កំហុស៖

- **បង្កើតសមាសភាគថ្មី**៖ ដំណើរការ `./sh/new.js [css|js] [component name]` ដើម្បីបង្កើតគ្រោងនៃការអភិវឌ្ឍន៍សមាសធាតុយ៉ាងរហ័ស (រួមទាំងទំព័រសាកល្បង និងការកំណត់រចនាសម្ព័ន្ធអន្តរជាតិ)។
- **ការបំបាត់កំហុសដោយឯករាជ្យ**៖ ដំណើរការ `./dev.js [component name]` (ឬដំណើរការ `./dev.js`ដោយផ្ទាល់) ដើម្បីចាប់ផ្តើមម៉ាស៊ីនមេអភិវឌ្ឍន៍ Vite និងបើកទំព័រសាកល្បងដោយស្វ័យប្រវត្តិនៃសមាសភាគនៅក្នុងកម្មវិធីរុករក។ គាំទ្រសមាសធាតុអាប់ដេតក្តៅ (HMR) ។ ដោយសារការអភិវឌ្ឍន៍ និងតំណភ្ជាប់ទិន្នន័យត្រូវបានផ្ដាច់ វាមានតំណផ្នែកខាងមុខនៃការអភិវឌ្ឍន៍សូន្យ ដែលស័ក្តិសមបំផុតសម្រាប់ AI (ដូចជា Claude Code, Google Antigravity ។ល។) ដើម្បីជួយក្នុងការអភិវឌ្ឍន៍ដោយស្វ័យប្រវត្តិយ៉ាងពេញលេញ និងការបំបាត់កំហុសកម្មវិធីរុករក។

### បោះពុម្ពបណ្ណាល័យសមាសធាតុ

#### webc dist

ដំណើរការវេចខ្ចប់ និងបោះពុម្ពគឺស្វ័យប្រវត្តិខ្ពស់៖
- **ការវេចខ្ចប់ធនធានឋិតិវន្ត**៖ ដំណើរការ `./sh/dist.js` (នោះគឺការប្រតិបត្តិ `webc dist`ភារកិច្ច) អាចស្កេនដោយស្វ័យប្រវត្តិនូវសមាសធាតុទំព័របណ្ដាញទាំងអស់នៅក្រោម `src/` ចងក្រង បង្ហាប់ និងបញ្ចេញពួកវាទៅថត `dist/`។ ថតនេះមានឯកសារ JS/CSS សម្រាប់ការចែកចាយផលិតកម្ម គេហទំព័រសាកល្បងដែលត្រៀមរួចជាស្រេចសម្រាប់ការដាក់ពង្រាយទៅកាន់ GitHub Pages ឬ Cloudflare Pages ហើយបានបង្កើតដោយស្វ័យប្រវត្តិ `_redirects` និង `404.html` ឯកសារកំណត់ផ្លូវ។
- ** NPM Publish**៖ ដំណើរការ `./sh/npmDist.js` ស្គ្រីបនឹងគ្រប់គ្រងភាពអាស្រ័យសមាសភាគដោយស្វ័យប្រវត្តិ បង្កើនលេខកំណែកញ្ចប់ និងប្រតិបត្តិ `npm publish` ដោយចុចតែម្តងបន្ទាប់ពីការវេចខ្ចប់ត្រូវបានបញ្ចប់ ដើម្បីបោះពុម្ពបណ្ណាល័យសមាសភាគទៅ NPM ។

### វេទិកានេះទទួលខុសត្រូវចំពោះការបកប្រែ

ក្របខ័ណ្ឌមានដំណើរការបកប្រែដោយស្វ័យប្រវត្តិដែលភ្ជាប់មកជាមួយ៖
1. កំណត់រចនាសម្ព័ន្ធច្បាប់នៃការបកប្រែ (ដូចជាភាសាប្រភព ភាសាគោលដៅ និងថតដែលត្រូវបកប្រែ) នៅក្នុង `tran.yml` នៅក្នុងថតឫសគម្រោង។
2. ប្រើកន្លែងដាក់ក្នុងសមាសធាតុ ឬឯកសារដោយមិនចាំបាច់ hardcode ច្រើនភាសានៅក្នុងកូដរបស់អ្នក។
3. ដំណើរការ `./sh/tran.js` ហើយម៉ាស៊ីនបកប្រែនឹងស្រង់អត្ថបទដោយស្វ័យប្រវត្តិ ហើយហៅទៅសេវាកម្មបកប្រែ WebC សម្រាប់ដំណើរការ បង្កើតធាតុបកប្រែ YAML ដោយស្វ័យប្រវត្តិ និងឯកសារ README ជាភាសាផ្សេងៗ។

### អថេរបរិស្ថាន

នៅពេលដំណើរការការបកប្រែដោយស្វ័យប្រវត្តិ អ្នកត្រូវកំណត់រចនាសម្ព័ន្ធអថេរបរិស្ថានខាងក្រោម ដើម្បីអនុញ្ញាតការចូលប្រើ API បកប្រែ៖

- **`WEBC_SITE_TOKEN`** (ឬ **`WEBC_TOKEN`**)៖ សោផ្ទៀងផ្ទាត់ដែលប្រើដើម្បីចូលប្រើ `webc.site` API ការបកប្រែដោយស្វ័យប្រវត្តិ។ អាចត្រូវបានចាក់ជាអថេរបរិស្ថាន ឬសរសេរ `token`នៅក្នុងឯកសារកំណត់រចនាសម្ព័ន្ធ `~/.config/webc.site.yml` នៅក្នុងថតផ្ទះរបស់អ្នកប្រើមូលដ្ឋាន។
