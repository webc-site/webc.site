# JS కోడ్ స్పెసిఫికేషన్

## కోడింగ్ శైలి

- అత్యంత ఆధునిక js సింటాక్స్‌ని ఉపయోగించి సరళమైనది, సొగసైనది మరియు సమర్థవంతమైనది
- మితిమీరిన పొడవైన విధులను విభజించండి మరియు ఇంటర్‌ఫేస్ డిజైన్ తక్కువ కలపడం మరియు అధిక సమన్వయంతో ఉండాలి.
- శ్రేణులు + లూప్‌లు/ప్రతి కోసం/మ్యాప్‌ని ఎక్కువగా ఉపయోగించండి మరియు తక్కువ పునరావృత కోడ్‌ను వ్రాయండి
- అధిక వ్యాఖ్యలను నివారించడానికి సాధారణ కోడ్‌లో ఫంక్షన్ వ్యాఖ్యలను వ్రాయవద్దు. సంక్లిష్ట తర్కం మరియు ప్రత్యేక అవసరాలు శుద్ధి చేయబడిన చైనీస్‌లో వివరించబడ్డాయి.
- రిడెండెన్సీని తగ్గించడానికి రిపీట్ కోడ్ ఫంక్షన్‌లలోకి సంగ్రహించబడుతుంది. ఉదాహరణకు, `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })`ని `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()`గా వ్రాయవచ్చు
- వేరియబుల్ డిక్లరేషన్: అనేక వరుస `const` డిక్లరేషన్‌లను ఒక స్టేట్‌మెంట్‌లో కలపండి. మూడు లైన్లలో వ్రాయడానికి బదులుగా, `const a=1, b=2, c=3;` అని వ్రాయండి
- అసమకాలిక ప్రాసెసింగ్: `await`ని ఉపయోగించండి, `.then` చైన్ కాల్‌ల వినియోగాన్ని నిషేధించండి
- మినహాయింపు నిర్వహణ కోడ్‌ని స్వయంచాలకంగా రూపొందించవద్దు, `try...catch`ని స్వయంచాలకంగా వ్రాయవద్దు (`try ... catch` మానవీయంగా నిర్వహించబడుతుంది, ఇప్పటికే ఉన్న `try catch` తొలగించబడకూడదు)
- ముందుగా ప్యూర్ ఫంక్షన్‌లు: స్వచ్ఛమైన ఫంక్షన్‌లను మాత్రమే రాయండి, క్లాస్‌లను ఎప్పటికీ రాయవద్దు
- బాణం ఫంక్షన్లను ఉపయోగించండి `const funcName = () => {}` మరియు `function` కీవర్డ్ (జనరేటర్లు మినహా) ఉపయోగించవద్దు; మీరు పారామితులను బంధించడానికి .bindని ఉపయోగించగలిగితే, విధులను నిర్వచించడాన్ని నివారించండి
- కోడ్ పునర్వినియోగం: పునర్వినియోగంపై శ్రద్ధ వహించండి, మరిన్ని చిన్న ఫంక్షన్‌లను సంగ్రహించండి మరియు పెద్ద సంఖ్యలో సారూప్య లేదా కాపీ-పేస్ట్ చేయబడిన కోడ్ నిర్మాణాలను నిశ్చయంగా నివారించండి.
- ఆబ్జెక్ట్ యాక్సెస్: అవసరమైన లక్షణాలను సంగ్రహించడానికి డిస్ట్రక్చరింగ్ అసైన్‌మెంట్‌ని ఉపయోగించడం ప్రాధాన్యతనివ్వండి, లోతైన మరియు సమూహ లక్షణాలను యాక్సెస్ చేయడానికి అంతర్గతంగా చుక్కలను పదేపదే ఉపయోగించకుండా ఉండండి మరియు పదేపదే ఐచ్ఛిక గొలుసు తీర్పులను విలీనం చేయండి
- ఫంక్షన్ పారామితుల కోసం వస్తువులను ఉపయోగించవద్దు, `{a,b,c}`ని వ్రాయవద్దు, `a, b, c`ని వ్రాయవద్దు; అనేక ఐచ్ఛిక పారామితులు ఉంటే, [[కాన్ఫిగరేషన్ ఐటెమ్‌ల సంఖ్య, కాన్ఫిగరేషన్ ఐటెమ్‌ల సంఖ్య], [కాన్ఫిగరేషన్ ఐటెమ్‌ల సంఖ్య, కాన్ఫిగరేషన్ ఐటెమ్‌లు],..] ఈ ఉదాహరణలో, కాన్ఫిగరేషన్ అంశాలు సంఖ్యా స్థిరాంకాలతో నిర్వచించబడతాయి.
- బహుళ-విలువ రిటర్న్‌ల కోసం, `{a,b,c}`కి బదులుగా అర్రే `[a,b,c]`ని ఉపయోగించండి. 3 కంటే ఎక్కువ రిటర్న్ విలువలు ఉంటే, స్థాన అర్థాన్ని నిర్వచించడానికి సంఖ్యా స్థిరాంకాలను ఉపయోగించండి.
- రాష్ట్రాన్ని సూచించడానికి స్ట్రింగ్ అవసరం లేదు, రాష్ట్రాన్ని నిర్వచించడానికి సంఖ్యా స్థిరాంకాలను ఉపయోగించండి
- స్ట్రింగ్ టెంప్లేట్ (``) ఉపయోగించబడదు, స్ట్రింగ్ కంకాటెనేషన్ (+) ఉపయోగించబడుతుంది, దిగుమతి మినహా (Vite స్టాటిక్ విశ్లేషణకు అనుకూలమైనది)
- లూప్ కోసం, మీకు క్రమ సంఖ్య అవసరమైతే, `i++`కి బదులుగా `++i`ని ఉపయోగించండి
- అనవసరమైన`?.`ని నివారించండి మరియు డిఫెన్సివ్ ప్రోగ్రామింగ్‌ను నిషేధించండి
- వేరియబుల్ పేరు బహువచనం కానవసరం లేదు మరియు జాబితాను సూచించడానికి `_li`తో ముగుస్తుంది.

## నామకరణ సమావేశం

- నామకరణం మినిమలిజంను అనుసరిస్తుంది. చిన్న కానీ అర్థవంతమైన పేర్లను ఉపయోగించండి, ఉదాహరణకు: `rm` బదులుగా `remove`, `delete`, `del`ని ఉపయోగించండి. అయితే, మీరు విపరీతమైన స్థితికి వెళ్లడం కూడా నివారించాలి, ఉదాహరణకు: `map`కి బదులుగా `m` అనే ఒక్క అక్షరాన్ని ఉపయోగించవద్దు
- ఫంక్షన్ నామకరణంలో క్రియలను మాత్రమే ఉపయోగించడానికి ప్రయత్నించండి. మీరు దానిని వ్యక్తీకరించడానికి ఒక పదాన్ని ఉపయోగించగలిగితే, రెండు పదాలను ఉపయోగించవద్దు. నామవాచకాలు ఫైల్ పేర్లలో ప్రతిబింబిస్తాయి. మీరు ఫైల్ పేరుకు క్రియను జోడించాలనుకుంటే, దయచేసి ముందుగా నామవాచకాన్ని మరియు చివరి క్రియను ఉంచండి. ఉదాహరణకు: `setProfile.js`కి బదులుగా `profileSet.js`
- వేరియబుల్ పేరు: అండర్లైన్ శైలిని ఉపయోగించండి (snake_case), ఉదాహరణకు `user_auth_token`; వేరియబుల్ ఒక ఫంక్షన్ అయితే, లోయర్‌కేస్ క్యామెల్ కేస్ నేమింగ్ స్టైల్‌ని ఉపయోగించండి
- ఫంక్షన్ పేరు: లోయర్‌కేస్ కామెల్‌కేస్ (ఒంటెకేస్) ఉపయోగించండి, ఉదాహరణకు `userData`
- ఫంక్షన్ పారామితులు: ఇది కాల్‌బ్యాక్ ఫంక్షన్ అయితే, `onChange` వంటి చిన్న కేస్ క్యామెల్ కేస్‌లో పేరు పెట్టండి
- మాడ్యూల్-స్థాయి స్థిరమైన నిర్వచనాలు పెద్ద అక్షరాల అండర్‌లైన్ శైలిని `UPPER_SNAKE_CASE` ఉపయోగిస్తాయి
- `get` వంటి అర్థరహిత ఉపసర్గలను వ్రాయవద్దు: `getCookie`కి బదులుగా `cookieByHeader` అని వ్రాయండి
- గ్లోబల్/మాడ్యూల్-స్థాయి స్థిరాంకాలు: పెద్ద అక్షరాల అండర్‌లైన్ శైలిని ఉపయోగించండి (UPPER_SNAKE_CASE), ఉదా. `CODE_TO_ID`, `ID_TO_LANG`

## మాడ్యులర్ మెకానిజం

- దిగుమతి: డిమాండ్‌పై ఖచ్చితంగా ఫంక్షన్‌లను దిగుమతి చేయండి, మొత్తం మాడ్యూల్ యొక్క ప్రత్యక్ష దిగుమతిని నిషేధించండి (`import * as x`ని నివారించండి లేదా పెద్ద వస్తువులను దిగుమతి చేయండి)
- ఎగుమతి: వస్తువుల ఎగుమతిని నిలిపివేయండి. `export const X=1, abc=()=>{};` వంటి ఫంక్షన్‌లు మరియు వేరియబుల్‌లను యూనిట్‌లుగా ఎగుమతి చేయండి. ఎగుమతి చేసిన కంటెంట్‌ను ప్రకటించడానికి వాటిని const + కామాతో కలపడానికి ప్రయత్నించండి. ఫైల్‌కి ఒకే ఒక ఫంక్షన్ ఉంటే, `export default`ని ఉపయోగించండి
- మీరు ఎగుమతి డిఫాల్ట్ ఫంక్షన్‌ను అంతర్గతంగా కాల్ చేయాల్సిన అవసరం లేనట్లయితే, ముందుగా స్థిరాంకాలను ప్రకటించి, ఆపై ఫైల్ చివరిలో వాటిని ఎగుమతి చేయకుండా ఉండండి.
- పాత్ రిజల్యూషన్: ప్రస్తుత డైరెక్టరీ పాత్‌ను పొందేటప్పుడు, `import.meta.dirname` తప్పనిసరిగా ఉపయోగించాలి
