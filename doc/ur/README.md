# WebC.site

## یہ کیا ہے؟

### یہ ایک مقامی ویب جزو لائبریری ہے۔

[ویب اجزاء](https://developer.mozilla.org/docs/Web/API/Web_components) کی بنیاد پر تیار کیا گیا، یہ رن ٹائم آزاد اور تمام فریم ورک کے لیے موزوں ہے۔

سائز کے لیے انتہائی بہتر بنایا گیا (اسی طرح کی مشہور لائبریریوں کے سائز کا دسواں حصہ)۔

75 زبانوں کی حمایت کرتا ہے۔ آپ آن لائن استعمال کے لیے براہ راست js اور css درآمد کر سکتے ہیں، یا ثانوی ترقی کے لیے ضرورت کے مطابق اپنے پروجیکٹ میں اجزاء شامل کر سکتے ہیں۔

<img src="https://i-01.eu.org/20260604114250265.avif" width="400">

### یہ جزو لائبریری کی ترقی کا فریم ورک ہے۔

یہ [سٹوری بک](https://storybook.js.org) کا جامد بلاگ ورژن ہے۔

اجزاء کی لائبریری کی تعمیر کمپنی کی اندرونی ترقی کی کارکردگی کو بہتر بنا سکتی ہے اور تکنیکی دائرے میں ایک برانڈ امیج بنا سکتی ہے۔

[webc.site](https://github.com/webc-site/webc.site) کی بنیاد پر، ٹیمیں آسانی سے اپنی اجزاء کی لائبریریاں بنا سکتی ہیں اور انہیں [github page](https://pages.github.com)، [cloudflare page](https://pages.cloudflare.com) وغیرہ پر شائع کر سکتی ہیں۔ شائع شدہ اجزاء کو [codepen.io](https://codepen.io) اور [jsfiddle](https://jsfiddle.net) پر آن لائن ڈیبگ کیا جا سکتا ہے۔

ایک ہی وقت میں، بلٹ ان ترجمے کا فریم ورک اجزاء اور دستاویزات کی بین الاقوامی کاری کو تیزی سے محسوس کر سکتا ہے، تاکہ آپ کے پروڈکٹ کے سامعین مزید زبان کے پابند نہ ہوں۔

### یہ مصنوعی ذہانت کے دور کے لیے فرنٹ اینڈ ڈیولپمنٹ کا ایک نیا نمونہ ہے۔

فریم ورک بلٹ ان ہے `.agents/skills`، مصنوعی ذہانت کو بہترین طریقوں کے مطابق ایک کلک کے ساتھ نئے اجزاء تیار کرنے میں آپ کی مدد کرنے کی اجازت دیتا ہے۔

<img src="https://i-01.eu.org/20260604121408079.avif" width="400">

فرنٹ اینڈ کوڈ کو ماحول کے پروگرامنگ کے غیر پائیدار کچرے کے پہاڑ سے ایک معیاری حصے میں تبدیل ہونے دیں جسے برقرار رکھا جا سکتا ہے اور طویل مدت کے دوران تکراری طور پر تیار کیا جا سکتا ہے۔

مصنوعی ذہانت کی مدد سے ترقی کی خصوصیات کے پیش نظر، ہم نے ایک نیا فرنٹ اینڈ ڈیولپمنٹ پیراڈائم تجویز کیا اور ڈیزائن کیا:

1. **ہر چیز ایک ویب جزو ہے**
2. **جز کو بیک اینڈ ڈیٹا لنک سے آزاد ہونا چاہیے، اور ڈیٹا انٹرفیس کال بیک فنکشن کے طور پر سامنے آتا ہے**

اس طرح، مصنوعی ذہانت جعلی ڈیٹا تیار کر سکتی ہے اور ڈیمو پیج پر اجزاء کی مختلف حالتیں ظاہر کر سکتی ہے۔

اجزاء کی ترقی اور ڈیٹا انٹرفیس کی علیحدگی، [webc.site جزو فریم ورک](https://github.com/webc-site/webc.site) کے ساتھ مل کر ایک اور اہم خصوصیت:

**ہر جزو ڈویلپمنٹ سرور کو آزادانہ طور پر شروع کر سکتا ہے اور ڈیمو پیج پر ڈیبگ کر سکتا ہے**۔

یہ ہر ایک جزو کی ترقی اور ڈیبگنگ کے لیے فرنٹ اینڈ لنک کو `0` تک کم کر دیتا ہے۔

اس طرح، مصنوعی ذہانت کی مدد سے ڈویلپمنٹ ٹولز ([کلاڈ کوڈ](https://code.claude.com)، [اوپن اے آئی کوڈیکس](https://openai.com)، [گوگل اینٹی گریویٹی](https://antigravity.google)، [کرسر](https://cursor.com)، [ونڈ سرف](https://windsurf.com)، [ڈیوین](https://devin.ai)، [اوپن ہینڈز](https://github.com/All-Hands-AI/OpenHands) وغیرہ، آپ کو خود بخود براؤزر کھولنے سے لطف اندوز نہیں کیا جاسکتا ہے مختلف گندے عوامل (بیک اینڈ ڈیٹا، لاگ ان اسٹیٹس، پاتھ روٹنگ، وغیرہ) کے ذریعے مسدود۔

اگر آپ مصنوعی ذہانت کو کنٹرول کرنا چاہتے ہیں اور لاگت کو کم کرنا چاہتے ہیں اور کارکردگی کو بڑھانا چاہتے ہیں، تو آپ کو ایک نئے نمونے کی ضرورت ہے جو ہر جزو کی ترقی اور ڈیبگنگ ماحول کو کم سے کم کرے۔

# میں نے ابھی TODO لکھنا شروع نہیں کیا ہے۔

### ویب اجزاء کی لائبریری

#### حتمی سائز کی اصلاح

مثال کے طور پر، ورچوئل اسکرول بار کے جزو کے لیے، `gz` کے بعد مقبول [OverlayScrollbars](https://github.com/KingSora/OverlayScrollbars) اسکرپٹ اور اسٹائل کے سائز کا موازنہ کریں، جیسا کہ درج ذیل جدول میں دکھایا گیا ہے:

|                   |   js    |  css   |     total      |
| :---------------- | :-----: | :----: | :------------: |
| overlayscrollbars | 14520 B | 2584 B | 17104 B (9.1X) |
| Scroll            | 1159 B  | 721 B  |   ⭐ 1880 B    |

`webc.site` کا ورچوئل اسکرول بار کا جزو صرف `OverlayScrollbars` کا `10%` ہے۔

#### رن ٹائم آزاد

معیارات [ویب اجزاء](https://developer.mozilla.org/docs/Web/API/Web_components) پر بنایا گیا، webc.site اجزاء کی لائبریری قدرتی طور پر رن ​​ٹائم ایگنوسٹک ہے۔ چاہے آپ React، Vue، Angular، Svelte، یا کوئی روایتی خالص HTML/JS پروجیکٹ استعمال کر رہے ہوں، تمام اجزاء کسی مخصوص فریم ورک کے لیے سیکنڈری پیکیجنگ کی ضرورت کے بغیر، مقامی HTML ٹیگز کی طرح براؤزر میں براہ راست چل سکتے ہیں۔

##### shadcn کے ساتھ موازنہ

[shadcn/ui](https://ui.shadcn.com) کوڈ ڈسٹری بیوشن موڈ کو اپناتا ہے، جس کے لیے آپ کو جزو کوڈ کو براہ راست اپنے پروجیکٹ میں کاپی کرنے کی ضرورت ہوتی ہے۔ یہ React ایکو سسٹم اور مخصوص کنسٹرکشن ٹول چینز (جیسے Tailwind CSS) کے ساتھ مضبوطی سے پابند ہے، اور اسے پورے فریم ورک میں استعمال نہیں کیا جا سکتا۔ اس میں پروجیکٹ انجینئرنگ کنفیگریشن کے لیے بھی کچھ تقاضے ہیں۔

اور `webc.site` کا جزو:
- **کراس فریم ورک یونیورسل**: ایک بار لکھیں اور اسے براہ راست متعارف کرایا جا سکتا ہے اور کسی بھی فرنٹ اینڈ فریم ورک (حتی کہ فریم ورک سے پاک ماحول) میں چلایا جا سکتا ہے۔
- **ماحول میں زیرو انحصار**: کسی مخصوص پیکیجنگ ٹولز یا سی ایس ایس پری پروسیسرز کی ضرورت نہیں ہے، بس مرتب شدہ جامد فائلیں درآمد کریں اور اسے باکس سے باہر استعمال کیا جا سکتا ہے۔

## خودکار ترقی اور ڈیبگنگ

### مقامی ترقی اور ڈیبگنگ

ہر ویب صفحہ کا جزو ڈیولپمنٹ سرور کو آزادانہ طور پر شروع کرنے کی حمایت کرتا ہے، کم سے کم ترقی اور ڈیبگنگ ماحول فراہم کرتا ہے:

- **ایک نیا جزو بنائیں**: `./sh/new.js [css|js] [جز کا نام]` چلائیں تاکہ فوری طور پر جزو کی ترقی کا ڈھانچہ تیار کیا جا سکے (بشمول ڈیمو صفحہ اور بین الاقوامی کاری کی ترتیب)۔
- **آزاد ڈیبگنگ**: وائٹ ڈیولپمنٹ سرور شروع کرنے کے لیے `./dev.js [component name]` چلائیں (یا `./dev.js` براہ راست چلائیں) اور براؤزر میں جزو کا ڈیمو صفحہ خود بخود کھولیں۔ جزو ہاٹ اپ ڈیٹ (HMR) کی حمایت کرتا ہے۔ چونکہ ڈیولپمنٹ اور ڈیٹا لنکس کو ڈیکپل کیا جاتا ہے، اس لیے صفر ڈیولپمنٹ فرنٹ اینڈ لنک ہے، جو کہ مکمل طور پر خودکار ڈیولپمنٹ اور براؤزر ڈیبگنگ میں مدد کرنے کے لیے AI (جیسے کلاڈ کوڈ، گوگل اینٹی گریویٹی، وغیرہ) کے لیے بہت موزوں ہے۔

### اجزاء کی لائبریری شائع کریں۔

#### webc dist

پیکیجنگ اور اشاعت کا عمل انتہائی خودکار ہے:
- **جامد وسائل کی پیکیجنگ**: `./sh/dist.js` (یعنی `webc dist` کام کو انجام دینا) خود بخود `src/` کے تحت ویب صفحہ کے تمام اجزاء کو اسکین کر سکتا ہے، انہیں کمپائل، کمپریس اور `dist/` ڈائریکٹری میں آؤٹ پٹ کر سکتا ہے۔ اس ڈائرکٹری میں پروڈکشن ڈسٹری بیوشن کے لیے JS/CSS فائلیں شامل ہیں، ایک ڈیمو سائٹ جو GitHub Pages یا Cloudflare Pages پر تعیناتی کے لیے تیار ہے، اور خود بخود `_redirects` اور `404.html` روٹنگ فائلیں تیار کرتی ہیں۔
- **NPM Publish**: `./sh/npmDist.js` چلائیں، اسکرپٹ اجزاء کی انحصار کو خود بخود ہینڈل کرے گا، پیکیج کے ورژن نمبر میں اضافہ کرے گا، اور پیکیجنگ مکمل ہونے کے بعد ایک کلک کے ساتھ `npm publish` کو NPM پر جزو لائبریری شائع کرنے کے لیے عمل میں لائے گا۔

### پلیٹ فارم ترجمہ کے لیے ذمہ دار ہے۔

فریم ورک میں خودکار ترجمے کے عمل شامل ہیں:
1. پروجیکٹ روٹ ڈائرکٹری میں `tran.yml` میں ترجمے کے قواعد (جیسے ماخذ کی زبان، ہدف کی زبان اور ڈائریکٹری کا ترجمہ کیا جانا ہے) کو ترتیب دیں۔
2. اپنے کوڈ میں متعدد زبانوں کو ہارڈ کوڈ کیے بغیر اجزاء یا دستاویزات میں پلیس ہولڈرز کا استعمال کریں۔
3. `./sh/tran.js` چلائیں، اور ترجمہ انجن خود کار طریقے سے متن کو نکالے گا اور پروسیسنگ کے لیے WebC ترجمہ سروس کو کال کرے گا، خود کار طریقے سے YAML ترجمہ کے اندراجات اور مختلف زبانوں میں README دستاویزات تیار کرے گا۔

### ماحولیاتی متغیرات

خودکار ترجمہ کرتے وقت، آپ کو ترجمہ API تک رسائی کی اجازت دینے کے لیے درج ذیل ماحولیاتی متغیرات کو ترتیب دینے کی ضرورت ہے:

- **`WEBC_SITE_TOKEN`** (یا **`WEBC_TOKEN`**): توثیق کی کلید `webc.site` خودکار ترجمہ API تک رسائی کے لیے استعمال کی جاتی ہے۔ ماحول کے متغیر کے طور پر انجکشن کیا جا سکتا ہے یا مقامی صارف کی ہوم ڈائرکٹری میں `~/.config/webc.site.yml` کنفیگریشن فائل میں `token` لکھا جا سکتا ہے۔
