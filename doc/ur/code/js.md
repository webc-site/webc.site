# جے ایس کوڈ کی تفصیلات

## کوڈنگ سٹائل

- جدید ترین js نحو کا استعمال کرتے ہوئے سادہ، خوبصورت اور موثر
- ضرورت سے زیادہ لمبے فنکشنز کو تقسیم کریں، اور انٹرفیس کا ڈیزائن کم کپلنگ اور اعلی ہم آہنگی والا ہونا چاہیے۔
- arrays + loops/forEach/map زیادہ استعمال کریں اور کم بار بار کوڈ لکھیں۔
- ضرورت سے زیادہ تبصروں سے بچنے کے لیے سادہ کوڈ میں فنکشن کمنٹس نہ لکھیں۔ پیچیدہ منطق اور خصوصی ضروریات کو بہتر چینی میں بیان کیا گیا ہے۔
- فالتو پن کو کم کرنے کے لیے دہرائے جانے والے کوڈ کو افعال میں خلاصہ کیا جاتا ہے۔ مثال کے طور پر، `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` کو `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()` لکھا جا سکتا ہے۔
- متغیر اعلامیہ: متعدد مسلسل `const` اعلانات کو ایک بیان میں یکجا کریں۔ اسے تین لائنوں میں لکھنے کے بجائے `const a=1, b=2, c=3;` لکھیں۔
- غیر مطابقت پذیر پروسیسنگ: `await` استعمال کریں، `.then` چین کالز کے استعمال پر پابندی لگائیں
- استثنیٰ ہینڈلنگ کوڈ خود بخود نہ بنائیں، خود بخود `try...catch` نہ لکھیں (`try ... catch` کو دستی طور پر برقرار رکھا جاتا ہے، موجودہ `try catch` کو حذف نہیں کیا جانا چاہیے)
- پہلے خالص فنکشنز: صرف خالص فنکشن لکھیں، کلاس کبھی نہیں۔
- تیر کے فنکشنز `const funcName = () => {}` استعمال کریں اور `function` کلیدی لفظ استعمال نہ کریں (سوائے جنریٹرز کے)؛ اگر آپ پیرامیٹرز کو باندھنے کے لیے .bind استعمال کر سکتے ہیں، تو افعال کی وضاحت کرنے سے گریز کریں۔
- کوڈ کا دوبارہ استعمال: دوبارہ استعمال کرنے پر توجہ دیں، مزید چھوٹے فنکشن نکالیں، اور بڑی تعداد میں ملتے جلتے یا کاپی پیسٹ شدہ کوڈ ڈھانچے سے پرہیز کریں۔
- آبجیکٹ تک رسائی: مطلوبہ صفات کو نکالنے کے لیے ڈیسٹرکچرنگ اسائنمنٹ کے استعمال کو ترجیح دیں، گہری اور نیسٹڈ اوصاف تک رسائی کے لیے اندرونی طور پر نقطوں کو بار بار استعمال کرنے سے گریز کریں، اور بار بار اختیاری سلسلہ کے فیصلوں کو ضم کریں۔
- فنکشن پیرامیٹرز کے لیے اشیاء کا استعمال نہ کریں، `{a,b,c}` نہ لکھیں، `a, b, c` لکھیں؛ اگر بہت سے اختیاری پیرامیٹرز ہیں تو، [[کنفیگریشن آئٹمز کی تعداد، کنفیگریشن آئٹمز]، [کنفیگریشن آئٹمز کی تعداد، کنفیگریشن آئٹمز]،..] استعمال کریں
- کثیر قدر کی واپسی کے لیے، `{a,b,c}` کی بجائے `[a,b,c]` کا استعمال کریں۔ اگر 3 سے زیادہ واپسی کی قدریں ہیں، تو پوزیشنی معنی کی وضاحت کے لیے عددی مستقل استعمال کریں۔
- ریاست کی نمائندگی کرنے کے لیے کسی تار کی ضرورت نہیں ہے، ریاست کی وضاحت کے لیے عددی مستقل استعمال کریں۔
- کوئی سٹرنگ ٹیمپلیٹ (``) استعمال نہیں کیا جاتا ہے، سٹرنگ کنکٹنیشن (+) استعمال کیا جاتا ہے، سوائے درآمد کے (Vite static analysis کے لیے آسان)
- لوپ کے لیے، اگر آپ کو سیریل نمبر کی ضرورت ہے، تو `i++` کی بجائے `++i` استعمال کریں۔
- غیر ضروری`?.` سے بچیں اور دفاعی پروگرامنگ سے منع کریں۔
- متغیر کا نام جمع ہونے کی ضرورت نہیں ہے، اور فہرست کی نشاندہی کرنے کے لیے `_li` کے ساتھ ختم ہوتا ہے۔

## نام دینے کا کنونشن

- نام کم سے کمیت کی پیروی کرتا ہے۔ مختصر لیکن معنی خیز نام استعمال کریں، مثال کے طور پر: `remove`، `delete`، `del` کی بجائے `rm` استعمال کریں۔ تاہم، آپ کو انتہا پر جانے سے بھی گریز کرنا چاہیے، مثال کے طور پر: `map` کی بجائے ایک حرف `m` استعمال نہ کریں۔
- فنکشن کے نام میں صرف فعل استعمال کرنے کی کوشش کریں۔ اگر آپ اس کے اظہار کے لیے ایک لفظ استعمال کر سکتے ہیں تو دو الفاظ استعمال نہ کریں۔ اسم فائل کے ناموں میں جھلکتے ہیں۔ اگر آپ کو فائل کے نام میں کوئی فعل شامل کرنے کی ضرورت ہے، تو براہ کرم اسم کو پہلے اور فعل کو آخر میں رکھیں۔ مثال کے طور پر: `setProfile.js` کی بجائے `profileSet.js`
- متغیر نام: انڈر لائن اسٹائل (snake_case) استعمال کریں، مثال کے طور پر `user_auth_token`؛ اگر متغیر ایک فنکشن ہے تو، چھوٹے اونٹ کے کیس کا نام دینے کا انداز استعمال کریں۔
- فنکشن کا نام: لوئر کیس کیمل کیس (کیمل کیس) استعمال کریں، مثال کے طور پر `userData`
- فنکشن پیرامیٹرز: اگر یہ کال بیک فنکشن ہے تو اسے چھوٹے اونٹ کے کیس میں نام دیں، جیسے `onChange`
- ماڈیول کی سطح کی مستقل تعریفیں بڑے حروف میں انڈر لائن اسٹائل `UPPER_SNAKE_CASE` استعمال کرتی ہیں۔
- `get` نہ لکھیں، ایسے بے معنی سابقے، جیسے: `getCookie` کی بجائے `cookieByHeader` لکھیں۔
- عالمی/ماڈیول سطح کے مستقل: بڑے حروف کی انڈر لائن اسٹائل (UPPER_SNAKE_CASE) استعمال کریں، جیسے `CODE_TO_ID`، `ID_TO_LANG`

## ماڈیولر میکانزم

- درآمد کریں: مطالبے پر درست طریقے سے افعال درآمد کریں، پورے ماڈیول کی براہ راست درآمد پر پابندی لگائیں (`import * as x` سے بچیں یا بڑی اشیاء درآمد کریں)
- برآمد کریں: اشیاء کی برآمد کو غیر فعال کریں۔ فنکشنز اور متغیرات کو اکائیوں کے طور پر برآمد کریں، جیسے `export const X=1, abc=()=>{};`۔ برآمد شدہ مواد کا اعلان کرنے کے لیے انہیں const + comma کے ساتھ جوڑنے کی کوشش کریں۔ اگر کسی فائل میں صرف ایک فنکشن ہے تو `export default` استعمال کریں۔
- جب تک کہ آپ کو ایکسپورٹ ڈیفالٹ فنکشن کو اندرونی طور پر کال کرنے کی ضرورت نہ ہو، پہلے مستقل کا اعلان کرنے اور پھر فائل کے آخر میں ان کو ایکسپورٹ کرنے سے گریز کریں۔
- پاتھ ریزولوشن: موجودہ ڈائرکٹری پاتھ حاصل کرتے وقت، `import.meta.dirname` استعمال کرنا ضروری ہے۔

## غلطی

- سٹرنگ کی غلطیوں سے بچیں اور مستقل ایرر کوڈز کا اعلان کرنے کے لیے const کا استعمال کریں۔
- اگر آپ کو ڈیٹا کی غلط معلومات واپس کرنے کی ضرورت ہے تو، استعمال کریں [ایرر کوڈ، ایرر میسج،...] اس طرح، ایرر میسج ٹیکسٹ ڈسکرپشن نہیں ہے بلکہ ایک عددی قدر ہے (جیسے [FILE_OVERSIZE, file_size, max_size])
