# জেএস কোড স্পেসিফিকেশন

## কোডিং শৈলী

- সবচেয়ে আধুনিক js সিনট্যাক্স ব্যবহার করে সহজ, মার্জিত এবং দক্ষ
- অত্যধিক দীর্ঘ ফাংশন বিভক্ত করুন, এবং ইন্টারফেস ডিজাইন কম কাপলিং এবং উচ্চ সমন্বয় হওয়া উচিত।
- আরো ব্যবহার করুন অ্যারে + loops/forEach/map এবং কম পুনরাবৃত্তি কোড লিখুন
- অতিরিক্ত মন্তব্য এড়াতে সাধারণ কোডে ফাংশন মন্তব্য লিখবেন না। জটিল যুক্তি এবং বিশেষ চাহিদা পরিমার্জিত চীনা ভাষায় ব্যাখ্যা করা হয়েছে।
- রিডানড্যান্সি কমাতে বারবার কোডকে ফাংশনে বিমূর্ত করা হয়। উদাহরণস্বরূপ, `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()` হিসাবে লেখা যেতে পারে
- পরিবর্তনশীল ঘোষণা: একটি বিবৃতিতে একাধিক পরপর `const` ঘোষণা একত্রিত করুন। তিন লাইনে না লিখে `const a=1, b=2, c=3;` লিখুন
- অ্যাসিঙ্ক্রোনাস প্রক্রিয়াকরণ: `await` ব্যবহার করুন, `.then` চেইন কল ব্যবহার নিষিদ্ধ করুন
- স্বয়ংক্রিয়ভাবে ব্যতিক্রম হ্যান্ডলিং কোড তৈরি করবেন না, স্বয়ংক্রিয়ভাবে `try...catch` লিখবেন না (`try ... catch` ম্যানুয়ালি রক্ষণাবেক্ষণ করা হয়, বিদ্যমান `try catch` মুছে ফেলা উচিত নয়)
- বিশুদ্ধ ফাংশন প্রথম: শুধুমাত্র বিশুদ্ধ ফাংশন লিখুন, ক্লাস না
- তীর ফাংশন ব্যবহার করুন `const funcName = () => {}` এবং `function` কীওয়ার্ড ব্যবহার করবেন না (জেনারেটর ছাড়া); যদি আপনি .bind ব্যবহার করতে পারেন পরামিতি আবদ্ধ করতে, ফাংশন সংজ্ঞায়িত করা এড়িয়ে চলুন
- কোড পুনঃব্যবহার: পুনঃব্যবহারের দিকে মনোযোগ দিন, আরও ছোট ফাংশন বের করুন এবং দৃঢ়ভাবে অনুরূপ বা কপি-পেস্ট করা কোড কাঠামোর একটি বড় সংখ্যা এড়িয়ে চলুন।
- অবজেক্ট অ্যাক্সেস: প্রয়োজনীয় অ্যাট্রিবিউট বের করার জন্য ডেস্ট্রাকচারিং অ্যাসাইনমেন্ট ব্যবহার করে অগ্রাধিকার দিন, গভীর এবং নেস্টেড অ্যাট্রিবিউট অ্যাক্সেস করতে বারবার অভ্যন্তরীণভাবে ডট ব্যবহার করা এড়িয়ে চলুন এবং বারবার ঐচ্ছিক চেইন বিচারকে একত্রিত করুন
- ফাংশন প্যারামিটারের জন্য বস্তু ব্যবহার করবেন না, `{a,b,c}` লিখবেন না, `a, b, c` লিখবেন না; যদি অনেক ঐচ্ছিক পরামিতি থাকে, তাহলে [[কনফিগারেশন আইটেমের সংখ্যা, কনফিগারেশন আইটেম], [কনফিগারেশন আইটেমের সংখ্যা, কনফিগারেশন আইটেম],..] ব্যবহার করুন
- মাল্টি-ভ্যালু রিটার্নের জন্য, `{a,b,c}` এর পরিবর্তে অ্যারে `[a,b,c]` ব্যবহার করুন। 3টির বেশি রিটার্ন মান থাকলে, অবস্থানগত অর্থ সংজ্ঞায়িত করতে সাংখ্যিক ধ্রুবক ব্যবহার করুন।
- রাষ্ট্রের প্রতিনিধিত্ব করার জন্য একটি স্ট্রিংয়ের প্রয়োজন নেই, রাষ্ট্রকে সংজ্ঞায়িত করতে সংখ্যাসূচক ধ্রুবক ব্যবহার করুন
- কোনো স্ট্রিং টেমপ্লেট (``) ব্যবহার করা হয় না, স্ট্রিং কনক্যাটেনেশন (+) ব্যবহার করা হয়, আমদানি ছাড়া (Vite স্ট্যাটিক বিশ্লেষণের জন্য সুবিধাজনক)
- লুপের জন্য, আপনার যদি সিরিয়াল নম্বরের প্রয়োজন হয়, `i++` এর পরিবর্তে `++i` ব্যবহার করুন
- অপ্রয়োজনীয়`?.` এড়িয়ে চলুন এবং প্রতিরক্ষামূলক প্রোগ্রামিং নিষিদ্ধ করুন
- পরিবর্তনশীল নামটি বহুবচন হতে হবে না, এবং একটি তালিকা নির্দেশ করতে `_li` দিয়ে শেষ হয়।

## নামকরণের রীতি

- নামকরণ minimalism অনুসরণ করে. ছোট কিন্তু অর্থপূর্ণ নাম ব্যবহার করুন, উদাহরণস্বরূপ: `remove`, `delete`, `del` এর পরিবর্তে `rm` ব্যবহার করুন৷ যাইহোক, আপনার চরম পর্যায়ে যাওয়া এড়ানো উচিত, উদাহরণস্বরূপ: `map` এর পরিবর্তে একটি অক্ষর `m` ব্যবহার করবেন না
- ফাংশন নামকরণে শুধুমাত্র ক্রিয়া ব্যবহার করার চেষ্টা করুন। আপনি যদি এটি প্রকাশ করতে একটি শব্দ ব্যবহার করতে পারেন তবে দুটি শব্দ ব্যবহার করবেন না। বিশেষ্যগুলি ফাইলের নামগুলিতে প্রতিফলিত হয়। আপনার যদি ফাইলের নামের সাথে একটি ক্রিয়া যুক্ত করার প্রয়োজন হয়, অনুগ্রহ করে বিশেষ্যটি প্রথমে এবং ক্রিয়াটি শেষে রাখুন। যেমন: `setProfile.js` এর পরিবর্তে `profileSet.js`
- পরিবর্তনশীল নাম: আন্ডারলাইন স্টাইল ব্যবহার করুন (snake_case), উদাহরণস্বরূপ `user_auth_token`; ভেরিয়েবল একটি ফাংশন হলে, ছোট হাতের ক্যামেল কেস নামকরণ শৈলী ব্যবহার করুন
- ফাংশনের নাম: ছোট হাতের ক্যামেলকেস (camelCase) ব্যবহার করুন, উদাহরণস্বরূপ `userData`৷
- ফাংশন প্যারামিটার: যদি এটি একটি কলব্যাক ফাংশন হয়, তাহলে এটিকে ছোট হাতের ক্যামেল কেসে নাম দিন, যেমন `onChange`
- মডিউল-স্তরের ধ্রুবক সংজ্ঞাগুলি বড় হাতের আন্ডারলাইন শৈলী `UPPER_SNAKE_CASE` ব্যবহার করে৷
- `get` লিখবেন না, যেমন অর্থহীন উপসর্গ, যেমন: `getCookie` এর পরিবর্তে `cookieByHeader` লিখুন
- গ্লোবাল/মডিউল-স্তরের ধ্রুবক: বড় হাতের আন্ডারলাইন স্টাইল ব্যবহার করুন (UPPER_SNAKE_CASE), যেমন `CODE_TO_ID`, `ID_TO_LANG`৷

## মডুলার মেকানিজম

- আমদানি: চাহিদা অনুযায়ী সঠিকভাবে ফাংশন আমদানি করুন, সম্পূর্ণ মডিউলের সরাসরি আমদানি নিষিদ্ধ করুন (`import * as x` এড়িয়ে চলুন বা বড় বস্তু আমদানি করুন)
- রপ্তানি: বস্তুর রপ্তানি অক্ষম করুন। ইউনিট হিসাবে ফাংশন এবং ভেরিয়েবল রপ্তানি করুন, যেমন `export const X=1, abc=()=>{};`৷ রপ্তানি করা বিষয়বস্তু ঘোষণা করতে একটি const + কমা দিয়ে তাদের একত্রিত করার চেষ্টা করুন। একটি ফাইলের শুধুমাত্র একটি ফাংশন থাকলে, `export default` ব্যবহার করুন৷
- আপনার অভ্যন্তরীণভাবে এক্সপোর্ট ডিফল্ট ফাংশন কল করার প্রয়োজন না হলে, প্রথমে ধ্রুবক ঘোষণা করা এড়িয়ে চলুন এবং তারপর ফাইলের শেষে রপ্তানি করুন।
- পাথ রেজোলিউশন: বর্তমান ডিরেক্টরি পাথ পাওয়ার সময়, `import.meta.dirname` ব্যবহার করতে হবে

## ভুল

- স্ট্রিং ত্রুটিগুলি এড়িয়ে চলুন এবং ধ্রুবক ত্রুটি কোডগুলি ঘোষণা করতে const ব্যবহার করুন।
- আপনি যদি ভুল তথ্য তথ্য ফেরত দিতে চান, তাহলে [ত্রুটির কোড, ত্রুটি বার্তা,...] ব্যবহার করুন এইভাবে, ত্রুটি বার্তাটি পাঠ্য বিবরণ নয়, তবে একটি সংখ্যাসূচক মান (যেমন [FILE_OVERSIZE, file_size, max_size])
