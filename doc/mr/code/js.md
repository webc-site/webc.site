# जेएस कोड तपशील

## कोडिंग शैली

- सर्वात आधुनिक js वाक्यरचना वापरून साधे, मोहक आणि कार्यक्षम
- जास्त लांब फंक्शन्स विभाजित करा आणि इंटरफेस डिझाइन कमी कपलिंग आणि उच्च एकसंध असावे.
- ॲरे + लूप/प्रत्येक/नकाशासाठी अधिक वापरा आणि कमी पुनरावृत्ती कोड लिहा
- जास्त टिप्पण्या टाळण्यासाठी फंक्शन टिप्पण्या सोप्या कोडमध्ये लिहू नका. जटिल तर्कशास्त्र आणि विशेष गरजा परिष्कृत चिनी भाषेत स्पष्ट केल्या आहेत.
- रिडंडंसी कमी करण्यासाठी पुनरावृत्ती कोड फंक्शन्समध्ये अमूर्त केला जातो. उदाहरणार्थ, `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()` असे लिहिले जाऊ शकते
- व्हेरिएबल डिक्लेरेशन: एका विधानात अनेक सलग `const` घोषणा एकत्र करा. तीन ओळींमध्ये लिहिण्याऐवजी `const a=1, b=2, c=3;` लिहा
- असिंक्रोनस प्रक्रिया: `await` वापरा, `.then` साखळी कॉलचा वापर प्रतिबंधित करा
- अपवाद हाताळणी कोड आपोआप व्युत्पन्न करू नका, आपोआप `try...catch` लिहू नका (`try ... catch` मॅन्युअली ठेवली जाते, विद्यमान `try catch` हटवू नये)
- प्रथम शुद्ध कार्ये: फक्त शुद्ध कार्ये लिहा, वर्ग कधीही
- बाण फंक्शन्स वापरा `const funcName = () => {}` आणि `function` कीवर्ड वापरू नका (जनरेटर वगळता); पॅरामीटर्स बांधण्यासाठी तुम्ही .bind वापरू शकत असल्यास, फंक्शन्स परिभाषित करणे टाळा
- कोडचा पुनर्वापर: पुनर्वापराकडे लक्ष द्या, अधिक लहान फंक्शन्स काढा आणि मोठ्या संख्येने समान किंवा कॉपी-पेस्ट केलेल्या कोड संरचना टाळा.
- ऑब्जेक्ट ऍक्सेस: आवश्यक गुणधर्म काढण्यासाठी डिस्ट्रक्चरिंग असाइनमेंट वापरण्यास प्राधान्य द्या, खोल आणि नेस्टेड विशेषतांमध्ये प्रवेश करण्यासाठी वारंवार डॉट्स वापरणे टाळा आणि वारंवार पर्यायी साखळी निर्णय विलीन करा
- फंक्शन पॅरामीटर्ससाठी ऑब्जेक्ट वापरू नका, `{a,b,c}` लिहू नका, `a, b, c` लिहा; जर अनेक पर्यायी पॅरामीटर्स असतील तर, [[कॉन्फिगरेशन आयटमची संख्या, कॉन्फिगरेशन आयटम], [कॉन्फिगरेशन आयटमची संख्या, कॉन्फिगरेशन आयटम],..] वापरा या पॅराडिझममध्ये, कॉन्फिगरेशन आयटम संख्यात्मक स्थिरांकांसह परिभाषित केले आहेत.
- मल्टी-व्हॅल्यू रिटर्नसाठी, `{a,b,c}` ऐवजी `[a,b,c]` ॲरे वापरा. 3 पेक्षा जास्त रिटर्न व्हॅल्यू असल्यास, स्थितीत्मक अर्थ परिभाषित करण्यासाठी अंकीय स्थिरांक वापरा.
- राज्याचे प्रतिनिधित्व करण्यासाठी स्ट्रिंगची आवश्यकता नाही, राज्य परिभाषित करण्यासाठी संख्यात्मक स्थिरांक वापरा
- कोणतेही स्ट्रिंग टेम्प्लेट (``) वापरलेले नाही, स्ट्रिंग जोडणी (+) वापरली जाते, आयात वगळता (विट स्टॅटिक विश्लेषणासाठी सोयीस्कर)
- लूपसाठी, तुम्हाला अनुक्रमांक हवा असल्यास, `i++` ऐवजी `++i` वापरा
- अनावश्यक`?.` टाळा आणि बचावात्मक प्रोग्रामिंग प्रतिबंधित करा
- व्हेरिएबलचे नाव अनेकवचनी असण्याची गरज नाही आणि सूची दर्शविण्यासाठी ते `_li` ने समाप्त होते.

## नामकरण परंपरा

- नामकरण मिनिमलिझमचा पाठपुरावा करते. लहान पण अर्थपूर्ण नावे वापरा, उदाहरणार्थ: `remove`, `delete`, `del` ऐवजी `rm` वापरा. तथापि, तुम्ही टोकाला जाणे देखील टाळले पाहिजे, उदाहरणार्थ: `map` ऐवजी `m` एकच अक्षर वापरू नका
- फंक्शन नेमिंगमध्ये फक्त क्रियापद वापरण्याचा प्रयत्न करा. आपण ते व्यक्त करण्यासाठी एक शब्द वापरू शकत असल्यास, दोन शब्द वापरू नका. संज्ञा फाइल नावांमध्ये प्रतिबिंबित होतात. जर तुम्हाला फाईलच्या नावात क्रियापद जोडायचे असेल तर कृपया नाम प्रथम आणि क्रियापद शेवटचे ठेवा. उदाहरणार्थ: `setProfile.js` ऐवजी `profileSet.js`
- व्हेरिएबलचे नाव: अधोरेखित शैली वापरा (snake_case), उदाहरणार्थ `user_auth_token`; व्हेरिएबल फंक्शन असल्यास, लोअरकेस कॅमल केस नेमिंग शैली वापरा
- फंक्शनचे नाव: लोअरकेस कॅमलकेस (camelCase) वापरा, उदाहरणार्थ `userData`
- फंक्शन पॅरामीटर्स: जर ते कॉलबॅक फंक्शन असेल, तर त्याला लोअरकेस कॅमल केसमध्ये नाव द्या, जसे की `onChange`
- मॉड्यूल-स्तरीय स्थिर व्याख्या अप्परकेस अधोरेखित शैली `UPPER_SNAKE_CASE` वापरतात
- `get` लिहू नका, असे निरर्थक उपसर्ग, जसे की: `getCookie` ऐवजी `cookieByHeader` लिहा
- ग्लोबल/मॉड्यूल-स्तरीय स्थिरांक: अप्परकेस अधोरेखित शैली वापरा (UPPER_SNAKE_CASE), उदा. `CODE_TO_ID`, `ID_TO_LANG`

## मॉड्यूलर यंत्रणा

- आयात करा: मागणीनुसार फंक्शन्स अचूकपणे आयात करा, संपूर्ण मॉड्यूल थेट आयात करण्यास प्रतिबंधित करा (`import * as x` टाळा किंवा मोठ्या वस्तू आयात करा)
- निर्यात: वस्तूंची निर्यात अक्षम करा. फंक्शन्स आणि व्हेरिएबल्स युनिट्स म्हणून एक्सपोर्ट करा, जसे की `export const X=1, abc=()=>{};`. निर्यात केलेली सामग्री घोषित करण्यासाठी त्यांना const + स्वल्पविरामाने एकत्र करण्याचा प्रयत्न करा. फाइलमध्ये एकच फंक्शन असल्यास, `export default` वापरा
- जोपर्यंत तुम्हाला एक्सपोर्ट डीफॉल्ट फंक्शनला अंतर्गत कॉल करण्याची आवश्यकता नाही तोपर्यंत, प्रथम स्थिरांक घोषित करणे टाळा आणि नंतर फाइलच्या शेवटी निर्यात करणे टाळा.
- पथ रिझोल्यूशन: वर्तमान निर्देशिका पथ प्राप्त करताना, `import.meta.dirname` वापरणे आवश्यक आहे

## चूक

- स्ट्रिंग त्रुटी टाळा आणि सतत त्रुटी कोड घोषित करण्यासाठी const वापरा.
- तुम्हाला चुकीची डेटा माहिती परत करायची असल्यास, [एरर कोड, एरर मेसेज,...] वापरा अशा प्रकारे, एरर मेसेज हे मजकूर वर्णन नसून संख्यात्मक मूल्य आहे (जसे की [FILE_OVERSIZE, file_size, max_size])
