# ข้อกำหนดรหัส JS

## สไตล์การเข้ารหัส

- เรียบง่าย หรูหรา และมีประสิทธิภาพ โดยใช้ไวยากรณ์ js ที่ทันสมัยที่สุด
- แยกฟังก์ชันที่ยาวเกินไป และการออกแบบอินเทอร์เฟซควรมีการเชื่อมต่อต่ำและมีการทำงานร่วมกันสูง
- ใช้ arrays + loops/forEach/map more และเขียนโค้ดที่ซ้ำกันน้อยลง
- อย่าเขียนความคิดเห็นเกี่ยวกับฟังก์ชันด้วยโค้ดง่ายๆ เพื่อหลีกเลี่ยงความคิดเห็นที่มากเกินไป ตรรกะที่ซับซ้อนและความต้องการพิเศษอธิบายเป็นภาษาจีนแบบละเอียด
- รหัสที่ซ้ำกันจะถูกสรุปเป็นฟังก์ชันเพื่อลดความซ้ำซ้อน ตัวอย่างเช่น `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` สามารถเขียนเป็น `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()`
- การประกาศตัวแปร: รวมการประกาศ `const` หลายรายการติดต่อกันเป็นคำสั่งเดียว แทนที่จะเขียนเป็นสามบรรทัด ให้เขียน `const a=1, b=2, c=3;`
- การประมวลผลแบบอะซิงโครนัส: ใช้ `await` ห้ามใช้ `.then` การเรียกลูกโซ่
- อย่าสร้างโค้ดการจัดการข้อยกเว้นโดยอัตโนมัติ อย่าเขียน `try...catch` โดยอัตโนมัติ (`try ... catch` ได้รับการดูแลด้วยตนเอง `try catch` ที่มีอยู่ไม่ควรถูกลบ)
- ฟังก์ชันเพียวต้องมาก่อน: เขียนเฉพาะฟังก์ชันเพียวเท่านั้น ไม่เคยมีคลาส
- ใช้ฟังก์ชันลูกศร `const funcName = () => {}` และอย่าใช้คีย์เวิร์ด `function` (ยกเว้นตัวสร้าง) หากคุณสามารถใช้ .bind เพื่อผูกพารามิเตอร์ ให้หลีกเลี่ยงการกำหนดฟังก์ชัน
- การใช้โค้ดซ้ำ: ให้ความสนใจกับการใช้ซ้ำ แยกฟังก์ชันเล็กๆ เพิ่มเติม และหลีกเลี่ยงโครงสร้างโค้ดที่คล้ายกันหรือคัดลอกวางจำนวนมากอย่างเด็ดเดี่ยว
- การเข้าถึงออบเจ็กต์: จัดลำดับความสำคัญโดยใช้การกำหนดการทำลายโครงสร้างเพื่อดึงคุณลักษณะที่ต้องการ หลีกเลี่ยงการใช้จุดภายในซ้ำๆ เพื่อเข้าถึงคุณลักษณะที่ลึกและซ้อนกัน และผสานการพิจารณาลูกโซ่ที่เป็นตัวเลือกซ้ำๆ
- อย่าใช้วัตถุสำหรับพารามิเตอร์ฟังก์ชัน ห้ามเขียน `{a,b,c}` เขียน `a, b, c`; หากมีพารามิเตอร์ทางเลือกหลายรายการ ให้ใช้ [[จำนวนรายการการกำหนดค่า รายการการกำหนดค่า], [จำนวนรายการการกำหนดค่า รายการการกำหนดค่า],..] ในกระบวนทัศน์นี้ รายการการกำหนดค่าจะถูกกำหนดด้วยค่าคงที่ตัวเลข
- สำหรับการส่งคืนหลายค่า ให้ใช้อาร์เรย์ `[a,b,c]` แทน `{a,b,c}` หากมีค่าส่งคืนมากกว่า 3 ค่า ให้ใช้ค่าคงที่ตัวเลขเพื่อกำหนดความหมายของตำแหน่ง
- ไม่จำเป็นต้องมีสตริงเพื่อแสดงสถานะ ใช้ค่าคงที่ตัวเลขเพื่อกำหนดสถานะ
- ไม่มีการใช้เทมเพลตสตริง (``) ใช้การต่อสตริง (+) ยกเว้นการนำเข้า (สะดวกสำหรับการวิเคราะห์แบบคงที่ของ Vite)
- สำหรับการวนซ้ำ หากคุณต้องการหมายเลขซีเรียล ให้ใช้ `++i` แทน `i++`
- หลีกเลี่ยง`?.`ที่ไม่จำเป็น และห้ามการเขียนโปรแกรมเชิงป้องกัน
- ชื่อตัวแปรไม่จำเป็นต้องเป็นพหูพจน์ และลงท้ายด้วย `_li` เพื่อระบุรายการ

## แบบแผนการตั้งชื่อ

- การตั้งชื่อเป็นไปตามความเรียบง่าย ใช้ชื่อที่สั้นแต่มีความหมาย เช่น ใช้ `rm` แทน `remove`, `delete`, `del` อย่างไรก็ตาม คุณควรหลีกเลี่ยงการใช้ภาษาที่รุนแรง เช่น: อย่าใช้ตัวอักษรตัวเดียว `m` แทน `map`
- พยายามใช้เฉพาะคำกริยาในการตั้งชื่อฟังก์ชัน ถ้าใช้คำเดียวแสดงออกได้ อย่าใช้สองคำ คำนามจะสะท้อนให้เห็นในชื่อไฟล์ หากคุณต้องการเพิ่มคำกริยาในชื่อไฟล์ โปรดใส่คำนามขึ้นหน้าและกริยาอยู่ท้ายชื่อไฟล์ ตัวอย่างเช่น: `profileSet.js` แทน `setProfile.js`
- ชื่อตัวแปร: ใช้รูปแบบขีดล่าง (snake_case) เช่น `user_auth_token`
- ชื่อฟังก์ชัน: ใช้ตัวพิมพ์เล็ก CamelCase (camelCase) เช่น `userData`
- พารามิเตอร์ฟังก์ชัน: หากเป็นฟังก์ชันเรียกกลับ ให้ตั้งชื่อโดยใช้ตัวพิมพ์เล็ก เช่น `onChange`
- คำจำกัดความคงที่ระดับโมดูลใช้รูปแบบขีดเส้นใต้ตัวพิมพ์ใหญ่ `UPPER_SNAKE_CASE`
- อย่าเขียน `get` คำนำหน้าที่ไม่มีความหมาย เช่น: เขียน `cookieByHeader` แทน `getCookie`
- ค่าคงที่ส่วนกลาง/ระดับโมดูล: ใช้รูปแบบขีดเส้นใต้ตัวพิมพ์ใหญ่ (UPPER_SNAKE_CASE) เช่น `CODE_TO_ID`, `ID_TO_LANG`

## กลไกแบบโมดูลาร์

- นำเข้า: นำเข้าฟังก์ชันได้อย่างถูกต้องตามความต้องการ ห้ามนำเข้าทั้งโมดูลโดยตรง (หลีกเลี่ยง `import * as x` หรือนำเข้าวัตถุขนาดใหญ่)
- ส่งออก: ปิดใช้งานการส่งออกออบเจ็กต์ ส่งออกฟังก์ชันและตัวแปรเป็นหน่วย เช่น `export const X=1, abc=()=>{};` ลองรวมเข้ากับ const + จุลภาคเพื่อประกาศเนื้อหาที่ส่งออก หากไฟล์มีเพียงฟังก์ชันเดียว ให้ใช้ `export default`
- ยกเว้นในกรณีที่คุณจำเป็นต้องเรียกใช้ฟังก์ชันเริ่มต้นการส่งออกเป็นการภายใน ให้หลีกเลี่ยงการประกาศค่าคงที่ก่อนแล้วจึงส่งออกไปที่ส่วนท้ายของไฟล์
- การแก้ไขเส้นทาง: เมื่อได้รับเส้นทางไดเรกทอรีปัจจุบัน ต้องใช้ `import.meta.dirname`

## ความผิดพลาด

- หลีกเลี่ยงข้อผิดพลาดของสตริงและใช้ const เพื่อประกาศรหัสข้อผิดพลาดคงที่
- หากคุณต้องการส่งคืนข้อมูลข้อมูลที่ไม่ถูกต้อง ให้ใช้ [รหัสข้อผิดพลาด ข้อความแสดงข้อผิดพลาด...] ด้วยวิธีนี้ ข้อความแสดงข้อผิดพลาดจะไม่ใช่คำอธิบายข้อความ แต่เป็นค่าตัวเลข (เช่น [FILE_OVERSIZE, file_size, max_size])
