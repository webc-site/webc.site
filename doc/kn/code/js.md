# JS ಕೋಡ್ ವಿವರಣೆ

## ಕೋಡಿಂಗ್ ಶೈಲಿ

- ಅತ್ಯಂತ ಆಧುನಿಕ js ಸಿಂಟ್ಯಾಕ್ಸ್ ಅನ್ನು ಬಳಸಿಕೊಂಡು ಸರಳ, ಸೊಗಸಾದ ಮತ್ತು ಪರಿಣಾಮಕಾರಿ
- ಹೆಚ್ಚು ಉದ್ದವಾದ ಕಾರ್ಯಗಳನ್ನು ವಿಭಜಿಸಿ, ಮತ್ತು ಇಂಟರ್ಫೇಸ್ ವಿನ್ಯಾಸವು ಕಡಿಮೆ ಜೋಡಣೆ ಮತ್ತು ಹೆಚ್ಚಿನ ಒಗ್ಗಟ್ಟಾಗಿರಬೇಕು.
- ಅರೇಗಳು + ಲೂಪ್‌ಗಳು/ಪ್ರತಿಯೊಂದೂ/ನಕ್ಷೆಯನ್ನು ಹೆಚ್ಚು ಬಳಸಿ ಮತ್ತು ಕಡಿಮೆ ಪುನರಾವರ್ತಿತ ಕೋಡ್ ಅನ್ನು ಬರೆಯಿರಿ
- ಅತಿಯಾದ ಕಾಮೆಂಟ್‌ಗಳನ್ನು ತಪ್ಪಿಸಲು ಸರಳ ಕೋಡ್‌ನಲ್ಲಿ ಫಂಕ್ಷನ್ ಕಾಮೆಂಟ್‌ಗಳನ್ನು ಬರೆಯಬೇಡಿ. ಸಂಕೀರ್ಣ ತರ್ಕ ಮತ್ತು ವಿಶೇಷ ಅಗತ್ಯಗಳನ್ನು ಸಂಸ್ಕರಿಸಿದ ಚೀನೀ ಭಾಷೆಯಲ್ಲಿ ವಿವರಿಸಲಾಗಿದೆ.
- ಪುನರಾವರ್ತಿತ ಕೋಡ್ ಪುನರಾವರ್ತನೆಯನ್ನು ಕಡಿಮೆ ಮಾಡಲು ಕಾರ್ಯಗಳಿಗೆ ಅಮೂರ್ತವಾಗಿದೆ. ಉದಾಹರಣೆಗೆ, `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` ಅನ್ನು `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()` ಎಂದು ಬರೆಯಬಹುದು
- ವೇರಿಯಬಲ್ ಡಿಕ್ಲರೇಶನ್: ಅನೇಕ ಸತತ `const` ಘೋಷಣೆಗಳನ್ನು ಒಂದು ಹೇಳಿಕೆಯಲ್ಲಿ ಸಂಯೋಜಿಸಿ. ಮೂರು ಸಾಲುಗಳಲ್ಲಿ ಬರೆಯುವ ಬದಲು `const a=1, b=2, c=3;` ಎಂದು ಬರೆಯಿರಿ
- ಅಸಮಕಾಲಿಕ ಪ್ರಕ್ರಿಯೆ: `await` ಬಳಸಿ, `.then` ಸರಣಿ ಕರೆಗಳ ಬಳಕೆಯನ್ನು ನಿಷೇಧಿಸಿ
- ಎಕ್ಸೆಪ್ಶನ್ ಹ್ಯಾಂಡ್ಲಿಂಗ್ ಕೋಡ್ ಅನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ರಚಿಸಬೇಡಿ, ಸ್ವಯಂಚಾಲಿತವಾಗಿ `try...catch` ಬರೆಯಬೇಡಿ (`try ... catch` ಅನ್ನು ಹಸ್ತಚಾಲಿತವಾಗಿ ನಿರ್ವಹಿಸಲಾಗುತ್ತದೆ, ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ `try catch` ಅನ್ನು ಅಳಿಸಬಾರದು)
- ಮೊದಲು ಶುದ್ಧ ಕಾರ್ಯಗಳು: ಶುದ್ಧ ಕಾರ್ಯಗಳನ್ನು ಮಾತ್ರ ಬರೆಯಿರಿ, ಎಂದಿಗೂ ವರ್ಗಗಳಿಲ್ಲ
- ಬಾಣದ ಕಾರ್ಯಗಳನ್ನು `const funcName = () => {}` ಬಳಸಿ ಮತ್ತು `function` ಕೀವರ್ಡ್ ಅನ್ನು ಬಳಸಬೇಡಿ (ಜನರೇಟರ್‌ಗಳನ್ನು ಹೊರತುಪಡಿಸಿ); ನೀವು ನಿಯತಾಂಕಗಳನ್ನು ಬೈಂಡ್ ಮಾಡಲು .bind ಅನ್ನು ಬಳಸಬಹುದಾದರೆ, ಕಾರ್ಯಗಳನ್ನು ವಿವರಿಸುವುದನ್ನು ತಪ್ಪಿಸಿ
- ಕೋಡ್ ಮರುಬಳಕೆ: ಮರುಬಳಕೆಗೆ ಗಮನ ಕೊಡಿ, ಹೆಚ್ಚು ಸಣ್ಣ ಕಾರ್ಯಗಳನ್ನು ಹೊರತೆಗೆಯಿರಿ ಮತ್ತು ಹೆಚ್ಚಿನ ಸಂಖ್ಯೆಯ ಒಂದೇ ರೀತಿಯ ಅಥವಾ ನಕಲಿಸಿ-ಅಂಟಿಸಲಾದ ಕೋಡ್ ರಚನೆಗಳನ್ನು ದೃಢವಾಗಿ ತಪ್ಪಿಸಿ.
- ಆಬ್ಜೆಕ್ಟ್ ಪ್ರವೇಶ: ಅಗತ್ಯವಿರುವ ಗುಣಲಕ್ಷಣಗಳನ್ನು ಹೊರತೆಗೆಯಲು ಡಿಸ್ಟ್ರಕ್ಚರಿಂಗ್ ಅಸೈನ್‌ಮೆಂಟ್ ಅನ್ನು ಆದ್ಯತೆ ನೀಡಿ, ಆಳವಾದ ಮತ್ತು ನೆಸ್ಟೆಡ್ ಗುಣಲಕ್ಷಣಗಳನ್ನು ಪ್ರವೇಶಿಸಲು ಆಂತರಿಕವಾಗಿ ಚುಕ್ಕೆಗಳನ್ನು ಪದೇ ಪದೇ ಬಳಸುವುದನ್ನು ತಪ್ಪಿಸಿ ಮತ್ತು ಪುನರಾವರ್ತಿತ ಐಚ್ಛಿಕ ಸರಣಿ ತೀರ್ಪುಗಳನ್ನು ವಿಲೀನಗೊಳಿಸಿ
- ಫಂಕ್ಷನ್ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳಿಗಾಗಿ ಆಬ್ಜೆಕ್ಟ್‌ಗಳನ್ನು ಬಳಸಬೇಡಿ, `{a,b,c}` ಬರೆಯಬೇಡಿ, `a, b, c` ಬರೆಯಬೇಡಿ; ಹಲವು ಐಚ್ಛಿಕ ನಿಯತಾಂಕಗಳಿದ್ದರೆ, ಬಳಸಿ [[ಸಂರಚನಾ ಐಟಂಗಳ ಸಂಖ್ಯೆ, ಕಾನ್ಫಿಗರೇಶನ್ ಐಟಂಗಳು], [ಕಾನ್ಫಿಗರೇಶನ್ ಐಟಂಗಳ ಸಂಖ್ಯೆ, ಕಾನ್ಫಿಗರೇಶನ್ ಐಟಂಗಳು],..] ಈ ಮಾದರಿಯಲ್ಲಿ, ಕಾನ್ಫಿಗರೇಶನ್ ಐಟಂಗಳನ್ನು ಸಂಖ್ಯಾ ಸ್ಥಿರಾಂಕಗಳೊಂದಿಗೆ ವ್ಯಾಖ್ಯಾನಿಸಲಾಗಿದೆ
- ಬಹು-ಮೌಲ್ಯದ ರಿಟರ್ನ್‌ಗಳಿಗಾಗಿ, `{a,b,c}` ಬದಲಿಗೆ ಅರೇ `[a,b,c]` ಅನ್ನು ಬಳಸಿ. 3 ರಿಟರ್ನ್ ಮೌಲ್ಯಗಳಿಗಿಂತ ಹೆಚ್ಚು ಇದ್ದರೆ, ಸ್ಥಾನಿಕ ಅರ್ಥವನ್ನು ವ್ಯಾಖ್ಯಾನಿಸಲು ಸಂಖ್ಯಾ ಸ್ಥಿರಾಂಕಗಳನ್ನು ಬಳಸಿ.
- ರಾಜ್ಯವನ್ನು ಪ್ರತಿನಿಧಿಸಲು ಸ್ಟ್ರಿಂಗ್ ಅಗತ್ಯವಿಲ್ಲ, ರಾಜ್ಯವನ್ನು ವ್ಯಾಖ್ಯಾನಿಸಲು ಸಂಖ್ಯಾ ಸ್ಥಿರಾಂಕಗಳನ್ನು ಬಳಸಿ
- ಯಾವುದೇ ಸ್ಟ್ರಿಂಗ್ ಟೆಂಪ್ಲೇಟ್ (``) ಅನ್ನು ಬಳಸಲಾಗಿಲ್ಲ, ಆಮದು ಹೊರತುಪಡಿಸಿ ಸ್ಟ್ರಿಂಗ್ ಸಂಯೋಜನೆಯನ್ನು (+) ಬಳಸಲಾಗಿದೆ (ವೈಟ್ ಸ್ಥಿರ ವಿಶ್ಲೇಷಣೆಗೆ ಅನುಕೂಲಕರವಾಗಿದೆ)
- ಲೂಪ್‌ಗಾಗಿ, ನಿಮಗೆ ಸರಣಿ ಸಂಖ್ಯೆ ಅಗತ್ಯವಿದ್ದರೆ, `i++` ಬದಲಿಗೆ `++i` ಬಳಸಿ
- ಅನಗತ್ಯ`?.` ಅನ್ನು ತಪ್ಪಿಸಿ ಮತ್ತು ರಕ್ಷಣಾತ್ಮಕ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಅನ್ನು ನಿಷೇಧಿಸಿ
- ವೇರಿಯಬಲ್ ಹೆಸರು ಬಹುವಚನವಾಗಿರಬೇಕಾಗಿಲ್ಲ ಮತ್ತು ಪಟ್ಟಿಯನ್ನು ಸೂಚಿಸಲು `_li` ನೊಂದಿಗೆ ಕೊನೆಗೊಳ್ಳುತ್ತದೆ.

## ಹೆಸರಿಸುವ ಸಂಪ್ರದಾಯ

- ಹೆಸರಿಸುವಿಕೆಯು ಕನಿಷ್ಠೀಯತಾವಾದವನ್ನು ಅನುಸರಿಸುತ್ತದೆ. ಚಿಕ್ಕದಾದ ಆದರೆ ಅರ್ಥಪೂರ್ಣ ಹೆಸರುಗಳನ್ನು ಬಳಸಿ, ಉದಾಹರಣೆಗೆ: `rm` ಬದಲಿಗೆ `remove`, `delete`, `del` ಬಳಸಿ. ಆದಾಗ್ಯೂ, ನೀವು ಅತಿರೇಕಕ್ಕೆ ಹೋಗುವುದನ್ನು ತಪ್ಪಿಸಬೇಕು, ಉದಾಹರಣೆಗೆ: `map` ಬದಲಿಗೆ ಒಂದೇ ಅಕ್ಷರವನ್ನು `m` ಬಳಸಬೇಡಿ
- ಕಾರ್ಯ ನಾಮಕರಣದಲ್ಲಿ ಕ್ರಿಯಾಪದಗಳನ್ನು ಮಾತ್ರ ಬಳಸಲು ಪ್ರಯತ್ನಿಸಿ. ಅದನ್ನು ವ್ಯಕ್ತಪಡಿಸಲು ನೀವು ಒಂದು ಪದವನ್ನು ಬಳಸಬಹುದಾದರೆ, ಎರಡು ಪದಗಳನ್ನು ಬಳಸಬೇಡಿ. ನಾಮಪದಗಳು ಫೈಲ್ ಹೆಸರುಗಳಲ್ಲಿ ಪ್ರತಿಫಲಿಸುತ್ತದೆ. ನೀವು ಫೈಲ್ ಹೆಸರಿಗೆ ಕ್ರಿಯಾಪದವನ್ನು ಸೇರಿಸಬೇಕಾದರೆ, ದಯವಿಟ್ಟು ನಾಮಪದವನ್ನು ಮೊದಲು ಮತ್ತು ಕ್ರಿಯಾಪದವನ್ನು ಕೊನೆಯದಾಗಿ ಇರಿಸಿ. ಉದಾಹರಣೆಗೆ: `setProfile.js` ಬದಲಿಗೆ `profileSet.js`
- ವೇರಿಯಬಲ್ ಹೆಸರು: ಅಂಡರ್‌ಲೈನ್ ಶೈಲಿಯನ್ನು ಬಳಸಿ (snake_case), ಉದಾಹರಣೆಗೆ `user_auth_token`; ವೇರಿಯೇಬಲ್ ಒಂದು ಕಾರ್ಯವಾಗಿದ್ದರೆ, ಲೋವರ್ಕೇಸ್ ಕ್ಯಾಮೆಲ್ ಕೇಸ್ ಹೆಸರಿಸುವ ಶೈಲಿಯನ್ನು ಬಳಸಿ
- ಕಾರ್ಯದ ಹೆಸರು: ಲೋವರ್‌ಕೇಸ್ ಕ್ಯಾಮೆಲ್‌ಕೇಸ್ (ಒಂಟೆ ಕೇಸ್) ಬಳಸಿ, ಉದಾಹರಣೆಗೆ `userData`
- ಫಂಕ್ಷನ್ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳು: ಇದು ಕಾಲ್‌ಬ್ಯಾಕ್ ಫಂಕ್ಷನ್ ಆಗಿದ್ದರೆ, `onChange` ನಂತಹ ಲೋವರ್‌ಕೇಸ್ ಕ್ಯಾಮೆಲ್ ಕೇಸ್‌ನಲ್ಲಿ ಹೆಸರಿಸಿ
- ಮಾಡ್ಯೂಲ್-ಮಟ್ಟದ ಸ್ಥಿರ ವ್ಯಾಖ್ಯಾನಗಳು ದೊಡ್ಡಕ್ಷರ ಅಂಡರ್‌ಲೈನ್ ಶೈಲಿಯನ್ನು ಬಳಸುತ್ತವೆ `UPPER_SNAKE_CASE`
- `get`, ಅಂತಹ ಅರ್ಥಹೀನ ಪೂರ್ವಪ್ರತ್ಯಯಗಳನ್ನು ಬರೆಯಬೇಡಿ, ಉದಾಹರಣೆಗೆ: `getCookie` ಬದಲಿಗೆ `cookieByHeader` ಬರೆಯಿರಿ
- ಜಾಗತಿಕ/ಮಾಡ್ಯೂಲ್-ಮಟ್ಟದ ಸ್ಥಿರಾಂಕಗಳು: ದೊಡ್ಡಕ್ಷರ ಅಂಡರ್‌ಲೈನ್ ಶೈಲಿಯನ್ನು ಬಳಸಿ (UPPER_SNAKE_CASE), ಉದಾ. `CODE_TO_ID`, `ID_TO_LANG`

## ಮಾಡ್ಯುಲರ್ ಯಾಂತ್ರಿಕತೆ

- ಆಮದು: ಬೇಡಿಕೆಯ ಮೇಲೆ ನಿಖರವಾಗಿ ಕಾರ್ಯಗಳನ್ನು ಆಮದು ಮಾಡಿ, ಸಂಪೂರ್ಣ ಮಾಡ್ಯೂಲ್‌ನ ನೇರ ಆಮದನ್ನು ನಿಷೇಧಿಸಿ (`import * as x` ತಪ್ಪಿಸಿ ಅಥವಾ ದೊಡ್ಡ ವಸ್ತುಗಳನ್ನು ಆಮದು ಮಾಡಿಕೊಳ್ಳಿ)
- ರಫ್ತು: ವಸ್ತುಗಳ ರಫ್ತು ನಿಷ್ಕ್ರಿಯಗೊಳಿಸಿ. `export const X=1, abc=()=>{};` ನಂತಹ ಕಾರ್ಯಗಳು ಮತ್ತು ವೇರಿಯೇಬಲ್‌ಗಳನ್ನು ಘಟಕಗಳಾಗಿ ರಫ್ತು ಮಾಡಿ. ರಫ್ತು ಮಾಡಿದ ವಿಷಯವನ್ನು ಘೋಷಿಸಲು ಅವುಗಳನ್ನು const + ಅಲ್ಪವಿರಾಮದೊಂದಿಗೆ ಸಂಯೋಜಿಸಲು ಪ್ರಯತ್ನಿಸಿ. ಫೈಲ್ ಒಂದೇ ಒಂದು ಕಾರ್ಯವನ್ನು ಹೊಂದಿದ್ದರೆ, `export default` ಅನ್ನು ಬಳಸಿ
- ನೀವು ರಫ್ತು ಡೀಫಾಲ್ಟ್ ಕಾರ್ಯವನ್ನು ಆಂತರಿಕವಾಗಿ ಕರೆಯುವ ಅಗತ್ಯವಿಲ್ಲದಿದ್ದರೆ, ಮೊದಲು ಸ್ಥಿರಾಂಕಗಳನ್ನು ಘೋಷಿಸುವುದನ್ನು ತಪ್ಪಿಸಿ ಮತ್ತು ನಂತರ ಫೈಲ್‌ನ ಕೊನೆಯಲ್ಲಿ ಅವುಗಳನ್ನು ರಫ್ತು ಮಾಡಬೇಡಿ.
- ಮಾರ್ಗ ರೆಸಲ್ಯೂಶನ್: ಪ್ರಸ್ತುತ ಡೈರೆಕ್ಟರಿ ಮಾರ್ಗವನ್ನು ಪಡೆದುಕೊಳ್ಳುವಾಗ, `import.meta.dirname` ಅನ್ನು ಬಳಸಬೇಕು

## ತಪ್ಪು

- ಸ್ಟ್ರಿಂಗ್ ದೋಷಗಳನ್ನು ತಪ್ಪಿಸಿ ಮತ್ತು ನಿರಂತರ ದೋಷ ಕೋಡ್‌ಗಳನ್ನು ಘೋಷಿಸಲು const ಅನ್ನು ಬಳಸಿ.
- ನೀವು ತಪ್ಪು ಡೇಟಾ ಮಾಹಿತಿಯನ್ನು ಹಿಂತಿರುಗಿಸಬೇಕಾದರೆ, ಬಳಸಿ [ದೋಷ ಕೋಡ್, ದೋಷ ಸಂದೇಶ,...] ಈ ರೀತಿಯಲ್ಲಿ, ದೋಷ ಸಂದೇಶವು ಪಠ್ಯ ವಿವರಣೆಯಲ್ಲ, ಆದರೆ ಸಂಖ್ಯಾತ್ಮಕ ಮೌಲ್ಯವಾಗಿದೆ (ಉದಾಹರಣೆಗೆ [FILE_OVERSIZE, file_size, max_size])

- ಬೈನರಿ ಡೇಟಾ: ಬೈನರಿಯೊಂದಿಗೆ ವ್ಯವಹರಿಸುವಾಗ, `Uint8Array` ಅನ್ನು ಏಕರೂಪವಾಗಿ ಬಳಸಲು ಪ್ರಯತ್ನಿಸಿ
