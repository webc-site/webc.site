# WebC.site

## ಇದೇನಿದು?

### ಇದು ಸ್ಥಳೀಯ ವೆಬ್ ಘಟಕ ಗ್ರಂಥಾಲಯವಾಗಿದೆ

[ವೆಬ್ ಘಟಕಗಳು](https://developer.mozilla.org/docs/Web/API/Web_components) ಆಧರಿಸಿ ಅಭಿವೃದ್ಧಿಪಡಿಸಲಾಗಿದೆ, ಇದು ರನ್‌ಟೈಮ್ ಸ್ವತಂತ್ರವಾಗಿದೆ ಮತ್ತು ಎಲ್ಲಾ ಚೌಕಟ್ಟುಗಳಿಗೆ ಸೂಕ್ತವಾಗಿದೆ.

ಗಾತ್ರಕ್ಕೆ ಅತ್ಯಂತ ಆಪ್ಟಿಮೈಸ್ ಮಾಡಲಾಗಿದೆ (ಇದೇ ರೀತಿಯ ಜನಪ್ರಿಯ ಲೈಬ್ರರಿಗಳ ಹತ್ತನೇ ಒಂದು ಭಾಗದಷ್ಟು ಗಾತ್ರ).

75 ಭಾಷೆಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. ಆನ್‌ಲೈನ್ ಬಳಕೆಗಾಗಿ ನೀವು ನೇರವಾಗಿ js ಮತ್ತು css ಅನ್ನು ಆಮದು ಮಾಡಿಕೊಳ್ಳಬಹುದು ಅಥವಾ ದ್ವಿತೀಯ ಅಭಿವೃದ್ಧಿಗೆ ಅಗತ್ಯವಿರುವಂತೆ ನಿಮ್ಮ ಯೋಜನೆಗೆ ಘಟಕಗಳನ್ನು ಸೇರಿಸಬಹುದು.

### ಇದು ಘಟಕ ಗ್ರಂಥಾಲಯ ಅಭಿವೃದ್ಧಿ ಚೌಕಟ್ಟು

ಇದು [ಸ್ಟೋರಿಬುಕ್](https://storybook.js.org) ನ ಸ್ಥಿರ ಬ್ಲಾಗ್ ಆವೃತ್ತಿಯಾಗಿದೆ.

ಘಟಕ ಗ್ರಂಥಾಲಯವನ್ನು ನಿರ್ಮಿಸುವುದು ಕಂಪನಿಯ ಆಂತರಿಕ ಅಭಿವೃದ್ಧಿ ದಕ್ಷತೆಯನ್ನು ಸುಧಾರಿಸಬಹುದು ಮತ್ತು ತಾಂತ್ರಿಕ ವಲಯದಲ್ಲಿ ಬ್ರ್ಯಾಂಡ್ ಇಮೇಜ್ ಅನ್ನು ನಿರ್ಮಿಸಬಹುದು.

[webc.site](https://github.com/webc-site/webc.site) ಅನ್ನು ಆಧರಿಸಿ, ತಂಡಗಳು ತಮ್ಮದೇ ಆದ ಘಟಕ ಲೈಬ್ರರಿಗಳನ್ನು ಸುಲಭವಾಗಿ ರಚಿಸಬಹುದು ಮತ್ತು ಅವುಗಳನ್ನು [ಗಿಥಬ್ ಪುಟ](https://pages.github.com), [ಕ್ಲೌಡ್‌ಫ್ಲೇರ್ ಪುಟ](https://pages.cloudflare.com), ಇತ್ಯಾದಿಗಳಿಗೆ ಪ್ರಕಟಿಸಬಹುದು. ಪ್ರಕಟಿತ ಘಟಕಗಳನ್ನು ಆನ್‌ಲೈನ್‌ನಲ್ಲಿ [codepen.io](https://codepen.io) ಮತ್ತು [jsfiddle](https://jsfiddle.net) ನಲ್ಲಿ ಡೀಬಗ್ ಮಾಡಬಹುದು.

ಅದೇ ಸಮಯದಲ್ಲಿ, ಅಂತರ್ನಿರ್ಮಿತ ಅನುವಾದ ಚೌಕಟ್ಟು ಘಟಕಗಳು ಮತ್ತು ದಾಖಲೆಗಳ ಅಂತರಾಷ್ಟ್ರೀಯೀಕರಣವನ್ನು ತ್ವರಿತವಾಗಿ ಅರಿತುಕೊಳ್ಳಬಹುದು, ಇದರಿಂದಾಗಿ ನಿಮ್ಮ ಉತ್ಪನ್ನ ಪ್ರೇಕ್ಷಕರು ಇನ್ನು ಮುಂದೆ ಭಾಷೆಗೆ ಬದ್ಧರಾಗಿರುವುದಿಲ್ಲ.

### ಇದು ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆಯ ಯುಗಕ್ಕೆ ಮುಂಭಾಗದ ಅಭಿವೃದ್ಧಿಯ ಹೊಸ ಮಾದರಿಯಾಗಿದೆ

ಫ್ರೇಮ್‌ವರ್ಕ್ ಅಂತರ್ನಿರ್ಮಿತವಾಗಿದೆ `.agents/skills`, ಉತ್ತಮ ಅಭ್ಯಾಸಗಳ ಪ್ರಕಾರ ಒಂದೇ ಕ್ಲಿಕ್‌ನಲ್ಲಿ ಹೊಸ ಘಟಕಗಳನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸಲು ನಿಮಗೆ ಸಹಾಯ ಮಾಡಲು ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆಯನ್ನು ಅನುಮತಿಸುತ್ತದೆ.

<img src="https://i-01.eu.org/20260604121408079.avif" width="400">

ಫ್ರಂಟ್-ಎಂಡ್ ಕೋಡ್ ಅನ್ನು ಸಮರ್ಥನೀಯವಲ್ಲದ ಕಸದ ಪರ್ವತದ ವಾತಾವರಣದಿಂದ ಪ್ರೋಗ್ರಾಮಿಂಗ್ ಪ್ರಮಾಣಿತ ಭಾಗವಾಗಿ ಬದಲಾಯಿಸಲು ಅವಕಾಶ ಮಾಡಿಕೊಡಿ, ಅದನ್ನು ದೀರ್ಘಕಾಲದವರೆಗೆ ನಿರ್ವಹಿಸಬಹುದು ಮತ್ತು ಪುನರಾವರ್ತಿತವಾಗಿ ವಿಕಸನಗೊಳಿಸಬಹುದು.

ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆ-ನೆರವಿನ ಅಭಿವೃದ್ಧಿಯ ಗುಣಲಕ್ಷಣಗಳ ದೃಷ್ಟಿಯಿಂದ, ನಾವು ಹೊಸ ಮುಂಭಾಗದ ಅಭಿವೃದ್ಧಿ ಮಾದರಿಯನ್ನು ಪ್ರಸ್ತಾಪಿಸಿದ್ದೇವೆ ಮತ್ತು ವಿನ್ಯಾಸಗೊಳಿಸಿದ್ದೇವೆ:

1. **ಎಲ್ಲವೂ ವೆಬ್ ಘಟಕವಾಗಿದೆ**
2. ** ಘಟಕವು ಬ್ಯಾಕ್-ಎಂಡ್ ಡೇಟಾ ಲಿಂಕ್‌ನಿಂದ ಸ್ವತಂತ್ರವಾಗಿರಬೇಕು ಮತ್ತು ಡೇಟಾ ಇಂಟರ್ಫೇಸ್ ಅನ್ನು ಕಾಲ್‌ಬ್ಯಾಕ್ ಕಾರ್ಯವಾಗಿ ಬಹಿರಂಗಪಡಿಸಲಾಗುತ್ತದೆ**

ಈ ರೀತಿಯಾಗಿ, ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆಯು ನಕಲಿ ಡೇಟಾವನ್ನು ರಚಿಸಬಹುದು ಮತ್ತು ಡೆಮೊ ಪುಟದಲ್ಲಿ ವಿಭಿನ್ನ ಸ್ಥಿತಿಯ ಘಟಕಗಳನ್ನು ಪ್ರದರ್ಶಿಸಬಹುದು.

ಕಾಂಪೊನೆಂಟ್ ಡೆವಲಪ್‌ಮೆಂಟ್ ಮತ್ತು ಡೇಟಾ ಇಂಟರ್‌ಫೇಸ್‌ನ ಪ್ರತ್ಯೇಕತೆ, [webc.site ಕಾಂಪೊನೆಂಟ್ ಫ್ರೇಮ್‌ವರ್ಕ್](https://github.com/webc-site/webc.site) ಜೊತೆಗೆ ಸಂಯೋಜಿಸಲ್ಪಟ್ಟ ಮತ್ತೊಂದು ಪ್ರಮುಖ ವೈಶಿಷ್ಟ್ಯ:

** ಪ್ರತಿಯೊಂದು ಘಟಕವು ಅಭಿವೃದ್ಧಿ ಸರ್ವರ್ ಅನ್ನು ಸ್ವತಂತ್ರವಾಗಿ ಪ್ರಾರಂಭಿಸಬಹುದು ಮತ್ತು ಡೆಮೊ ಪುಟದಲ್ಲಿ ಅಭಿವೃದ್ಧಿಪಡಿಸಬಹುದು ಮತ್ತು ಡೀಬಗ್ ಮಾಡಬಹುದು**.

<img src="https://i-01.eu.org/webcSiteDemo.svg" width="400">
<img src="https://i-01.eu.org/20260604112614499.avif" width="400">

ಇದು ಪ್ರತಿ ಘಟಕದ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಡೀಬಗ್‌ಗಾಗಿ ಫ್ರಂಟ್-ಎಂಡ್ ಲಿಂಕ್ ಅನ್ನು `0` ಗೆ ಕಡಿಮೆ ಮಾಡುತ್ತದೆ.

ಈ ರೀತಿಯಾಗಿ, ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆಯ ನೆರವಿನ ಅಭಿವೃದ್ಧಿ ಸಾಧನಗಳು ([Claude Code](https://code.claude.com), [OpenAI Codex](https://openai.com), [Google Antigravity](https://antigravity.google), [Cursor](https://cursor.com), [Windsurf](https://windsurf.com), [Devin](https://devin.ai), [OpenHands](https://github.com/All-Hands-AI/OpenHands) ಇತ್ಯಾದಿ), ನೀವು ಇನ್ನು ಮುಂದೆ ವಿವಿಧ ಅಸ್ತವ್ಯಸ್ತವಾಗಿರುವ ಡೆವಲಪ್‌ಮೆಂಟ್‌ಗಳನ್ನು ತೆರೆದು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಡೆವಲಪ್‌ಮೆಂಟ್ ಅನ್ನು ಮುಕ್ತಗೊಳಿಸಬಹುದು ಮತ್ತು ಡೆವಲಪ್‌ಮೆಂಟ್ ಅನ್ನು ಸಂಪೂರ್ಣವಾಗಿ ಆನಂದಿಸಬಹುದು. (ಬ್ಯಾಕೆಂಡ್ ಡೇಟಾ, ಲಾಗಿನ್ ಸ್ಥಿತಿ, ಮಾರ್ಗ ರೂಟಿಂಗ್, ಇತ್ಯಾದಿ).

ನೀವು ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆಯನ್ನು ನಿಯಂತ್ರಿಸಲು ಮತ್ತು ವೆಚ್ಚವನ್ನು ಕಡಿಮೆ ಮಾಡಲು ಮತ್ತು ದಕ್ಷತೆಯನ್ನು ಹೆಚ್ಚಿಸಲು ಬಯಸಿದರೆ, ಪ್ರತಿ ಘಟಕದ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಡೀಬಗ್ ಮಾಡುವ ಪರಿಸರವನ್ನು ಕಡಿಮೆ ಮಾಡುವ ಹೊಸ ಮಾದರಿಯ ಅಗತ್ಯವಿದೆ.

### ವೆಬ್ ಘಟಕ ಗ್ರಂಥಾಲಯ

#### ಅಂತಿಮ ಗಾತ್ರದ ಆಪ್ಟಿಮೈಸೇಶನ್

ಉದಾಹರಣೆಗೆ, ವರ್ಚುವಲ್ ಸ್ಕ್ರಾಲ್ ಬಾರ್ ಕಾಂಪೊನೆಂಟ್‌ಗಾಗಿ, ಕೆಳಗಿನ ಕೋಷ್ಟಕದಲ್ಲಿ ತೋರಿಸಿರುವಂತೆ `gz` ನಂತರ ಜನಪ್ರಿಯ [OverlayScrollbars](https://github.com/KingSora/OverlayScrollbars) ಸ್ಕ್ರಿಪ್ಟ್ ಮತ್ತು ಶೈಲಿಯ ಗಾತ್ರವನ್ನು ಹೋಲಿಕೆ ಮಾಡಿ:

|                   |   js    |  css   |     total      |
| :---------------- | :-----: | :----: | :------------: |
| overlayscrollbars | 14520 B | 2584 B | 17104 B (9.1X) |
| Scroll            | 1159 B  | 721 B  |   ⭐ 1880 ಬಿ    |

`webc.site`ನ ವರ್ಚುವಲ್ ಸ್ಕ್ರಾಲ್ ಬಾರ್ ಘಟಕವು ಕೇವಲ `OverlayScrollbars`ನ `10%` ಆಗಿದೆ.

#### ಚಾಲನಾಸಮಯ ಸ್ವತಂತ್ರ

ಮಾನದಂಡಗಳ ಮೇಲೆ ನಿರ್ಮಿಸಲಾಗಿದೆ [ವೆಬ್ ಘಟಕಗಳು](https://developer.mozilla.org/docs/Web/API/Web_components), webc.site ಕಾಂಪೊನೆಂಟ್ ಲೈಬ್ರರಿಯು ಸ್ವಾಭಾವಿಕವಾಗಿ ರನ್‌ಟೈಮ್-ಅಜ್ಞೇಯತಾವಾದಿಯಾಗಿದೆ. ನೀವು React, Vue, Angular, Svelte, ಅಥವಾ ಸಾಂಪ್ರದಾಯಿಕ ಶುದ್ಧ HTML/JS ಪ್ರಾಜೆಕ್ಟ್ ಅನ್ನು ಬಳಸುತ್ತಿರಲಿ, ನಿರ್ದಿಷ್ಟ ಫ್ರೇಮ್‌ವರ್ಕ್‌ಗಾಗಿ ದ್ವಿತೀಯ ಪ್ಯಾಕೇಜಿಂಗ್ ಅಗತ್ಯವಿಲ್ಲದೇ, ಸ್ಥಳೀಯ HTML ಟ್ಯಾಗ್‌ಗಳಂತಹ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ಎಲ್ಲಾ ಘಟಕಗಳು ನೇರವಾಗಿ ರನ್ ಆಗಬಹುದು.

##### Shadcn ನೊಂದಿಗೆ ಹೋಲಿಕೆ

[shadcn/ui](https://ui.shadcn.com) ಕೋಡ್ ವಿತರಣಾ ಮೋಡ್ ಅನ್ನು ಅಳವಡಿಸಿಕೊಳ್ಳುತ್ತದೆ, ಇದು ನಿಮ್ಮ ಸ್ವಂತ ಯೋಜನೆಗೆ ನೇರವಾಗಿ ಘಟಕ ಕೋಡ್ ಅನ್ನು ನಕಲಿಸುವ ಅಗತ್ಯವಿದೆ. ಇದು ರಿಯಾಕ್ಟ್ ಪರಿಸರ ವ್ಯವಸ್ಥೆ ಮತ್ತು ನಿರ್ದಿಷ್ಟ ನಿರ್ಮಾಣ ಸಾಧನ ಸರಪಳಿಗಳಿಗೆ (ಟೈಲ್‌ವಿಂಡ್ ಸಿಎಸ್‌ಎಸ್‌ನಂತಹ) ಬಿಗಿಯಾಗಿ ಬಂಧಿಸಲ್ಪಟ್ಟಿದೆ ಮತ್ತು ಚೌಕಟ್ಟುಗಳಾದ್ಯಂತ ಬಳಸಲಾಗುವುದಿಲ್ಲ. ಇದು ಪ್ರಾಜೆಕ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಸಂರಚನೆಗೆ ಕೆಲವು ಅವಶ್ಯಕತೆಗಳನ್ನು ಹೊಂದಿದೆ.

ಮತ್ತು `webc.site` ನ ಘಟಕ:
- **ಕ್ರಾಸ್-ಫ್ರೇಮ್‌ವರ್ಕ್ ಸಾರ್ವತ್ರಿಕ**: ಒಮ್ಮೆ ಬರೆಯಿರಿ ಮತ್ತು ನೇರವಾಗಿ ಪರಿಚಯಿಸಬಹುದು ಮತ್ತು ಯಾವುದೇ ಮುಂಭಾಗದ ಚೌಕಟ್ಟಿನಲ್ಲಿ ಚಲಾಯಿಸಬಹುದು (ಫ್ರೇಮ್‌ವರ್ಕ್-ಮುಕ್ತ ಪರಿಸರವೂ ಸಹ).
- **ಪರಿಸರ ಶೂನ್ಯ ಅವಲಂಬನೆ**: ಯಾವುದೇ ನಿರ್ದಿಷ್ಟ ಪ್ಯಾಕೇಜಿಂಗ್ ಉಪಕರಣಗಳು ಅಥವಾ CSS ಪ್ರಿಪ್ರೊಸೆಸರ್‌ಗಳ ಅಗತ್ಯವಿಲ್ಲ, ಸಂಕಲಿಸಿದ ಸ್ಥಿರ ಫೈಲ್‌ಗಳನ್ನು ಆಮದು ಮಾಡಿಕೊಳ್ಳಿ ಮತ್ತು ಅದನ್ನು ಬಾಕ್ಸ್‌ನಿಂದ ಹೊರಗೆ ಬಳಸಬಹುದು.

## ಸ್ವಯಂಚಾಲಿತ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಡೀಬಗ್ ಮಾಡುವಿಕೆ

### ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಡೀಬಗ್ ಮಾಡುವಿಕೆ

ಪ್ರತಿಯೊಂದು ವೆಬ್ ಪುಟ ಘಟಕವು ಅಭಿವೃದ್ಧಿ ಸರ್ವರ್ ಅನ್ನು ಸ್ವತಂತ್ರವಾಗಿ ಪ್ರಾರಂಭಿಸುವುದನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ, ಕನಿಷ್ಠ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಡೀಬಗ್ ಮಾಡುವ ಪರಿಸರವನ್ನು ಒದಗಿಸುತ್ತದೆ:

- **ಹೊಸ ಘಟಕವನ್ನು ರಚಿಸಿ**: ಘಟಕ ಅಭಿವೃದ್ಧಿಯ ಅಸ್ಥಿಪಂಜರವನ್ನು ತ್ವರಿತವಾಗಿ ರಚಿಸಲು `./sh/new.js [css|js] [ಘಟಕ ಹೆಸರು]` ರನ್ ಮಾಡಿ (ಡೆಮೊ ಪುಟ ಮತ್ತು ಅಂತರಾಷ್ಟ್ರೀಯೀಕರಣ ಕಾನ್ಫಿಗರೇಶನ್ ಸೇರಿದಂತೆ).
- **ಸ್ವತಂತ್ರ ಡೀಬಗ್ ಮಾಡುವಿಕೆ**: Vite ಡೆವಲಪ್‌ಮೆಂಟ್ ಸರ್ವರ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸಲು ಮತ್ತು ಬ್ರೌಸರ್‌ನಲ್ಲಿ ಕಾಂಪೊನೆಂಟ್‌ನ ಡೆಮೊ ಪುಟವನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ತೆರೆಯಲು `./dev.js [ಕಾಂಪೊನೆಂಟ್ ಹೆಸರು]` (ಅಥವಾ ನೇರವಾಗಿ `./dev.js` ರನ್ ಮಾಡಿ) ರನ್ ಮಾಡಿ. ಕಾಂಪೊನೆಂಟ್ ಹಾಟ್ ಅಪ್‌ಡೇಟ್ (HMR) ಅನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಡೇಟಾ ಲಿಂಕ್‌ಗಳನ್ನು ಬೇರ್ಪಡಿಸಲಾಗಿರುವುದರಿಂದ, ಸಂಪೂರ್ಣ ಸ್ವಯಂಚಾಲಿತ ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಬ್ರೌಸರ್ ಡೀಬಗ್ ಮಾಡುವಿಕೆಯಲ್ಲಿ ಸಹಾಯ ಮಾಡಲು AI ಗೆ (ಕ್ಲಾಡ್ ಕೋಡ್, ಗೂಗಲ್ ಆಂಟಿಗ್ರಾವಿಟಿ, ಇತ್ಯಾದಿ) ಅತ್ಯಂತ ಸೂಕ್ತವಾದ ಅಭಿವೃದ್ಧಿಯ ಮುಂಭಾಗದ ಲಿಂಕ್ ಶೂನ್ಯವಾಗಿದೆ.

### ಕಾಂಪೊನೆಂಟ್ ಲೈಬ್ರರಿಯನ್ನು ಪ್ರಕಟಿಸಿ

#### webc dist

ಪ್ಯಾಕೇಜಿಂಗ್ ಮತ್ತು ಪ್ರಕಾಶನ ಪ್ರಕ್ರಿಯೆಯು ಹೆಚ್ಚು ಸ್ವಯಂಚಾಲಿತವಾಗಿದೆ:
- **ಸ್ಟಾಟಿಕ್ ರಿಸೋರ್ಸ್ ಪ್ಯಾಕೇಜಿಂಗ್**: `./sh/dist.js` (ಅಂದರೆ, `webc dist` ಕಾರ್ಯವನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸುವುದರಿಂದ) `src/` ಅಡಿಯಲ್ಲಿ ಎಲ್ಲಾ ವೆಬ್ ಪುಟದ ಘಟಕಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸ್ಕ್ಯಾನ್ ಮಾಡಬಹುದು, ಕಂಪೈಲ್ ಮಾಡಬಹುದು, ಸಂಕುಚಿತಗೊಳಿಸಬಹುದು ಮತ್ತು ಅವುಗಳನ್ನು `dist/` ಡೈರೆಕ್ಟರಿಗೆ ಔಟ್‌ಪುಟ್ ಮಾಡಬಹುದು. ಈ ಡೈರೆಕ್ಟರಿಯು ಉತ್ಪಾದನಾ ವಿತರಣೆಗಾಗಿ JS/CSS ಫೈಲ್‌ಗಳನ್ನು ಹೊಂದಿದೆ, GitHub ಪುಟಗಳು ಅಥವಾ ಕ್ಲೌಡ್‌ಫ್ಲೇರ್ ಪುಟಗಳಿಗೆ ನಿಯೋಜನೆಗಾಗಿ ಸಿದ್ಧವಾಗಿರುವ ಡೆಮೊ ಸೈಟ್, ಮತ್ತು ಸ್ವಯಂಚಾಲಿತವಾಗಿ `_redirects` ಮತ್ತು `404.html` ರೂಟಿಂಗ್ ಫೈಲ್‌ಗಳನ್ನು ರಚಿಸಲಾಗಿದೆ.
- **NPM ಪ್ರಕಟಿಸಿ**: `./sh/npmDist.js` ಅನ್ನು ರನ್ ಮಾಡಿ, ಸ್ಕ್ರಿಪ್ಟ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಘಟಕ ಅವಲಂಬನೆಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ, ಪ್ಯಾಕೇಜ್ ಆವೃತ್ತಿ ಸಂಖ್ಯೆಯನ್ನು ಹೆಚ್ಚಿಸುತ್ತದೆ ಮತ್ತು NPM ಗೆ ಕಾಂಪೊನೆಂಟ್ ಲೈಬ್ರರಿಯನ್ನು ಪ್ರಕಟಿಸಲು ಪ್ಯಾಕೇಜಿಂಗ್ ಪೂರ್ಣಗೊಂಡ ನಂತರ ಒಂದು ಕ್ಲಿಕ್‌ನಲ್ಲಿ `npm publish` ಅನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸುತ್ತದೆ.

### ವೇದಿಕೆಯು ಅನುವಾದದ ಜವಾಬ್ದಾರಿಯನ್ನು ಹೊಂದಿದೆ

ಫ್ರೇಮ್‌ವರ್ಕ್ ಅಂತರ್ನಿರ್ಮಿತ ಸ್ವಯಂಚಾಲಿತ ಅನುವಾದ ಪ್ರಕ್ರಿಯೆಗಳನ್ನು ಹೊಂದಿದೆ:
1. ಪ್ರಾಜೆಕ್ಟ್ ರೂಟ್ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ `tran.yml` ನಲ್ಲಿ ಅನುವಾದ ನಿಯಮಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ (ಉದಾಹರಣೆಗೆ ಮೂಲ ಭಾಷೆ, ಗುರಿ ಭಾಷೆ ಮತ್ತು ಅನುವಾದಿಸಬೇಕಾದ ಡೈರೆಕ್ಟರಿ).
2. ನಿಮ್ಮ ಕೋಡ್‌ನಲ್ಲಿ ಬಹು ಭಾಷೆಗಳನ್ನು ಹಾರ್ಡ್‌ಕೋಡ್ ಮಾಡದೆಯೇ ಘಟಕಗಳು ಅಥವಾ ಡಾಕ್ಯುಮೆಂಟ್‌ಗಳಲ್ಲಿ ಪ್ಲೇಸ್‌ಹೋಲ್ಡರ್‌ಗಳನ್ನು ಬಳಸಿ.
3. `./sh/tran.js` ಅನ್ನು ರನ್ ಮಾಡಿ, ಮತ್ತು ಅನುವಾದ ಎಂಜಿನ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪಠ್ಯವನ್ನು ಹೊರತೆಗೆಯುತ್ತದೆ ಮತ್ತು ಪ್ರಕ್ರಿಯೆಗಾಗಿ WebC ಅನುವಾದ ಸೇವೆಗೆ ಕರೆ ಮಾಡುತ್ತದೆ, ಸ್ವಯಂಚಾಲಿತವಾಗಿ YAML ಅನುವಾದ ನಮೂದುಗಳನ್ನು ಮತ್ತು ವಿವಿಧ ಭಾಷೆಗಳಲ್ಲಿ README ದಸ್ತಾವೇಜನ್ನು ಉತ್ಪಾದಿಸುತ್ತದೆ.

### ಪರಿಸರ ಅಸ್ಥಿರ

ಸ್ವಯಂಚಾಲಿತ ಅನುವಾದವನ್ನು ನಿರ್ವಹಿಸುವಾಗ, ಅನುವಾದ API ಗೆ ಪ್ರವೇಶವನ್ನು ದೃಢೀಕರಿಸಲು ನೀವು ಈ ಕೆಳಗಿನ ಪರಿಸರ ವೇರಿಯೇಬಲ್‌ಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಬೇಕಾಗುತ್ತದೆ:

- **`WEBC_SITE_TOKEN`** (ಅಥವಾ **`WEBC_TOKEN`**): `webc.site` ಸ್ವಯಂಚಾಲಿತ ಅನುವಾದ API ಅನ್ನು ಪ್ರವೇಶಿಸಲು ದೃಢೀಕರಣ ಕೀಯನ್ನು ಬಳಸಲಾಗುತ್ತದೆ. ಸ್ಥಳೀಯ ಬಳಕೆದಾರರ ಹೋಮ್ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿರುವ `~/.config/webc.site.yml` ಕಾನ್ಫಿಗರೇಶನ್ ಫೈಲ್‌ನಲ್ಲಿ ಪರಿಸರ ವೇರಿಯೇಬಲ್ ಅಥವಾ `token` ಎಂದು ಬರೆಯಬಹುದು.
