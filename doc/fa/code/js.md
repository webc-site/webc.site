# مشخصات کد JS

## سبک کدنویسی

- ساده، زیبا و کارآمد، با استفاده از مدرن ترین نحو js
- عملکردهای بیش از حد طولانی را تقسیم کنید، و طراحی رابط باید جفت کم و انسجام بالا باشد.
- از آرایه‌ها + حلقه‌ها/forEach/map بیشتر استفاده کنید و کدهای تکراری کمتری بنویسید
- برای جلوگیری از کامنت های بیش از حد، نظرات تابع را با کد ساده ننویسید. منطق پیچیده و نیازهای ویژه به زبان چینی تصفیه شده توضیح داده شده است.
- کدهای مکرر برای کاهش افزونگی به توابع انتزاع می‌شوند. برای مثال، `cosnt A=new Table({ style: { border: NO_BORDER } }), B=new Table({ style: { border: NO_BORDER } })` را می توان به صورت `const newTable=()=>new Table({ style: { border: NO_BORDER } }), A=newTable(), B=newTable()` نوشت
- اعلان متغیر: چندین اعلان `const` متوالی را در یک عبارت ترکیب کنید. به جای اینکه آن را در سه خط بنویسید، `const a=1, b=2, c=3;` را بنویسید
- پردازش ناهمزمان: از `await` استفاده کنید، استفاده از تماس های زنجیره ای `.then` را ممنوع کنید
- کد مدیریت استثنا را به طور خودکار ایجاد نکنید، `try...catch` را به طور خودکار ننویسید (`try ... catch` به صورت دستی نگهداری می شود، `try catch` موجود نباید حذف شود)
- ابتدا توابع خالص: فقط توابع خالص بنویسید، هرگز کلاسها
- از توابع پیکان `const funcName = () => {}` استفاده کنید و از کلمه کلیدی `function` استفاده نکنید (به جز ژنراتورها). اگر می توانید از .bind برای اتصال پارامترها استفاده کنید، از تعریف توابع اجتناب کنید
- استفاده مجدد از کد: به استفاده مجدد توجه کنید، توابع کوچک بیشتری را استخراج کنید، و قاطعانه از تعداد زیادی از ساختارهای کد مشابه یا کپی پیست شده خودداری کنید.
- دسترسی به شی: اولویت بندی با استفاده از تخصیص تخریب ساختار برای استخراج ویژگی های مورد نیاز، اجتناب از استفاده مکرر از نقاط در داخل برای دسترسی به ویژگی های عمیق و تودرتو، و ادغام قضاوت های زنجیره ای اختیاری مکرر
- از اشیاء برای پارامترهای تابع استفاده نکنید، `{a,b,c}` ننویسید، `a, b, c` را بنویسید. اگر پارامترهای اختیاری زیادی وجود دارد، از [[تعداد آیتم های پیکربندی، آیتم های پیکربندی]، [تعداد آیتم های پیکربندی، آیتم های پیکربندی]،...] در این پارادایم، آیتم های پیکربندی با ثابت های عددی تعریف می شوند.
- برای برگرداندن چند مقدار، به جای `{a,b,c}` از آرایه `[a,b,c]` استفاده کنید. اگر بیش از 3 مقدار بازگشتی وجود دارد، از ثابت های عددی برای تعریف معنای موقعیتی استفاده کنید.
- نیازی به رشته ای برای نمایش وضعیت نیست، برای تعریف حالت از ثابت های عددی استفاده کنید
- هیچ الگوی رشته ای (``) استفاده نمی شود، از الحاق رشته (+) استفاده می شود، به جز برای واردات (مناسب برای تجزیه و تحلیل استاتیک Vite)
- برای حلقه، اگر به شماره سریال نیاز دارید، به جای `i++` از `++i` استفاده کنید.
- از`?.` غیرضروری اجتناب کنید و برنامه‌نویسی تدافعی را ممنوع کنید
- نام متغیر نیازی به جمع ندارد و برای نشان دادن لیست به `_li` ختم می شود.

## قرارداد نامگذاری

- نامگذاری مینیمالیسم را دنبال می کند. از نام های کوتاه اما معنی دار استفاده کنید، به عنوان مثال: به جای `remove`، `delete`، `del` از `rm` استفاده کنید. با این حال، باید از افراط و تفریط نیز اجتناب کنید، برای مثال: به جای `map` از یک حرف `m` استفاده نکنید.
- سعی کنید در نامگذاری توابع فقط از افعال استفاده کنید. اگر می توانید از یک کلمه برای بیان آن استفاده کنید، از دو کلمه استفاده نکنید. اسم ها در نام فایل ها منعکس می شوند. اگر نیاز به اضافه کردن فعل به نام فایل دارید، لطفا اسم را اول و فعل را آخر قرار دهید. برای مثال: `profileSet.js` به جای `setProfile.js`
- نام متغیر: از سبک زیر خط (snake_case) استفاده کنید، برای مثال `user_auth_token`; اگر متغیر یک تابع است، از سبک نامگذاری شتر با حروف کوچک استفاده کنید
- نام تابع: از حرف کوچک camelCase (camelCase) برای مثال `userData` استفاده کنید
- پارامترهای تابع: اگر تابع پاسخ به تماس است، آن را با حروف کوچک شتر نامگذاری کنید، مانند `onChange`
- تعاریف ثابت سطح ماژول از سبک زیر خط بزرگ `UPPER_SNAKE_CASE` استفاده می کنند
- از نوشتن `get`، پیشوندهای بی معنی مانند: بجای `getCookie` `cookieByHeader` بنویسید
- ثابت های سراسری/سطح ماژول: از سبک زیر خط بزرگ (UPPER_SNAKE_CASE) استفاده کنید، به عنوان مثال. `CODE_TO_ID`، `ID_TO_LANG`

## مکانیزم مدولار

- واردات: عملکردها را به صورت دقیق وارد کنید، واردات مستقیم کل ماژول را ممنوع کنید (از `import * as x` اجتناب کنید یا اشیاء بزرگ را وارد کنید)
- Export: غیرفعال کردن صادرات اشیا. توابع و متغیرها را به عنوان واحد صادر کنید، مانند `export const X=1, abc=()=>{};`. سعی کنید آنها را با یک const + کاما ترکیب کنید تا محتوای صادر شده را اعلام کنید. اگر یک فایل فقط یک تابع دارد، از `export default` استفاده کنید
- مگر اینکه لازم باشد تابع پیش‌فرض صادرات را به صورت داخلی فراخوانی کنید، ابتدا از اعلان ثابت‌ها و سپس صادرات آنها در انتهای فایل خودداری کنید.
- وضوح مسیر: هنگام بدست آوردن مسیر دایرکتوری فعلی، باید از `import.meta.dirname` استفاده شود

## اشتباه
