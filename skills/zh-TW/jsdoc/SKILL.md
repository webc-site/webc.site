## 運行步驟

1. 執行 `./sh/jsDoc.js`，查看缺少文件的函數（若無輸出，則無需補充）。
2. 根據輸出的文件列表，逐一讀取現有文件與程式碼並補全（依序處理，逐一擊破）。

## 文件要求

如果導出是函數，寫
- 參數與回​​傳值：
  - 僅在有參數或傳回值時寫對應說明，無則不寫。
  - 參數使用無序列表介紹。
  - 若傳回值為數組，使用無序列表逐一說明（否則，直接寫，`傳回值: 描述`，一行即可）。
  - 若參數或傳回值為函數，其參數與傳回值使用子無序列表說明（無則不寫）。

如果導出是常數，就介紹這是什麼，一行即可

- 用語簡潔、專業，避免使用形容詞。
- 要在現有的文件上修訂，不要完全重寫

## 格式模板

每個名字只寫一次，避免重複，盡量一行表達清楚，參考格式如下

```
# 檔案名稱

## 函數名(參數)

用途描述

參數:
- `xx` : 用途
  - 參數
    - `a` : 用途
  - 傳回值 : 說明

傳回值: 說明
```
