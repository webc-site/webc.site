# internasionalisasi

Kasus: Lihat `src/webc/BoxX` untuk mengimplementasikan internasionalisasi komponen dan demo.

aturan:
- `yml` Jangan tambahkan `:` di akhir entri, masukkan `:` ke dalam template pug dan kode js, lalu sambungkan
- Gunakan `|-` untuk teks multi-baris `yml` untuk menghapus spasi tambahan dengan mudah.
- Jika tidak ada entri internasional, hapus file `I18N.js` (jangan tulis apa pun, jangan tulis `export`)
- Entri harus dalam satu bahasa, dan dilarang keras mencampurkan beberapa bahasa.

proses:

1. Lihat kode bahasa sumber yang dikonfigurasi di direktori root `tran.yml` → `tran` → `from`

2. Masuk ke direktori `src/webc/nama komponen`

3. Internasionalisasi komponen
   - Tulis teks yang digunakan dalam `nama komponen.js` sebagai entri ke dalam `src/webc/nama komponen/i18n/source bahasa/js.yml`.
   - Ubah `src/webc/component name/I18N.js` dan konfigurasikan konstanta `yml` nama kunci entri, seperti `export const KEY = "ymlKey"`
   - Pengenalan dan pengikatan di `src/webc/component name.js` (jika ada `js.yml`, jika tidak maka tidak perlu):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     impor i18nUrl, { KUNCI } dari "./nama komponen/I18N.js"; // Kerangka kerja akan secara otomatis menyuntikkan i18nUrl, default ekspor default, jangan membuatnya sendiri

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Cobalah untuk mengikat secara berkelompok untuk mengurangi panggilan fLang
    `nama komponen.js` komponen tidak boleh memiliki string yang berisi teks

4. Internasionalisasi presentasi
   - Di `demo/_.pug`, teks label yang perlu diterjemahkan diisi dalam bentuk `nama kunci c-t` (ini adalah label khusus, baris terpisah), dan entri ditulis ke dalam `demo/i18n/source bahasa/pug.yml`
   - `demo/_.js` Tulis entri teks asli yang digunakan dalam `demo/i18n/source bahasa/js.yml` dan letakkan di `demo/I18N.js`
   - Masukkan `fetchLang` ke dalam `demo/_.js` dan muat `./I18N.js` (ini adalah `demo/I18N.js`), lihat proses internasionalisasi komponen di atas (misalnya, `demo/i18n` tidak memiliki `js.yml` di bawah, lewati langkah ini)
   - `demo/I18N.js` Konfigurasikan konstanta nama kunci di `demo/i18n/source bahasa/js.yml` dan ekspor

5. Tulis deskripsi komponen di `src/webc/component name/i18n/source bahasa/README.md`. Prosesnya adalah sebagai berikut
  5.1 Tulis judul utama. Judul harus dimulai dengan `#`, menjelaskan tujuan komponen dalam satu kalimat (tidak menulis nama komponen), dan tidak menambahkan tanda baca di akhir.
  5.2 Sebutkan ciri-ciri utama komponen, dimulai dengan judul utama, dan jangan menulis judul sekunder. Jika terdapat beberapa karakteristik, gunakan daftar tidak berurutan (jika uraian daftar tidak berurutan relatif pendek, tidak akan ada titik di akhir).
  5.3 Tulis `## Menggunakan demo`, kode demo dimulai dengan ` ```html ` (pada kode demo, tag elemen tunggal tidak ditutup), jika ada `js`, tulis `html` secara internal, gunakan `<script type="module">` Tag, `js` Tidak memiliki lekukan pada baris pertama konten.
  Jika js murni, mulailah dengan ` ```js `. `js` Dalam demonstrasi, jika Anda perlu mengimpor `component/_.js`, silakan gunakan `import xxx from "webc.site/component name.js";`. Ini adalah `js` yang telah dikompilasi, yang secara otomatis akan memasukkan js dan css.
  `js` Gunakan tulisan ringkas, seperti `const` untuk mendeklarasikan beberapa konstanta

Teks utama memperkenalkan cara menggunakan komponen, parameter antarmuka, beralih antara `css` dan `class` di status berbeda, dll.
Selanjutnya jelaskan parameter, nilai kembalian, status gaya, dll. (jika ada), dan tambahkan judul sekunder.

Gunakan kata-kata secara ringkas dan hindari kata sifat.

Copywriting harus menghindari hal-hal yang tidak masuk akal dan tidak menulis kata-kata yang tidak bermakna (seperti: universal, container, dll).

6. Jalankan `./sh/tran.js` dari direktori root, terjemahan
