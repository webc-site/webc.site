# umiędzynarodowienie

Przypadek: Zobacz `src/webc/BoxX`, aby wdrożyć internacjonalizację komponentów i wersji demonstracyjnej.

reguła:
- `yml` Nie dodawaj `:` na końcu wpisu, wstaw `:` do szablonu pug i kodu js i połącz to
- Użyj `|-` w przypadku wielowierszowego tekstu `yml`, aby łatwo usunąć końcowe spacje.
- Jeśli nie ma wpisu umiędzynarodowionego, wyczyść plik `I18N.js` (nic nie pisz, nie pisz `export`)
- Zgłoszenia muszą być w jednym języku, a mieszanie wielu języków jest surowo zabronione.

proces:

1. Wyświetl kod języka źródłowego skonfigurowany w katalogu głównym `tran.yml` → `tran` → `from`

2. Wejdź do katalogu `src/webc/component name`

3. Internacjonalizacja komponentów
   - Zapisz tekst używany w pliku `nazwa komponentu.js` jako wpis w pliku `src/webc/nazwa komponentu/i18n/język źródłowy/js.yml`.
   - Zmodyfikuj `src/webc/component name/I18N.js` i skonfiguruj stałą nazwy klucza wpisu `yml`, np. `export const KEY = "ymlKey"`
   - Wprowadzenie i powiązanie w `src/webc/component name.js` (jeśli jest `js.yml`, jeśli nie, to nie jest to konieczne):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     zaimportuj i18nUrl, {KLUCZ } z „./nazwa komponentu/I18N.js”; // Framework automatycznie wstrzyknie i18nUrl, domyślne ustawienie domyślne eksportu, nie twórz go samodzielnie

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Spróbuj wiązać partiami, aby zmniejszyć liczbę wywołań fLang
    `nazwa komponentu.js` komponentu nie powinna zawierać ciągu zawierającego tekst

4. Umiędzynarodowienie prezentacji
   - W `demo/_.pug` tekst etykiety, który należy przetłumaczyć, jest zapisywany w postaci `nazwy klucza c-t` (jest to etykieta niestandardowa, osobna linia), a wpis jest zapisywany w `demo/i18n/sourcelang/pug.yml`
   - `demo/_.js` Wpisz oryginalny wpis tekstowy używany w `demo/i18n/sourcelang/js.yml` i umieść go w `demo/I18N.js`
   - Wprowadź `fetchLang` do `demo/_.js` i załaduj `./I18N.js` (to jest `demo/I18N.js`), zapoznaj się z powyższym procesem internacjonalizacji komponentu (na przykład `demo/i18n` nie ma `js.yml` poniżej, pomiń ten krok)
   - `demo/I18N.js` Skonfiguruj stałe nazw kluczy w `demo/i18n/sourcelang/js.yml` i wyeksportuj

5. Zapisz opis komponentu w pliku `src/webc/component name/i18n/sourcelang/README.md`. Proces jest następujący
  5.1 Wpisz tytuł główny. Tytuł powinien zaczynać się od `#`, opisywać przeznaczenie komponentu w jednym zdaniu (nie podawać nazwy komponentu), a na końcu nie dodawać znaków interpunkcyjnych.
  5.2 Wymień główne cechy komponentu, zaczynając od tytułu głównego i nie wpisuj tytułów drugorzędnych. Jeśli cech jest wiele, użyj listy nieuporządkowanej (jeśli opis listy nieuporządkowanej jest stosunkowo krótki, na końcu nie będzie kropki).
  5.3 Zapis `## Używając demo`, kod demo zaczyna się od ` ```html ` (w kodzie demonstracyjnym znacznik pojedynczego elementu nie jest zamknięty), jeśli jest `js`, napisz `html` wewnętrznie, użyj `<script type="module">` Tag, `js` Nie zawiera wcięcia w pierwszym wierszu treści.
  Jeśli jest to czysty js, zacznij od ` ```js `. `js` Jeśli w demonstracji chcesz zaimportować `component/_.js`, użyj `import xxx z „webc.site/component name.js”;`. To jest skompilowany `js`, który automatycznie wstrzyknie js i css.
  `js` Używaj zwięzłego zapisu, np. `const`, aby zadeklarować wiele stałych

Tekst główny przedstawia sposób użycia komponentu, parametry interfejsu, przełączanie pomiędzy `css` i `class` w różnych stanach itp.
Następnie opisz parametry, zwracaną wartość, status stylu itp. (jeśli istnieje) i dodaj tytuł dodatkowy.

Używaj słów zwięźle i unikaj przymiotników.

Copywriting powinien unikać bzdur i nie pisać słów bez znaczenia (takich jak: uniwersalny, kontener itp.).

6. Uruchom `./sh/tran.js` katalogu głównego, tłumaczenie
