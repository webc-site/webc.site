# internacionalización

Caso: consulte `src/BoxX` para implementar a internacionalización dos compoñentes e a demostración.

regra:
- `yml` Non engadas `:` ao final da entrada, pon `:` no modelo pug e no código js, ​​e empalmao
- Usa `|-` para o texto de varias liñas de `yml` para eliminar facilmente os espazos ao final.
- Se non hai ningunha entrada internacionalizada, borre o ficheiro `I18N.js` (non escriba nada, non escriba `export`)
- As entradas deben estar nun só idioma e está terminantemente prohibido mesturar varios idiomas.

proceso:

1. Ver o código da lingua fonte configurado no directorio raíz `tran.yml` → `tran` → `from`

2. Introduza o directorio `src/component name`

3. Internacionalizar compoñentes
   - Escribe o texto usado por `_.js` como entrada en `src/component name/i18n/source language/js.yml`.
   - Modifique `Nome do compoñente/I18N.js` e configure a constante do `yml` nome da chave de entrada, como `export const KEY = "ymlKey"`
   - `Introducido e vinculado no nome do compoñente/_.js` (se hai `js.yml`, non é necesario se non):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     importar i18nUrl, { KEY } de "./I18N.js"; // O framework inxectará automaticamente i18nUrl, o valor predeterminado de exportación, non o cree vostede mesmo

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Tenta vincular por lotes para reducir as chamadas fLang
    O `_.js` do compoñente non debe ter cadeas que conteñan texto

4. Internacionalización de presentacións
   - En `demo/_.pug`, o texto da etiqueta que se debe traducir está ocupado en forma de `nome da chave c-t` (esta é unha etiqueta personalizada, unha liña separada) e a entrada está escrita en `demo/i18n/source language/pug.yml`
   - `demo/_.js` Escribe a entrada de texto orixinal utilizada en `demo/i18n/source language/js.yml` e colócaa en `demo/I18N.js`
   - Introduce `fetchLang` en `demo/_.js` e carga `./I18N.js` (isto é `demo/I18N.js`), consulta o proceso de internacionalización de compoñentes anterior (por exemplo, `demo/i18n` non ten `js.yml` a continuación, omite este paso)
   - `demo/I18N.js` Configure as constantes do nome da chave en `demo/i18n/source language/js.yml` e exporte

5. Escriba a descrición do compoñente en `Nome do compoñente/i18n/source language/README.md`. O proceso é o seguinte
  5.1 Escribe o título principal. O título debe comezar por `#`, describir o propósito do compoñente nunha frase (non escribir o nome do compoñente) e non engadir puntuación ao final.
  5.2 Enumera as características principais do compoñente, comezando polo título principal, e non escribe títulos secundarios. Se hai varias características, use unha lista sen ordenar (se a descrición da lista non ordenada é relativamente curta, non haberá punto ao final).
  5.3 Escriba `## Usando a demostración`, o código de demostración comeza por ` ```html ` (no código de demostración, a etiqueta de elemento único non está pechada), se hai `js`, escriba `class="CODE">, escriba `class="CODE"> internamente ` id="CODE"> etiqueta;
  Se é js puro, comeza por ` ```js `. `js` Na demostración, se precisa importar `component/_.js`, use `import xxx from "webc.site/component/_.js";`.
  `js` Use escritura concisa, como `const` para declarar varias constantes

O texto principal presenta como usar o compoñente, os parámetros da interface, o cambio entre `css` e `class` en diferentes estados, etc.
Posteriormente, describa os parámetros, o valor de retorno, o estado do estilo, etc. (se os houbera) e engada un título secundario.

Usa palabras concisas e evita os adxectivos.

A redacción debe evitar tonterías e non escribir palabras sen sentido (como: universal, contedor, etc.).

6. Executar `./sh/tran.js` do directorio raíz, tradución
