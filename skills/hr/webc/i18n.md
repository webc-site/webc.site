# internacionalizacija

Slučaj: pogledajte `src/BoxX` za implementaciju internacionalizacije komponenti i demo.

pravilo:
- `yml` Nemojte dodavati `:` na kraju unosa, stavite `:` u predložak pug i js kod i spojite ga
- Koristite `|-` za višeredni tekst od `yml` za jednostavno uklanjanje razmaka na kraju.
- Ako nema internacionaliziranog unosa, izbrišite datoteku `I18N.js` (nemojte pisati ništa, nemojte pisati `export`)
- Prijave moraju biti na jednom jeziku, a miješanje više jezika je strogo zabranjeno.

proces:

1. Pogledajte kôd izvornog jezika konfiguriran u korijenskom direktoriju `tran.yml` → `tran` → `from`

2. Unesite direktorij `src/component name`

3. Internacionalizirati komponente
   - Napišite tekst koji koristi `_.js` kao unos u `src/component name/i18n/source language/js.yml`.
   - Izmijenite `Naziv komponente/I18N.js` i konfigurirajte konstantu `yml` naziva ulaznog ključa, kao što je `export const KEY = "ymlKey"`
   - `Uvedeno i vezano u naziv komponente/_.js` (ako postoji `js.yml`, nije potrebno ako nije):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     import i18nUrl, { KEY } from "./I18N.js"; // Okvir će automatski ubaciti i18nUrl, zadanu zadanu vrijednost izvoza, nemojte je sami stvarati

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Pokušajte vezati u serijama kako biste smanjili fLang pozive
    `_.js` komponente ne smije sadržavati nizove koji sadrže tekst

4. Internacionalizacija prezentacija
   - U `demo/_.pug`, tekst oznake koji treba prevesti je zauzet u obliku `c-t key name` (ovo je prilagođena oznaka, zaseban red), a unos je zapisan u `demo/i18n/source language/pug.yml`
   - `demo/_.js` Napišite izvorni unos teksta korišten u `demo/i18n/source language/js.yml` i smjestite ga u `demo/I18N.js`
   - Uvedite `fetchLang` u `demo/_.js` i učitajte `./I18N.js` (ovo je `demo/I18N.js`), pogledajte gornji proces internacionalizacije komponente (na primjer, `demo/i18n` nema `js.yml` ispod, preskočite ovaj korak)
   - `demo/I18N.js` Konfigurirajte konstante naziva ključa u `demo/i18n/source language/js.yml` i izvezite

5. Napišite opis komponente u `Naziv komponente/i18n/izvorni jezik/README.md`. Proces je sljedeći
  5.1 Napišite glavni naslov. Naslov treba započeti s `#`, opisivati ​​svrhu komponente u jednoj rečenici (ne pisati naziv komponente) i ne dodavati interpunkcijske znakove na kraju.
  5.2 Navedite glavne značajke sastavnice, počevši od glavnog naslova, a nemojte pisati sekundarne naslove. Ako postoji više karakteristika, koristite nesređeni popis (ako je opis nesređenog popisa relativno kratak, neće biti točke na kraju).
  5.3 Zapišite `## Koristeći demo`, demo kod počinje s ` ```html ` (u demo kodu, oznaka jednog elementa nije zatvorena), ako postoji `js`, interno napišite `html`, koristite `<script type="module">` oznaka;
  Ako je čisti js, počnite s ` ```js `. `js` U demonstraciji, ako trebate uvesti `component/_.js`, koristite `import xxx iz "webc.site/component name.js";`.
  `js` Koristite sažeto pisanje, kao što je `const` za deklariranje više konstanti

Glavni tekst predstavlja kako koristiti komponentu, parametre sučelja, prebacivanje između `css` i `class` u različitim stanjima, itd.
Zatim opišite parametre, povratnu vrijednost, status stila itd. (ako postoji) i dodajte sekundarni naslov.

Koristite riječi sažeto i izbjegavajte pridjeve.

Copywriting treba izbjegavati besmislice i ne pisati besmislene riječi (kao što su: univerzalni, spremnik i sl.).
