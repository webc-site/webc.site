# Internationaliséierung

Fall: Referéiert op `src/webc/BoxX` fir Internationaliséierung vu Komponenten an Demo ëmzesetzen.

Regel:
- `yml` Füügt net `:` um Enn vun der Entrée bäi, setzt `:` an d'Pug-Schabloun an de js-Code, a verbënnt et
- Benotzt `|-` fir Multi-Linn Text vun `yml` fir einfach hannendrun Plazen ze läschen.
- Wann et keng internationaliséierter Entrée gëtt, läscht d'`I18N.js` Datei (schreiwen näischt, schreiwt net `export`)
- Entréen mussen an enger eenzeger Sprooch sinn, a verschidde Sprooche vermëschen ass strikt verbueden.

Prozess:

1. Kuckt de Quellsproochcode deen am Rootverzeichnis konfiguréiert ass `tran.yml` → `tran` → `from`

2. Gitt den `src/webc/component name` Verzeichnis an

3. Komponente internationaliséieren
   - Schreift den Text deen an `Component name.js` benotzt gëtt als Entrée an `src/webc/component name/i18n/source language/js.yml`.
   - Ännert `src/webc/component name/I18N.js` a konfiguréiert d'Konstante vum `yml` Entréesschlësselnumm, wéi `export const KEY = "ymlKey"`
   - Aféierung a Bindung an `src/webc/component name.js` (wann et `js.yml` gëtt, wann net, ass et net néideg):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     importéieren i18nUrl, {KEY} vun "./Component name/I18N.js"; // De Kader gëtt automatesch i18nUrl injizéiert, de Standard Export Standard, erstellt et net selwer

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Probéiert a Chargen ze binden fir fLang Uriff ze reduzéieren
    Den `Komponentname.js` vum Komponent däerf keng String mat Text hunn

4. Internationaliséierung vun Presentatiounen
   - An `demo/_.pug` ass de Labeltext, deen iwwersat muss ginn, besat a Form vum `c-t Schlësselnumm` (dëst ass e personaliséierte Label, eng separat Linn), an den Entrée gëtt an `demo/i18n/source language/pug.yml` geschriwwen.
   - `demo/_.js` Schreift den ursprénglechen Textentrée deen an `Demo/i18n/source language/js.yml` benotzt gëtt a setzt se op `demo/I18N.js`
   - Aféierung `fetchLang` an `demo/_.js` a lued `./I18N.js` (dëst ass `demo/I18N.js`), kuckt op de Komponent Internationaliséierungsprozess uewen (zum Beispill, `demo/i18n` huet net `js.yml` drënner, sprangen dëse Schrëtt)
   - `demo/I18N.js` Konfiguréieren Schlësselnummkonstanten an `Demo/i18n/source language/js.yml` an exportéieren

5. Schreift d'Komponentbeschreiwung an `src/webc/component name/i18n/source language/README.md`. De Prozess ass wéi follegt
  5.1 Schreift den Haapttitel. Den Titel soll mat `#` ufänken, den Zweck vun der Komponent an engem Saz beschreiwen (net den Numm vum Komponent schreiwen), a keng Punktuatioun um Enn derbäisetzen.
  5.2 Lëscht d'Haaptmerkmale vun der Komponent, ugefaange mam Haapttitel, a schreift keng sekundär Titelen. Wann et e puer Charakteristiken, benotzt eng unordered Lëscht (wann der unordered Lëscht Beschreiwung relativ kuerz ass, gëtt et keng Period um Enn).
  5.3 Schreift `## Mat der Demo` fänkt den Demo-Code mat ` ```html ` (am Demo-Code ass den eenzegen Element-Tag net zou), wann et `js` ass, schreift `html<b class="CODE3"> intern`html<b class =" CODE3"> intern. class="CODE"> Tag, `js` Hutt keng Indentatioun an der éischter Zeil vum Inhalt.
  Wann et pure js ass, fänkt mat ` ```js ` un. `js` An der Demonstratioun, wann Dir `Component/_.js` importéiere musst, benotzt w.e.g. `import xxx vun "webc.site/component name.js";`. Dëst ass de kompiléierten `js`, deen automatesch js an css injizéiert.
  `js` Benotzt präzis Schreiwen, wéi e `const` fir verschidde Konstanten ze deklaréieren

Den Haapttext stellt vir wéi d'Komponente benotzt ginn, Interface Parameteren, Wiessel tëscht `css` an `class` a verschiddene Staaten, etc.
Beschreift duerno d'Parameteren, Retourwäert, Stilstatus, etc.. (wann iwwerhaapt), a füügt e Secondaire Titel.

Benotzt Wierder präzis a vermeit Adjektiver.

Copywriting soll Nonsens vermeiden a keng sënnlos Wierder schreiwen (wéi: Universal, Container, asw.).

6. Run `./sh/tran.js` vum Root-Verzeichnis, Iwwersetzung
