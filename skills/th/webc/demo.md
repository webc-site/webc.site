# การสาธิต

อ้างถึง `src/BoxX/demo` และพัฒนาสาธิตส่วนประกอบใน `src/component name/demo/`

`demo/_.pug` เทมเพลตย่อยสาธิต

รวมเฉพาะแท็กคอมโพเนนต์แบบมินิมอลเท่านั้น ไม่จำเป็นต้องมีหน้าที่สมบูรณ์ และห้ามนำเข้าสไตล์และสคริปต์ กรอบงานจะฉีดโดยอัตโนมัติ (เช่น `demo/_.styl`, `demo/_.js`)

ห้ามเขียนชื่อ (เช่น ห้ามเขียน xxx demo), ห้ามเขียนชื่อส่วนประกอบ, ห้ามแนะนำส่วนประกอบ, เขียนเฉพาะสถานะของกรณีการใช้งานการสาธิตเท่านั้น (หากไม่มีสถานะ, ไม่ต้องเขียนชื่อเรื่อง)

องค์ประกอบรูทของ `_.pug` ใช้ `main.demo.Lg` ซึ่งได้กำหนดสไตล์ใน `demo/_.styl` และไม่ได้กำหนดสไตล์ของ `main.demo.Lg` เอง

คอนเทนเนอร์ขององค์ประกอบการแสดงผลควรสอดคล้องกับ `main.demo.Lg` และไม่ควรมี `margin` และ `padding` อยู่ระหว่างกัน เนื่องจาก `main.demo` ตั้งค่า `padding` ไว้แล้ว และการตั้งค่าว่างจะถูกทำซ้ำ

`demo/_.styl` หากมีองค์ประกอบการแสดงผลเพียงองค์ประกอบเดียว คอนเทนเนอร์ของมันจะไม่มีกรอบด้านนอก และตั้งค่าเฉพาะ `max-width` (เนื่องจากคอนเทนเนอร์การนำเสนอจะเพิ่มกรอบด้านนอกโดยอัตโนมัติ) หากมีหลายอัน คอนเทนเนอร์สามารถแยกออกจากกันโดยใช้กรอบด้านนอกได้

อย่าเขียน `img` ใน `demo/_.pug` ให้ใช้ `demo/_.styl` เพื่อตั้งค่าภาพพื้นหลัง

หากส่วนประกอบมีหลายรูปแบบ โปรดแสดงตามลำดับ (เช่น กำลังโหลด ข้อมูลว่าง ข้อมูล ล้มเหลว ฯลฯ) เค้าโครงแนวตั้งแบบยืดหยุ่น

`demo/_.styl`: `stylus` รูปแบบของหน้าสาธิต โปรดปฏิบัติตามข้อกำหนดโค้ดและความสวยงามในการออกแบบของ [./styl.md](./styl.md)

อย่าเขียน `@import '../_.styl'` เพื่อนำเข้าส่วนประกอบที่จะสาธิต เพราะส่วนประกอบนั้นจะถูกแทรกโดยอัตโนมัติ

โปรดนำเข้า `@import '../../Btn/_.styl'` ใน `demo/_.styl` และใช้ `.Btn`

`demo/_.js`: เรียกสคริปต์ รูปแบบ `export default (root) => { ... }` พารามิเตอร์ `root` เป็นโฮสต์ (`document` หรือองค์ประกอบรากของคอนเทนเนอร์แซนด์บ็อกซ์)

`D.createElement` เป็นสิ่งต้องห้าม ใช้ `newEl` กล่าวถึงโดย [./js.md](./js.md)

`js` ห้ามใช้ใน `c-t` และสำหรับการทำให้เป็นสากล `fLang`

หาก `../_.js` (`../_.js` คือการแสดงส่วนประกอบนั้นเอง ส่วนประกอบที่ต้องพึ่งพาที่จำเป็นสำหรับการสาธิตอื่น ๆ ยังคงจำเป็นต้องนำเข้า) ต้องใช้เพียง `import` เท่านั้น และไม่มีการนำเข้าเนื้อหาเฉพาะเจาะจง ซึ่งสามารถละเว้นได้ (กรอบงานจะแทรกโดยอัตโนมัติ)

`pug` ไม่จำเป็นต้องนำเข้า `js` และ `styl` กรอบงานจะแทรกเข้าไปโดยอัตโนมัติ
