# ความเป็นสากล

กรณี: อ้างถึง `src/BoxX` เพื่อปรับใช้ส่วนประกอบและการสาธิตให้เป็นสากล

กฎ:
- `yml` อย่าเพิ่ม `:` ที่ส่วนท้ายของรายการ ใส่ `:` ลงในเทมเพลต pug และโค้ด js แล้วประกบกัน
- ใช้ `|-` สำหรับข้อความหลายบรรทัดของ `yml` เพื่อลบช่องว่างต่อท้ายได้อย่างง่ายดาย
- หากไม่มีรายการที่เป็นสากล ให้ล้างไฟล์ `I18N.js` (ไม่ต้องเขียนอะไรเลย ไม่ต้องเขียน `export`)
- ผลงานจะต้องเป็นภาษาเดียว และห้ามผสมหลายภาษาโดยเด็ดขาด

กระบวนการ:

1. ดูรหัสภาษาต้นฉบับที่กำหนดค่าไว้ในไดเรกทอรีราก `tran.yml` → `tran` → `from`

2. ป้อนไดเร็กทอรี `src/component name`

3. ทำให้ส่วนประกอบเป็นสากล
   - เขียนข้อความที่ใช้โดย `_.js` เป็นรายการใน `src/component name/i18n/source language/js.yml`
   - แก้ไข `ชื่อคอมโพเนนต์/I18N.js` และกำหนดค่าค่าคงที่ของชื่อคีย์รายการ `yml` เช่น `export const KEY = "ymlKey"`
   - `แนะนำและผูกไว้ในชื่อส่วนประกอบ/_.js` (หากมี `js.yml` ก็ไม่จำเป็นถ้าไม่มี):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     นำเข้า i18nUrl, { คีย์ } จาก "./I18N.js"; // กรอบงานจะแทรก i18nUrl โดยอัตโนมัติ ซึ่งเป็นค่าเริ่มต้นในการส่งออก อย่าสร้างมันขึ้นมาเอง

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    พยายามรวมเป็นชุดเพื่อลดการโทร fLang
    `_.js` ของส่วนประกอบไม่ควรมีสตริงที่มีข้อความ

4. ความเป็นสากลของการนำเสนอ
   - ใน `demo/_.pug` ข้อความป้ายกำกับที่ต้องการแปลจะอยู่ในรูปแบบของ `ชื่อคีย์ c-t` (นี่คือป้ายกำกับที่กำหนดเอง บรรทัดแยกต่างหาก) และรายการจะถูกเขียนลงใน `demo/i18n/source language/pug.yml`
   - `demo/_.js` เขียนรายการข้อความต้นฉบับที่ใช้ใน `demo/i18n/source language/js.yml` และวางไว้ใน `demo/I18N.js`
   - แนะนำ `fetchLang` ลงใน `demo/_.js` และโหลด `./I18N.js` (นี่คือ `demo/I18N.js`) อ้างถึงกระบวนการทำให้เป็นสากลของส่วนประกอบด้านบน (เช่น `demo/i18n` ไม่มี `js.yml` ด้านล่าง ข้ามขั้นตอนนี้)
   - `demo/I18N.js` กำหนดค่าคงที่ชื่อคีย์ใน `demo/i18n/source language/js.yml` และส่งออก

5. เขียนคำอธิบายส่วนประกอบใน `ชื่อส่วนประกอบ/i18n/source language/README.md` กระบวนการมีดังนี้
  5.1 เขียนหัวเรื่องหลัก ชื่อควรขึ้นต้นด้วย `#` อธิบายวัตถุประสงค์ของส่วนประกอบในหนึ่งประโยค (อย่าเขียนชื่อส่วนประกอบ) และไม่เพิ่มเครื่องหมายวรรคตอนที่ท้าย
  5.2 ระบุคุณสมบัติหลักขององค์ประกอบ โดยเริ่มจากชื่อหลัก และห้ามเขียนชื่อรอง หากมีลักษณะหลายรายการ ให้ใช้รายการแบบไม่เรียงลำดับ (หากคำอธิบายรายการแบบไม่เรียงลำดับค่อนข้างสั้น ก็จะไม่มีจุดต่อท้าย)
  5.3 เขียน `## โดยใช้การสาธิต` รหัสสาธิตจะเริ่มต้นด้วย ` ```html ` (ในโค้ดสาธิต แท็กองค์ประกอบเดียวจะไม่ถูกปิด) หากมี `js` ให้เขียน `html` ภายใน ให้ใช้แท็ก `<script type="module">`;
  หากเป็น js ล้วนๆ ให้เริ่มต้นด้วย ` ```js ` `js` ในการสาธิต หากคุณต้องการนำเข้า `component/_.js` โปรดใช้ `import xxx จาก "webc.site/component name.js";`
  `js` ใช้การเขียนที่กระชับ เช่น `const` เพื่อประกาศค่าคงที่หลายค่า

ข้อความหลักแนะนำวิธีใช้ส่วนประกอบ พารามิเตอร์อินเทอร์เฟซ การสลับระหว่าง `css` และ `class` ในสถานะที่แตกต่างกัน ฯลฯ
ต่อมาให้อธิบายพารามิเตอร์ ค่าที่ส่งคืน สถานะสไตล์ ฯลฯ (ถ้ามี) และเพิ่มชื่อรอง

ใช้คำให้กระชับและหลีกเลี่ยงคำคุณศัพท์

การเขียนคำโฆษณาควรหลีกเลี่ยงเรื่องไร้สาระและอย่าเขียนคำที่ไม่มีความหมาย (เช่น: สากล, คอนเทนเนอร์ ฯลฯ )

6. เรียกใช้ `./sh/tran.js` ของไดเรกทอรีราก การแปล
