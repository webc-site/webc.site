# ສາທິດ

ອ້າງອີງໃສ່ `src/webc/BoxX/demo` ແລະພັດທະນາສາທິດອົງປະກອບໃນ `src/webc/component name/demo/`

`demo/_.pug`, ແມ່ແບບຍ່ອຍສາທິດ

ມີແຕ່ແທັກອົງປະກອບທີ່ໜ້ອຍທີ່ສຸດເທົ່ານັ້ນ, ບໍ່ຈຳເປັນຕ້ອງມີໜ້າເຕັມ, ແລະການນຳເຂົ້າສະໄຕລ໌ ແລະສະຄຣິບຖືກຫ້າມ. ກອບຈະໃສ່ອັດຕະໂນມັດ (ເຊັ່ນ: `demo/_.styl`, `demo/_.js`).

ຫ້າມຂຽນຊື່ (ຕົວຢ່າງ, ຢ່າຂຽນ xxx demo), ບໍ່ຂຽນຊື່ອົງປະກອບ, ບໍ່ແນະນຳອົງປະກອບ, ພຽງແຕ່ຂຽນສະຖານະຂອງ demo use case (ຖ້າບໍ່ມີສະຖານະ, ຂຽນບໍ່ມີຊື່)

ອົງປະກອບຮາກຂອງ `_.pug`ໃຊ້ `main.demo.Lg` , ເຊິ່ງໄດ້ກໍານົດຮູບແບບໃນ `demo/_.styl` ແລະບໍ່ໄດ້ກໍານົດຮູບແບບຂອງ `main.demo.Lg`ຕົວຂອງມັນເອງ.

ກ່ອງບັນຈຸຂອງອົງປະກອບສະແດງຄວນຈະສອດຄ່ອງກັບ `main.demo.Lg`, ແລະບໍ່ຄວນມີ `margin` ແລະ `padding`ລະຫວ່າງພວກມັນ, ເພາະວ່າ`main.demo`ຕົວຂອງມັນເອງໄດ້ຕັ້ງໄວ້ແລ້ວ `padding`, ແລະການຕັ້ງຄ່າມັນຫວ່າງເປົ່າຈະຖືກເຮັດຊ້ຳ.

`demo/_.styl` ຖ້າ​ຫາກ​ວ່າ​ມີ​ພຽງ​ແຕ່​ຫນຶ່ງ​ອົງ​ປະ​ກອບ​ການ​ສະ​ແດງ​ຜົນ​, container ຂອງ​ມັນ​ຈະ​ບໍ່​ມີ​ກອບ​ພາຍ​ນອກ​, ແລະ​ຕັ້ງ​ພຽງ​ແຕ່ `max-width` (ເນື່ອງ​ຈາກ​ວ່າ​ຕົວ​ບັນ​ຈຸ​ການ​ນໍາ​ສະ​ເຫນີ​ຈະ​ເພີ່ມ​ກອບ​ນອກ​ອັດ​ຕະ​ໂນ​ມັດ​)​. ຖ້າມີຫຼາຍອັນ, ບັນຈຸສາມາດແຍກອອກດ້ວຍກອບນອກ.

ຢ່າຂຽນ `img`ໃນ `demo/_.pug`, ໃຊ້ `demo/_.styl`ເພື່ອຕັ້ງຮູບພື້ນຫຼັງ

ຖ້າອົງປະກອບມີຫຼາຍຮູບແບບ, ກະລຸນາສະແດງພວກມັນຕາມລໍາດັບ (ເຊັ່ນ: ການໂຫຼດ, ຂໍ້ມູນຫວ່າງເປົ່າ, ຂໍ້ມູນ, ລົ້ມເຫລວ, ແລະອື່ນໆ), ຮູບແບບແນວຕັ້ງ flex.

`demo/_.styl`: `stylus`ຮູບແບບຂອງໜ້າສາທິດ, ກະລຸນາປະຕິບັດຕາມລະຫັດສະເພາະ ແລະຄວາມງາມຂອງການອອກແບບ [./styl.md](./styl.md)

ຖ້າ​ຫາກ​ວ່າ​ທ່ານ​ບໍ່​ໄດ້​ຂຽນ `@import '../_.styl'`ການ​ນໍາ​ເຂົ້າ​ອົງ​ປະ​ກອບ​ທີ່​ຈະ​ສະ​ແດງ​ໃຫ້​ເຫັນ​, ມັນ​ຈະ​ໄດ້​ຮັບ​ການ​ສັກ​ຢາ​ອັດ​ຕະ​ໂນ​ມັດ​; ມັນຖືກຫ້າມບໍ່ໃຫ້ໃຊ້ `@import`ໃນ `demo/_.styl` ເພື່ອນໍາເຂົ້າຮູບແບບອົງປະກອບອື່ນໆ. ຖ້າການສາທິດຂຶ້ນກັບອົງປະກອບອື່ນໆ (ເຊັ່ນ: ປຸ່ມ `.Btn`), ກະລຸນານໍາເຂົ້າອົງປະກອບທີ່ຂຶ້ນກັບຜ່ານ `import "../../Btn.js"`ໃນ `demo/_.js`.

`demo/_.js`: ໂທຫາສະຄຣິບ, ຮູບແບບ `export default (root) => { ... }`, ພາຣາມິເຕີ `root`ເປັນເຈົ້າພາບ (`document` ຫຼືອົງປະກອບຮາກຂອງກ່ອງຊາຍ)

`D.createElement`ຖືກຫ້າມ, ໃຊ້ `newEl`ກ່າວໂດຍ [./js.md](./js.md)

`js`ຖືກຫ້າມສໍາລັບການນໍາໃຊ້ໃນ `c-t`, ແລະສໍາລັບການເປັນສາກົນ `fLang`

ຖ້າ `../component name.js` (`../component name.js` ແມ່ນເພື່ອສະແດງອົງປະກອບຂອງຕົວມັນເອງ, ແລະອົງປະກອບທີ່ຂຶ້ນກັບອື່ນໆທີ່ຕ້ອງການໂດຍສາທິດຍັງຈໍາເປັນຕ້ອງໄດ້ນໍາເຂົ້າ), ພຽງແຕ່ `import` ຈໍາເປັນຕ້ອງໄດ້, ແລະບໍ່ມີການນໍາເຂົ້າເນື້ອໃນສະເພາະ, ເຊິ່ງສາມາດຖືກຍົກເວັ້ນອັດຕະໂນມັດ (ກອບຈະປະຕິເສດ)

`pug`ບໍ່​ຈໍາ​ເປັນ​ຕ້ອງ​ນໍາ​ເຂົ້າ `js` ແລະ `styl` , ກອບ​ຈະ​ສັກ​ໃຫ້​ເຂົາ​ເຈົ້າ​ອັດ​ຕະ​ໂນ​ມັດ
