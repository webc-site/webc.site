
  ສໍາລັບອົງປະກອບ `js`, ປະຕິບັດອົງປະກອບໂດຍໃຊ້ `src/webc/componentname.js`.

  ໃຊ້ເສັ້ນທາງທີ່ກ່ຽວຂ້ອງເພື່ອນໍາເຂົ້າອົງປະກອບອື່ນໆ, ເຊັ່ນ: `ນໍາເຂົ້າ "./component name.js"` (ໃນໄຟລ໌ JS ຂອງແຕ່ລະອົງປະກອບ, ນໍາເຂົ້າການເພິ່ງພາອາໄສທີ່ກົງກັນກ່ອນ, ແລະຫຼັງຈາກນັ້ນນໍາເຂົ້າ `ນໍາເຂົ້າ "./component name/_.styl";`).

  `this`ພຽງ​ແຕ່​ກຳ​ນົດ​ໜ້າ​ທີ່​ໂທ​ກັບ​ພື້ນ​ທີ່​ຂອງ​ອົງ​ປະ​ກອບ​ເວັບ​ເທົ່າ​ນັ້ນ, ຂຽນ​ອັນ​ອື່ນ​ເປັນ​ໜ້າ​ທີ່​ອິດ​ສະ​ລະ, ຜ່ານ​ໃນ `this` ແລະ​ພາ​ລາ​ມິ​ເຕີ​ທີ່​ຕ້ອງ​ການ.

  ນອກເໜືອໄປຈາກສ່ວນຕິດຕໍ່ທີ່ເປີດເຜີຍພາຍນອກ, ເພື່ອເຮັດໃຫ້ການເພີ່ມປະສິດທິພາບການສັ່ນສະເທືອນຂອງຕົ້ນໄມ້ ແລະ ການບີບອັດລະຫັດ, ມັນຖືກຫ້າມບໍ່ໃຫ້ຜູກມັດຄຸນລັກສະນະສ່ວນຕົວແບບກຳນົດເອງໃນ `this` (ໃຊ້ `this.$`ສຳລັບຟັງຊັນການໂຫຼດ), ແລະຟັງຊັນສັ່ງສູງສາມາດຖືກໃຊ້ເພື່ອກັບຄືນການປິດເພື່ອຖືຕົວແປລະດັບປານກາງ.

  ຢ່າຂຽນ `constructor`, ຄຸນສົມບັດແມ່ນໄດ້ຮັບໃນ `connectedCallback`.


  ອ້າງເຖິງ [./js.md](./js.md) ເພື່ອນຳໃຊ້ໜ້າທີ່ສາທາລະນະຄືນໃໝ່ພາຍໃຕ້ `src/js/`

  ສຳລັບບັນຈຸ, ໃຫ້ໃຊ້ `slot` ແທນຄຸນສົມບັດ.
  ຖ້າທ່ານຕ້ອງການໂຫຼດຂໍ້ມູນ, ເປີດເຜີຍສ່ວນຕິດຕໍ່ການໂຫຼດໄປຫາຊັ້ນເທິງໃນຮູບແບບຂອງຟັງຊັນການໂທຄືນແບບບໍ່ກົງກັນ, ແລະໃຊ້ຄໍາຄິດຄໍາເຫັນເພື່ອຊີ້ບອກຮູບແບບຂອງມູນຄ່າກັບຄືນ.
  ອົງປະກອບຕ້ອງຫຼີກເວັ້ນການຮົ່ວໄຫຼຂອງຫນ່ວຍຄວາມຈໍາ. `disconnectedCallback` ລວມ​ເຫດການ​ຖອນ​ການ​ຕິດ​ຕັ້ງ​ເຂົ້າ​ເປັນ​ໜຶ່ງ​ຟັງ​ຊັນ `this.$` ແລະ​ຢ່າ​ຂຽນ​ຟັງ​ຊັນ​ຖອນ​ຕິດ​ຕັ້ງ​ຫຼາຍ​ອັນ.
  ຄ່າກັບຄືນຂອງການໂຕ້ຕອບສາມາດເປັນຄ່າດຽວ, array, ຫຼື array ຂອງ array, ບໍ່ແມ່ນວັດຖຸ.
  ຖ້າອົງປະກອບມີຫຼາຍລັດ, ໃຫ້ໃຊ້ຄຳເຫັນຫຼາຍແຖວເພື່ອອະທິບາຍຮູບແບບຂໍ້ມູນຂອງແຕ່ລະລັດຢ່າງຈະແຈ້ງ. (ຢ່ານຳໜ້າແຕ່ລະແຖວຄຳເຫັນດ້ວຍ `*`)
  ສະຫຼັບສະຖານະ, ໂດຍໃຊ້ຄ່າຄົງທີ່ຕົວເລກ (ກຳນົດເປັນ `const.js`).
  ເມື່ອການເອີ້ນຄືນກຳລັງໂຫລດ, ໃຊ້ `const xx = newEl('b');xx.className="Ing"`ເພື່ອສະແດງພາບເຄື່ອນໄຫວ, ແລະແນະນຳການອ້າງອີງຜ່ານ `import "./Ing.js"`ໃນໄຟລ໌ JS ອົງປະກອບ.
