# ຄວາມເປັນສາກົນ

ກໍລະນີ: ອ້າງເຖິງ `src/webc/BoxX` ເພື່ອປະຕິບັດການເປັນສາກົນຂອງອົງປະກອບ ແລະສາທິດ.

ກົດ​ລະ​ບຽບ:
- `yml`ບໍ່​ເພີ່ມ `:`​ໃນ​ຕອນ​ທ້າຍ​ຂອງ​ການ​ປ້ອນ​ຂໍ້​ມູນ, ເອົາ `:`ເຂົ້າ​ໄປ​ໃນ pug template ແລະ js code, ແລະ splice ມັນ.
- ໃຊ້ `|-` ສໍາລັບຂໍ້ຄວາມຫຼາຍແຖວຂອງ `yml` ເພື່ອເອົາພື້ນທີ່ຕໍ່ທ້າຍອອກໄດ້ຢ່າງງ່າຍດາຍ.
- ຖ້າ​ຫາກ​ວ່າ​ບໍ່​ມີ​ການ​ເຂົ້າ​ສູ່​ລະ​ດັບ​ສາ​ກົນ​, ລົບ​ໄຟລ​໌ `I18N.js` (ຢ່າ​ຂຽນ​ຫຍັງ, ບໍ່​ຂຽນ `export`)
- ການເຂົ້າຕ້ອງຢູ່ໃນພາສາດຽວ, ແລະການປະສົມຫຼາຍພາສາແມ່ນຖືກຫ້າມຢ່າງເຂັ້ມງວດ.

ຂະບວນການ:

1. ເບິ່ງລະຫັດພາສາຕົ້ນສະບັບທີ່ກຳນົດຄ່າຢູ່ໃນລະບົບຮາກ `tran.yml` → `tran` → `from`

2. ປ້ອນ `src/webc/ຊື່ອົງປະກອບ` ໄດເຣັກທໍຣີ

3. Internationalize ອົງ​ປະ​ກອບ​
   - ຂຽນຂໍ້ຄວາມທີ່ໃຊ້ໃນ `component name.js` ເປັນການເຂົ້າສູ່ `src/webc/component name/i18n/source language/js.yml`.
   - ແກ້ໄຂ `src/webc/component name/I18N.js` ແລະກຳນົດຄ່າຄົງທີ່ຂອງ `yml`ຊື່ກະແຈເຂົ້າ, ເຊັ່ນ `export const KEY = "ymlKey"`
   - ການແນະນຳ ແລະຜູກມັດໃນ `src/webc/component name.js` (ຖ້າມີ `js.yml`, ຖ້າບໍ່ແມ່ນ, ມັນບໍ່ຈຳເປັນ):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     ນໍາເຂົ້າ i18nUrl, { KEY } ຈາກ "./component name/I18N.js"; // ກອບຈະໃສ່ i18nUrl ອັດຕະໂນມັດ, ການສົ່ງອອກມາດຕະຖານເລີ່ມຕົ້ນ, ບໍ່ໄດ້ສ້າງມັນເອງ

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    ພະຍາຍາມຜູກມັດເປັນຊຸດເພື່ອຫຼຸດການໂທ fLang
    ອົງປະກອບຂອງ `component name.js` ບໍ່ຄວນມີສະຕຣິງທີ່ມີຂໍ້ຄວາມ.

4. ການນໍາສະເຫນີສາກົນ
   - ໃນ `demo/_.pug`, ຂໍ້ຄວາມປ້າຍກຳກັບທີ່ຕ້ອງການແປແມ່ນຖືກຄອບຄອງໃນຮູບແບບຂອງ `ຊື່ລະຫັດ c-t` (ນີ້ແມ່ນປ້າຍກຳກັບແບບກຳນົດເອງ, ເປັນແຖວແຍກຕ່າງຫາກ), ແລະລາຍການຈະຖືກຂຽນໃສ່ໃນ `demo/i18n/source language/pug.yml`
   - `demo/_.js`ຂຽນຂໍ້ຄວາມຕົ້ນສະບັບທີ່ໃຊ້ໃນ `demo/i18n/source language/js.yml` ແລະວາງໄວ້ໃນ `demo/I18N.js`
   - ແນະນໍາ `fetchLang`ເຂົ້າໄປໃນ `demo/_.js` ແລະໂຫຼດ `./I18N.js` (ນີ້ແມ່ນ `demo/I18N.js`), ອ້າງອີງເຖິງຂະບວນການສາກົນຂອງອົງປະກອບຂ້າງເທິງ (ຕົວຢ່າງ: `demo/i18n`ບໍ່ມີ `js.yml`ຂ້າງລຸ່ມນີ້, ໃຫ້ຂ້າມຂັ້ນຕອນນີ້)
   - `demo/I18N.js`ກຳນົດຄ່າຄົງທີ່ຂອງຊື່ຫຼັກໃນ `demo/i18n/source language/js.yml` ແລະສົ່ງອອກ

5. ຂຽນຄໍາອະທິບາຍອົງປະກອບໃນ `src/webc/component name/i18n/source language/README.md`. ຂະບວນການແມ່ນດັ່ງຕໍ່ໄປນີ້
  5.1 ຂຽນຫົວຂໍ້ຕົ້ນຕໍ. ຫົວຂໍ້ຄວນເລີ່ມຕົ້ນດ້ວຍ `#`, ອະທິບາຍຈຸດປະສົງຂອງອົງປະກອບໃນປະໂຫຍກໜຶ່ງ (ຢ່າຂຽນຊື່ອົງປະກອບ), ແລະຢ່າເພີ່ມເຄື່ອງໝາຍວັກຕອນທ້າຍ.
  5.2 ບອກລັກສະນະຕົ້ນຕໍຂອງອົງປະກອບ, ໂດຍເລີ່ມຕົ້ນດ້ວຍຫົວຂໍ້ຕົ້ນຕໍ, ແລະບໍ່ຂຽນຫົວຂໍ້ທີສອງ. ຖ້າມີຫຼາຍລັກສະນະ, ໃຫ້ໃຊ້ບັນຊີລາຍຊື່ທີ່ບໍ່ມີຄໍາສັ່ງ (ຖ້າຄໍາອະທິບາຍລາຍຊື່ທີ່ບໍ່ໄດ້ສັ່ງແມ່ນຂ້ອນຂ້າງສັ້ນ, ຈະບໍ່ມີໄລຍະເວລາໃນຕອນທ້າຍ).
  5.3 ຂຽນ `## ການໃຊ້ demo`, ລະຫັດສາທິດເລີ່ມຕົ້ນດ້ວຍ ` ```html ` (ໃນລະຫັດສາທິດ, ແທັກອົງປະກອບດຽວບໍ່ໄດ້ປິດ), ຖ້າມີ `js`, ຂຽນ `html<b class="CODE`<b>id" > ພາຍໃນ, `js` ບໍ່ມີການຫຍໍ້ໜ້າຢູ່ໃນແຖວທຳອິດຂອງເນື້ອຫາ.
  ຖ້າມັນເປັນ js ບໍລິສຸດ, ເລີ່ມຕົ້ນດ້ວຍ ` ```js `. `js`ໃນ​ການ​ສາ​ທິດ, ຖ້າ​ຫາກ​ວ່າ​ທ່ານ​ຕ້ອງ​ການ​ນໍາ​ເຂົ້າ `component/_.js`, ກະ​ລຸ​ນາ​ໃຊ້ `import xxx ຈາກ "webc.site/component name.js";`. ນີ້ແມ່ນການລວບລວມ `js`, ເຊິ່ງຈະໃສ່ js ແລະ css ໂດຍອັດຕະໂນມັດ.
  `js` ໃຊ້ການຂຽນຫຍໍ້ ເຊັ່ນ `const` ເພື່ອປະກາດຄ່າຄົງທີ່ຫຼາຍອັນ

ຂໍ້​ຄວາມ​ຫຼັກ​ແນະ​ນໍາ​ວິ​ທີ​ການ​ນໍາ​ໃຊ້​ອົງ​ປະ​ກອບ​, ຕົວ​ກໍາ​ນົດ​ການ​ໃນ​ການ​ໂຕ້​ຕອບ​, ສະ​ຫຼັບ​ລະ​ຫວ່າງ `css` ແລະ `class`​ໃນ​ລັດ​ທີ່​ແຕກ​ຕ່າງ​ກັນ​, ແລະ​ອື່ນໆ​.
ຕໍ່ມາອະທິບາຍພາລາມິເຕີ, ຄ່າກັບຄືນ, ສະຖານະຮູບແບບ, ແລະອື່ນໆ (ຖ້າມີ), ແລະເພີ່ມຫົວຂໍ້ທີສອງ.

ໃຊ້ຄໍາສັບທີ່ຊັດເຈນແລະຫຼີກເວັ້ນການ adjective.

Copywriting ຄວນຫຼີກເວັ້ນບໍ່ມີຄວາມຫມາຍແລະບໍ່ຂຽນຄໍາທີ່ບໍ່ມີຄວາມຫມາຍ (ເຊັ່ນ: ທົ່ວໄປ, ບັນຈຸ, ແລະອື່ນໆ).
