# சர்வதேசமயமாக்கல்

வழக்கு: கூறுகள் மற்றும் டெமோவின் சர்வதேசமயமாக்கலைச் செயல்படுத்த `src/BoxX` ஐப் பார்க்கவும்.

விதி:
- `yml` உள்ளீட்டின் முடிவில் `:`ஐச் சேர்க்க வேண்டாம், `:`ஐ pug டெம்ப்ளேட் மற்றும் js குறியீட்டில் வைத்து, அதை பிரிக்கவும்.
- ட்ரெய்லிங் ஸ்பேஸ்களை எளிதாக அகற்ற, `yml`ன் பல வரி உரைக்கு `|-`ஐப் பயன்படுத்தவும்.
- சர்வதேசமயமாக்கப்பட்ட நுழைவு இல்லை என்றால், `I18N.js` கோப்பை அழிக்கவும் (எதையும் எழுத வேண்டாம், `export` எழுத வேண்டாம்)
- உள்ளீடுகள் ஒரே மொழியில் இருக்க வேண்டும், மேலும் பல மொழிகளைக் கலப்பது கண்டிப்பாக தடைசெய்யப்பட்டுள்ளது.

செயல்முறை:

1. மூல கோப்பகத்தில் உள்ளமைக்கப்பட்ட மூல மொழிக் குறியீட்டைப் பார்க்கவும் `tran.yml` → `tran` → `from`

2. `src/component name` கோப்பகத்தை உள்ளிடவும்

3. கூறுகளை சர்வதேசமயமாக்குங்கள்
   - `src/component name/i18n/source language/js.yml` இல் உள்ள நுழைவாக `_.js` பயன்படுத்திய உரையை எழுதவும்.
   - `கூறு பெயர்/I18N.js` ஐ மாற்றி, `yml` நுழைவு விசைப் பெயரின் மாறிலியை உள்ளமைக்கவும், அதாவது `export const KEY = "ymlKey"`
   - `உள்கூறு பெயர்/_.js` இல் அறிமுகப்படுத்தப்பட்டது மற்றும் பிணைக்கப்பட்டுள்ளது (`js.yml` இருந்தால், அது தேவையில்லை என்றால்):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     i18nUrl, { KEY } ஐ "./I18N.js" இலிருந்து இறக்குமதி செய்யவும்; // கட்டமைப்பு தானாகவே i18nUrl ஐ செலுத்தும், இயல்புநிலை ஏற்றுமதி இயல்புநிலை, அதை நீங்களே உருவாக்க வேண்டாம்

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    FLang அழைப்புகளைக் குறைக்க, தொகுதிகளாக இணைக்க முயற்சிக்கவும்
    கூறுகளின் `_.js` உரையைக் கொண்ட சரங்களைக் கொண்டிருக்கக்கூடாது

4. விளக்கக்காட்சிகளின் சர்வதேசமயமாக்கல்
   - `demo/_.pug`ல், மொழிபெயர்க்கப்பட வேண்டிய லேபிள் உரை `c-t முக்கிய பெயர்` (இது தனிப்பயன் லேபிள், ஒரு தனி வரி) வடிவத்தில் ஆக்கிரமிக்கப்பட்டுள்ளது, மேலும் உள்ளீடு `demo/i18n/source language/pug.yml` இல் எழுதப்பட்டுள்ளது.
   - `demo/_.js` `demo/i18n/source language/js.yml` இல் பயன்படுத்தப்பட்ட அசல் உரை உள்ளீட்டை எழுதி `demo/I18N.js` இல் வைக்கவும்
   - `fetchLang`ஐ `demo/_.js`க்கு அறிமுகப்படுத்தி, `./I18N.js`ஐ ஏற்றவும் (இது `demo/I18N.js`), மேலே உள்ள கூறு சர்வதேசமயமாக்கல் செயல்முறையைப் பார்க்கவும் (எடுத்துக்காட்டாக, `demo/i18n`க்கு கீழே `js.yml` இல்லை, இந்தப் படியைத் தவிர்க்கவும்)
   - `demo/I18N.js` முக்கிய பெயர் மாறிலிகளை `demo/i18n/source language/js.yml` இல் உள்ளமைத்து ஏற்றுமதி செய்யவும்

5. கூறுகளின் விளக்கத்தை `கூறு பெயர்/i18n/source language/README.md` இல் எழுதவும். செயல்முறை பின்வருமாறு
  5.1 முக்கிய தலைப்பை எழுதவும். தலைப்பு `#` என்று தொடங்க வேண்டும், ஒரு வாக்கியத்தில் கூறுகளின் நோக்கத்தை விவரிக்க வேண்டும் (கூறு பெயரை எழுத வேண்டாம்), இறுதியில் நிறுத்தற்குறிகளை சேர்க்க வேண்டாம்.
  5.2 முக்கிய தலைப்பில் தொடங்கி, கூறுகளின் முக்கிய அம்சங்களை பட்டியலிடுங்கள் மற்றும் இரண்டாம் நிலை தலைப்புகளை எழுத வேண்டாம். பல குணாதிசயங்கள் இருந்தால், வரிசைப்படுத்தப்படாத பட்டியலைப் பயன்படுத்தவும் (வரிசைப்படுத்தப்படாத பட்டியல் விவரம் ஒப்பீட்டளவில் சிறியதாக இருந்தால், முடிவில் காலம் இருக்காது).
  5.3 `## டெமோவைப் பயன்படுத்தி` எழுதவும், டெமோ குறியீடு ` ```html ` எனத் தொடங்கும் (டெமோ குறியீட்டில், ஒற்றை உறுப்பு குறிச்சொல் மூடப்படவில்லை), `js` இருந்தால், உள்நாட்டில் CO DE ஐப் பயன்படுத்தவும் `<script type="module">` குறிச்சொல்;
  இது தூய js எனில், ` ```js ` என்று தொடங்கவும். `js` ஆர்ப்பாட்டத்தில், நீங்கள் `component/_.js`ஐ இறக்குமதி செய்ய வேண்டும் என்றால், "webc.site/component/_.js";` இலிருந்து `இறக்குமதி xxx ஐப் பயன்படுத்தவும்.
  `js` பல மாறிலிகளை அறிவிக்க `const` போன்ற சுருக்கமான எழுத்தைப் பயன்படுத்தவும்

கூறு, இடைமுக அளவுருக்கள், வெவ்வேறு நிலைகளில் `css` மற்றும் `class` ஆகியவற்றுக்கு இடையே மாறுதல் போன்றவற்றை முக்கிய உரை அறிமுகப்படுத்துகிறது.
பின்னர் அளவுருக்கள், திரும்ப மதிப்பு, நடை நிலை போன்றவற்றை விவரிக்கவும் (ஏதேனும் இருந்தால்), மற்றும் இரண்டாம் தலைப்பைச் சேர்க்கவும்.

சொற்களை சுருக்கமாகப் பயன்படுத்தவும், உரிச்சொற்களைத் தவிர்க்கவும்.

நகல் எழுதுதல் முட்டாள்தனத்தைத் தவிர்க்க வேண்டும் மற்றும் அர்த்தமற்ற வார்த்தைகளை எழுதக்கூடாது (அதாவது: உலகளாவிய, கொள்கலன், முதலியன).

6. ரூட் டைரக்டரியின் `./sh/tran.js`ஐ இயக்கவும்
