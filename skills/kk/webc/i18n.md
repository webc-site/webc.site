# интернационалдандыру

Жағдай: Құрамдастарды интернационалдандыруды және демонстрацияны жүзеге асыру үшін `src/webc/BoxX` бөлімін қараңыз.

ереже:
- `yml` Жазбаның соңына `:` қоспаңыз, pug үлгісіне және JS кодына `:` енгізіп, оны біріктіріңіз.
- Артқы бос орындарды оңай жою үшін `yml` көп жолды мәтін үшін `|-` пайдаланыңыз.
- Егер халықаралық жазба болмаса, `I18N.js` файлын тазалаңыз (ештеңе жазбаңыз, `export` жазбаңыз)
- Жазбалар бір тілде болуы керек және бірнеше тілді араластыруға қатаң тыйым салынады.

процесс:

1. Түбірлік каталогта конфигурацияланған бастапқы тіл кодын қараңыз `tran.yml` → `tran` → `from`

2. `src/webc/component name` каталогын енгізіңіз

3. Компоненттерді интернационалдандыру
   - `компонент атауы.js` ішінде пайдаланылған мәтінді `src/webc/component name/i18n/source language/js.yml` ішіне жазба ретінде жазыңыз.
   - `src/webc/component name/I18N.js` өзгертіңіз және `export const KEY = "ymlKey"` сияқты `yml` енгізу кілті атауының тұрақты мәнін конфигурациялаңыз
   - `src/webc/component name.js` ішінде кіріспе және байланыстыру (егер `js.yml` бар болса, жоқ болса, қажет емес):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     "./component name/I18N.js" ішінен i18nUrl, { KEY } импорттау; // Рамка автоматты түрде i18nUrl енгізеді, әдепкі экспорттау, оны өзіңіз жасамаңыз

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    FLang қоңырауларын азайту үшін топтамалармен байланыстырып көріңіз
    Компоненттің `компонент атауы.js` құрамында мәтіні бар жол болмауы керек

4. Презентацияларды интернационалдандыру
   - `demo/_.pug` ішінде аударылуы қажет белгі мәтіні `c-t кілт атауы` (бұл реттелетін белгі, бөлек жол) түрінде орналасқан және жазба `demo/i18n/source language/pug.yml` ішіне жазылған.
   - `demo/_.js` `demo/i18n/source language/js.yml` ішінде пайдаланылған түпнұсқа мәтін жазбасын жазыңыз және оны `demo/I18N.js` ішіне орналастырыңыз.
   - `fetchLang` мәнін `demo/_.js` ішіне енгізіп, `./I18N.js` жүктеңіз (бұл `demo/I18N.js`), жоғарыдағы құрамдас интернационалдандыру процесін қараңыз (мысалы, `demo/i18n` төменде `js.yml` жоқ, бұл қадамды өткізіп жіберіңіз)
   - `demo/I18N.js` `demo/i18n/source language/js.yml` ішінде кілт атауының тұрақты мәндерін конфигурациялаңыз және экспорттаңыз

5. Компонент сипаттамасын `src/webc/component name/i18n/source language/README.md` ішінде жазыңыз. Процесс келесідей
  5.1 Негізгі тақырыпты жазыңыз. Тақырып `#`-ден басталып, компоненттің мақсатын бір сөйлеммен сипаттау керек (компонент атын жазбаңыз), соңында тыныс белгілерін қоймаңыз.
  5.2 Негізгі тақырыптан бастап компоненттің негізгі белгілерін көрсетіңіз және қосалқы тақырыптарды жазбаңыз. Бірнеше сипаттамалар болса, ретсіз тізімді пайдаланыңыз (егер реттелмеген тізім сипаттамасы салыстырмалы түрде қысқа болса, соңында нүкте болмайды).
  5.3 Жазыңыз `## Демонстрацияны пайдаланып`, демо-код ` ```html `-ден басталады (демо кодта жалғыз элемент тегі жабылмайды), `js` бар болса, `DE<b>іштей `DECODE"> class="html"<b> пайдаланыңыз. id="3">` Тег, `js` Мазмұнның бірінші жолында шегініс болмауы керек.
  Егер ол таза js болса, ` ```js ` деп бастаңыз. `js` Көрсетілімде `component/_.js` импорттау қажет болса, `import xxx файлын "webc.site/component name.js";` пайдаланыңыз. Бұл құрастырылған `js`, ол автоматты түрде js және css енгізеді.
  `js` Бірнеше тұрақты мәндерді жариялау үшін `const` сияқты қысқаша жазуды пайдаланыңыз

Негізгі мәтін компонентті қалай пайдалану керектігін, интерфейс параметрлерін, әртүрлі күйлердегі `css` және `class` арасында ауысуды және т.б.
Кейіннен параметрлерді, қайтарылатын мәнді, мәнер күйін және т.б. сипаттаңыз (бар болса) және қосымша тақырып қосыңыз.

Сөздерді қысқаша қолданып, сын есімдерден аулақ болыңыз.

Копирайтинг бос сөзден аулақ болу керек және мағынасыз сөздерді жазбау керек (мысалы: әмбебап, контейнер және т.б.).

6. Түбірлік каталогтың `./sh/tran.js` іске қосыңыз, аударма
