# rahvusvahelistumine

Juhtum: komponentide rahvusvahelistumise ja demo rakendamiseks vaadake `src/webc/BoxX`.

reegel:
- `yml` Ärge lisage `:` kirje lõppu, sisestage `:` pug-malli ja js-koodi ning ühendage see
- Kasutage `|-` mitmerealise teksti `yml` jaoks, et hõlpsasti eemaldada lõpus olevad tühikud.
- Kui rahvusvahelist kirjet pole, tühjendage fail `I18N.js` (ärge kirjutage midagi, ärge kirjutage `export`)
- Kanded peavad olema ühes keeles ja mitme keele segamine on rangelt keelatud.

protsess:

1. Vaadake juurkataloogis `tran.yml` → `tran` → `from` seadistatud lähtekeele koodi

2. Sisestage kataloog `src/webc/component name`

3. Komponendid rahvusvahelistuvad
   - Kirjutage failis `komponendi nimi.js` kasutatav tekst kirjena `src/webc/component name/i18n/source language/js.yml`.
   - Muutke `src/webc/component name/I18N.js` ja konfigureerige `yml` sisestusvõtme nime konstant, nt `export const KEY = "ymlKey"`
   - Sissejuhatus ja sidumine failis `src/webc/component name.js` (kui on `js.yml`, kui ei, siis pole see vajalik):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     importida i18nUrl, { KEY } from "./component name/I18N.js"; // Raamistik sisestab automaatselt ekspordi vaikeväärtuse i18nUrl, ärge looge seda ise

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Flangi kõnede vähendamiseks proovige siduda partiidena
    Komponendi `komponendi nimi.js` ei tohiks sisaldada teksti sisaldavat stringi

4. Ettekannete rahvusvahelistumine
   - Lahtris `demo/_.pug` on tõlkimist vajav sildi tekst hõivatud kujul `c-t võtme nimi` (see on kohandatud silt, eraldi rida) ja kirje kirjutatakse faili `demo/i18n/source language/pug.yml`
   - `demo/_.js` Kirjutage failis `demo/i18n/source language/js.yml` kasutatud originaalteksti kirje ja asetage see kausta `demo/I18N.js`
   - Sisestage `fetchLang` jaotisesse `demo/_.js` ja laadige `./I18N.js` (see on `demo/I18N.js`), vaadake ülaltoodud komponentide rahvusvahelistumise protsessi (näiteks `demo/i18n` ei sisalda allolevat `js.yml`, jätke see samm vahele)
   - `demo/I18N.js` Seadistage võtmenimekonstandid failis `demo/i18n/source language/js.yml` ja eksportige

5. Kirjutage komponendi kirjeldus faili `src/webc/component name/i18n/source language/README.md`. Protsess on järgmine
  5.1 Kirjutage põhipealkiri. Pealkiri peaks algama tähega `#`, kirjeldama ühe lausega komponendi eesmärki (ära kirjuta komponendi nime) ja ära lisa lõppu kirjavahemärke.
  5.2 Loetlege komponendi põhiomadused, alustades põhipealkirjast, ja ärge kirjutage kõrvalpealkirju. Kui tunnuseid on mitu, kasutage järjestamata loendit (kui järjestamata loendi kirjeldus on suhteliselt lühike, ei ole lõpus punkti).
  5.3 Kirjutage `## Demot kasutades` algab demokood ` ```html ` (demokoodis ei ole üheelemendiline märgend suletud), `js` olemasolul kirjutage `html="CODE"> Tag = ="CODE"> ><b" class="CODE"> sisemiselt, kasutage `js` Esimesel sisureal ei ole taanet.
  Kui see on puhas js, alustage koodiga ` ```js `. `js` Kui peate demonstratsioonis importima `component/_.js`, kasutage käsku `import xxx saidilt "webc.site/component name.js";`. See on kompileeritud `js`, mis sisestab automaatselt js ja css.
  `js` Kasutage mitme konstandi deklareerimiseks lühikest kirja, näiteks `const`

Põhitekst tutvustab komponendi kasutamist, liidese parameetreid, erinevates olekutes `css` ja `class` vaheldumisi jne.
Seejärel kirjeldage parameetreid, tagastusväärtust, stiili olekut jne (kui on) ja lisage teisene pealkiri.

Kasutage sõnu lühidalt ja vältige omadussõnu.

Copywriting peaks vältima jamasid ja mitte kirjutama mõttetuid sõnu (näiteks: universaalne, konteiner jne).

6. Käivitage juurkataloogi tõlge `./sh/tran.js`
