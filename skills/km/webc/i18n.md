# អន្តរជាតិ

ករណី៖ យោងទៅ `src/webc/BoxX` ដើម្បីអនុវត្តអន្តរភាវូបនីយកម្មនៃសមាសធាតុ និងការបង្ហាញ។

ច្បាប់៖
- `yml`កុំបន្ថែម `:`នៅចុងបញ្ចប់នៃធាតុ សូមដាក់ `:`ទៅក្នុងគំរូ pug និងកូដ js ហើយបំបែកវា
- ប្រើ `|-` សម្រាប់​អត្ថបទ​ច្រើន​បន្ទាត់​នៃ `yml` ដើម្បី​ងាយស្រួល​លុប​ដកឃ្លា​ពីក្រោយ។
- ប្រសិនបើ​មិន​មាន​ការ​បញ្ចូល​លក្ខណៈ​អន្តរជាតិ​ទេ សូម​សម្អាត​ឯកសារ `I18N.js` (កុំ​សរសេរ​អ្វី​ទាំងអស់ កុំ​សរសេរ `export`)
- ការបញ្ចូលត្រូវតែជាភាសាតែមួយ ហើយការលាយភាសាច្រើនត្រូវបានហាមឃាត់យ៉ាងតឹងរ៉ឹង។

ដំណើរការ៖

1. មើលកូដភាសាប្រភពដែលបានកំណត់រចនាសម្ព័ន្ធនៅក្នុងថត root `tran.yml` → `tran` → `from`

2. បញ្ចូលថត `src/webc/component name`

3. ធ្វើអន្តរកម្មសមាសធាតុ
   - សរសេរអត្ថបទដែលប្រើក្នុង `component name.js` ជាធាតុចូលទៅក្នុង `src/webc/component name/i18n/source language/js.yml`។
   - កែប្រែ `src/webc/component name/I18N.js` ហើយ​កំណត់​រចនាសម្ព័ន្ធ​ថេរ​នៃ `yml`ឈ្មោះ​សោចូល ដូចជា `export const KEY = "ymlKey"`
   - សេចក្តីផ្តើម និងការចងភ្ជាប់នៅក្នុង `src/webc/component name.js` (ប្រសិនបើមាន `js.yml` បើមិនដូច្នោះទេ វាមិនចាំបាច់ទេ៖
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     នាំចូល i18nUrl, { KEY } ពី "./component name/I18N.js"; // ក្របខណ្ឌនឹងចាក់ដោយស្វ័យប្រវត្តិ i18nUrl ដែលជាលំនាំដើមនាំចេញ មិនត្រូវបង្កើតវាដោយខ្លួនឯងទេ។

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    ព្យាយាមចងជាបាច់ ដើម្បីកាត់បន្ថយការហៅ fLang
    `សមាសភាគ name.js` របស់សមាសភាគមិនគួរមានខ្សែអក្សរដែលមានអត្ថបទទេ

4. ការធ្វើបទបង្ហាញអន្តរជាតិ
   - នៅក្នុង `demo/_.pug` អត្ថបទស្លាកដែលត្រូវបកប្រែត្រូវបានកាន់កាប់ជាទម្រង់ `ឈ្មោះគន្លឹះ c-t` (នេះជាស្លាកផ្ទាល់ខ្លួន បន្ទាត់ដាច់ដោយឡែក) ហើយធាតុត្រូវបានសរសេរទៅក្នុង `demo/i18n/source language/pug.yml`
   - `demo/_.js` សរសេរធាតុអត្ថបទដើមដែលប្រើក្នុង `demo/i18n/source language/js.yml` ហើយដាក់វាក្នុង `demo/I18N.js`
   - ណែនាំ `fetchLang`ទៅក្នុង `demo/_.js` ហើយផ្ទុក `./I18N.js` (នេះគឺជា `demo/I18N.js`) យោងទៅដំណើរការអន្តរកម្មនៃសមាសភាគខាងលើ (ឧទាហរណ៍ `demo/i18n`មិនមាន `js.yml`ខាងក្រោម សូមរំលងជំហាននេះ)
   - `demo/I18N.js`កំណត់​រចនាសម្ព័ន្ធ​ឈ្មោះ​គន្លឹះ​ក្នុង `demo/i18n/source language/js.yml` និង​នាំចេញ

5. សរសេរការពិពណ៌នាសមាសភាគនៅក្នុង `src/webc/component name/i18n/source language/README.md` ។ ដំណើរការមានដូចខាងក្រោម
  5.1 សរសេរចំណងជើងសំខាន់។ ចំណងជើងគួរតែចាប់ផ្តើមដោយ `#` ពិពណ៌នាអំពីគោលបំណងនៃសមាសភាគក្នុងប្រយោគមួយ (កុំសរសេរឈ្មោះសមាសភាគ) ហើយកុំបន្ថែមវណ្ណយុត្តិនៅចុងបញ្ចប់។
  5.2 រាយបញ្ជីលក្ខណៈសំខាន់ៗនៃសមាសភាគ ដោយចាប់ផ្តើមពីចំណងជើងមេ ហើយកុំសរសេរចំណងជើងបន្ទាប់បន្សំ។ ប្រសិនបើមានលក្ខណៈច្រើន សូមប្រើបញ្ជីដែលមិនបានបញ្ជាទិញ (ប្រសិនបើការពិពណ៌នាបញ្ជីដែលមិនមានលំដាប់លំដោយគឺខ្លី វានឹងមិនមានរយៈពេលនៅចុងបញ្ចប់ទេ)។
  5.3 សរសេរ `## ដោយប្រើការបង្ហាញ` កូដសាកល្បងចាប់ផ្តើមដោយ ` ```html ` (នៅក្នុងកូដសាកល្បង ស្លាកធាតុតែមួយមិនត្រូវបានបិទទេ) ប្រសិនបើមាន `js` សរសេរ `html` Tag ប្រើប្រាស់ខាងក្នុង។ `js` កុំមានការចូលបន្ទាត់ក្នុងជួរទីមួយនៃមាតិកា។
  ប្រសិនបើវាជា js សុទ្ធ សូមចាប់ផ្តើមជាមួយ ` ```js `។ `js`នៅក្នុងការបង្ហាញ ប្រសិនបើអ្នកត្រូវការនាំចូល`component/_.js` សូមប្រើ`នាំចូល xxx ពី "webc.site/component name.js";`។ នេះគឺជាការចងក្រង `js` ដែលនឹងចាក់បញ្ចូល js និង css ដោយស្វ័យប្រវត្តិ។
  `js` ប្រើការសរសេរសង្ខេប ដូចជា `const` ដើម្បីប្រកាសចំនួនថេរច្រើន

អត្ថបទចម្បងណែនាំពីរបៀបប្រើសមាសភាគ ប៉ារ៉ាម៉ែត្រចំណុចប្រទាក់ ការផ្លាស់ប្តូររវាង `css` និង `class` នៅក្នុងស្ថានភាពផ្សេងគ្នា។ល។
បន្ទាប់មកពណ៌នាអំពីប៉ារ៉ាម៉ែត្រ តម្លៃត្រឡប់ ស្ថានភាពរចនាប័ទ្ម។ល។ (ប្រសិនបើមាន) ហើយបន្ថែមចំណងជើងបន្ទាប់បន្សំ។

ប្រើពាក្យដោយសង្ខេប និងជៀសវាងគុណនាម។

ការ​សរសេរ​ចម្លង​គួរ​ជៀស​វាង​មិន​សមហេតុសមផល ហើយ​កុំ​សរសេរ​ពាក្យ​គ្មាន​ន័យ (ដូចជា៖ សកល កុងតឺន័រ ជាដើម)។

6. ដំណើរការ `./sh/tran.js` នៃ​ថត​ឫស ការបកប្រែ
