# আন্তর্জাতিকীকরণ

কেস: উপাদান এবং ডেমোর আন্তর্জাতিকীকরণ বাস্তবায়ন করতে `src/BoxX` পড়ুন।

নিয়ম:
- `yml` এন্ট্রির শেষে `:` যোগ করবেন না, pug টেমপ্লেট এবং js কোডে `:` রাখুন এবং এটিকে বিভক্ত করুন
- ট্রেলিং স্পেস সহজে সরাতে `yml` এর বহু-লাইন পাঠ্যের জন্য `|-` ব্যবহার করুন৷
- যদি কোনও আন্তর্জাতিক এন্ট্রি না থাকে তবে `I18N.js` ফাইলটি সাফ করুন (কিছু লিখবেন না, `export` লিখবেন না)
- এন্ট্রি একটি একক ভাষায় হতে হবে, এবং একাধিক ভাষা মেশানো কঠোরভাবে নিষিদ্ধ।

প্রক্রিয়া:

1. রুট ডিরেক্টরিতে কনফিগার করা উৎস ভাষার কোড দেখুন `tran.yml` → `tran` → `from`

2. `src/component name` ডিরেক্টরি লিখুন

3. উপাদান আন্তর্জাতিকীকরণ
   - `src/component name/i18n/source language/js.yml`-এ এন্ট্রি হিসাবে `_.js` দ্বারা ব্যবহৃত পাঠ্যটি লিখুন।
   - `কম্পোনেন্টের নাম/I18N.js` পরিবর্তন করুন এবং `yml` এন্ট্রি কী নামের ধ্রুবক কনফিগার করুন, যেমন `export const KEY = "ymlKey"`
   - `প্রবর্তিত এবং কম্পোনেন্ট নামে আবদ্ধ/_.js` (যদি `js.yml` থাকে, না থাকলে এটি প্রয়োজনীয় নয়):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     "./I18N.js" থেকে i18nUrl, { KEY } আমদানি করুন; // ফ্রেমওয়ার্ক স্বয়ংক্রিয়ভাবে i18nUrl ইনজেক্ট করবে, ডিফল্ট এক্সপোর্ট ডিফল্ট, এটি নিজে তৈরি করবেন না

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    fLang কল কমাতে ব্যাচে বাঁধার চেষ্টা করুন
    কম্পোনেন্টের `_.js` টেক্সট ধারণকারী স্ট্রিং থাকা উচিত নয়

4. উপস্থাপনা আন্তর্জাতিকীকরণ
   - `demo/_.pug`-এ, যে লেবেল পাঠ্যটিকে অনুবাদ করতে হবে সেটি `c-t কী নামের` আকারে দখল করা হয়েছে (এটি একটি কাস্টম লেবেল, একটি পৃথক লাইন), এবং এন্ট্রিটি `demo/i18n/source language/pug.yml`-এ লেখা আছে
   - `demo/_.js` `demo/i18n/source language/js.yml`-এ ব্যবহৃত মূল পাঠ্য এন্ট্রি লিখুন এবং এটি `demo/I18N.js` এ রাখুন
   - `fetchLang`কে `demo/_.js` এ পরিচয় করিয়ে দিন এবং `./I18N.js` লোড করুন (এটি `demo/I18N.js`), উপরের উপাদান আন্তর্জাতিকীকরণ প্রক্রিয়াটি দেখুন (উদাহরণস্বরূপ, `demo/i18n` এর নিচে `js.yml` নেই, এই ধাপটি এড়িয়ে যান)
   - `demo/I18N.js` `demo/i18n/source language/js.yml`-এ কী নামের ধ্রুবক কনফিগার করুন এবং রপ্তানি করুন

5. `কম্পোনেন্টের নাম/i18n/source language/README.md`-এ কম্পোনেন্টের বিবরণ লিখুন। প্রক্রিয়াটি নিম্নরূপ
  5.1 মূল শিরোনাম লিখুন। শিরোনামটি `#` দিয়ে শুরু হওয়া উচিত, একটি বাক্যে উপাদানটির উদ্দেশ্য বর্ণনা করুন (উপাদানের নাম লিখবেন না), এবং শেষে বিরামচিহ্ন যোগ করবেন না।
  5.2 মূল শিরোনাম দিয়ে শুরু করে উপাদানটির প্রধান বৈশিষ্ট্যগুলি তালিকাভুক্ত করুন এবং গৌণ শিরোনাম লিখবেন না। যদি একাধিক বৈশিষ্ট্য থাকে, তাহলে একটি ক্রমবিন্যস্ত তালিকা ব্যবহার করুন (যদি অপরিবর্তিত তালিকার বিবরণ তুলনামূলকভাবে সংক্ষিপ্ত হয়, তাহলে শেষে কোনো সময় থাকবে না)।
  5.3 `## ডেমো ব্যবহার করে` লিখুন, ডেমো কোডটি ` ```html ` দিয়ে শুরু হয় (ডেমো কোডে, একক উপাদান ট্যাগ বন্ধ থাকে না), যদি `js` থাকে, তাহলে লিখুন `id="`<b><b><b><b><b><b> class="`<b><b><b><b><b><b>` class="`<b><b>`<b><b>``````` ক্লাস ব্যবহার করুন।" ট্যাগ;
  যদি এটি খাঁটি js হয়, তাহলে ` ```js ` দিয়ে শুরু করুন৷ `js` প্রদর্শনে, আপনার যদি `component/_.js` আমদানি করতে হয়, অনুগ্রহ করে `"webc.site/component name.js" থেকে xxx আমদানি করুন;` ব্যবহার করুন৷
  `js` সংক্ষিপ্ত লেখা ব্যবহার করুন, যেমন `const` একাধিক ধ্রুবক ঘোষণা করতে

মূল পাঠ্যটি কীভাবে কম্পোনেন্ট ব্যবহার করতে হয়, ইন্টারফেস প্যারামিটার, বিভিন্ন রাজ্যে `css` এবং `class` এর মধ্যে স্যুইচিং ইত্যাদির পরিচয় দেয়।
পরবর্তীতে পরামিতি, রিটার্ন মান, স্টাইল স্ট্যাটাস ইত্যাদি বর্ণনা করুন (যদি থাকে), এবং একটি গৌণ শিরোনাম যোগ করুন।

সংক্ষিপ্তভাবে শব্দ ব্যবহার করুন এবং বিশেষণ এড়িয়ে চলুন।

কপিরাইটিং এড়িয়ে চলা উচিত অর্থহীন শব্দ (যেমন: সর্বজনীন, ধারক, ইত্যাদি)।

6. রুট ডিরেক্টরি, অনুবাদের `./sh/tran.js` চালান
