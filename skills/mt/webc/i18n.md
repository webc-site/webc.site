# internazzjonalizzazzjoni

Każ: Irreferi għal `src/webc/BoxX` biex timplimenta l-internazzjonalizzazzjoni tal-komponenti u d-demo.

regola:
- `yml` Iżżidx `:` fl-aħħar tad-dħul, poġġi `:` fil-mudell pug u l-kodiċi js, u ssieħebha
- Uża `|-` għal test b'ħafna linji ta' `yml` biex tneħħi faċilment l-ispazji ta' wara.
- Jekk ma jkun hemm l-ebda dħul internazzjonalizzat, neħħi l-fajl `I18N.js` (tikteb xejn, tikteb `export`)
- L-entrati għandhom ikunu f'lingwa waħda, u t-taħlit ta' lingwi multipli huwa strettament ipprojbit.

proċess:

1. Ara l-kodiċi tal-lingwa tas-sors konfigurat fid-direttorju tal-għeruq `tran.yml` → `tran` → `from`

2. Daħħal id-direttorju `src/webc/component name`

3. Internationalize komponenti
   - Ikteb it-test użat f'`komponent name.js` bħala dħul f'`src/webc/component name/i18n/source language/js.yml`.
   - Immodifika `src/webc/component name/I18N.js` u kkonfigura l-kostanti ta' `yml` isem taċ-ċavetta tad-dħul, bħal `export const KEY = "ymlKey"`
   - Introduzzjoni u rbit f'`src/webc/component name.js` (jekk ikun hemm `js.yml`, jekk le, mhux meħtieġ):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     importazzjoni i18nUrl, { KEY } minn "./isem tal-komponent/I18N.js"; // Il-qafas awtomatikament jinjetta i18nUrl, id-default default tal-esportazzjoni, ma toħloqx lilek innifsek

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Ipprova torbot f'lottijiet biex tnaqqas is-sejħiet flang
    `isem tal-komponent.js` tal-komponent m'għandux ikollu string li jkun fiha test

4. Internazzjonalizzazzjoni ta' preżentazzjonijiet
   - F'`demo/_.pug`, it-test tat-tikketta li jeħtieġ li jiġi tradott huwa okkupat fil-forma ta' `isem taċ-ċavetta c-t` (din hija tikketta apposta, linja separata), u l-entrata hija miktuba f'`demo/i18n/source language/pug.yml`
   - `demo/_.js` Ikteb l-entrata tat-test oriġinali użata f'`demo/i18n/source language/js.yml` u poġġiha f'`demo/I18N.js`
   - Introduċi `fetchLang` f'`demo/_.js` u tagħbija `./I18N.js` (dan huwa `demo/I18N.js`), irreferi għall-proċess ta' internazzjonalizzazzjoni tal-komponenti hawn fuq (per eżempju, `demo/i18n` m'għandux `js.yml` hawn taħt, aqbeż dan il-pass)
   - `demo/I18N.js` Ikkonfigura l-kostanti tal-ismijiet ewlenin f'`demo/i18n/source language/js.yml` u esporta

5. Ikteb id-deskrizzjoni tal-komponent f'`src/webc/isem tal-komponent/i18n/source language/README.md`. Il-proċess huwa kif ġej
  5.1 Ikteb it-titlu prinċipali. It-titlu għandu jibda b'`#`, jiddeskrivi l-iskop tal-komponent f'sentenza waħda (tiktebx l-isem tal-komponent), u ma żżidx punteġġjatura fl-aħħar.
  5.2 Elenka l-karatteristiċi ewlenin tal-komponent, billi tibda bit-titlu prinċipali, u tiktebx titoli sekondarji. Jekk ikun hemm karatteristiċi multipli, uża lista mhux ordnata (jekk id-deskrizzjoni tal-lista mhux ordnata hija relattivament qasira, ma jkun hemm l-ebda perjodu fl-aħħar).
  5.3 Ikteb `## Bl-użu tad-demo`, il-kodiċi demo jibda b'` ```html ` (fil-kodiċi demo, it-tikketta tal-element wieħed mhix magħluqa), jekk ikun hemm `js`, ikteb `id="CODE">, ikteb `class="CODE"> `internment ` <b>` class="CODE"> Tag, `js` M'għandekx indentazzjoni fl-ewwel linja tal-kontenut.
  Jekk huwa js pur, ibda b'` ```js `. `js` Fid-dimostrazzjoni, jekk għandek bżonn timporta `component/_.js`, jekk jogħġbok uża `import xxx minn "webc.site/component name.js";`. Dan huwa l-`js` ikkumpilat, li awtomatikament jinjetta js u css.
  `js` Uża kitba konċiża, bħal `const` biex tiddikjara kostanti multipli

It-test ewlieni jintroduċi kif tuża l-komponent, il-parametri tal-interface, il-bidla bejn `css` u `class` fi stati differenti, eċċ.
Sussegwentement iddeskrivi l-parametri, il-valur tar-ritorn, l-istatus tal-istil, eċċ. (jekk ikun hemm), u żid titolu sekondarju.

Uża l-kliem b’mod konċiż u evita l-aġġettivi.

Il-Kitba għandha tevita bla sens u ma tiktebx kliem bla sens (bħal: universali, kontenitur, eċċ.).
