# デモ

`src/webc/BoxX/demo`を参照し、`src/webc/component name/demo/` にあるコンポーネントのデモを開発します。

`demo/_.pug`、デモ サブテンプレート

最小限のコンポーネント タグのみが含まれており、完全なページは必要なく、スタイルとスクリプトのインポートは禁止されています。フレームワークは自動的に挿入します (`demo/_.styl`、`demo/_.js` など)。

タイトルは書かないでください（例えば、xxxデモは書かないでください）、コンポーネント名は書かないでください、コンポーネントの紹介は書かないでください、デモのユースケースのステータスのみを書いてください（ステータスがない場合は、タイトルを書かないでください）

`_.pug` のルート要素は `main.demo.Lg` を使用します。これは `demo/_.styl` でスタイルを定義しており、 `main.demo.Lg` 自体のスタイルは定義しません。

表示コンポーネントのコンテナは `main.demo.Lg` と整列する必要があり、その間に `margin` と `padding` があってはなりません。これは、`main.demo` 自体がすでに `padding` を設定しており、空白の設定が繰り返されるためです。

`demo/_.styl` 表示コンポーネントが 1 つだけの場合、そのコンテナには外枠が設定されず、`max-width` のみが設定されます (プレゼンテーション コンテナが自動的に外枠を追加するため)。複数ある場合は外枠でコンテナを区切ることも可能です。

`demo/_.pug` には `img` を書かず、 `demo/_.styl` を使用して背景画像を設定してください

コンポーネントに複数のフォームがある場合は、それらを順番に表示してください (読み込み、空のデータ、データ、失敗など)。フレックス垂直レイアウト

`demo/_.styl`: デモ ページの `stylus` スタ​​イル。[./styl.md](./styl.md) のコード仕様とデザイン美学に従ってください。

デモするコンポーネントをインポートするために`@import '../_.styl'`を記述しない場合、コンポーネントは自動的に挿入されます。 `demo/_.styl` で `@import` を使用して他のコンポーネント スタイルをインポートすることは禁止されています。デモが他のコンポーネント (ボタン `.Btn` など) に依存している場合は、`demo/_.js` の `import "../../Btn.js"` を通じて依存コンポーネントをインポートしてください。

`demo/_.js`: スクリプトを呼び出します。形式は `export default (root) => { ... }`、パラメータ `root` はホスト (`document` またはサンドボックス コンテナのルート要素) です。

`D.createElement` は禁止されています。[./js.md](./js.md) で言及されている `newEl` を使用してください。

`js` は `c-t` および国際化 `fLang` での使用が禁止されています。

`../component name.js` (`../component name.js` はコンポーネント自体を表示するためのもので、デモに必要な他の依存コンポーネントは引き続きインポートする必要があります) の場合、`import` のみが必要であり、特定のコンテンツはインポートされないため、省略できます (フレームワークが自動的に挿入します)。

`pug` `js` と `styl` をインポートする必要はありません。フレームワークによって自動的に挿入されます。
