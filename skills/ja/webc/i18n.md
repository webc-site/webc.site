# 国際化

ケース: コンポーネントとデモの国際化を実装するには、 `src/BoxX` を参照してください。

ルール：
- `yml` エントリの末尾に `:` を追加せず、pug テンプレートと js コードに `:` を挿入してつなぎ合わせます。
- `yml` の複数行テキストに `|-` を使用すると、末尾のスペースを簡単に削除できます。
- 国際化されたエントリがない場合は、`I18N.js` ファイルをクリアします (何も書き込まないでください。`export` も書き込まないでください)。
- エントリーは単一言語で行う必要があり、複数の言語を混合することは固く禁止されています。

プロセス：

1. ルート ディレクトリに設定されているソース言語コードを表示します `tran.yml` → `tran` → `from`

2. `src/コンポーネント名` ディレクトリを入力します。

3. コンポーネントの国際化
   - `_.js` で使用されるテキストを `src/コンポーネント名/i18n/ソース言語/js.yml` のエントリとして記述します。
   - `コンポーネント名/I18N.js` を変更し、`yml` エントリ キー名の定数 (`export const KEY = "ymlKey"` など) を構成します。
   - `コンポーネント名/_.js` に導入およびバインドされます (`js.yml` がある場合、ない場合は必要ありません)。
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     「./I18N.js」から i18nUrl, { KEY } をインポートします。 // フレームワークは、エクスポートのデフォルトである i18nUrl を自動的に挿入します。自分で作成しないでください。

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    fLang 呼び出しを減らすためにバッチでバインドしてみてください
    コンポーネントの`_.js`にはテキストを含む文字列を含めないでください

4. プレゼンテーションの国際化
   - `demo/_.pug` では、翻訳する必要があるラベル テキストは `c-t キー名` の形式で占有され (これはカスタム ラベルであり、別の行になります)、エントリは `demo/i18n/source language/pug.yml` に書き込まれます。
   - `demo/_.js` `demo/i18n/source language/js.yml` で使用される元のテキスト エントリを書き込み、`demo/I18N.js` に配置します。
   - `fetchLang` を `demo/_.js` に導入して `./I18N.js` を読み込みます (これは `demo/I18N.js`)。上記のコンポーネントの国際化プロセスを参照してください (たとえば、`demo/i18n` には以下の `js.yml` がありません。この手順はスキップしてください)。
   - `demo/I18N.js` `demo/i18n/source language/js.yml` でキー名定数を構成し、エクスポートします

5. コンポーネントの説明を`コンポーネント名/i18n/ソース言語/README.md` に記述します。プロセスは次のとおりです
  5.1 メインタイトルを書きます。タイトルは `#` で始まり、コンポーネントの目的を 1 文で説明し (コンポーネント名は書かないでください)、最後に句読点を追加しないでください。
  5.2 コンポーネントの主な機能をメインタイトルから列挙し、サブタイトルは書かないでください。複数の特性がある場合は、順序なしリストを使用します (順序なしリストの説明が比較的短い場合は、末尾にピリオドがありません)。
  5.3 `## を記述する デモ` を使用して、デモ コードは ` ```html ` で始まります (デモ コードでは、単一要素タグが閉じられていません)。 `js` がある場合は、 `html` を内部的に記述し、 `<script type="module">` タグを使用します。
  純粋な js の場合は、` ```js ` から始めます。 `js` デモで`component/_.js`をインポートする必要がある場合は、 `import xxx from "webc.site/component/_.js";` を使用してください。
  `js` 複数の定数を宣言するには、`const` などの簡潔な記述を使用します。

本文では、コンポーネントの使用方法、インターフェースパラメータ、 状態の`css`と`class`の切り替えなどを紹介します。
その後、パラメータ、戻り値、スタイルのステータスなど (存在する場合) を記述し、補助タイトルを追加します。

言葉は簡潔に使用し、形容詞は避けてください。

コピーライティングではナンセンスを避け、意味のない言葉 (ユニバーサル、コンテナなど) を書かないでください。

6. ルート ディレクトリの `./sh/tran.js` を実行し、翻訳します。
