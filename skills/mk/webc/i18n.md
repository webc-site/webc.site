# интернационализација

Случај: погледнете во `src/BoxX` за спроведување на интернационализација на компоненти и демо.

правило:
- `yml` Не додавајте `:` на крајот од записот, ставете `:` во шаблонот на pug и js кодот и спојте го
- Користете `|-` за текст со повеќе линии од `yml` за лесно отстранување на празни места.
- Ако нема интернационализиран запис, исчистете ја датотеката `I18N.js` (не пишувајте ништо, не пишувајте `export`)
- Записите мора да бидат на еден јазик, а мешањето на повеќе јазици е строго забрането.

процес:

1. Погледнете го кодот на изворниот јазик конфигуриран во root директориумот `tran.yml` → `tran` → `from`

2. Внесете го директориумот `src/име на компонента`

3. Интернационализирајте ги компонентите
   - Напишете го текстот што го користи `_.js` како запис во `src/component name/i18n/source language/js.yml`.
   - Изменете го `Име на компонента/I18N.js` и конфигурирајте ја константата на `yml` името на влезниот клуч, како што е `export const KEY = "ymlKey"`
   - `Воведен и врзан во име на компонентата/_.js` (ако има `js.yml`, не е потребно ако не):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     увези i18nUrl, { KEY } од „./I18N.js“; // Рамката автоматски ќе го инјектира i18nUrl, стандардниот стандард за извоз, не го креирајте сами

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Обидете се да се поврзете во серии за да ги намалите повиците fLang
    `_.js` на компонентата не треба да има низи што содржат текст

4. Интернационализација на презентации
   - Во `demo/_.pug`, текстот на етикетата што треба да се преведе е зафатен во форма на `c-t име на клучот` (ова е прилагодена етикета, посебна линија), а записот е запишан во `demo/i18n/source language/pug.yml`
   - `demo/_.js` Напишете го оригиналниот текст што се користи во `demo/i18n/source language/js.yml` и ставете го во `demo/I18N.js`
   - Воведете го `fetchLang` во `demo/_.js` и вчитајте го `./I18N.js` (ова е `demo/I18N.js`), погледнете го процесот на интернационализација на компонентите погоре (на пример, `demo/i18n` нема `js.yml` подолу, прескокнете го овој чекор)
   - `demo/I18N.js` Конфигурирајте ги константите на името на клучот во `demo/i18n/source language/js.yml` и извезете

5. Напишете го описот на компонентата во `Име на компонентата/i18n/изворниот јазик/README.md`. Процесот е како што следува
  5.1 Напишете го главниот наслов. Насловот треба да започне со `#`, да ја опише целта на компонентата во една реченица (не пишувајте го името на компонентата) и не додавајте интерпункциски знаци на крајот.
  5.2 Наведете ги главните карактеристики на компонентата, почнувајќи од главниот наслов и не пишувајте секундарни наслови. Ако има повеќе карактеристики, користете неуреден список (ако описот на неподредениот список е релативно краток, на крајот нема да има точка).
  5.3 Напишете `## Користејќи го демото`, демо-кодот започнува со ` ```html ` (во демо-кодот, ознаката за еден елемент не е затворена), ако има `js`, напишете `внатрешно `CODE"> id="3">` ознака;
  Ако е чист js, почнете со ` ```js `. `js` Во демонстрацијата, ако треба да увезете `компонента/_.js`, ве молиме користете `увоз xxx од "webc.site/component name.js";`.
  `js` Користете концизно пишување, како што е `const` за да декларирате повеќе константи

Главниот текст воведува како да се користи компонентата, параметрите на интерфејсот, префрлањето помеѓу `css` и `class` во различни состојби итн.
Последователно опишете ги параметрите, повратната вредност, статусот на стилот итн. (ако ги има), и додадете секундарен наслов.

Користете ги зборовите концизно и избегнувајте придавки.

Пишувањето треба да избегнува глупости и да не пишува бесмислени зборови (како: универзален, контејнер итн.).

6. Стартувај `./sh/tran.js` од root директориумот, превод
