# internacionalizācija

Gadījums: skatiet `src/BoxX`, lai ieviestu komponentu internacionalizāciju un demonstrāciju.

noteikums:
- `yml` Nepievienojiet `:` ieraksta beigās, ievietojiet `:` mops veidnē un js kodā un savienojiet to
- Izmantojiet `|-` `yml` vairāku rindiņu tekstam, lai viegli noņemtu beigu atstarpes.
- Ja nav internacionalizēta ieraksta, notīriet `I18N.js` failu (neko nerakstiet, nerakstiet `export`)
- Ierakstiem ir jābūt vienā valodā, un vairāku valodu sajaukšana ir stingri aizliegta.

process:

1. Skatiet avota valodas kodu, kas konfigurēts saknes direktorijā `tran.yml` → `tran` → `from`

2. Ievadiet `src/komponenta nosaukuma` direktoriju

3. Internacionalizējiet komponentus
   - Ierakstiet tekstu, ko izmanto `_.js` kā ierakstu failā `src/component name/i18n/source language/js.yml`.
   - Modificējiet `komponenta nosaukums/I18N.js` un konfigurējiet `yml` ievades atslēgas nosaukuma konstanti, piemēram, `export const KEY = "ymlKey"`
   - `Ieviests un saistīts komponenta nosaukumā/_.js` (ja ir `js.yml`, nav nepieciešams, ja nav):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     importēt i18nUrl, { KEY } no "./I18N.js"; // Ietvars automātiski ievadīs i18nUrl, kas ir noklusējuma eksporta noklusējuma vērtība. Neveidojiet to pats

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Mēģiniet saistīt pa daļām, lai samazinātu fLang zvanus
    Komponenta `_.js` nedrīkst būt virknes, kas satur tekstu

4. Prezentāciju internacionalizācija
   - Programmā `demo/_.pug` tulkojamais etiķetes teksts ir aizņemts kā `c-t atslēgas nosaukums` (šī ir pielāgota iezīme, atsevišķa rindiņa), un ieraksts tiek ierakstīts failā `demo/i18n/source language/pug.yml`.
   - `demo/_.js` Uzrakstiet oriģinālo teksta ierakstu, kas izmantots `demo/i18n/source language/js.yml` un ievietojiet to `demo/I18N.js`
   - Ievadiet `fetchLang` `demo/_.js` un ielādējiet `./I18N.js` (tas ir `demo/I18N.js`), skatiet iepriekš minēto komponentu internacionalizācijas procesu (piemēram, `demo/i18n` tālāk nav `js.yml`, izlaidiet šo darbību)
   - `demo/I18N.js` Konfigurējiet atslēgas nosaukumu konstantes programmā `demo/i18n/source language/js.yml` un eksportējiet

5. Ierakstiet komponenta aprakstu sadaļā `Komponenta nosaukums/i18n/source language/README.md`. Process ir šāds
  5.1. Uzrakstiet galveno nosaukumu. Virsrakstam jāsākas ar `#`, vienā teikumā jāapraksta komponenta mērķis (nerakstiet komponenta nosaukumu) un beigās nepievienojiet pieturzīmes.
  5.2 Uzskaitiet komponenta galvenās iezīmes, sākot ar galveno nosaukumu, un nerakstiet sekundāros nosaukumus. Ja ir vairāki raksturlielumi, izmantojiet nesakārtotu sarakstu (ja nesakārtotā saraksta apraksts ir salīdzinoši īss, beigās nebūs punkta).
  5.3. Rakstīt `## Izmantojot demonstrāciju`, demonstrācijas kods sākas ar ` ```html ` (demonstrācijas kodā viena elementa tags nav aizvērts), ja ir `js`, ierakstiet `html` iekšēji ` ><b` ;
  Ja tas ir tīrs js, sāciet ar ` ```js `. `js` Demonstrācijā, ja nepieciešams importēt `component/_.js`, lūdzu, izmantojiet `import xxx no "webc.site/component name.js";`.
  `js` Izmantojiet īsu rakstīšanu, piemēram, `const`, lai deklarētu vairākas konstantes

Galvenais teksts iepazīstina ar komponenta lietošanu, saskarnes parametriem, pārslēgšanos starp `css` un `class` dažādos stāvokļos utt.
Pēc tam aprakstiet parametrus, atgriešanas vērtību, stila statusu utt. (ja tāds ir) un pievienojiet sekundāro nosaukumu.

Lietojiet vārdus īsi un izvairieties no īpašības vārdiem.

Rakstot tekstu, vajadzētu izvairīties no muļķībām un nerakstīt bezjēdzīgus vārdus (piemēram, universāls, konteiners utt.).

6. Palaidiet `./sh/tran.js` saknes direktoriju, tulkojumu
