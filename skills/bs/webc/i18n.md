# internacionalizacija

Slučaj: Pogledajte `src/BoxX` za implementaciju internacionalizacije komponenti i demonstracije.

pravilo:
- `yml` Nemojte dodavati `:` na kraju unosa, stavite `:` u predložak pug i js kod i spojite ga
- Koristite `|-` za višeredni tekst od `yml` da lako uklonite razmake na kraju.
- Ako nema internacionaliziranog unosa, obrišite datoteku `I18N.js` (ne pišite ništa, nemojte pisati `export`)
- Unosi moraju biti na jednom jeziku, a miješanje više jezika je strogo zabranjeno.

proces:

1. Pogledajte kod izvornog jezika konfiguriran u korijenskom direktoriju `tran.yml` → `tran` → `from`

2. Unesite `src/ime komponente` direktorij

3. Internacionalizirajte komponente
   - Napišite tekst koji koristi `_.js` kao unos u `src/ime komponente/i18n/izvorni jezik/js.yml`.
   - Izmijenite `ime komponente/I18N.js` i konfigurirajte konstantu `yml` imena ključa za unos, kao što je `export const KEY = "ymlKey"`
   - `Uvedeno i uvezano u naziv komponente/_.js` (ako postoji `js.yml`, nije potrebno ako ne):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     import i18nUrl, { KEY } iz "./I18N.js"; // Okvir će automatski ubaciti i18nUrl, zadani zadani izvoz, nemojte ga sami kreirati

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Pokušajte da se povežete u grupama kako biste smanjili fLang pozive
    Komponenta `_.js` ne bi trebala imati nizove koji sadrže tekst

4. Internacionalizacija prezentacija
   - U `demo/_.pug`, tekst oznake koji treba prevesti je zauzet u obliku `c-t naziv ključa` (ovo je prilagođena oznaka, poseban red), a unos je upisan u `demo/i18n/source language/pug.yml`
   - `demo/_.js` Napišite originalni tekst koji se koristi u `demo/i18n/source language/js.yml` i stavite ga u `demo/I18N.js`
   - Uvedite `fetchLang` u `demo/_.js` i učitajte `./I18N.js` (ovo je `demo/I18N.js`), pogledajte gornji proces internacionalizacije komponente (na primjer, `demo/i18n` nema `js.yml` ispod, preskočite ovaj korak)
   - `demo/I18N.js` Konfigurirajte konstante imena ključa u `demo/i18n/source language/js.yml` i izvezite

5. Napišite opis komponente u `ime komponente/i18n/izvorni jezik/README.md`. Proces je sljedeći
  5.1 Napišite glavni naslov. Naslov treba početi sa `#`, opisati svrhu komponente u jednoj rečenici (ne pisati naziv komponente) i ne dodavati interpunkciju na kraju.
  5.2 Navedite glavne karakteristike komponente, počevši od glavnog naslova, a nemojte pisati sekundarne naslove. Ako postoji više karakteristika, koristite neuređenu listu (ako je opis neuređene liste relativno kratak, na kraju neće biti tačke).
  5.3 Napišite `## Koristeći demo`, demo kod počinje sa ` ```html ` (u demo kodu, oznaka jednog elementa nije zatvorena), ako postoji `js`, upišite internu klasu `><b>html"` id="3">` oznaka;
  Ako je čisti js, počnite s ` ```js `. `js` U demonstraciji, ako trebate uvesti `component/_.js`, koristite `import xxx iz "webc.site/component/_.js";`.
  `js` Koristite sažeto pisanje, kao što je `const` da deklarirate više konstanti

Glavni tekst uvodi kako koristiti komponentu, parametre interfejsa, prebacivanje između `css` i `class` u različitim stanjima, itd.
Zatim opišite parametre, povratnu vrijednost, status stila itd. (ako ih ima) i dodajte sekundarni naslov.

Koristite riječi sažeto i izbjegavajte prideve.

Copywriting treba izbjegavati besmislice i ne pisati besmislene riječi (kao što su: univerzalno, kontejner, itd.).

6. Pokrenite `./sh/tran.js` korijenskog direktorija, prijevod
