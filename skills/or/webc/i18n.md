# ଆନ୍ତର୍ଜାତୀୟକରଣ

ମାମଲା: ଉପାଦାନ ଏବଂ ଡେମୋର ଆନ୍ତର୍ଜାତୀୟକରଣ କାର୍ଯ୍ୟକାରୀ କରିବାକୁ `src/BoxX` କୁ ଅନୁସରଣ କରନ୍ତୁ |

ନିୟମ:
- `yml` ପ୍ରବେଶ ଶେଷରେ `:` ଯୋଡନ୍ତୁ ନାହିଁ, ପଗ୍ ଟେମ୍ପଲେଟ୍ ଏବଂ js କୋଡ୍ ରେ `:` ରଖନ୍ତୁ, ଏବଂ ଏହାକୁ ଦୁଇଭାଗ କରନ୍ତୁ |
- ସହଜରେ ଟ୍ରେଲିଂ ସ୍ପେସ୍ ଅପସାରଣ କରିବାକୁ `yml` ର ମଲ୍ଟି ଲାଇନ୍ ଟେକ୍ସଟ୍ ପାଇଁ `|-` ବ୍ୟବହାର କରନ୍ତୁ |
- ଯଦି କ international ଣସି ଆନ୍ତର୍ଜାତୀୟକରଣ ପ୍ରବେଶ ନାହିଁ, `I18N.js` ଫାଇଲ୍ ସଫା କରନ୍ତୁ (କିଛି ଲେଖନ୍ତୁ ନାହିଁ, `export` ଲେଖନ୍ତୁ ନାହିଁ)
- ଏଣ୍ଟ୍ରିଗୁଡିକ ଗୋଟିଏ ଭାଷାରେ ହେବା ଜରୁରୀ, ଏବଂ ଏକାଧିକ ଭାଷା ମିଶ୍ରଣ କରିବା ନିଷେଧ |

ପ୍ରକ୍ରିୟା:

1. ମୂଳ ଡିରେକ୍ଟୋରୀରେ ବିନ୍ୟାସିତ ଉତ୍ସ ଭାଷା କୋଡ୍ ଦେଖନ୍ତୁ `tran.yml` → `tran` → `from`

2. <B ଶ୍ରେଣୀ = "କୋଡ୍"> src / ଉପାଦାନ ନାମ </ b> ଡିରେକ୍ଟୋରୀ ପ୍ରବେଶ କରନ୍ତୁ |

3. ଉପାଦାନଗୁଡ଼ିକୁ ଆନ୍ତର୍ଜାତୀୟ କର |
   - <B ଶ୍ରେଣୀ = "CODE"> src / ଉପାଦାନ ନାମ / i18n / ଉତ୍ସ ଭାଷା / js.yml ` ରେ ଏଣ୍ଟ୍ରି ଭାବରେ `_.js` ଦ୍ୱାରା ବ୍ୟବହୃତ ପାଠ ଲେଖ |
   - <B class = "CODE"> ଉପାଦାନ ନାମ / I18N.js ` କୁ ପରିବର୍ତ୍ତନ କରନ୍ତୁ ଏବଂ `yml` ପ୍ରବେଶ କୀ ନାମର ସ୍ଥିରକୁ ବିନ୍ୟାସ କରନ୍ତୁ, ଯେପରିକି `export const KEY = "ymlKey"`
   - <b class = "CODE"> ଉପାଦାନ ନାମ / _। js ` ରେ ପରିଚିତ ଏବଂ ବନ୍ଧା (ଯଦି `js.yml` ଅଛି, ଯଦି ଏହା ଆବଶ୍ୟକ ନୁହେଁ):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     "./I18N.js" ରୁ i18nUrl, {KEY import ଆମଦାନି କରନ୍ତୁ; // framework ାଞ୍ଚା ସ୍ୱୟଂଚାଳିତ ଭାବରେ i18nUrl ଇଞ୍ଜେକ୍ସନ ଦେବ, ଡିଫଲ୍ଟ ରପ୍ତାନି ଡିଫଲ୍ଟ, ଏହାକୁ ନିଜେ ସୃଷ୍ଟି କରନ୍ତୁ ନାହିଁ |

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    FLang କଲ୍ ହ୍ରାସ କରିବାକୁ ବ୍ୟାଚ୍ ରେ ବାନ୍ଧିବାକୁ ଚେଷ୍ଟା କରନ୍ତୁ |
    ଉପାଦାନର `_.js` ପାଠ୍ୟ ଧାରଣ କରିଥିବା ଷ୍ଟ୍ରିଙ୍ଗ୍ ରହିବା ଉଚିତ୍ ନୁହେଁ |

4. ଉପସ୍ଥାପନାଗୁଡ଼ିକର ଆନ୍ତର୍ଜାତୀୟକରଣ |
   - `demo/_.pug` ରେ, ଅନୁବାଦ ହେବା ଆବଶ୍ୟକ ଥିବା ଲେବଲ୍ ପାଠ୍ୟ <b ଶ୍ରେଣୀ = "CODE"> c-t କି ନାମ </ b> (ଏହା ଏକ କଷ୍ଟମ୍ ଲେବଲ୍, ଏକ ପୃଥକ ରେଖା) ଆକାରରେ ଦଖଲ କରାଯାଇଛି ଏବଂ ଏଣ୍ଟ୍ରି <b ଶ୍ରେଣୀ = "CODE"> ଡେମୋ / i18n / ଉତ୍ସ ଭାଷା / pug.yml ` ରେ ଲେଖାଯାଇଛି |
   - `demo/_.js` <b class = "CODE"> demo / i18n / source language / js.yml ` ରେ ବ୍ୟବହୃତ ମୂଳ ପାଠ୍ୟ ଏଣ୍ଟ୍ରି ଲେଖ ଏବଂ ଏହାକୁ `demo/I18N.js` ରେ ରଖ |
   - `fetchLang` କୁ `demo/_.js` ରେ ପରିଚିତ କର ଏବଂ `./I18N.js` (ଏହା ହେଉଛି `demo/I18N.js`), ଉପରୋକ୍ତ ଉପାଦାନ ଆନ୍ତର୍ଜାତୀୟକରଣ ପ୍ରକ୍ରିୟାକୁ ଅନୁସରଣ କରନ୍ତୁ (ଉଦାହରଣ ସ୍ୱରୂପ, `demo/i18n` ତଳେ `js.yml` ନାହିଁ, ଏହି ପଦକ୍ଷେପକୁ ଏଡ଼ାଇ ଦିଅନ୍ତୁ)
   - `demo/I18N.js` <b class = "CODE"> demo / i18n / source language / js.yml ` ରେ କୀ ନାମ କନଷ୍ଟାଣ୍ଟଗୁଡ଼ିକୁ ବିନ୍ୟାସ କରନ୍ତୁ ଏବଂ ରପ୍ତାନି କରନ୍ତୁ |

5. ଉପାଦାନ ବର୍ଣ୍ଣନାକୁ <b ଶ୍ରେଣୀ = "କୋଡ୍"> ଉପାଦାନ ନାମ / i18n / ଉତ୍ସ ଭାଷା / README.md ` ରେ ଲେଖନ୍ତୁ | ପ୍ରକ୍ରିୟା ନିମ୍ନଲିଖିତ ଅଟେ |
  5.1 ମୁଖ୍ୟ ଆଖ୍ୟା ଲେଖ | ଶିରୋନାମାଟି `#` ରୁ ଆରମ୍ଭ ହେବା ଉଚିତ, ଗୋଟିଏ ବାକ୍ୟରେ ଉପାଦାନର ଉଦ୍ଦେଶ୍ୟ ବର୍ଣ୍ଣନା କରିବା ଉଚିତ (ଉପାଦାନ ନାମ ଲେଖନ୍ତୁ ନାହିଁ) ଏବଂ ଶେଷରେ ବିରାମଚିହ୍ନ ଯୋଡନ୍ତୁ ନାହିଁ |
  5.2 ମୁଖ୍ୟ ଆଖ୍ୟା ଠାରୁ ଆରମ୍ଭ କରି ଉପାଦାନର ମୁଖ୍ୟ ବ features ଶିଷ୍ଟ୍ୟଗୁଡିକ ତାଲିକାଭୁକ୍ତ କର, ଏବଂ ଦଳୀୟ ଆଖ୍ୟା ଲେଖ ନାହିଁ | ଯଦି ଏକାଧିକ ବ characteristics ଶିଷ୍ଟ୍ୟ ଅଛି, ଏକ ଅଣସଂରକ୍ଷିତ ତାଲିକା ବ୍ୟବହାର କରନ୍ତୁ (ଯଦି ଅଣସଂରକ୍ଷିତ ତାଲିକା ବର୍ଣ୍ଣନା ଅପେକ୍ଷାକୃତ ସ୍ୱଳ୍ପ, ଶେଷରେ କ period ଣସି ଅବଧି ରହିବ ନାହିଁ) |
  5.3 ଲେଖନ୍ତୁ <b ଶ୍ରେଣୀ = "କୋଡ୍"> ## ଡେମୋ ବ୍ୟବହାର କରି </ b>, ଡେମୋ କୋଡ୍ ` ` `` html <b class = "CODE"> ରୁ ଆରମ୍ଭ ହୁଏ (ଡେମୋ କୋଡ୍ ରେ, ଏକକ ଉପାଦାନ ଟ୍ୟାଗ୍ ବନ୍ଦ ହୋଇନାହିଁ), ଯଦି </ b> js <b ଶ୍ରେଣୀ = "CODE">, ତେବେ </ b> <d> ` <script type="module"> `ଟ୍ୟାଗ୍;
  ଯଦି ଏହା ଶୁଦ୍ଧ js, ` ` `` js <b class = "CODE"> ରୁ ଆରମ୍ଭ କରନ୍ତୁ | ` js <b class = "CODE"> ପ୍ରଦର୍ଶନରେ, ଯଦି ଆପଣ ଆମଦାନି କରିବା ଆବଶ୍ୟକ କରନ୍ତି ` ଉପାଦାନ / _। js <b class = "CODE">, ଦୟାକରି </ b> ଆମଦାନି xxx କୁ "webc.site/component name.js";
  `js` ସଂକ୍ଷିପ୍ତ ଲେଖା ବ୍ୟବହାର କରନ୍ତୁ, ଯେପରିକି `const` ଏକାଧିକ ସ୍ଥିର ଘୋଷଣା କରିବାକୁ |

ବିଭିନ୍ନ ପାଠ୍ୟରେ ଉପାଦାନ, ଇଣ୍ଟରଫେସ୍ ପାରାମିଟରଗୁଡିକ, `css` ଏବଂ `class` ମଧ୍ୟରେ ସୁଇଚ୍ ଇତ୍ୟାଦି କିପରି ବ୍ୟବହାର କରାଯିବ ତାହା ମୁଖ୍ୟ ପାଠ୍ୟ ପରିଚିତ କରେ |
ପରବର୍ତ୍ତୀ ସମୟରେ ପାରାମିଟରଗୁଡିକ, ରିଟର୍ନ ମୂଲ୍ୟ, ଶ style ଳୀ ସ୍ଥିତି ଇତ୍ୟାଦି ବର୍ଣ୍ଣନା କର (ଯଦି ଥାଏ), ଏବଂ ଏକ ଦଳୀୟ ଆଖ୍ୟା ଯୋଡ |

ସଂକ୍ଷେପରେ ଶବ୍ଦ ବ୍ୟବହାର କରନ୍ତୁ ଏବଂ ବିଶେଷଣରୁ ଦୂରେଇ ରୁହନ୍ତୁ |

କପି ରାଇଟ୍ ନିର୍ବନ୍ଧରୁ ଦୂରେଇ ରହିବା ଉଚିତ ଏବଂ ଅର୍ଥହୀନ ଶବ୍ଦ ଲେଖିବା ଉଚିତ୍ ନୁହେଁ (ଯେପରିକି: ସର୍ବଭାରତୀୟ, ପାତ୍ର, ଇତ୍ୟାଦି) |
