ପ୍ରକ୍ରିୟା ଅନୁସରଣ କରନ୍ତୁ, ଏକ ଯୋଜନା ପ୍ରସ୍ତୁତ କରନ୍ତୁ, ଏବଂ ତ୍ରୁଟିଗୁଡିକୁ ଏଡାଇବା ପାଇଁ ପର୍ଯ୍ୟାୟ କ୍ରମେ ବିକାଶ କରନ୍ତୁ |

1. <B class = "CODE"> ./ sh / new.js ପ୍ରୋଜେକ୍ଟ ପ୍ରକାର ଉପାଦାନ ନାମ </ b> ଚଲାନ୍ତୁ ଏବଂ `src/webc` ଡିରେକ୍ଟୋରୀ ତଳେ ଉପାଦାନ ସୃଷ୍ଟି କରନ୍ତୁ |
   ପ୍ରୋଜେକ୍ଟ ପ୍ରକାର `css` (ଶୁଦ୍ଧ ଶ style ଳୀ ଉପାଦାନ), `js` (ୱେବ୍ ପୃଷ୍ଠା ଉପାଦାନ) ହୋଇପାରେ |
   ଉପାଦାନ ନାମର ପ୍ରଥମ ଅକ୍ଷରକୁ କ୍ୟାପିଟାଲ୍ କରନ୍ତୁ |

2. `.agents/doc/code` ପାଇଁ କୋଡିଂ ନିର୍ଦ୍ଦିଷ୍ଟତା ପ Read ଏବଂ ତାପରେ ବିକାଶ କର |

   ଉପାଦାନ ଶ style ଳୀ `_.styl` ରେ ଲେଖାଯାଇଛି | ପ୍ରଦର୍ଶନ ପାଇଁ ବ୍ୟବହୃତ ଶ style ଳୀ `demo/_.styl` ରେ ଲେଖାଯାଇଛି | ଏହାକୁ <b ଶ୍ରେଣୀ = "କୋଡ୍"> ଉପାଦାନ ନାମ / _ ଷ୍ଟାଇଲ୍ </ b> ରେ ଲେଖିବା ନିଷେଧ |

  ଅନ୍ୟ ଉପାଦାନରୁ ଶ yles ଳୀ ଆମଦାନୀ କରିବାକୁ `@import` ବ୍ୟବହାର କରିବା ଉଭୟ `_.styl` ଏବଂ `demo/_.styl` ରେ ନିଷେଧ |

  କୋଡିଂ ମାନାଙ୍କ ଅନୁସରଣ କରନ୍ତୁ ଏବଂ [./styl.md)(/0) ର ସ est ନ୍ଦର୍ଯ୍ୟକରଣ ଡିଜାଇନ୍ କରନ୍ତୁ |

  ନିର୍ଭରଶୀଳ ଉପାଦାନଗୁଡିକ (ଏବଂ ସେମାନଙ୍କର ଶ yles ଳୀଗୁଡିକ) <b ଶ୍ରେଣୀ = "CODE"> ଆମଦାନି "./other components.js" ` <b class = "CODE"> webc / componentname.js `; ପ୍ରଦର୍ଶନ ପାଇଁ ନିର୍ଭରଶୀଳତା <b ଶ୍ରେଣୀ = "CODE"> ଆମଦାନି "../../other components.js" ` `demo/_.js` ରେ ଆମଦାନୀ ହୋଇଥାଏ |

   ସମସ୍ତ ଉପାଦାନଗୁଡିକ (ଶୁଦ୍ଧ ଶ style ଳୀ ଉପାଦାନଗୁଡିକ ଅନ୍ତର୍ଭୂକ୍ତ କରି) ନିଶ୍ଚିତ ଭାବରେ ଏକ <b ଶ୍ରେଣୀ = "CODE"> ଉପାଦାନ ନାମ। Js ` ସ୍କ୍ରିପ୍ଟ ଫାଇଲ୍ ରହିବା ଆବଶ୍ୟକ | ଶୁଦ୍ଧ ଶ style ଳୀ ଉପାଦାନଗୁଡ଼ିକର <b ଶ୍ରେଣୀ = "କୋଡ୍"> ଉପାଦାନ ନାମ। Js ` କେବଳ ସଂପୃକ୍ତ `_.styl` ଆମଦାନୀ କରେ | JS ତର୍କ ସହିତ ଉପାଦାନଗୁଡ଼ିକ ପାଇଁ, ବିକାଶ ପାଇଁ [webc.js.md] (/ 1) ପ୍ରକ୍ରିୟାକୁ ଅନୁସରଣ କରନ୍ତୁ |
