# Internationalisierung

Fall: Informationen zur Implementierung der Internationalisierung von Komponenten und Demo finden Sie unter `src/BoxX`.

Regel:
- `yml` Fügen Sie nicht `:` am Ende des Eintrags hinzu, fügen Sie `:` in die Pug-Vorlage und den JS-Code ein und fügen Sie es zusammen
- Verwenden Sie `|-` für mehrzeiligen Text von `yml`, um nachfolgende Leerzeichen einfach zu entfernen.
- Wenn kein internationalisierter Eintrag vorhanden ist, löschen Sie die Datei `I18N.js` (schreiben Sie nichts, schreiben Sie nicht `export`).
- Beiträge müssen in einer einzigen Sprache verfasst sein. Das Mischen mehrerer Sprachen ist strengstens untersagt.

Verfahren:

1. Zeigen Sie den im Stammverzeichnis `tran.yml` → `tran` → `from` konfigurierten Quellsprachencode an

2. Geben Sie das Verzeichnis `src/component name` ein

3. Komponenten internationalisieren
   - Schreiben Sie den von `_.js` verwendeten Text als Eintrag in `src/component name/i18n/source language/js.yml`.
   - Ändern Sie `Komponentenname/I18N.js` und konfigurieren Sie die Konstante des `yml`Eintragsschlüsselnamens, z. B. `export const KEY = "ymlKey"`
   - `Eingeführt und gebunden in Komponentenname/_.js` (wenn `js.yml` vorhanden ist, ist es nicht erforderlich, wenn nicht):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     import i18nUrl, { KEY } from „./I18N.js“; // Das Framework injiziert automatisch i18nUrl, den Standardexportstandard, erstellen Sie ihn nicht selbst

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Versuchen Sie, stapelweise zu binden, um fLang-Aufrufe zu reduzieren
    `_.js` der Komponente sollte keine Zeichenfolgen enthalten, die Text enthalten

4. Internationalisierung von Präsentationen
   - In `demo/_.pug` wird der zu übersetzende Beschriftungstext in Form eines `c-t-Schlüsselnamens` belegt (dies ist eine benutzerdefinierte Beschriftung, eine separate Zeile) und der Eintrag wird in `demo/i18n/source language/pug.yml` geschrieben
   - `demo/_.js` Schreiben Sie den ursprünglichen Texteintrag, der in `demo/i18n/source language/js.yml` verwendet wird, und platzieren Sie ihn in `demo/I18N.js`
   - Führen Sie `fetchLang` in `demo/_.js` ein und laden Sie `./I18N.js` (dies ist `demo/I18N.js`). Beachten Sie den obigen Komponenten-Internationalisierungsprozess (z. B. verfügt `demo/i18n` unten nicht über `js.yml`, überspringen Sie diesen Schritt).
   - `demo/I18N.js`Konfigurieren Sie Schlüsselnamenkonstanten in `demo/i18n/source language/js.yml` und exportieren Sie sie

5. Schreiben Sie die Komponentenbeschreibung in `Component name/i18n/source language/README.md`. Der Prozess ist wie folgt
  5.1 Schreiben Sie den Haupttitel. Der Titel sollte mit `#` beginnen, den Zweck der Komponente in einem Satz beschreiben (schreiben Sie nicht den Komponentennamen) und am Ende keine Satzzeichen hinzufügen.
  5.2 Listen Sie die Hauptmerkmale der Komponente auf, beginnend mit dem Haupttitel, und schreiben Sie keine Nebentitel. Wenn mehrere Merkmale vorhanden sind, verwenden Sie eine ungeordnete Liste (wenn die Beschreibung der ungeordneten Liste relativ kurz ist, wird am Ende kein Punkt angezeigt).
  5.3 Schreiben Sie `## unter Verwendung der Demo`. Der Democode beginnt mit ` ```html ` (im Democode ist das Einzelelement-Tag nicht geschlossen). Wenn `js` vorhanden ist, schreiben Sie `html` intern und verwenden Sie das Tag `<script type="module">`.
  Wenn es reines JS ist, beginnen Sie mit ` ```js `. `js` Wenn Sie in der Demonstration `component/_.js` importieren müssen, verwenden Sie bitte `import xxx from "webc.site/component/_.js";`.
  `js` Verwenden Sie eine prägnante Schreibweise, z. B. einen `const`, um mehrere Konstanten zu deklarieren

Der Haupttext stellt die Verwendung der Komponente, Schnittstellenparameter, das Umschalten zwischen `css` und `class` in verschiedenen Zuständen usw. vor.
Beschreiben Sie anschließend die Parameter, den Rückgabewert, den Stilstatus usw. (falls vorhanden) und fügen Sie einen sekundären Titel hinzu.

Verwenden Sie Wörter prägnant und vermeiden Sie Adjektive.

Beim Verfassen von Texten sollte Unsinn vermieden und keine bedeutungslosen Wörter geschrieben werden (z. B. „Universal“, „Container“ usw.).

6. Führen Sie `./sh/tran.js` des Stammverzeichnisses Übersetzung aus
