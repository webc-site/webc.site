Schreiben Sie formell und vermeiden Sie gesprochene Sprache: Verwenden Sie für offizielle technische Dokumente einen strengen und professionellen Sprechstil und verbieten Sie die Verwendung von Modalpartikeln oder alltäglicher gesprochener Sprache im Dokument.

Eliminieren Sie Pronomen der dritten Person: Es ist verboten, Pronomen der dritten Person wie „it“ und „they“ zu verwenden, um sich auf Tools, Skripte oder Module zu beziehen. Sie sollten sich direkt auf „Ausführungszeit“, „das Skript“, „dieses Tool“ oder den spezifischen Namen beziehen.

Seien Sie realistisch in Bezug auf Funktionen und vermeiden Sie Schnickschnack: Vermeiden Sie bei der Beschreibung von Tools oder Skripten die Verwendung von Marketing- oder abstrakten Modifikatoren wie „intelligent“ und „fortgeschritten“ und beschreiben Sie direkt deren spezifische Funktionen und Verwendungszwecke.

Kontrollieren Sie die Satzlänge und verwenden Sie Zeilenumbrüche: Vermeiden Sie das Schreiben langer, komplexer Sätze. Teilen Sie den Text sinnvoll in kurze Sätze auf und verwenden Sie Listen und mehrere Zeilenumbrüche, um das Lese- und visuelle Layout-Erlebnis zu optimieren.

Reduzieren Sie ausländische Begriffe und lokalisieren Sie die Übersetzung: Verwenden Sie so wenige englische Begriffe wie möglich und verwenden Sie einfache chinesische Erklärungen (z. B. verwenden Sie „Codeunterschied“ anstelle von „Git diff“ oder „diff“; verwenden Sie „Commit-Informationen“ anstelle von „Commit-Nachricht“).
