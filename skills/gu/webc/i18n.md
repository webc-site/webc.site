# આંતરરાષ્ટ્રીયકરણ

કેસ: ઘટકો અને ડેમોના આંતરરાષ્ટ્રીયકરણને અમલમાં મૂકવા માટે `src/BoxX` નો સંદર્ભ લો.

નિયમ:
- `yml` એન્ટ્રીના અંતે `:` ઉમેરશો નહીં, પગ ટેમ્પલેટ અને js કોડમાં `:` મૂકો અને તેને વિભાજિત કરો
- પાછળની જગ્યાઓ સરળતાથી દૂર કરવા માટે `yml`ના બહુ-લાઇન ટેક્સ્ટ માટે `|-` નો ઉપયોગ કરો.
- જો ત્યાં કોઈ આંતરરાષ્ટ્રીયકૃત એન્ટ્રી ન હોય, તો `I18N.js` ફાઈલ સાફ કરો (કંઈ પણ લખશો નહીં, `export` લખશો નહીં)
- એન્ટ્રીઓ એક જ ભાષામાં હોવી જોઈએ, અને બહુવિધ ભાષાઓનું મિશ્રણ સખત રીતે પ્રતિબંધિત છે.

પ્રક્રિયા:

1. રૂટ ડિરેક્ટરીમાં ગોઠવેલ સ્રોત ભાષા કોડ જુઓ `tran.yml` → `tran` → `from`

2. `src/component name` ડિરેક્ટરી દાખલ કરો

3. ઘટકોનું આંતરરાષ્ટ્રીયકરણ કરો
   - `src/component name/i18n/source language/js.yml` માં એન્ટ્રી તરીકે `_.js` દ્વારા વપરાયેલ ટેક્સ્ટ લખો.
   - `ઘટકનું નામ/I18N.js` સંશોધિત કરો અને `yml` એન્ટ્રી કી નામ, જેમ કે `export const KEY = "ymlKey"` ની સ્થિરતાને ગોઠવો
   - `પરિચય અને ઘટક નામમાં બંધાયેલ/_.js` (જો ત્યાં `js.yml` હોય, તો તે જરૂરી નથી જો ન હોય તો):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     "./I18N.js" માંથી i18nUrl, { KEY } આયાત કરો; // ફ્રેમવર્ક આપમેળે i18nUrl ઇન્જેક્ટ કરશે, ડિફોલ્ટ નિકાસ ડિફોલ્ટ, તેને જાતે બનાવશો નહીં

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    ફ્લેંગ કોલ્સ ઘટાડવા માટે બેચમાં બાંધવાનો પ્રયાસ કરો
    ઘટકના `_.js`માં ટેક્સ્ટ ધરાવતી સ્ટ્રિંગ્સ હોવી જોઈએ નહીં

4. પ્રસ્તુતિઓનું આંતરરાષ્ટ્રીયકરણ
   - `demo/_.pug`માં, જે લેબલ ટેક્સ્ટનું ભાષાંતર કરવાની જરૂર છે તે `c-t કી નામ` (આ એક કસ્ટમ લેબલ છે, એક અલગ લાઇન છે), અને એન્ટ્રી `demo/i18n/source language/pug.yml`માં લખેલી છે.
   - `demo/_.js` `demo/i18n/source language/js.yml` માં વપરાયેલ મૂળ ટેક્સ્ટ એન્ટ્રી લખો અને તેને `demo/I18N.js` માં મૂકો
   - `demo/_.js` માં `fetchLang`નો પરિચય આપો અને `./I18N.js` લોડ કરો (આ `demo/I18N.js` છે), ઉપરની ઘટક આંતરરાષ્ટ્રીયકરણ પ્રક્રિયાનો સંદર્ભ લો (ઉદાહરણ તરીકે, `demo/i18n` નીચે `js.yml` નથી, આ પગલું છોડો)
   - `demo/I18N.js` કી નામ સ્થિરાંકોને `demo/i18n/source language/js.yml` માં ગોઠવો અને નિકાસ કરો

5. ઘટકનું વર્ણન `ઘટકનું નામ/i18n/source language/README.md` માં લખો. પ્રક્રિયા નીચે મુજબ છે
  5.1 મુખ્ય શીર્ષક લખો. શીર્ષક `#` થી શરૂ થવું જોઈએ, એક વાક્યમાં ઘટકના હેતુનું વર્ણન કરો (ઘટકનું નામ લખશો નહીં), અને અંતમાં વિરામચિહ્નો ઉમેરશો નહીં.
  5.2 મુખ્ય શીર્ષકથી શરૂ કરીને, ઘટકની મુખ્ય લાક્ષણિકતાઓની સૂચિ બનાવો અને ગૌણ શીર્ષકો લખશો નહીં. જો ત્યાં બહુવિધ લાક્ષણિકતાઓ હોય, તો અવ્યવસ્થિત સૂચિનો ઉપયોગ કરો (જો અવ્યવસ્થિત સૂચિનું વર્ણન પ્રમાણમાં ટૂંકું હોય, તો અંતે કોઈ અવધિ હશે નહીં).
  5.3 `## ડેમોનો ઉપયોગ કરીને` લખો, ડેમો કોડ ` ```html ` થી શરૂ થાય છે (ડેમો કોડમાં, સિંગલ એલિમેન્ટ ટેગ બંધ નથી), જો ત્યાં `js` હોય, તો લખો `CODE="`<b><b><b><b>` class">``` class="`` ઉપયોગ કરો ટેગ;
  જો તે શુદ્ધ js હોય, તો ` ```js ` થી પ્રારંભ કરો. `js` પ્રદર્શનમાં, જો તમારે `component/_.js` આયાત કરવાની જરૂર હોય, તો કૃપા કરીને `"webc.site/component/_.js";` માંથી xxx આયાત કરો.
  `js` બહુવિધ સ્થિરાંકો જાહેર કરવા માટે સંક્ષિપ્ત લેખનનો ઉપયોગ કરો, જેમ કે `const`

મુખ્ય ટેક્સ્ટ કમ્પોનન્ટનો ઉપયોગ કેવી રીતે કરવો, ઈન્ટરફેસ પેરામીટર્સ, વિવિધ રાજ્યોમાં `css` અને `class` વચ્ચે સ્વિચ કરવું વગેરેનો પરિચય આપે છે.
ત્યારબાદ પરિમાણો, વળતર મૂલ્ય, શૈલી સ્થિતિ વગેરેનું વર્ણન કરો (જો કોઈ હોય તો), અને ગૌણ શીર્ષક ઉમેરો.

સંક્ષિપ્તમાં શબ્દોનો ઉપયોગ કરો અને વિશેષણો ટાળો.

કૉપિરાઇટિંગ એ બકવાસ ટાળવા જોઈએ અને અર્થહીન શબ્દો ન લખવા જોઈએ (જેમ કે: યુનિવર્સલ, કન્ટેનર, વગેરે).

6. રૂટ ડાયરેક્ટરી, અનુવાદનું `./sh/tran.js` ચલાવો
