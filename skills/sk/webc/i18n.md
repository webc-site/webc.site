# internacionalizácie

Prípad: Ak chcete implementovať internacionalizáciu komponentov a ukážku, pozrite si `src/BoxX`.

pravidlo:
- `yml` Nepridávajte `:` na koniec záznamu, vložte `:` do šablóny pug a kódu js a spojte ho
- Použite `|-` pre viacriadkový text z `yml` na jednoduché odstránenie medzier na konci.
- Ak neexistuje žiadna internacionalizovaná položka, vymažte súbor `I18N.js` (nič nepíšte, nepíšte `export`)
- Príspevky musia byť v jednom jazyku a miešanie viacerých jazykov je prísne zakázané.

proces:

1. Zobraziť kód zdrojového jazyka nakonfigurovaný v koreňovom adresári `tran.yml` → `tran` → `from`

2. Zadajte adresár `src/názov komponentu`

3. Internacionalizujte komponenty
   - Napíšte text, ktorý používa `_.js` ako položku v `src/názov komponentu/i18n/zdrojový jazyk/js.yml`.
   - Upravte `Názov komponentu/I18N.js` a nakonfigurujte konštantu názvu vstupného kľúča `yml`, napríklad `export const KEY = "ymlKey"`
   - `Zavedené a viazané v názve komponentu/_.js` (ak existuje `js.yml`, nie je potrebné, ak nie):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     import i18nUrl, { KEY } z "./I18N.js"; // Rámec automaticky vloží i18nUrl, predvolené predvolené nastavenie exportu, nevytvárajte ho sami

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Pokúste sa viazať v dávkach, aby ste znížili hovory fLang
    Komponent `_.js` by nemal obsahovať reťazce obsahujúce text

4. Internacionalizácia prezentácií
   - V `demo/_.pug` je text označenia, ktorý je potrebné preložiť, obsadený v tvare `názov kľúča c-t` (toto je vlastný štítok, samostatný riadok) a záznam sa zapíše do `demo/i18n/source language/pug.yml`
   - `demo/_.js` Napíšte pôvodný textový záznam použitý v `demo/i18n/source language/js.yml` a umiestnite ho do `demo/I18N.js`
   - Zaveďte `fetchLang` do `demo/_.js` a načítajte `./I18N.js` (toto je `demo/I18N.js`), pozrite si proces internacionalizácie komponentu vyššie (napríklad `demo/i18n` nižšie nemá `js.yml`, tento krok preskočte)
   - `demo/I18N.js` Nakonfigurujte konštanty názvu kľúča v `demo/i18n/source language/js.yml` a exportujte

5. Napíšte popis komponentu do `Názov komponentu/i18n/zdrojový jazyk/README.md`. Postup je nasledovný
  5.1 Napíšte hlavný názov. Názov by mal začínať na `#`, jednou vetou opísať účel komponentu (nepíšte názov komponentu) a na koniec nepridávajte interpunkciu.
  5.2 Uveďte hlavné črty komponentu, počnúc hlavným názvom a nepíšte vedľajšie názvy. Ak existuje viacero charakteristík, použite nezoradený zoznam (ak je popis neusporiadaného zoznamu relatívne krátky, na konci nebude bodka).
  5.3 Napíšte `## Pomocou ukážky` sa ukážkový kód začína znakom ` ```html ` (v ukážkovom kóde nie je značka jedného prvku uzavretá), ak existuje `js`, interne napíšte značku `html``<b">, použite ;``
  Ak ide o čistý js, začnite s ` ```js `. `js` Ak v ukážke potrebujete importovať `component/_.js`, použite `import xxx z "webc.site/component/_.js";`.
  `js` Na deklarovanie viacerých konštánt použite stručné písanie, ako napríklad `const`

Hlavný text predstavuje spôsob používania komponentu, parametre rozhrania, prepínanie medzi `css` a `class` v rôznych stavoch atď.
Následne popíšte parametre, návratovú hodnotu, stav štýlu atď. (ak existujú) a pridajte sekundárny názov.

Používajte slová stručne a vyhýbajte sa prídavným menám.

Copywriting by sa mal vyhýbať nezmyslom a nepísať nezmyselné slová (ako: univerzálny, kontajnerový atď.).
