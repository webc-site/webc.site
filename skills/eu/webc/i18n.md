# nazioartekotzea

Kasua: ikusi `src/webc/BoxX` osagaien eta demoaren nazioartekotzea ezartzeko.

araua:
- `yml` Ez gehitu `:` sarreraren amaieran, jarri `:` pug txantiloian eta js kodean, eta elkartu
- Erabili `|-` `yml`-ren lerro anitzeko testurako, amaierako zuriuneak erraz kentzeko.
- Nazioarteko sarrerarik ez badago, garbitu `I18N.js` fitxategia (ez idatzi ezer, ez idatzi `export`)
- Sarrerak hizkuntza bakarrean egin behar dira, eta hizkuntza anitzak nahastea erabat debekatuta dago.

prozesua:

1. Ikusi erroko direktorioan konfiguratutako iturburu-hizkuntza-kodea `tran.yml` → `tran` → `from`

2. Sartu `src/webc/component name` direktorioa

3. Osagaiak nazioartekotzea
   - Idatzi `osagaiaren izena.js` atalean erabilitako testua `src/webc/componentearen izena/i18n/source language/js.yml`-en sarrera gisa.
   - Aldatu `src/webc/component name/I18N.js` eta konfiguratu `yml` sarrera-gakoaren izenaren konstantea, adibidez, `export const KEY = "ymlKey"`
   - `src/webc/component name.js`-en sarrera eta lotura (`js.yml` badago, ez bada, ez da beharrezkoa):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     inportatu i18nUrl, { KEY } "./component name/I18N.js"-tik; // Esparruak automatikoki injektatuko du i18nUrl, esportazio lehenetsia, ez sortu zuk zeuk

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Saiatu loteka lotzen fLang deiak murrizteko
    Osagaiaren `osagaiaren izena.js`ek ez luke testua duen katerik izan behar

4. Aurkezpenak nazioartekotzea
   - `demo/_.pug`-n, itzuli behar den etiketa-testua `c-t gako-izena` moduan okupatuta dago (etiketa pertsonalizatua da, lerro bereizia), eta sarrera `demo/i18n/source language/pug.yml`-n idatzita dago.
   - `demo/_.js` Idatzi `demo/i18n/source language/js.yml`n erabilitako jatorrizko testu-sarrera eta jarri `demo/I18N.js` atalean
   - Sartu `fetchLang` `demo/_.js`-n eta kargatu `./I18N.js` (hau `demo/I18N.js` da), ikusi goiko osagaien nazioartekotze-prozesua (adibidez, `demo/i18n`k ez dauka `js.yml` behean, saltatu urrats hau)
   - `demo/I18N.js` Konfiguratu gako-izenen konstanteak `demo/i18n/source language/js.yml`n eta esportatu

5. Idatzi osagaiaren deskribapena `src/webc/component name/i18n/source language/README.md` atalean. Prozesua honakoa da
  5.1 Idatzi izenburu nagusia. Izenburuak `#`-rekin hasi behar du, osagaiaren helburua esaldi batean deskribatu (ez idatzi osagaiaren izena) eta ez jarri puntuaziorik amaieran.
  5.2 Zerrendatu osagaiaren ezaugarri nagusiak, izenburu nagusitik hasita, eta ez idatzi bigarren mailako titulurik. Ezaugarri anitz badaude, erabili ordenatu gabeko zerrenda bat (ordenatu gabeko zerrendaren deskribapena nahiko laburra bada, ez da punturik egongo amaieran).
  5.3 Idatzi `## Demo-a erabiliz, demo-kodea ` ```html `-rekin hasten da (demo-kodean, elementu bakarreko etiketa ez dago itxita), `js` badago, idatzi `DE html<b>"><b>`, idatzi `html"<b>><b="CODE"> barne. class="CODE"> Etiketa, `js` Ez dute koskarik edukiaren lehen lerroan.
  Js hutsa bada, hasi ` ```js `. `js` Erakusketan, `component/_.js` inportatu behar baduzu, erabili `import xxx "webc.site/component name.js";`. Hau konpilatutako `js da, automatikoki js eta css injektatuko dituena.
  `js` Erabili idazkera zehatza, adibidez, `const` konstante bat baino gehiago deklaratzeko

Testu nagusiak osagaia nola erabili, interfaze-parametroak, `css` eta `class` artean aldatzea egoera ezberdinetan, etab.
Ondoren, deskribatu parametroak, itzulera-balioa, estilo-egoera, etab. (halakorik badago), eta gehitu bigarren mailako titulu bat.

Erabili hitzak zehatz-mehatz eta saihestu adjektiboak.

Copywritingak zentzugabekeriak saihestu behar ditu eta ez idatzi esanahirik gabeko hitzak (adibidez: unibertsala, edukiontzia, etab.).
