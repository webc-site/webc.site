# internacionalizacija

Atvejis: Norėdami įdiegti komponentų internacionalizavimą ir demonstracinę versiją, žr. `src/BoxX`.

taisyklė:
- `yml` Nepridėkite `:` įrašo pabaigoje, įdėkite `:` į pug šabloną ir js kodą ir sujunkite
- Naudokite `|-` kelių eilučių `yml` tekstui, kad lengvai pašalintumėte tarpus.
- Jei nėra internacionalizuoto įrašo, išvalykite `I18N.js` failą (nieko nerašykite, nerašykite `export`)
- Įrašai turi būti pateikiami viena kalba, o kelių kalbų maišymas yra griežtai draudžiamas.

procesas:

1. Peržiūrėkite šaltinio kalbos kodą, sukonfigūruotą šakniniame kataloge `tran.yml` → `tran` → `from`

2. Įveskite `src/komponento pavadinimo` katalogą

3. Internacionalizuoti komponentus
   - Parašykite `_.js` naudojamą tekstą kaip įrašą `src/component name/i18n/source language/js.yml`.
   - Pakeiskite `Komponento pavadinimą/I18N.js` ir sukonfigūruokite `yml` įvesties rakto pavadinimo konstantą, pvz., `export const KEY = "ymlKey"`
   - `Įvestas ir susietas komponente name/_.js` (jei yra `js.yml`, nebūtina, jei ne):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     importuoti i18nUrl, { KEY } iš "./I18N.js"; // Sistema automatiškai įves i18nUrl, numatytąjį eksporto parametrą, nekurkite jo patys

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Pabandykite susieti paketais, kad sumažintumėte „fLang“ skambučius
    Komponento `_.js` neturėtų būti eilučių su tekstu

4. Prezentacijų internacionalizavimas
   - `demo/_.pug` etiketės tekstas, kurį reikia išversti, yra užimtas `c-t rakto pavadinimo` forma (tai yra tinkinta etiketė, atskira eilutė), o įrašas įrašomas į `demo/i18n/source language/pug.yml`
   - `demo/_.js` Parašykite originalų teksto įrašą, naudojamą `demo/i18n/source language/js.yml` ir įdėkite jį į `demo/I18N.js`
   - Įveskite `fetchLang` į `demo/_.js` ir įkelkite `./I18N.js` (tai yra `demo/I18N.js`), žr. anksčiau pateiktą komponento internacionalizavimo procesą (pvz., `demo/i18n` toliau neturi `js.yml`, praleiskite šį veiksmą)
   - `demo/I18N.js` Konfigūruokite rakto pavadinimo konstantas `demo/i18n/source language/js.yml` ir eksportuokite

5. Parašykite komponento aprašą `Komponento pavadinimas/i18n/source language/README.md`. Procesas yra toks
  5.1 Parašykite pagrindinį pavadinimą. Pavadinimas turi prasidėti `#`, vienu sakiniu apibūdinti komponento paskirtį (nerašykite komponento pavadinimo), o pabaigoje nedėkite skyrybos ženklų.
  5.2 Išvardykite pagrindines komponento savybes, pradedant pagrindiniu pavadinimu, ir nerašykite antrinių pavadinimų. Jei yra kelios charakteristikos, naudokite nerūšiuotą sąrašą (jei nerūšiuoto sąrašo aprašymas yra palyginti trumpas, pabaigoje nebus taško).
  5.3 Rašykite `## Naudodami demonstracinę versiją` demonstracinis kodas prasideda ` ```html ` (demonstraciniame kode vieno elemento žyma neuždaryta), jei yra `js`, parašykite `html` viduje ` `<b> tag, naudokite ;
  Jei tai grynas js, pradėkite nuo ` ```js `. `js` Demonstracijoje, jei reikia importuoti `component/_.js`, naudokite `import xxx iš "webc.site/component name.js";`.
  `js` Norėdami deklaruoti kelias konstantas, naudokite glaustą rašymą, pvz., `const`

Pagrindiniame tekste supažindinama su komponento naudojimu, sąsajos parametrais, perjungimu tarp `css` ir `class` skirtingose ​​būsenose ir kt.
Vėliau aprašykite parametrus, grąžinimo reikšmę, stiliaus būseną ir kt. (jei yra) ir pridėkite antrinį pavadinimą.

Vartokite žodžius glaustai ir venkite būdvardžių.

Rašant tekstą reikėtų vengti nesąmonių ir nerašyti bereikšmių žodžių (pvz.: universalus, konteineris ir pan.).
