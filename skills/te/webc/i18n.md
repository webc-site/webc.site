# అంతర్జాతీయీకరణ

కేస్: భాగాలు మరియు డెమో అంతర్జాతీయీకరణను అమలు చేయడానికి `src/webc/BoxX`ని చూడండి.

నియమం:
- `yml` ఎంట్రీ చివరిలో `:`ని జోడించవద్దు, `:`ని pug టెంప్లేట్ మరియు js కోడ్‌లో ఉంచండి మరియు దానిని స్ప్లైస్ చేయండి
- వెనుకంజలో ఉన్న ఖాళీలను సులభంగా తొలగించడానికి `yml` యొక్క బహుళ-లైన్ వచనం కోసం `|-`ని ఉపయోగించండి.
- అంతర్జాతీయీకరించిన ఎంట్రీ లేనట్లయితే, `I18N.js` ఫైల్‌ను క్లియర్ చేయండి (ఏమీ వ్రాయవద్దు, `export` అని వ్రాయవద్దు)
- ఎంట్రీలు తప్పనిసరిగా ఒకే భాషలో ఉండాలి మరియు బహుళ భాషలను కలపడం ఖచ్చితంగా నిషేధించబడింది.

ప్రక్రియ:

1. రూట్ డైరెక్టరీలో కాన్ఫిగర్ చేయబడిన సోర్స్ లాంగ్వేజ్ కోడ్‌ను వీక్షించండి `tran.yml` → `tran` → `from`

2. `src/webc/component name` డైరెక్టరీని నమోదు చేయండి

3. భాగాలను అంతర్జాతీయీకరించండి
   - `component name.js`లో ఉపయోగించిన వచనాన్ని `src/webc/component name/i18n/source language/js.yml`కి ఎంట్రీగా వ్రాయండి.
   - `src/webc/component name/I18N.js`ని సవరించండి మరియు `export const KEY = "ymlKey"` వంటి `yml` ఎంట్రీ కీ పేరు యొక్క స్థిరాంకాన్ని కాన్ఫిగర్ చేయండి
   - `src/webc/component name.js`లో పరిచయం మరియు బైండింగ్ (`js.yml` ఉంటే, అది అవసరం లేదు):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     "./component name/I18N.js" నుండి i18nUrl, { KEY }ని దిగుమతి చేయండి; // ఫ్రేమ్‌వర్క్ స్వయంచాలకంగా i18nUrlని ఇంజెక్ట్ చేస్తుంది, డిఫాల్ట్ ఎగుమతి డిఫాల్ట్, దీన్ని మీరే సృష్టించవద్దు

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    ఫ్లాంగ్ కాల్‌లను తగ్గించడానికి బ్యాచ్‌లలో బైండ్ చేయడానికి ప్రయత్నించండి
    కాంపోనెంట్ యొక్క `component name.js`లో టెక్స్ట్ ఉన్న స్ట్రింగ్ ఉండకూడదు

4. ప్రదర్శనల అంతర్జాతీయీకరణ
   - `demo/_.pug`లో, అనువదించాల్సిన లేబుల్ టెక్స్ట్ `c-t కీ పేరు` రూపంలో ఆక్రమించబడింది (ఇది అనుకూల లేబుల్, ప్రత్యేక లైన్), మరియు ఎంట్రీ `demo/i18n/source language/pug.yml`లో వ్రాయబడింది.
   - `demo/_.js` `demo/i18n/source language/js.yml`లో ఉపయోగించిన ఒరిజినల్ టెక్స్ట్ ఎంట్రీని వ్రాసి `demo/I18N.js`లో ఉంచండి
   - `fetchLang`ని `demo/_.js`లో ప్రవేశపెట్టి, `./I18N.js`ని లోడ్ చేయండి (ఇది `demo/I18N.js`), ఎగువ భాగం అంతర్జాతీయీకరణ ప్రక్రియను చూడండి (ఉదాహరణకు, `demo/i18n` క్రింద `js.yml` లేదు, ఈ దశను దాటవేయి)
   - `demo/I18N.js` `demo/i18n/source language/js.yml`లో కీ పేరు స్థిరాంకాలను కాన్ఫిగర్ చేయండి మరియు ఎగుమతి చేయండి

5. `src/webc/component name/i18n/source language/README.md`లో భాగం వివరణను వ్రాయండి. ప్రక్రియ క్రింది విధంగా ఉంది
  5.1 ప్రధాన శీర్షికను వ్రాయండి. శీర్షిక `#`తో ప్రారంభం కావాలి, ఒక వాక్యంలో భాగం యొక్క ప్రయోజనాన్ని వివరించాలి (కంపోనెంట్ పేరును వ్రాయవద్దు) మరియు చివర విరామ చిహ్నాలను జోడించవద్దు.
  5.2 ప్రధాన శీర్షికతో ప్రారంభించి, భాగం యొక్క ప్రధాన లక్షణాలను జాబితా చేయండి మరియు ద్వితీయ శీర్షికలను వ్రాయవద్దు. బహుళ లక్షణాలు ఉంటే, క్రమం లేని జాబితాను ఉపయోగించండి (క్రమం చేయని జాబితా వివరణ సాపేక్షంగా తక్కువగా ఉంటే, ముగింపులో వ్యవధి ఉండదు).
  6 `<script type="module">` ట్యాగ్, `js` కంటెంట్ యొక్క మొదటి పంక్తిలో ఇండెంటేషన్ లేదు.
  ఇది స్వచ్ఛమైన js అయితే, ` ```js `తో ప్రారంభించండి. `js` ప్రదర్శనలో, మీరు `component/_.js`ని దిగుమతి చేయవలసి వస్తే, దయచేసి "webc.site/component name.js";` నుండి `దిగుమతి xxxని ఉపయోగించండి. ఇది కంపైల్ చేయబడిన `js`, ఇది స్వయంచాలకంగా js మరియు cssని ఇంజెక్ట్ చేస్తుంది.
  `js` బహుళ స్థిరాంకాలను ప్రకటించడానికి `const` వంటి సంక్షిప్త రచనను ఉపయోగించండి

ప్రధాన వచనం కాంపోనెంట్, ఇంటర్‌ఫేస్ పారామితులను ఎలా ఉపయోగించాలో పరిచయం చేస్తుంది, వివిధ రాష్ట్రాల్లో `css` మరియు `class` మధ్య మారడం మొదలైనవి.
తదనంతరం పరామితులు, రిటర్న్ విలువ, స్టైల్ స్థితి మొదలైనవాటిని వివరించండి (ఏదైనా ఉంటే), మరియు ద్వితీయ శీర్షికను జోడించండి.

పదాలను సంక్షిప్తంగా ఉపయోగించండి మరియు విశేషణాలను నివారించండి.

కాపీ రైటింగ్ అర్ధంలేని పదాలను నివారించాలి మరియు అర్థరహిత పదాలను వ్రాయకూడదు (ఉదా: యూనివర్సల్, కంటైనర్, మొదలైనవి).
