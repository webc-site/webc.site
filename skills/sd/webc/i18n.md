# بين الاقواميت

ڪيس: حوالو ڏيو `src/webc/BoxX` اجزاء ۽ ڊيمو جي بين الاقواميت کي لاڳو ڪرڻ لاء.

قاعدو:
- `yml` شامل نه ڪريو `:` داخل ٿيڻ جي آخر ۾، `:` وجھو پگ ٽيمپليٽ ۽ js ڪوڊ ۾، ۽ ان کي ورهايو
- استعمال ڪريو `|-` جي گھڻ-لائن واري متن لاءِ `yml` آسانيءَ سان پٺتي پيل اسپيس کي هٽائڻ لاءِ.
- جيڪڏهن ڪا بين الاقوامي داخلا نه آهي، صاف ڪريو `I18N.js` فائل (ڪجهه به نه لکو، نه لکو `export`)
- داخل ٿيڻ لازمي آھي ھڪڙي ٻولي ۾، ۽ گھڻن ٻولين کي گڏ ڪرڻ سختي سان منع ٿيل آھي.

عمل:

1. روٽ ڊاريڪٽري ۾ ترتيب ڏنل ماخذ ٻولي ڪوڊ ڏسو `tran.yml` → `tran` → `from`

2. داخل ڪريو `src/webc/component name` ڊاريڪٽري

3. بين الاقوامي اجزاء
   - `component name.js` ۾ استعمال ٿيل متن لکو جيئن `src/webc/component name/i18n/source language/js.yml` ۾ داخل ٿيو.
   - تبديلي آڻيو `src/webc/component name/I18N.js` ۽ ترتيب ڏيو مسلسل جي `yml` داخلا ڪي جو نالو، جهڙوڪ `export const KEY = "ymlKey"`
   - تعارف ۽ پابند `src/webc/component name.js` ۾ (جيڪڏهن اتي آهي `js.yml`، جيڪڏهن نه، اهو ضروري ناهي):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     درآمد ڪريو i18nUrl، { KEY } مان "./component name/I18N.js"؛ // فريم ورڪ خود بخود i18nUrl انجڻ ڪندو، ڊفالٽ ايڪسپورٽ ڊفالٽ، ان کي پنهنجو پاڻ ٺاهي نه ڪريو

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    fLang ڪالن کي گهٽائڻ لاءِ بيچ ۾ پابند ڪرڻ جي ڪوشش ڪريو
    جزو جي `component name.js` ۾ متن تي مشتمل اسٽرنگ نه هجڻ گهرجي

4. پيشڪش جي بين الاقواميت
   - ۾ `demo/_.pug`، ليبل متن جنهن کي ترجمو ڪرڻ جي ضرورت آهي `c-t اهم نالو` جي صورت ۾ قبضو ڪيو ويو آهي (هي هڪ ڪسٽم ليبل آهي، هڪ الڳ لائن)، ۽ داخلا `demo/i18n/source language/pug.yml` ۾ لکيل آهي.
   - `demo/_.js` `demo/i18n/source language/js.yml` ۾ استعمال ٿيل اصل ٽيڪسٽ انٽري لکو ۽ ان کي `demo/I18N.js` ۾ رکو
   - متعارف ڪرايو `fetchLang` ۾ `demo/_.js` ۽ لوڊ ڪريو `./I18N.js` (هي آهي `demo/I18N.js`)، مٿي ڏنل جزو جي بين الاقواميت واري عمل ڏانهن رجوع ڪريو (مثال طور، `demo/i18n` وٽ هيٺ ڏنل `js.yml` ناهي، هن قدم کي ڇڏي ڏيو)
   - `demo/I18N.js` ڪنجي نالن جي مستقل ترتيب کي `demo/i18n/source language/js.yml` ۾ ترتيب ڏيو ۽ برآمد ڪريو

5. جزو جي وضاحت `src/webc/component name/i18n/source language/README.md` ۾ لکو. عمل هن ريت آهي
  5.1 مکيه عنوان لکو. عنوان `#` سان شروع ٿيڻ گهرجي، هڪ جملي ۾ جزو جو مقصد بيان ڪريو (جزيو جو نالو نه لکو)، ۽ آخر ۾ اوقاف شامل نه ڪريو.
  5.2 جزو جي مکيه خصوصيتن جي فهرست ڪريو، بنيادي عنوان سان شروع ڪريو، ۽ ثانوي عنوان نه لکو. جيڪڏھن گھڻا خاصيتون آھن، ھڪڙي غير ترتيب ڏنل فهرست استعمال ڪريو (جيڪڏھن غير ترتيب ڏنل فهرست جي وضاحت نسبتا مختصر آھي، آخر ۾ ڪو عرصو نه ھوندو).
  5.3 لکو `## ڊيمو استعمال ڪندي`، ڊيمو ڪوڊ ` ```html ` سان شروع ٿئي ٿو (ڊيمو ڪوڊ ۾، واحد عنصر ٽيگ بند نه آهي)، جيڪڏهن آهي `js`، لکو `js`، لکو `id="1">```CODE">``<b><b><b>` class="demo` استعمال ڪريو. class="CODE"> ٽيگ، `js` مواد جي پهرين قطار ۾ انڊينٽيشن نه آهي.
  جيڪڏھن اھو خالص js آھي، شروع ڪريو ` ```js ` سان. `js` مظاهري ۾، جيڪڏهن توهان کي درآمد ڪرڻ جي ضرورت آهي `component/_.js`، مهرباني ڪري استعمال ڪريو `xxx from "webc.site/component name.js";`. هي آهي مرتب ڪيل `js`، جيڪو خودڪار طريقي سان js ۽ css کي داخل ڪندو.
  `js` مختصر لکڻ استعمال ڪريو، جيئن ته `const` گھڻن مستقلن کي بيان ڪرڻ لاءِ

مکيه متن تعارف ڪري ٿو ته جزو کي ڪيئن استعمال ڪجي، انٽرفيس پيٽرولر، مختلف رياستن ۾ `css` ۽ `class` جي وچ ۾ سوئچنگ وغيره.
بعد ۾ وضاحت ڪريو پيٽرولر، واپسي جي قيمت، انداز جي حيثيت، وغيره. (جيڪڏهن ڪو به هجي)، ۽ ثانوي عنوان شامل ڪريو.

لفظن کي اختصار سان استعمال ڪريو ۽ صفتن کان پاسو ڪريو.

ڪاپي رائيٽنگ کي بي معنيٰ لفظن کان پاسو ڪرڻ گهرجي ۽ بي معنيٰ لفظ نه لکڻ گهرجي (جهڙوڪ: عالمگير، ڪنٽينر، وغيره).

6. رن `./sh/tran.js` روٽ ڊاريڪٽري جو، ترجمو
