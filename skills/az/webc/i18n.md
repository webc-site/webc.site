# beynəlmiləlləşmə

Case: Komponentlərin beynəlmiləlləşdirilməsini və nümayişi həyata keçirmək üçün `src/BoxX`-ə baxın.

qayda:
- `yml` Girişin sonuna `:` əlavə etməyin, pug şablonuna və js koduna `:` qoyun və onu bir-birinə əlavə edin
- Arxadakı boşluqları asanlıqla silmək üçün `yml` çoxsətirli mətni üçün `|-` istifadə edin.
- Əgər beynəlxalqləşdirilmiş giriş yoxdursa, `I18N.js` faylını silin (heç nə yazmayın, `export` yazmayın)
- Yazılar bir dildə olmalıdır və birdən çox dili qarışdırmaq qəti qadağandır.

proses:

1. Kök kataloqunda konfiqurasiya edilmiş mənbə dil koduna baxın `tran.yml` → `tran` → `from`

2. `src/komponent adı` kataloqunu daxil edin

3. Komponentləri beynəlmiləlləşdirin
   - `_.js` tərəfindən giriş kimi istifadə olunan mətni `src/component name/i18n/source language/js.yml`-də yazın.
   - `Komponent adı/I18N.js`-i dəyişdirin və `export const KEY = "ymlKey"` kimi `yml` giriş açarı adının sabitini konfiqurasiya edin
   - `Təqdim edilib və komponent adı/_.js` ilə bağlanıb (əgər `js.yml` varsa, lazım deyil):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     i18nUrl, { KEY } "./I18N.js"-dən idxal edin; // Çərçivə avtomatik olaraq defolt ixrac standartı olan i18nUrl inyeksiya edəcək, onu özünüz yaratmayın

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    FLang zənglərini azaltmaq üçün qruplar şəklində bağlamağa çalışın
    Komponentin `_.js` tərkibində mətn olan sətirlər olmamalıdır

4. Təqdimatların beynəlmiləlləşdirilməsi
   - `demo/_.pug`-də tərcümə edilməli olan etiket mətni `c-t açar adı` (bu fərdi etiket, ayrıca sətirdir) şəklində tutulur və giriş `demo/i18n/source language/pug.yml`-a yazılır.
   - `demo/_.js` `demo/i18n/source language/js.yml`-də istifadə edilən orijinal mətn daxiletməsini yazın və `demo/I18N.js`-də yerləşdirin.
   - `fetchLang`-ni `demo/_.js`-ə daxil edin və `./I18N.js` yükləyin (bu, `demo/I18N.js`), yuxarıdakı komponent beynəlxalqləşdirmə prosesinə baxın (məsələn, `demo/i18n` aşağıda `js.yml` yoxdur, bu addımı atlayın)
   - `demo/I18N.js` `demo/i18n/source language/js.yml`-də əsas ad sabitlərini konfiqurasiya edin və ixrac edin

5. Komponent təsvirini `Komponent adı/i18n/source language/README.md`-də yazın. Proses aşağıdakı kimidir
  5.1 Əsas başlığı yazın. Başlıq `#` ilə başlamalı, komponentin məqsədini bir cümlə ilə təsvir etməli (komponentin adını yazmayın) və sonunda durğu işarələri qoymayın.
  5.2 Əsas başlıqdan başlayaraq komponentin əsas xüsusiyyətlərini sadalayın və ikinci dərəcəli başlıqlar yazmayın. Bir neçə xüsusiyyət varsa, sıralanmamış siyahıdan istifadə edin (əgər sıralanmamış siyahının təsviri nisbətən qısadırsa, sonunda nöqtə olmayacaq).
  5.3 Yazın `## Demodan` istifadə edərək, demo kodu ` ```html ` ilə başlayır (demo kodunda, tək element teqi bağlanmır), `js` varsa, `DEMOD <b>daxili olaraq yazın. id="3">` teq;
  Əgər təmiz js-dirsə, ` ```js ` ilə başlayın. `js` Nümayişdə, əgər `component/_.js` idxal etmək lazımdırsa, lütfən, "webc.site/component name.js" ünvanından `import xxx-dən istifadə edin;`.
  `js` Çoxlu sabitləri elan etmək üçün `const` kimi qısa yazıdan istifadə edin

Əsas mətn komponentdən necə istifadə ediləcəyini, interfeys parametrlərini, müxtəlif dövlətlərdə `css` və `class` arasında keçidi və s.
Daha sonra parametrləri, qaytarılan dəyəri, stil statusunu və s. (əgər varsa) təsvir edin və ikinci dərəcəli başlıq əlavə edin.

Sözləri yığcam şəkildə istifadə edin və sifətlərdən qaçın.

Kopirayter cəfəngiyyatdan qaçmalı və mənasız sözlər yazmamalıdır (məsələn: universal, konteyner və s.).

6. Kök qovluğunun `./sh/tran.js`-ni işlədin, tərcümə edin
