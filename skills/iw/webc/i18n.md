# בִּנאוּם

מקרה: עיין ב`src/BoxX` ליישום בינלאומי של רכיבים והדגמה.

כְּלָל:
- `yml` אל תוסיף `:` בסוף הערך, הכניסו `:` לתבנית הפאג ולקוד ה-js, וחבר אותו
- השתמש ב-`|-` לטקסט מרובה שורות של `yml` כדי להסיר בקלות רווחים עוקבים.
- אם אין ערך בינלאומי, נקה את הקובץ `I18N.js` (אל תכתוב כלום, אל תכתוב `export`)
- הערכים חייבים להיות בשפה אחת, וערבוב של מספר שפות אסור בהחלט.

תַהֲלִיך:

1. הצג את קוד שפת המקור שהוגדר בספריית השורש `tran.yml` → `tran` → `from`

2. הזן את ספריית `src/component name`

3. עשה רכיבים בינלאומיים
   - כתוב את הטקסט המשמש את `_.js` כערך ב-`src/component name/i18n/source language/js.yml`.
   - שנה את `שם הרכיב/I18N.js` והגדר את הקבוע של `yml` שם מפתח הכניסה, כגון `export const KEY = "ymlKey"`
   - `מוצג ומאוגד בשם רכיב/_.js` (אם יש `js.yml`, אין צורך אם לא):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     ייבוא ​​i18nUrl, { KEY } מ-"./I18N.js"; // המסגרת תזריק אוטומטית i18nUrl, ברירת המחדל של הייצוא, אל תיצור אותה בעצמך

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    נסה לאגד בקבוצות כדי לצמצם קריאות fLang
    `_.js` של רכיב לא אמורות לכלול מחרוזות המכילות טקסט

4. בינלאומי של מצגות
   - ב-`demo/_.pug`, טקסט התווית שיש לתרגם תפוס בצורה של `שם מפתח c-t` (זוהי תווית מותאמת אישית, שורה נפרדת), והערך נכתב ב`demo/i18n/source language/pug.yml`
   - `demo/_.js` כתוב את רשומת הטקסט המקורית בשימוש ב-`demo/i18n/source language/js.yml` והצב אותו ב-`demo/I18N.js`
   - הכנס את `fetchLang` ל-`demo/_.js` וטען את `./I18N.js` (זהו `demo/I18N.js`), עיין בתהליך הבינאום של הרכיבים למעלה (לדוגמה, ל`demo/i18n` אין `js.yml` למטה, דלג על שלב זה)
   - `demo/I18N.js` הגדר קבועים של שם מפתח ב-`demo/i18n/source language/js.yml` וייצא

5. כתוב את תיאור הרכיב ב-`שם הרכיב/i18n/שפת המקור/README.md`. התהליך הוא כדלקמן
  5.1 כתוב את הכותרת הראשית. הכותרת צריכה להתחיל ב`#`, לתאר את מטרת הרכיב במשפט אחד (אל תכתוב את שם הרכיב), ואל תוסיף סימני פיסוק בסוף.
  5.2 רשום את המאפיינים העיקריים של הרכיב, החל מהכותרת הראשית, ואל תכתוב כותרות משניות. אם יש מספר מאפיינים, השתמש ברשימה לא מסודרת (אם תיאור הרשימה הלא מסודרת קצר יחסית, לא תהיה נקודה בסוף).
  5.3 כתוב `## באמצעות ההדגמה`, קוד ההדגמה מתחיל ב-` ```html ` (בקוד ההדגמה, תג הרכיב הבודד אינו סגור), אם יש `js`, כתוב `><b">תג`><b">תג פנימי`><b">`html<b">`html` internly="CODE">
  אם זה js טהור, התחל עם ` ```js `. `js` בהדגמה, אם אתה צריך לייבא את `component/_.js`, אנא השתמש ב`ייבוא ​​xxx מ-"webc.site/component/_.js";`.
  `js` השתמש בכתיבה תמציתית, כגון `const` כדי להכריז על מספר קבועים

הטקסט הראשי מציג את אופן השימוש ברכיב, פרמטרי ממשק, מעבר בין `css` ל`class` במצבים שונים וכו'.
לאחר מכן תאר את הפרמטרים, ערך ההחזרה, סטטוס הסגנון וכו' (אם יש), והוסף כותרת משנית.

השתמש במילים בצורה תמציתית והימנע משמות תואר.
