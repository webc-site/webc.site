# internasyonalisasyon

Kaso: Sumangguni sa `src/BoxX` upang ipatupad ang internasyonalisasyon ng mga bahagi at demo.

tuntunin:
- `yml` Huwag magdagdag ng `:` sa dulo ng entry, ilagay ang `:` sa template ng pug at js code, at i-splice ito
- Gamitin ang `|-` para sa multi-line na text ng `yml` upang madaling alisin ang mga trailing space.
- Kung walang internationalized na entry, i-clear ang `I18N.js` file (huwag magsulat ng kahit ano, huwag magsulat ng `export`)
- Dapat na nasa iisang wika ang mga entry, at mahigpit na ipinagbabawal ang paghahalo ng maraming wika.

proseso:

1. Tingnan ang source language code na na-configure sa root directory `tran.yml` → `tran` → `from`

2. Ilagay ang direktoryo ng `src/component name`

3. I-internationalize ang mga bahagi
   - Isulat ang text na ginamit ng `_.js` bilang isang entry sa `src/component name/i18n/source language/js.yml`.
   - Baguhin ang `Component name/I18N.js` at i-configure ang constant ng `yml` entry key name, gaya ng `export const KEY = "ymlKey"`
   - `Ipinakilala at nakatali sa pangalan ng bahagi/_.js` (kung mayroong `js.yml`, hindi kinakailangan kung hindi):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     mag-import ng i18nUrl, { KEY } mula sa "./I18N.js"; // Awtomatikong mag-i-inject ang framework ng i18nUrl, ang default na export default, huwag mo itong likhain

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Subukang mag-bind sa mga batch upang bawasan ang mga tawag sa fLang
    Ang `_.js` ng bahagi ay hindi dapat magkaroon ng mga string na naglalaman ng teksto

4. Internasyonalisasyon ng mga presentasyon
   - Sa `demo/_.pug`, ang text ng label na kailangang isalin ay inookupahan sa anyo ng `c-t key name` (ito ay isang custom na label, isang hiwalay na linya), at ang entry ay nakasulat sa `demo/i18n/source language/pug.yml`
   - `demo/_.js` Isulat ang orihinal na text entry na ginamit sa `demo/i18n/source language/js.yml` at ilagay ito sa `demo/I18N.js`
   - Ipasok ang `fetchLang` sa `demo/_.js` at i-load ang `./I18N.js` (ito ay `demo/I18N.js`), sumangguni sa bahaging proseso ng internationalization sa itaas (halimbawa, ang `demo/i18n` ay walang `js.yml` sa ibaba, laktawan ang hakbang na ito)
   - `demo/I18N.js` I-configure ang key name constants sa `demo/i18n/source language/js.yml` at i-export

5. Isulat ang paglalarawan ng bahagi sa `Pangalan ng bahagi/i18n/source language/README.md`. Ang proseso ay ang mga sumusunod
  5.1 Isulat ang pangunahing pamagat. Ang pamagat ay dapat magsimula sa `#`, ilarawan ang layunin ng bahagi sa isang pangungusap (huwag isulat ang pangalan ng bahagi), at huwag magdagdag ng bantas sa dulo.
  5.2 Ilista ang mga pangunahing katangian ng bahagi, simula sa pangunahing pamagat, at huwag sumulat ng pangalawang pamagat. Kung maraming katangian, gumamit ng hindi nakaayos na listahan (kung medyo maikli ang paglalarawan ng hindi nakaayos na listahan, walang tuldok sa dulo).
  5.3 Sumulat ng `## Gamit ang demo`, magsisimula ang demo code sa ` ```html ` (sa demo code, hindi nakasara ang solong tag ng elemento), kung mayroong `js`, isulat ang `html</b class="CODE"> internally;><b class="i-CODE">
  Kung ito ay purong js, magsimula sa ` ```js `. `js` Sa demonstration, kung kailangan mong mag-import ng `component/_.js`, mangyaring gumamit ng `import xxx mula sa "webc.site/component/_.js";`.
  `js` Gumamit ng maigsi na pagsulat, gaya ng `const` para magdeklara ng maraming constants

Ipinapakilala ng pangunahing text kung paano gamitin ang bahagi, mga parameter ng interface, pagpapalit sa pagitan ng `css` at `class` sa iba't ibang mga estado, atbp.
Kasunod na ilarawan ang mga parameter, return value, style status, atbp. (kung mayroon), at magdagdag ng pangalawang pamagat.

Gumamit ng mga salita nang maigsi at iwasan ang mga pang-uri.

Dapat iwasan ng copywriting ang walang kapararakan at huwag magsulat ng mga walang kabuluhang salita (tulad ng: unibersal, lalagyan, atbp.).
