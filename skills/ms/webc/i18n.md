# pengantarabangsaan

Kes: Rujuk `src/webc/BoxX` untuk melaksanakan pengantarabangsaan komponen dan demo.

peraturan:
- `yml` Jangan tambah `:` pada penghujung entri, letakkan `:` ke dalam templat pug dan kod js, dan sambungkannya
- Gunakan `|-` untuk teks berbilang baris `yml` untuk mengalih keluar ruang belakang dengan mudah.
- Jika tiada kemasukan antarabangsa, kosongkan fail `I18N.js` (jangan tulis apa-apa, jangan tulis `export`)
- Penyertaan mestilah dalam satu bahasa, dan pencampuran berbilang bahasa adalah dilarang sama sekali.

proses:

1. Lihat kod bahasa sumber yang dikonfigurasikan dalam direktori akar `tran.yml` → `tran` → `from`

2. Masukkan direktori `src/webc/component name`

3. Mengantarabangsakan komponen
   - Tulis teks yang digunakan dalam `nama komponen.js` sebagai entri ke dalam `src/webc/component name/i18n/source language/js.yml`.
   - Ubah suai `src/webc/component name/I18N.js` dan konfigurasikan pemalar `yml` nama kunci masukan, seperti `export const KEY = "ymlKey"`
   - Pengenalan dan pengikatan dalam `src/webc/component name.js` (jika terdapat `js.yml`, jika tidak, ia tidak perlu):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     import i18nUrl, { KEY } daripada "./component name/I18N.js"; // Rangka kerja akan menyuntik i18nUrl secara automatik, lalai eksport lalai, jangan buat sendiri

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Cuba ikat dalam kelompok untuk mengurangkan panggilan fLang
    `nama komponen.js` komponen tidak sepatutnya mempunyai rentetan yang mengandungi teks

4. Pengantarabangsaan pembentangan
   - Dalam `demo/_.pug`, teks label yang perlu diterjemahkan diduduki dalam bentuk `nama kunci c-t` (ini ialah label tersuai, baris yang berasingan) dan entri ditulis ke dalam `demo/i18n/source language/pug.yml`
   - `demo/_.js` Tulis entri teks asal yang digunakan dalam `demo/i18n/source language/js.yml` dan letakkannya dalam `demo/I18N.js`
   - Perkenalkan `fetchLang` ke dalam `demo/_.js` dan muatkan `./I18N.js` (ini ialah `demo/I18N.js`), rujuk proses pengantarabangsaan komponen di atas (contohnya, `demo/i18n` tidak mempunyai `js.yml` di bawah, langkau langkah ini)
   - `demo/I18N.js` Konfigurasikan pemalar nama kunci dalam `demo/i18n/source language/js.yml` dan eksport

5. Tulis penerangan komponen dalam `src/webc/component name/i18n/source language/README.md`. Prosesnya adalah seperti berikut
  5.1 Tulis tajuk utama. Tajuk hendaklah bermula dengan `#`, terangkan tujuan komponen dalam satu ayat (jangan tulis nama komponen), dan jangan tambah tanda baca di hujungnya.
  5.2 Senaraikan ciri utama komponen, bermula dengan tajuk utama, dan jangan tulis tajuk sekunder. Jika terdapat berbilang ciri, gunakan senarai tidak tertib (jika perihalan senarai tidak tersusun agak pendek, tiada noktah pada penghujungnya).
  5.3 Tulis `## Menggunakan demo`, kod demo bermula dengan ` ```html ` (dalam kod demo, teg elemen tunggal tidak ditutup), jika ada `js`, tulis `html` dalaman class="CODE"> Tag, `js` Tidak mempunyai lekukan dalam baris pertama kandungan.
  Jika ia adalah js tulen, mulakan dengan ` ```js `. `js` Dalam demonstrasi, jika anda perlu mengimport `komponen/_.js`, sila gunakan `import xxx daripada "webc.site/component name.js";`. Ini ialah `js` yang disusun, yang akan menyuntik js dan css secara automatik.
  `js` Gunakan tulisan ringkas, seperti `const` untuk mengisytiharkan berbilang pemalar

Teks utama memperkenalkan cara menggunakan komponen, parameter antara muka, bertukar antara `css` dan `class` dalam keadaan berbeza, dsb.
Seterusnya, huraikan parameter, nilai pulangan, status gaya, dsb. (jika ada), dan tambahkan tajuk kedua.

Gunakan perkataan dengan ringkas dan elakkan kata adjektif.

Copywriting harus mengelakkan perkara karut dan tidak menulis perkataan yang tidak bermakna (seperti: universal, bekas, dll.).

6. Jalankan `./sh/tran.js` direktori akar, terjemahan
