# internationalisation

Cas : reportez-vous à `src/webc/BoxX` pour implémenter l'internationalisation des composants et de la démo.

règle:
- `yml` N'ajoutez pas `:` à la fin de l'entrée, mettez `:` dans le modèle pug et le code js, et collez-le
- Utilisez `|-` pour le texte multiligne de `yml` afin de supprimer facilement les espaces de fin.
- S'il n'y a pas d'entrée internationalisée, effacez le fichier `I18N.js` (n'écrivez rien, n'écrivez pas `export`)
- Les candidatures doivent être rédigées dans une seule langue et le mélange de plusieurs langues est strictement interdit.

processus:

1. Afficher le code de langue source configuré dans le répertoire racine `tran.yml` → `tran` → `from`

2. Entrez le répertoire `src/webc/component name`

3. Internationaliser les composants
   - Écrivez le texte utilisé dans `component name.js` comme entrée dans `src/webc/component name/i18n/source language/js.yml`.
   - Modifiez `src/webc/component name/I18N.js` et configurez la constante du nom de la clé d'entrée `yml`, telle que `export const KEY = "ymlKey"`
   - Introduction et liaison dans `src/webc/component name.js` (s'il y a `js.yml`, sinon, ce n'est pas nécessaire) :
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     importer i18nUrl, { KEY } depuis "./nom du composant/I18N.js" ; // Le framework injectera automatiquement i18nUrl, la valeur par défaut de l'export, ne le créez pas vous-même

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Essayez de lier par lots pour réduire les appels fLang
    Le `nom du composant.js` du composant ne doit pas contenir de chaîne contenant du texte.

4. Internationalisation des présentations
   - Dans `demo/_.pug`, le texte de l'étiquette qui doit être traduit est occupé sous la forme de `nom de clé c-t` (il s'agit d'une étiquette personnalisée, une ligne séparée), et l'entrée est écrite dans `demo/i18n/source language/pug.yml`
   - `demo/_.js` Écrivez l'entrée de texte originale utilisée dans `demo/i18n/source language/js.yml` et placez-la dans `demo/I18N.js`
   - Introduisez `fetchLang` dans `demo/_.js` et chargez `./I18N.js` (il s'agit de `demo/I18N.js`), reportez-vous au processus d'internationalisation des composants ci-dessus (par exemple, `demo/i18n` n'a pas `js.yml` ci-dessous, ignorez cette étape)
   - `demo/I18N.js` Configurer les constantes de nom de clé dans `demo/i18n/source language/js.yml` et exporter

5. Écrivez la description du composant dans `src/webc/component name/i18n/source language/README.md`. Le processus est le suivant
  5.1 Écrivez le titre principal. Le titre doit commencer par `#`, décrire l'objectif du composant en une phrase (n'écrivez pas le nom du composant) et n'ajoutez pas de ponctuation à la fin.
  5.2 Répertoriez les principales caractéristiques du composant, en commençant par le titre principal, et n'écrivez pas de titres secondaires. S'il y a plusieurs caractéristiques, utilisez une liste non ordonnée (si la description de la liste non ordonnée est relativement courte, il n'y aura pas de point à la fin).
  5.3 Écrire `## En utilisant la démo`, le code de la démo commence par ` ```html ` (dans le code de la démo, la balise d'un seul élément n'est pas fermée), s'il y a `js`, écrivez `html` en interne, utilisez `<script type="module">` Balise, `js` Ne pas avoir d'indentation dans la première ligne du contenu.
  S'il s'agit de pur js, commencez par ` ```js `. `js` Dans la démonstration, si vous devez importer `component/_.js`, veuillez utiliser `import xxx from "webc.site/component name.js";`. Il s'agit du `js` compilé, qui injectera automatiquement js et css.
  `js` Utilisez une écriture concise, telle qu'un `const` pour déclarer plusieurs constantes

Le texte principal présente comment utiliser le composant, les paramètres d'interface, le basculement entre `css` et `class` dans différents états, etc.
Décrivez ensuite les paramètres, la valeur de retour, l'état du style, etc. (le cas échéant), et ajoutez un titre secondaire.

Utilisez les mots de manière concise et évitez les adjectifs.

La rédaction doit éviter les bêtises et n'écrire pas de mots dénués de sens (tels que : universel, conteneur, etc.).

6. Exécutez `./sh/tran.js` du répertoire racine, traduction
