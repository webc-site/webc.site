# დემო

იხილეთ `src/BoxX/demo` და განავითარეთ კომპონენტის დემო ვერსია `src/component name/demo/`

`demo/_.pug`, დემო ქვეთარგი

მხოლოდ მინიმალისტური კომპონენტის ტეგები შედის, სრული გვერდი არ არის საჭირო და სტილისა და სკრიპტების იმპორტი აკრძალულია. ჩარჩო ავტომატურად იქნება ინექცია (როგორიცაა `demo/_.styl`, `demo/_.js`).

არ დაწეროთ სათაური (მაგალითად, არ დაწეროთ xxx დემო), არ დაწეროთ კომპონენტის სახელი, არ შეიტანოთ კომპონენტი, დაწეროთ მხოლოდ დემო გამოყენების შემთხვევის სტატუსი (თუ არ არის სტატუსი, ჩაწერეთ სათაურის გარეშე)

`_.pug`-ის ძირეული ელემენტი იყენებს `main.demo.Lg` , რომელმაც განსაზღვრა სტილი `demo/_.styl`-ში და არ განსაზღვრავს თავად `main.demo.Lg` სტილს.

ჩვენების კომპონენტის კონტეინერი უნდა იყოს გასწორებული `main.demo.Lg`-თან და მათ შორის არ უნდა იყოს `margin` და `padding`, რადგან თავად `main.demo` უკვე დააყენა `padding` და მისი ცარიელი დაყენება განმეორდება.

`demo/_.styl` თუ ეკრანის მხოლოდ ერთი კომპონენტია, მის კონტეინერს არ ექნება გარე ჩარჩო და მხოლოდ დააყენებს `max-width` (რადგან პრეზენტაციის კონტეინერი ავტომატურად დაამატებს გარე ჩარჩოს). თუ რამდენიმე მათგანია, კონტეინერი შეიძლება გამოიყოს გარე ჩარჩოთი.

არ დაწეროთ `img` `demo/_.pug`-ში, გამოიყენეთ `demo/_.styl` ფონის სურათის დასაყენებლად

თუ კომპონენტს აქვს მრავალი ფორმა, გთხოვთ აჩვენოთ ისინი თანმიმდევრობით (როგორიცაა ჩატვირთვა, ცარიელი მონაცემები, მონაცემები, წარუმატებელი და ა.შ.), მოქნილი ვერტიკალური განლაგება

`demo/_.styl`: `stylus` დემო გვერდის სტილი, გთხოვთ, მიჰყვეთ [./styl.md](./styl.md) კოდის სპეციფიკაციებს და დიზაინის ესთეტიკას.

არ დაწეროთ `@import '../_.styl'` დემონსტრირებადი კომპონენტის იმპორტისთვის, ის ავტომატურად იქნება ინექცია

გთხოვთ შემოიტანოთ `@import '../../Btn/_.styl'` `demo/_.styl`-ში და გამოიყენოთ `.Btn`

`demo/_.js`: სკრიპტის გამოძახება, ფორმატი `export default (root) => { ... }`, პარამეტრი `root` არის ჰოსტი (`document` ან sandbox კონტეინერის ძირეული ელემენტი)

`D.createElement` აკრძალულია, გამოიყენეთ [./js.md](./js.md) მიერ აღნიშნული `newEl`

`js` აკრძალულია გამოყენება `c-t` და ინტერნაციონალიზაციისთვის `fLang`

თუ `../_.js` (`../_.js` უნდა აჩვენოს თავად კომპონენტი, სხვა დემოსთვის საჭირო დამოკიდებულ კომპონენტებს ჯერ კიდევ სჭირდება იმპორტი) საჭიროა მხოლოდ `import` და არ არის იმპორტირებული კონკრეტული კონტენტი, რომლის გამოტოვებაც შესაძლებელია (ჩარჩო ავტომატურად ჩაუშვებს მას)

`pug` არ არის საჭირო `js` და `styl` იმპორტი, ფრეიმვორკი მათ ავტომატურად გაუკეთებს
