# ინტერნაციონალიზაცია

შემთხვევა: იხილეთ `src/BoxX` კომპონენტებისა და დემო ინტერნაციონალიზაციის განსახორციელებლად.

წესი:
- `yml` არ დაამატოთ `:` ჩანაწერის ბოლოს, ჩასვით `:` pug შაბლონში და js კოდში და გააერთიანეთ იგი
- გამოიყენეთ `|-` `yml`-ის მრავალსტრიქონიანი ტექსტისთვის, რათა ადვილად წაშალოთ უკანა სივრცეები.
- თუ არ არის ინტერნაციონალიზებული ჩანაწერი, გაასუფთავეთ `I18N.js` ფაილი (არაფერი დაწეროთ, არ დაწეროთ `export`)
- ჩანაწერები უნდა იყოს ერთ ენაზე და რამდენიმე ენის შერევა მკაცრად აკრძალულია.

პროცესი:

1. იხილეთ საწყისი ენის კოდი, რომელიც კონფიგურირებულია root დირექტორიაში `tran.yml` → `tran` → `from`

2. შეიყვანეთ `src/კომპონენტის სახელი` დირექტორია

3. კომპონენტების ინტერნაციონალიზაცია
   - ჩაწერეთ ტექსტი, რომელსაც იყენებს `_.js`, როგორც ჩანაწერი `src/component name/i18n/source language/js.yml`.
   - შეცვალეთ `კომპონენტის სახელი/I18N.js` და დააკონფიგურირეთ `yml` შეყვანის გასაღების სახელის მუდმივი, როგორიცაა `export const KEY = "ymlKey"`
   - `შეტანილი და შეკრული კომპონენტის სახელში/_.js` (თუ არის `js.yml`, არ არის აუცილებელი, თუ არა):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     იმპორტი i18nUrl, { KEY } „./I18N.js“-დან; // Framework ავტომატურად შეაქვს i18nUrl, ნაგულისხმევი ექსპორტის ნაგულისხმევი, ნუ შექმნით მას

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    სცადეთ დაკავშირება ჯგუფურად, რათა შეამციროთ fLang ზარები
    კომპონენტის `_.js` არ უნდა იყოს ტექსტის შემცველი სტრიქონები

4. პრეზენტაციების ინტერნაციონალიზაცია
   - `demo/_.pug`-ში, ლეიბლის ტექსტი, რომელიც უნდა ითარგმნოს, არის დაკავებული `c-t გასაღების სახელით` (ეს არის მორგებული ლეიბლი, ცალკე ხაზი) ​​და ჩანაწერი იწერება `demo/i18n/source language/pug.yml`
   - `demo/_.js` ჩაწერეთ ორიგინალური ტექსტის ჩანაწერი, რომელიც გამოიყენება `დემო/i18n/source language/js.yml`-ში და მოათავსეთ `demo/I18N.js`-ში
   - შეიტანეთ `fetchLang` `demo/_.js`-ში და ჩატვირთეთ `./I18N.js` (ეს არის `demo/I18N.js`), იხილეთ კომპონენტის ინტერნაციონალიზაციის პროცესი ზემოთ (მაგალითად, `demo/i18n` არ აქვს `js.yml` ქვემოთ, გამოტოვეთ ეს ნაბიჯი)
   - `demo/I18N.js` გასაღების სახელების მუდმივების კონფიგურაცია `demo/i18n/source language/js.yml`-ში და ექსპორტი

5. ჩაწერეთ კომპონენტის აღწერა `კომპონენტის სახელი/i18n/source language/README.md`. პროცესი შემდეგია
  5.1 დაწერეთ მთავარი სათაური. სათაური უნდა იწყებოდეს `#`-ით, აღწეროთ კომპონენტის დანიშნულება ერთი წინადადებით (არ დაწეროთ კომპონენტის სახელი) და არ დაამატოთ პუნქტუაცია ბოლოს.
  5.2 ჩამოთვალეთ კომპონენტის ძირითადი მახასიათებლები ძირითადი სათაურით დაწყებული და არ დაწეროთ მეორადი სათაურები. თუ არსებობს მრავალი მახასიათებელი, გამოიყენეთ შეუკვეთავი სია (თუ შეუკვეთავი სიის აღწერა შედარებით მოკლეა, ბოლოს პერიოდი არ იქნება).
  5.3 ჩაწერეთ `## დემო-ს გამოყენებით`, დემო კოდი იწყება ` ```html ` (დემო კოდში, ერთი ელემენტის ტეგი არ არის დახურული), თუ არის `js`, ჩაწერეთ `interally class="CODE"> <b>class="CODE">. id="3">` ტეგი;
  თუ ის არის სუფთა js, დაიწყეთ ` ```js `-ით. `js` დემონსტრაციაში, თუ გჭირდებათ `კომპონენტის/_.js` იმპორტი, გთხოვთ, გამოიყენოთ ` xxx-ის იმპორტი "webc.site/component name.js";`-დან.
  `js` გამოიყენეთ ლაკონური წერა, როგორიცაა `const` მრავალი მუდმივის გამოსაცხადებლად

ძირითადი ტექსტი წარმოგიდგენთ კომპონენტის გამოყენებას, ინტერფეისის პარამეტრებს, `css`-სა და `class`-ს შორის გადართვას სხვადასხვა მდგომარეობაში და ა.შ.
შემდეგ აღწერეთ პარამეტრები, დაბრუნებული მნიშვნელობა, სტილის სტატუსი და ა.შ. (ასეთის არსებობის შემთხვევაში) და დაამატეთ მეორადი სათაური.

გამოიყენეთ სიტყვები ლაკონურად და მოერიდეთ ზედსართავ სახელებს.
