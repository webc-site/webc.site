# 国际化

案例: 参考 `src/BoxX` 实现组件与 Demo 的国际化。

规则：
- `yml` 词条不在结尾加 `:`，把 `:` 放到 pug 模板、js 代码中，拼接
- `yml` 的多行文本用 `|-`，方便去掉结尾空格
- 如无国际化词条，清空 `I18N.js` 文件（什么内容不要写，不要写 `export`）
- 词条用单一语言，严禁多语言混杂

流程:

1. 查看根目录下 `tran.yml` → `tran` → `from` 配置的源语言代码

2. 进入 `src/组件名`目录

3. 对组件国际化
   - 将 `_.js` 用到的文本，作为词条写入 `src/组件名/i18n/源语言/js.yml`。
   - 修改 `组件名/I18N.js`，配置 `yml` 词条键名的常量，如 `export const KEY = "ymlKey"`
   - `组件名/_.js` 中引入并绑定（如有 `js.yml`，没有不需要）：
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     import i18nUrl, { KEY } from "./I18N.js"; // 框架会自动注入 i18nUrl 这个默认的 export default，不要自己创建

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    尽量批量绑定，减少 fLang 的调用
    组件的 `_.js` 不要有包含文字的字符串

4. 演示的国际化
   - `demo/_.pug` 中，需要翻译的标签文本通过 `c-t 键名` 形式占位（这是自定义标签，单独一行 )，并将词条写入 `demo/i18n/源语言/pug.yml`
   - `demo/_.js` 用到的原文词条写入 `demo/i18n/源语言/js.yml`，并在 `demo/I18N.js`
   - `demo/_.js` 中引入 `fetchLang` 加载 `./I18N.js` （这是 `demo/I18N.js`），参考上面的组件国际化流程（如 `demo/i18n` 下面没有 `js.yml`，跳过这一步）
   - `demo/I18N.js` 配置 `demo/i18n/源语言/js.yml` 中的键名常量，导出

5. 在 `组件名/i18n/源语言/README.md` 写组件说明，流程如下
  5.1 写主标题， 标题以 `#` 开头，一句话描述组件用途 (不写组件名)，结尾不加标点。
  5.2 列出组件的主要特性，接着主标题开始，不写二级标题。如有多条特性，用无序列表(无序列表描述如果比较短，结尾不加句号)。
  5.3 写`## 使用演示`，演示代码用 ` ```html ` 开头(演示代码中，单元素标签不加闭合)，如有 `js`，写 `html` 内部，用`<script type="module">` 标签；
  如果是纯 js，用` ```js `开头。`js` 演示中，如需导入 `组件/_.js`，请用 `import xxx from "webc.site/组件/_.js";`。
  `js` 用简洁的写法，比如一个 `const` 声明多个常量

正文介绍组件如何使用、接口参数、不同状态 `css` 的 `class` 的切换等。
后续描述参数、返回值、样式状态等(如有)，加二级标题。

用词要简洁，不写形容词。

文案要避免废话，不写没有意义的词（比如：通用、容器等等）。

6. 运行根目录的 `./sh/tran.js`，翻译
