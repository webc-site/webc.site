# internacionalização

Caso: Consulte `src/BoxX` para implementar a internacionalização de componentes e demonstração.

regra:
- `yml` Não adicione `:` no final da entrada, coloque `:` no modelo pug e no código js e emende-o
- Use `|-` para texto multilinha de `yml` para remover facilmente espaços finais.
- Se não houver entrada internacionalizada, limpe o arquivo `I18N.js` (não escreva nada, não escreva `export`)
- As inscrições devem ser em um único idioma, sendo estritamente proibida a mistura de vários idiomas.

processo:

1. Visualize o código do idioma fonte configurado no diretório raiz `tran.yml` → `tran` → `from`

2. Insira o diretório `src/nome do componente`

3. Internacionalizar componentes
   - Escreva o texto usado por `_.js` como uma entrada em `src/nome do componente/i18n/source language/js.yml`.
   - Modifique `Nome do componente/I18N.js` e configure a constante do nome da chave de entrada `yml`, como `export const KEY = "ymlKey"`
   - `Introduzido e vinculado no componente name/_.js` (se houver `js.yml`, caso contrário não é necessário):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     importar i18nUrl, {KEY} de "./I18N.js"; // O framework injetará automaticamente i18nUrl, o padrão de exportação padrão, não crie você mesmo

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Tente vincular em lotes para reduzir chamadas de fLang
    O `_.js` do componente não deve conter strings contendo texto

4. Internacionalização de apresentações
   - Em `demo/_.pug`, o texto do rótulo que precisa ser traduzido é ocupado na forma de `nome da chave c-t` (este é um rótulo personalizado, uma linha separada) e a entrada é escrita em `demo/i18n/source language/pug.yml`
   - `demo/_.js` Escreva a entrada de texto original usada em `demo/i18n/source language/js.yml` e coloque-a em `demo/I18N.js`
   - Insira `fetchLang` em `demo/_.js` e carregue `./I18N.js` (este é `demo/I18N.js`), consulte o processo de internacionalização do componente acima (por exemplo, `demo/i18n` não tem `js.yml` abaixo, pule esta etapa)
   - `demo/I18N.js` Configure constantes de nome de chave em `demo/i18n/source language/js.yml` e exporte

5. Escreva a descrição do componente em `Nome do componente/i18n/idioma de origem/README.md`. O processo é o seguinte
  5.1 Escreva o título principal. O título deve começar com `#`, descrever a finalidade do componente em uma frase (não escrever o nome do componente) e não adicionar pontuação no final.
  5.2 Liste as principais características do componente, começando pelo título principal, e não escreva títulos secundários. Se houver múltiplas características, use uma lista não ordenada (se a descrição da lista não ordenada for relativamente curta, não haverá ponto final).
  5.3 Escreva `## Usando a demonstração`, o código de demonstração começa com ` ```html ` (no código de demonstração, a tag de elemento único não é fechada), se houver `js`, escreva `html` internamente, use a tag `<script type="module">`;
  Se for js puro, comece com ` ```js `. `js` Na demonstração, se você precisar importar `component/_.js`, use `import xxx from "webc.site/component/_.js";`.
  `js` Use uma escrita concisa, como `const` para declarar múltiplas constantes

O texto principal apresenta como usar o componente, parâmetros de interface, alternar entre `css` e `class` em diferentes estados, etc.
Posteriormente, descreva os parâmetros, valor de retorno, status do estilo, etc. (se houver) e adicione um título secundário.

Use palavras de forma concisa e evite adjetivos.

A redação deve evitar bobagens e não escrever palavras sem sentido (como: universal, container, etc.).

6. Execute `./sh/tran.js` do diretório raiz, tradução
