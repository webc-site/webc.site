# միջազգայնացումը

Դեպք՝ տես `src/BoxX`՝ բաղադրիչների միջազգայնացում և ցուցադրություն իրականացնելու համար:

կանոն.
- `yml` Մի ավելացրեք `:` մուտքի վերջում, դրեք `:` pug ձևանմուշի և js կոդի մեջ և միացրեք այն:
- Օգտագործեք `|-` `yml`-ի բազմատող տեքստի համար, որպեսզի հեշտությամբ հեռացնեք վերջին բացատները:
- Եթե ​​միջազգայնացված գրառում չկա, մաքրեք `I18N.js` ֆայլը (ոչինչ մի գրեք, մի գրեք `export`)
- Գրառումները պետք է լինեն մեկ լեզվով, և մի քանի լեզուների խառնումը խստիվ արգելվում է:

գործընթաց:

1. Դիտեք արմատային գրացուցակում կազմաձևված սկզբնաղբյուր լեզվի կոդը `tran.yml` → `tran` → `from`

2. Մուտքագրեք `src/component name` գրացուցակը

3. Միջազգայնացնել բաղադրիչները
   - Գրեք `_.js`-ի կողմից օգտագործված տեքստը որպես մուտք `src/component name/i18n/source language/js.yml`-ում:
   - Փոփոխեք `Բաղադրիչի անունը/I18N.js` և կազմաձևեք `yml` մուտքի բանալի անվան հաստատունը, ինչպիսին է `export const KEY = "ymlKey"`
   - `Ներդրված և ամրացված բաղադրիչի անունով/_.js` (եթե կա `js.yml`, անհրաժեշտ չէ, եթե ոչ):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     ներմուծել i18nUrl, { KEY } «./I18N.js»-ից; // Շրջանակը ավտոմատ կերպով կներարկի i18nUrl, արտահանման լռելյայն լռելյայն, մի ստեղծեք այն ինքներդ

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Փորձեք խմբաքանակով կապել՝ fLang զանգերը նվազեցնելու համար
    Բաղադրիչի `_.js`-ը չպետք է ունենա տեքստ պարունակող տողեր

4. Ներկայացումների միջազգայնացում
   - `demo/_.pug`-ում պիտակի տեքստը, որը պետք է թարգմանվի, գրված է `c-t բանալի անվան` տեսքով (սա հատուկ պիտակ է, առանձին տող), և մուտքը գրված է `demo/i18n/source language/pug.yml`:
   - `demo/_.js` Գրեք բնօրինակ տեքստի մուտքագրումը, որն օգտագործվում է `demo/i18n/source language/js.yml`-ում և տեղադրեք այն `demo/I18N.js`-ում:
   - Ներդրեք `fetchLang`-ը `demo/_.js`-ում և բեռնեք `./I18N.js` (սա `demo/I18N.js` է), նայեք վերը նշված բաղադրիչի միջազգայնացման գործընթացին (օրինակ, `demo/i18n`-ը չունի `js.yml` ստորև, բաց թողեք այս քայլը)
   - `demo/I18N.js` Կարգավորեք բանալիների անվան հաստատունները `demo/i18n/source language/js.yml`-ում և արտահանեք

5. Բաղադրիչի նկարագրությունը գրեք `Բաղադրիչի անվանում/i18n/source language/README.md`-ում: Գործընթացը հետեւյալն է
  5.1 Գրեք հիմնական վերնագիրը: Վերնագիրը պետք է սկսվի `#`-ով, նկարագրի բաղադրիչի նպատակը մեկ նախադասությամբ (մի գրեք բաղադրիչի անունը) և վերջում չավելացնեք կետադրական նշաններ:
  5.2 Նշե՛ք բաղադրիչի հիմնական հատկանիշները՝ սկսած հիմնական վերնագրից, և մի՛ գրեք երկրորդական վերնագրեր: Եթե ​​կան մի քանի բնութագրեր, օգտագործեք չդասավորված ցուցակը (եթե չպատվիրված ցանկի նկարագրությունը համեմատաբար կարճ է, վերջում ժամանակաշրջան չի լինի):
  5.3 Գրեք `## Օգտագործելով ցուցադրությունը`, ցուցադրական կոդը սկսվում է ` ```html `-ով (դեմո կոդում մեկ տարրի պիտակը փակված չէ), եթե կա `js`, գրեք `internal class="CODE"> `internal class="CODE">: id="3">` պիտակ;
  Եթե ​​այն մաքուր js է, սկսեք ` ```js `-ով: `js` Ցուցադրման ժամանակ, եթե Ձեզ անհրաժեշտ է ներմուծել `բաղադրիչ/_.js`, խնդրում ենք օգտագործել `ներմուծել xxx «webc.site/component name.js»;`-ից:
  `js` Օգտագործեք հակիրճ գրություն, օրինակ `const` մի քանի հաստատուններ հայտարարելու համար

Հիմնական տեքստը ներկայացնում է, թե ինչպես օգտագործել բաղադրիչը, ինտերֆեյսի պարամետրերը, անցումը `css`-ի և `class`-ի միջև տարբեր վիճակներում և այլն:
Հետագայում նկարագրեք պարամետրերը, վերադարձի արժեքը, ոճի կարգավիճակը և այլն (եթե այդպիսիք կան), և ավելացրեք երկրորդական վերնագիր:

Օգտագործեք բառերը հակիրճ և խուսափեք ածականներից:

Copywriting-ից պետք է խուսափել անհեթեթությունից և չգրել անիմաստ բառեր (օրինակ՝ ունիվերսալ, կոնտեյներ և այլն):

6. Գործարկեք `./sh/tran.js` արմատային գրացուցակը, թարգմանությունը
