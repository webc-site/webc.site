# internaţionalizare

Caz: Consultați `src/BoxX` pentru a implementa internaționalizarea componentelor și demonstrația.

regulă:
- `yml` Nu adăugați `:` la sfârșitul intrării, introduceți `:` în șablonul pug și codul js și îmbinați-l
- Folosiți `|-` pentru textul pe mai multe rânduri din `yml` pentru a elimina cu ușurință spațiile din urmă.
- Dacă nu există o intrare internaționalizată, ștergeți fișierul `I18N.js` (nu scrieți nimic, nu scrieți `export`)
- Intrările trebuie să fie într-o singură limbă, iar amestecarea mai multor limbi este strict interzisă.

proces:

1. Vizualizați codul limbii sursă configurat în directorul rădăcină `tran.yml` → `tran` → `from`

2. Introduceți directorul `src/component name`

3. Internaționalizați componente
   - Scrieți textul folosit de `_.js` ca intrare în `src/component name/i18n/source language/js.yml`.
   - Modificați `Numele componentei/I18N.js` și configurați constanta `yml` numele cheii de intrare, cum ar fi `export const KEY = "ymlKey"`
   - `Introdus și legat în numele componentei/_.js` (dacă există `js.yml`, nu este necesar dacă nu):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     import i18nUrl, { KEY } din „./I18N.js”; // Cadrul va injecta automat i18nUrl, implicit implicit de export, nu îl creați singur

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Încercați să legați în loturi pentru a reduce apelurile fLang
    Componenta `_.js` nu trebuie să aibă șiruri de caractere care să conțină text

4. Internaționalizarea prezentărilor
   - În `demo/_.pug`, textul etichetei care trebuie tradus este ocupat sub forma `nume cheie c-t` (aceasta este o etichetă personalizată, o linie separată), iar intrarea este scrisă în `demo/i18n/source language/pug.yml`
   - `demo/_.js` Scrieți textul original folosit în `demo/i18n/source language/js.yml` și plasați-o în `demo/I18N.js`
   - Introduceți `fetchLang` în `demo/_.js` și încărcați `./I18N.js` (acesta este `demo/I18N.js`), consultați procesul de internaționalizare a componentelor de mai sus (de exemplu, `demo/i18n` nu are `js.yml` mai jos, omiteți acest pas)
   - `demo/I18N.js` Configurați constantele numelui cheii în `demo/i18n/source language/js.yml` și exportați

5. Scrieți descrierea componentei în `Numele componentei/i18n/sursă limba/README.md`. Procesul este după cum urmează
  5.1 Scrieți titlul principal. Titlul trebuie să înceapă cu `#`, să descrie scopul componentei într-o singură propoziție (nu scrieți numele componentei) și să nu adăugați semne de punctuație la sfârșit.
  5.2 Enumerați caracteristicile principale ale componentei, începând cu titlul principal și nu scrieți titluri secundare. Dacă există mai multe caracteristici, utilizați o listă neordonată (dacă descrierea listei neordonate este relativ scurtă, nu va exista un punct la sfârșit).
  5.3 Scrieți `## Folosind demo`, codul demo începe cu ` ```html ` (în codul demo, eticheta cu un singur element nu este închisă), dacă există `js`, scrieți `de html<b>"id="CODE"> în mod intern`><`3 etichetă;
  Dacă este js pur, începeți cu ` ```js `. `js` În demonstrație, dacă trebuie să importați `component/_.js`, vă rugăm să utilizați `import xxx din „webc.site/component/_.js";`.
  `js` Utilizați scrierea concisă, cum ar fi un `const` pentru a declara mai multe constante

Textul principal prezintă modul de utilizare a componentei, parametrii interfeței, comutarea între `css` și `class` în diferite stări etc.
Descrieți ulterior parametrii, valoarea returnată, starea stilului etc. (dacă există) și adăugați un titlu secundar.
