# internacionalizace

Případ: Pro implementaci internacionalizace komponent a ukázky viz `src/webc/BoxX`.

pravidlo:
- `yml` Nepřidávejte `:` na konec záznamu, vložte `:` do šablony pug a kódu js a spojte jej
- Použijte `|-` pro víceřádkový text `yml`, abyste snadno odstranili mezery na konci.
- Pokud neexistuje žádný internacionalizovaný záznam, vymažte soubor `I18N.js` (nic nepište, nepište `export`)
- Příspěvky musí být v jednom jazyce a míchání více jazyků je přísně zakázáno.

proces:

1. Zobrazit kód zdrojového jazyka nakonfigurovaný v kořenovém adresáři `tran.yml` → `tran` → `from`

2. Zadejte adresář `src/webc/component name`

3. Internacionalizujte komponenty
   - Napište text použitý v `název komponenty.js` jako vstup do `src/webc/název komponenty/i18n/source language/js.yml`.
   - Upravte `src/webc/component name/I18N.js` a nakonfigurujte konstantu názvu vstupního klíče `yml`, například `export const KEY = "ymlKey"`
   - Úvod a vazba v `src/webc/component name.js` (pokud existuje `js.yml`, pokud ne, není nutné):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     import i18nUrl, { KEY } z "./název komponenty/I18N.js"; // Framework automaticky vloží i18nUrl, výchozí výchozí export, nevytvářejte jej sami

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Pokuste se vázat v dávkách, abyste snížili volání fLang
    `název komponenty.js` komponenty by neměl obsahovat řetězec obsahující text

4. Internacionalizace prezentací
   - V `demo/_.pug` je text štítku, který je třeba přeložit, obsazen ve tvaru `název klíče c-t` (toto je vlastní štítek, samostatný řádek) a záznam je zapsán do `demo/i18n/source language/pug.yml`
   - `demo/_.js` Napište původní text použitý v `demo/i18n/source language/js.yml` a vložte jej do `demo/I18N.js`
   - Zaveďte `fetchLang` do `demo/_.js` a načtěte `./I18N.js` (toto je `demo/I18N.js`), viz výše uvedený proces internacionalizace komponenty (například `demo/i18n` níže nemá `js.yml`, tento krok přeskočte)
   - `demo/I18N.js` Nakonfigurujte konstanty názvu klíče v `demo/i18n/source language/js.yml` a exportujte

5. Napište popis komponenty do `src/webc/component name/i18n/source language/README.md`. Postup je následující
  5.1 Napište hlavní název. Název by měl začínat `#`, popisovat účel komponenty jednou větou (nepište název komponenty) a na konec nepřidávejte interpunkci.
  5.2 Uveďte hlavní rysy komponenty, počínaje hlavním názvem a nepište vedlejší názvy. Pokud existuje více charakteristik, použijte neuspořádaný seznam (pokud je popis neuspořádaného seznamu relativně krátký, na konci nebude tečka).
  5.3 Napište `## Pomocí ukázky`, ukázkový kód začíná ` ```html ` (v ukázkovém kódu není značka jednoho prvku uzavřena), pokud existuje `js`, interně napište `html`"3> tag, použijte id ` `js` Na prvním řádku obsahu není odsazení.
  Pokud se jedná o čistý js, začněte s ` ```js `. `js` Pokud v ukázce potřebujete importovat `component/_.js`, použijte prosím `import xxx z "webc.site/component name.js";`. Toto je zkompilovaný `js`, který automaticky vloží js a css.
  `js` Použijte stručné psaní, například `const` k deklaraci více konstant

Hlavní text popisuje použití komponenty, parametry rozhraní, přepínání mezi `css` a `class` v různých stavech atd.
Následně popište parametry, návratovou hodnotu, stav stylu atd. (pokud existuje) a přidejte sekundární název.

Používejte slova stručně a vyhýbejte se přídavným jménům.
