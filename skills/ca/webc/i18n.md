# internacionalització

Cas: consulteu `src/BoxX` per implementar la internacionalització dels components i la demostració.

regla:
- `yml` No afegiu `:` al final de l'entrada, poseu `:` a la plantilla pug i al codi js, i empalmeu-lo
- Utilitzeu `|-` per al text de diverses línies de `yml` per eliminar fàcilment els espais al final.
- Si no hi ha cap entrada internacionalitzada, esborreu el fitxer `I18N.js` (no escriviu res, no escriviu `export`)
- Les entrades han de ser en un sol idioma i està totalment prohibit barrejar diversos idiomes.

procés:

1. Veure el codi de l'idioma font configurat al directori arrel `tran.yml` → `tran` → `from`

2. Introduïu el directori `src/component name`

3. Internacionalitzar components
   - Escriu el text utilitzat per `_.js` com a entrada a `src/nom del component/i18n/source language/js.yml`.
   - Modifiqueu `Nom del component/I18N.js` i configureu la constant del `yml` nom de la clau d'entrada, com ara `export const KEY = "ymlKey"`
   - `Introduït i vinculat al nom del component/_.js` (si hi ha `js.yml`, no és necessari si no):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     importar i18nUrl, { KEY } de "./I18N.js"; // El marc injectarà automàticament i18nUrl, el valor per defecte d'exportació, no el creeu vosaltres mateixos

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Intenteu enllaçar per lots per reduir les trucades de flang
    El `_.js` del component no hauria de tenir cadenes que continguin text

4. Internacionalització de presentacions
   - A `demo/_.pug`, el text de l'etiqueta que cal traduir està ocupat en forma de `nom de clau c-t` (aquesta és una etiqueta personalitzada, una línia separada) i l'entrada s'escriu a `demo/i18n/source language/pug.yml`
   - `demo/_.js` Escriviu l'entrada de text original utilitzada a `demo/i18n/source language/js.yml` i col·loqueu-la a `demo/I18N.js`
   - Introduïu `fetchLang` a `demo/_.js` i carregueu `./I18N.js` (això és `demo/I18N.js`), consulteu el procés d'internacionalització dels components anterior (per exemple, `demo/i18n` no té `js.yml` a continuació, ometeu aquest pas).
   - `demo/I18N.js` Configureu les constants del nom de la clau a `demo/i18n/source language/js.yml` i exporteu

5. Escriviu la descripció del component a `Nom del component/i18n/source language/README.md`. El procés és el següent
  5.1 Escriu el títol principal. El títol ha de començar per `#`, descriure el propòsit del component en una frase (no escriviu el nom del component) i no afegir puntuació al final.
  5.2 Enumereu les característiques principals del component, començant pel títol principal, i no escriviu títols secundaris. Si hi ha diverses característiques, utilitzeu una llista no ordenada (si la descripció de la llista no ordenada és relativament curta, no hi haurà cap punt al final).
  5.3 Escriu `## Amb la demostració`, el codi de demostració comença amb ` ```html ` (al codi de demostració, l'etiqueta d'element únic no està tancada), si hi ha `js`, escriviu `de html<b>"id="3" internament. etiqueta;
  Si és pur js, comença amb ` ```js `. `js` A la demostració, si necessiteu importar `component/_.js`, utilitzeu `import xxx des de "webc.site/component/_.js";`.
  `js` Utilitzeu una escriptura concisa, com ara un `const` per declarar múltiples constants

El text principal presenta com utilitzar el component, els paràmetres de la interfície, canviar entre `css` i `class` en diferents estats, etc.
Posteriorment, descriu els paràmetres, el valor de retorn, l'estat de l'estil, etc. (si n'hi ha) i afegiu un títol secundari.

Utilitza les paraules de manera concisa i evita els adjectius.

La redacció ha d'evitar el disbarat i no escriure paraules sense sentit (com ara: universal, contenidor, etc.).

6. Executeu `./sh/tran.js` del directori arrel, traducció
