# ಅಂತರಾಷ್ಟ್ರೀಯೀಕರಣ

ಪ್ರಕರಣ: ಘಟಕಗಳು ಮತ್ತು ಡೆಮೊಗಳ ಅಂತರಾಷ್ಟ್ರೀಯೀಕರಣವನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸಲು `src/BoxX` ಅನ್ನು ನೋಡಿ.

ನಿಯಮ:
- `yml` ಪ್ರವೇಶದ ಕೊನೆಯಲ್ಲಿ `:` ಅನ್ನು ಸೇರಿಸಬೇಡಿ, `:` ಅನ್ನು pug ಟೆಂಪ್ಲೇಟ್ ಮತ್ತು js ಕೋಡ್‌ಗೆ ಹಾಕಿ ಮತ್ತು ಅದನ್ನು ಸ್ಪ್ಲೈಸ್ ಮಾಡಿ
- ಟ್ರೇಲಿಂಗ್ ಸ್ಪೇಸ್‌ಗಳನ್ನು ಸುಲಭವಾಗಿ ತೆಗೆದುಹಾಕಲು `yml` ಬಹು-ಸಾಲಿನ ಪಠ್ಯಕ್ಕಾಗಿ `|-` ಬಳಸಿ.
- ಯಾವುದೇ ಅಂತರಾಷ್ಟ್ರೀಯ ನಮೂದು ಇಲ್ಲದಿದ್ದರೆ, `I18N.js` ಫೈಲ್ ಅನ್ನು ತೆರವುಗೊಳಿಸಿ (ಏನನ್ನೂ ಬರೆಯಬೇಡಿ, `export` ಬರೆಯಬೇಡಿ)
- ನಮೂದುಗಳು ಒಂದೇ ಭಾಷೆಯಲ್ಲಿರಬೇಕು ಮತ್ತು ಬಹು ಭಾಷೆಗಳನ್ನು ಮಿಶ್ರಣ ಮಾಡುವುದನ್ನು ಕಟ್ಟುನಿಟ್ಟಾಗಿ ನಿಷೇಧಿಸಲಾಗಿದೆ.

ಪ್ರಕ್ರಿಯೆ:

1. ಮೂಲ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಕಾನ್ಫಿಗರ್ ಮಾಡಲಾದ ಮೂಲ ಭಾಷಾ ಕೋಡ್ ಅನ್ನು ವೀಕ್ಷಿಸಿ `tran.yml` → `tran` → `from`

2. `src/component name` ಡೈರೆಕ್ಟರಿಯನ್ನು ನಮೂದಿಸಿ

3. ಘಟಕಗಳನ್ನು ಅಂತರರಾಷ್ಟ್ರೀಯಗೊಳಿಸಿ
   - `src/component name/i18n/source language/js.yml` ನಲ್ಲಿ ಪ್ರವೇಶವಾಗಿ `_.js` ಬಳಸಿದ ಪಠ್ಯವನ್ನು ಬರೆಯಿರಿ.
   - `ಕಾಂಪೊನೆಂಟ್ ಹೆಸರು/I18N.js` ಅನ್ನು ಮಾರ್ಪಡಿಸಿ ಮತ್ತು `yml` ಪ್ರವೇಶ ಕೀ ಹೆಸರಿನ ಸ್ಥಿರಾಂಕವನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ, ಉದಾಹರಣೆಗೆ `export const KEY = "ymlKey"`
   - `ಕಾಂಪೊನೆಂಟ್ ಹೆಸರು/_.js` ನಲ್ಲಿ ಪರಿಚಯಿಸಲಾಗಿದೆ ಮತ್ತು ಬಂಧಿಸಲಾಗಿದೆ (`js.yml` ಇದ್ದರೆ, ಇಲ್ಲದಿದ್ದರೆ ಅದು ಅಗತ್ಯವಿಲ್ಲ):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     "./I18N.js" ನಿಂದ i18nUrl, { KEY } ಆಮದು ಮಾಡಿ; // ಫ್ರೇಮ್‌ವರ್ಕ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ i18nUrl ಅನ್ನು ಇಂಜೆಕ್ಟ್ ಮಾಡುತ್ತದೆ, ಡೀಫಾಲ್ಟ್ ರಫ್ತು ಡೀಫಾಲ್ಟ್, ಅದನ್ನು ನೀವೇ ರಚಿಸಬೇಡಿ

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    ಫ್ಲ್ಯಾಂಗ್ ಕರೆಗಳನ್ನು ಕಡಿಮೆ ಮಾಡಲು ಬ್ಯಾಚ್‌ಗಳಲ್ಲಿ ಬಂಧಿಸಲು ಪ್ರಯತ್ನಿಸಿ
    ಕಾಂಪೊನೆಂಟ್‌ನ `_.js` ಪಠ್ಯವನ್ನು ಒಳಗೊಂಡಿರುವ ಸ್ಟ್ರಿಂಗ್‌ಗಳನ್ನು ಹೊಂದಿರಬಾರದು

4. ಪ್ರಸ್ತುತಿಗಳ ಅಂತರಾಷ್ಟ್ರೀಯೀಕರಣ
   - `demo/_.pug` ನಲ್ಲಿ, ಅನುವಾದಿಸಬೇಕಾದ ಲೇಬಲ್ ಪಠ್ಯವನ್ನು `c-t ಕೀ ಹೆಸರು` ರೂಪದಲ್ಲಿ ಆಕ್ರಮಿಸಲಾಗಿದೆ (ಇದು ಕಸ್ಟಮ್ ಲೇಬಲ್, ಪ್ರತ್ಯೇಕ ಸಾಲು), ಮತ್ತು ಪ್ರವೇಶವನ್ನು `demo/i18n/source language/pug.yml` ಗೆ ಬರೆಯಲಾಗಿದೆ
   - `demo/_.js` `demo/i18n/source language/js.yml` ನಲ್ಲಿ ಬಳಸಲಾದ ಮೂಲ ಪಠ್ಯ ನಮೂದನ್ನು ಬರೆಯಿರಿ ಮತ್ತು ಅದನ್ನು `demo/I18N.js` ನಲ್ಲಿ ಇರಿಸಿ
   - `fetchLang` ಅನ್ನು `demo/_.js` ಗೆ ಪರಿಚಯಿಸಿ ಮತ್ತು `./I18N.js` ಅನ್ನು ಲೋಡ್ ಮಾಡಿ (ಇದು `demo/I18N.js`), ಮೇಲಿನ ಘಟಕ ಅಂತರಾಷ್ಟ್ರೀಯೀಕರಣ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಉಲ್ಲೇಖಿಸಿ (ಉದಾಹರಣೆಗೆ, `demo/i18n` ಕೆಳಗೆ `js.yml` ಹೊಂದಿಲ್ಲ, ಈ ಹಂತವನ್ನು ಬಿಟ್ಟುಬಿಡಿ)
   - `demo/I18N.js` `demo/i18n/source language/js.yml` ನಲ್ಲಿ ಕೀ ಹೆಸರು ಸ್ಥಿರಾಂಕಗಳನ್ನು ಕಾನ್ಫಿಗರ್ ಮಾಡಿ ಮತ್ತು ರಫ್ತು ಮಾಡಿ

5. `ಕಾಂಪೊನೆಂಟ್ ಹೆಸರು/i18n/source language/README.md` ನಲ್ಲಿ ಘಟಕ ವಿವರಣೆಯನ್ನು ಬರೆಯಿರಿ. ಪ್ರಕ್ರಿಯೆಯು ಈ ಕೆಳಗಿನಂತಿರುತ್ತದೆ
  5.1 ಮುಖ್ಯ ಶೀರ್ಷಿಕೆಯನ್ನು ಬರೆಯಿರಿ. ಶೀರ್ಷಿಕೆಯು `#` ನೊಂದಿಗೆ ಪ್ರಾರಂಭವಾಗಬೇಕು, ಒಂದು ವಾಕ್ಯದಲ್ಲಿ ಘಟಕದ ಉದ್ದೇಶವನ್ನು ವಿವರಿಸಬೇಕು (ಸಂಘಟನೆಯ ಹೆಸರನ್ನು ಬರೆಯಬೇಡಿ), ಮತ್ತು ಕೊನೆಯಲ್ಲಿ ವಿರಾಮಚಿಹ್ನೆಯನ್ನು ಸೇರಿಸಬೇಡಿ.
  5.2 ಮುಖ್ಯ ಶೀರ್ಷಿಕೆಯಿಂದ ಪ್ರಾರಂಭವಾಗುವ ಘಟಕದ ಮುಖ್ಯ ಲಕ್ಷಣಗಳನ್ನು ಪಟ್ಟಿ ಮಾಡಿ ಮತ್ತು ದ್ವಿತೀಯ ಶೀರ್ಷಿಕೆಗಳನ್ನು ಬರೆಯಬೇಡಿ. ಬಹು ಗುಣಲಕ್ಷಣಗಳಿದ್ದರೆ, ಕ್ರಮಿಸದ ಪಟ್ಟಿಯನ್ನು ಬಳಸಿ (ಆಕ್ರಮಿಸದ ಪಟ್ಟಿಯ ವಿವರಣೆಯು ತುಲನಾತ್ಮಕವಾಗಿ ಚಿಕ್ಕದಾಗಿದ್ದರೆ, ಕೊನೆಯಲ್ಲಿ ಯಾವುದೇ ಅವಧಿ ಇರುವುದಿಲ್ಲ).
  5.3 `## ಡೆಮೊ ಬಳಸಿ` ಬರೆಯಿರಿ, ಡೆಮೊ ಕೋಡ್ ` ```html ` ನೊಂದಿಗೆ ಪ್ರಾರಂಭವಾಗುತ್ತದೆ (ಡೆಮೊ ಕೋಡ್‌ನಲ್ಲಿ, ಸಿಂಗಲ್ ಎಲಿಮೆಂಟ್ ಟ್ಯಾಗ್ ಮುಚ್ಚಿಲ್ಲ), `js` ಇದ್ದರೆ, ಆಂತರಿಕವಾಗಿ CO DE, CODE ಅನ್ನು ಬರೆಯಿರಿ `<script type="module">` ಟ್ಯಾಗ್;
  ಇದು ಶುದ್ಧ js ಆಗಿದ್ದರೆ, ` ```js ` ನೊಂದಿಗೆ ಪ್ರಾರಂಭಿಸಿ. `js` ಪ್ರಾತ್ಯಕ್ಷಿಕೆಯಲ್ಲಿ, ನೀವು `component/_.js` ಅನ್ನು ಆಮದು ಮಾಡಿಕೊಳ್ಳಬೇಕಾದರೆ, ದಯವಿಟ್ಟು "webc.site/component/_.js";` ನಿಂದ xxx ಆಮದು ಮಾಡಿ.
  `js` ಬಹು ಸ್ಥಿರಾಂಕಗಳನ್ನು ಘೋಷಿಸಲು `const` ನಂತಹ ಸಂಕ್ಷಿಪ್ತ ಬರವಣಿಗೆಯನ್ನು ಬಳಸಿ

ಮುಖ್ಯ ಪಠ್ಯವು ಕಾಂಪೊನೆಂಟ್ ಅನ್ನು ಹೇಗೆ ಬಳಸುವುದು, ಇಂಟರ್ಫೇಸ್ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳು, ವಿವಿಧ ಸ್ಥಿತಿಗಳಲ್ಲಿ `css` ಮತ್ತು `class` ನಡುವೆ ಬದಲಾಯಿಸುವುದು ಇತ್ಯಾದಿಗಳನ್ನು ಪರಿಚಯಿಸುತ್ತದೆ.
ತರುವಾಯ ನಿಯತಾಂಕಗಳನ್ನು ವಿವರಿಸಿ, ಮೌಲ್ಯವನ್ನು ಹಿಂತಿರುಗಿಸಿ, ಶೈಲಿಯ ಸ್ಥಿತಿ, ಇತ್ಯಾದಿ (ಯಾವುದಾದರೂ ಇದ್ದರೆ), ಮತ್ತು ದ್ವಿತೀಯ ಶೀರ್ಷಿಕೆಯನ್ನು ಸೇರಿಸಿ.

ಪದಗಳನ್ನು ಸಂಕ್ಷಿಪ್ತವಾಗಿ ಬಳಸಿ ಮತ್ತು ವಿಶೇಷಣಗಳನ್ನು ತಪ್ಪಿಸಿ.

ಕಾಪಿರೈಟಿಂಗ್ ಅಸಂಬದ್ಧತೆಯನ್ನು ತಪ್ಪಿಸಬೇಕು ಮತ್ತು ಅರ್ಥಹೀನ ಪದಗಳನ್ನು ಬರೆಯಬಾರದು (ಉದಾಹರಣೆಗೆ: ಸಾರ್ವತ್ರಿಕ, ಕಂಟೇನರ್, ಇತ್ಯಾದಿ).
