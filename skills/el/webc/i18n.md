# διεθνοποίηση

Περίπτωση: Ανατρέξτε στο `src/BoxX` για να εφαρμόσετε τη διεθνοποίηση στοιχείων και επίδειξης.

κανόνας:
- `yml` Μην προσθέσετε `:` στο τέλος της καταχώρισης, βάλτε το `:` στο πρότυπο pug και τον κώδικα js και συνδέστε το
- Χρησιμοποιήστε το `|-` για κείμενο πολλών γραμμών του `yml` για να αφαιρέσετε εύκολα τα τελικά κενά.
- Εάν δεν υπάρχει διεθνοποιημένη καταχώριση, διαγράψτε το αρχείο `I18N.js` (μην γράψετε τίποτα, μην γράψετε `export`)
- Οι συμμετοχές πρέπει να είναι σε μία μόνο γλώσσα και η μίξη πολλών γλωσσών απαγορεύεται αυστηρά.

διαδικασία:

1. Δείτε τον κώδικα της γλώσσας πηγής που έχει διαμορφωθεί στον ριζικό κατάλογο `tran.yml` → `tran` → `from`

2. Εισαγάγετε τον κατάλογο `src/όνομα στοιχείου`

3. Διεθνοποίηση εξαρτημάτων
   - Γράψτε το κείμενο που χρησιμοποιείται από το `_.js` ως καταχώρηση στο `src/component name/i18n/source language/js.yml`.
   - Τροποποιήστε το `Όνομα στοιχείου/I18N.js` και διαμορφώστε τη σταθερά του ονόματος κλειδιού εισαγωγής `yml`, όπως `export const KEY = "ymlKey"`
   - `Εισάγεται και δεσμεύεται στο όνομα στοιχείου/_.js` (εάν υπάρχει `js.yml`, δεν είναι απαραίτητο, εάν όχι):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     εισαγωγή i18nUrl, { KEY } από το "./I18N.js"; // Το πλαίσιο θα εισάγει αυτόματα το i18nUrl, την προεπιλεγμένη προεπιλογή εξαγωγής, μην το δημιουργήσετε μόνοι σας

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Προσπαθήστε να δεσμεύσετε σε παρτίδες για να μειώσετε τις κλήσεις fLang
    Το `_.js` του στοιχείου δεν πρέπει να έχει συμβολοσειρές που να περιέχει κείμενο

4. Διεθνοποίηση παρουσιάσεων
   - Στο `demo/_.pug`, το κείμενο της ετικέτας που πρέπει να μεταφραστεί είναι κατειλημμένο με τη μορφή `c-t όνομα κλειδιού` (αυτή είναι μια προσαρμοσμένη ετικέτα, μια ξεχωριστή γραμμή) και η καταχώρηση γράφεται στο `demo/i18n/source language/pug.yml`
   - `demo/_.js` Γράψτε την αρχική καταχώριση κειμένου που χρησιμοποιείται στο `demo/i18n/source language/js.yml` και τοποθετήστε την στο `demo/I18N.js`
   - Εισαγάγετε το `fetchLang` στο `demo/_.js` και φορτώστε το `./I18N.js` (αυτό είναι `demo/I18N.js`), ανατρέξτε στη διαδικασία διεθνοποίησης στοιχείων παραπάνω (για παράδειγμα, το `demo/i18n` δεν έχει `js.yml` παρακάτω, παραλείψτε αυτό το βήμα)
   - `demo/I18N.js` Διαμόρφωση σταθερών ονομάτων κλειδιών στο `demo/i18n/source language/js.yml` και εξαγωγή

5. Γράψτε την περιγραφή του στοιχείου στο `Όνομα στοιχείου/i18n/source language/README.md`. Η διαδικασία είναι η εξής
  5.1 Γράψτε τον κύριο τίτλο. Ο τίτλος πρέπει να ξεκινά με `#`, να περιγράφει τον σκοπό του στοιχείου με μία πρόταση (μην γράψετε το όνομα του στοιχείου) και να μην προσθέσετε σημεία στίξης στο τέλος.
  5.2 Καταχωρίστε τα κύρια χαρακτηριστικά του στοιχείου, ξεκινώντας από τον κύριο τίτλο και μην γράφετε δευτερεύοντες τίτλους. Εάν υπάρχουν πολλά χαρακτηριστικά, χρησιμοποιήστε μια λίστα χωρίς σειρά (εάν η περιγραφή της λίστας χωρίς σειρά είναι σχετικά σύντομη, δεν θα υπάρχει τελεία στο τέλος).
  5.3 Γράψτε `## Χρησιμοποιώντας την επίδειξη`, ο κώδικας επίδειξης ξεκινά με ` ```html ` (στον κώδικα επίδειξης, η ετικέτα ενός στοιχείου δεν είναι κλειστή), εάν υπάρχει `js`, γράψτε <b>εσωτερικά class="CODE"> `χρήση `html id="3">` ετικέτα;
  Εάν είναι καθαρό js, ξεκινήστε με ` ```js `. `js` Στην επίδειξη, εάν χρειάζεται να εισαγάγετε `component/_.js`, χρησιμοποιήστε `εισαγωγή xxx από το "webc.site/component/_.js";`.
  `js` Χρησιμοποιήστε συνοπτική γραφή, όπως `const` για να δηλώσετε πολλές σταθερές

Το κύριο κείμενο εισάγει τον τρόπο χρήσης του στοιχείου, τις παραμέτρους διεπαφής, την εναλλαγή μεταξύ `css` και `class` σε διαφορετικές καταστάσεις, κ.λπ.
Στη συνέχεια, περιγράψτε τις παραμέτρους, την επιστρεφόμενη τιμή, την κατάσταση στυλ κ.λπ. (εάν υπάρχουν) και προσθέστε έναν δευτερεύοντα τίτλο.

Χρησιμοποιήστε τις λέξεις συνοπτικά και αποφύγετε τα επίθετα.

Το copywriting θα πρέπει να αποφεύγει τις ανοησίες και να μην γράφει λέξεις χωρίς νόημα (όπως: καθολικό, δοχείο κ.λπ.).
