# internacionalización

Caso: Consulte `src/webc/BoxX` para implementar la internacionalización de componentes y la demostración.

regla:
- `yml` No agregue `:` al final de la entrada, coloque `:` en la plantilla pug y el código js, ​​y empalme
- Utilice `|-` para texto de varias líneas de `yml` para eliminar fácilmente los espacios finales.
- Si no hay ninguna entrada internacionalizada, borre el archivo `I18N.js` (no escriba nada, no escriba `export`)
- Las participaciones deben estar en un solo idioma y está estrictamente prohibido mezclar varios idiomas.

proceso:

1. Ver el código del idioma fuente configurado en el directorio raíz `tran.yml` → `tran` → `from`

2. Introduzca el directorio `src/webc/component name`

3. Internacionalizar componentes
   - Escriba el texto utilizado en `component name.js` como una entrada en `src/webc/component name/i18n/source language/js.yml`.
   - Modifique `src/webc/component name/I18N.js` y configure la constante del `yml` nombre de clave de entrada, como `export const KEY = "ymlKey"`
   - Introducción y vinculación en `src/webc/component name.js` (si hay `js.yml`, si no, no es necesario):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     importar i18nUrl, { CLAVE } de "./nombre del componente/I18N.js"; // El marco inyectará automáticamente i18nUrl, el valor predeterminado de exportación predeterminado, no lo cree usted mismo

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Intente vincularse en lotes para reducir las llamadas fLang
    El `nombre del componente.js` del componente no debe tener una cadena que contenga texto.

4. Internacionalización de presentaciones.
   - En `demo/_.pug`, el texto de la etiqueta que debe traducirse se ocupa en el formato `nombre de clave c-t` (esta es una etiqueta personalizada, una línea separada) y la entrada se escribe en `demo/i18n/source language/pug.yml`.
   - `demo/_.js` Escriba la entrada de texto original utilizada en `demo/i18n/source language/js.yml` y colóquela en `demo/I18N.js`
   - Introduzca `fetchLang` en `demo/_.js` y cargue `./I18N.js` (esto es `demo/I18N.js`), consulte el proceso de internacionalización de componentes anterior (por ejemplo, `demo/i18n` no tiene `js.yml` a continuación, omita este paso)
   - `demo/I18N.js` Configure constantes de nombre de clave en `demo/i18n/source language/js.yml` y exporte

5. Escriba la descripción del componente en `src/webc/component name/i18n/source language/README.md`. El proceso es el siguiente
  5.1 Escribe el título principal. El título debe comenzar con `#`, describir el propósito del componente en una oración (no escriba el nombre del componente) y no agregue puntuación al final.
  5.2 Enumere las características principales del componente, comenzando con el título principal y no escriba títulos secundarios. Si hay varias características, utilice una lista desordenada (si la descripción de la lista desordenada es relativamente corta, no habrá un punto al final).
  5.3 Escriba `## Usando la demostración`, el código de demostración comienza con ` ```html ` (en el código de demostración, la etiqueta de elemento único no está cerrada), si hay `js`, escriba `html` internamente, use `<script type="module">` Etiqueta, `js` No tiene sangría en la primera línea de contenido.
  Si es js puro, comience con ` ```js `. `js` En la demostración, si necesita importar `component/_.js`, utilice `import xxx desde "webc.site/component name.js";`. Este es el `js` compilado, que inyectará automáticamente js y css.
  `js` Utilice escritura concisa, como `const` para declarar múltiples constantes

El texto principal presenta cómo usar el componente, los parámetros de la interfaz, cómo cambiar entre `css` y `class` en diferentes estados, etc.
Posteriormente describa los parámetros, valor de retorno, estado del estilo, etc. (si corresponde) y agregue un título secundario.

Utilice palabras de forma concisa y evite los adjetivos.

La redacción publicitaria debe evitar tonterías y no escribir palabras sin sentido (como: universal, contenedor, etc.).

6. Ejecute `./sh/tran.js` del directorio raíz, traducción
