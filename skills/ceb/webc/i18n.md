# internasyonalisasyon

Kaso: Tan-awa ang `src/webc/BoxX` aron ipatuman ang internasyonalisasyon sa mga sangkap ug demo.

lagda:
- `yml` Ayaw idugang `:` sa katapusan sa entry, ibutang `:` sa pug template ug js code, ug idugtong kini
- Gamita ang `|-` para sa multi-line nga text sa `yml` aron daling matangtang ang mga trailing space.
- Kung walay internasyonal nga entry, hawanan ang `I18N.js` file (ayaw pagsulat og bisan unsa, ayaw pagsulat `export`)
- Ang mga entri kinahanglan anaa sa usa ka pinulongan, ug ang pagsagol sa daghang mga pinulongan hugot nga gidili.

proseso:

1. Tan-awa ang source language code nga gi-configure sa root directory `tran.yml` → `tran` → `from`

2. Isulod ang `src/webc/component name` nga direktoryo

3. I-internasyonal ang mga sangkap
   - Isulat ang teksto nga gigamit sa `component name.js` isip entry ngadto sa `src/webc/component name/i18n/source language/js.yml`.
   - Usba ang `src/webc/component name/I18N.js` ug i-configure ang constant sa `yml` entry key name, sama sa `export const KEY = "ymlKey"`
   - Pasiuna ug pagbugkos sa `src/webc/component name.js` (kon adunay `js.yml`, kung wala, dili kinahanglan):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     import i18nUrl, {KEY } gikan sa "./component name/I18N.js"; // Ang balangkas awtomatik nga mag-inject sa i18nUrl, ang default export default, ayaw paghimo niini sa imong kaugalingon

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Sulayi ang pagbugkos sa mga batch aron makunhuran ang mga tawag sa fLang
    Ang `ngalan sa component.js` sa component kinahanglan walay string nga adunay sulod nga teksto

4. Internasyonalisasyon sa mga presentasyon
   - Sa `demo/_.pug`, ang label nga teksto nga kinahanglang hubaron giokupar sa porma sa `c-t key name` (kini usa ka custom label, usa ka bulag nga linya), ug ang entry gisulat sa `demo/i18n/source language/pug.yml`
   - `demo/_.js` Isulat ang orihinal nga text entry nga gigamit sa `demo/i18n/source language/js.yml` ug ibutang kini sa `demo/I18N.js`
   - Ipaila ang `fetchLang` sa `demo/_.js` ug i-load ang `./I18N.js` (kini mao ang `demo/I18N.js`), i-refer ang component nga proseso sa internationalization sa ibabaw (pananglitan, `demo/i18n` walay `js.yml` sa ubos, laktawan kini nga lakang)
   - `demo/I18N.js` I-configure ang key name constants sa `demo/i18n/source language/js.yml` ug export

5. Isulat ang deskripsyon sa component sa `src/webc/component name/i18n/source language/README.md`. Ang proseso mao ang mosunod
  5.1 Isulat ang nag-unang titulo. Ang titulo kinahanglang magsugod sa `#`, ihulagway ang katuyoan sa component sa usa ka sentence (ayaw isulat ang component name), ug ayaw idugang ang punctuation sa katapusan.
  5.2 Ilista ang mga nag-unang bahin sa component, sugod sa nag-unang titulo, ug ayaw pagsulat ug ikaduhang titulo. Kung adunay daghang mga kinaiya, gamita ang usa ka wala masunud nga lista (kung ang wala masunud nga paghulagway sa lista medyo mubo, wala’y panahon sa katapusan).
  5.3 Isulat ang `## Gamit ang demo`, ang demo code magsugod sa ` ```html ` (sa demo code, ang usa ka elemento nga tag dili sirado), kon adunay `js`, isulat ang `html<b class="iCODE"> internally,><3> class="CODE"> Tag, `js` Walay indentasyon sa unang linya sa sulod.
  Kung puro js, ​​sugdi sa ` ```js `. `js` Sa demonstrasyon, kung kinahanglan nimong i-import ang `component/_.js`, palihug gamita ang `import xxx gikan sa "webc.site/component name.js";`. Kini ang gihugpong nga `js`, nga awtomatikong mag-inject sa js ug css.
  `js` Gamita ang mugbo nga pagsulat, sama sa `const` sa pagdeklarar og daghang mga constants

Ang panguna nga teksto nagpaila kung giunsa paggamit ang sangkap, mga parameter sa interface, pagbalhin tali sa `css` ug `class` sa lainlaing mga estado, ug uban pa.
Pagkahuman ihulagway ang mga parameter, kantidad sa pagbalik, kahimtang sa istilo, ug uban pa (kung naa), ug pagdugang usa ka ikaduha nga titulo.
