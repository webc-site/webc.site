# আন্তৰ্জাতিককৰণ

ক্ষেত্ৰ: উপাদানসমূহ আৰু ডেমোৰ আন্তৰ্জাতিককৰণ প্ৰণয়ন কৰিবলে `src/BoxX` চাওক।

নিয়ম:
- `yml` প্ৰৱেশৰ শেষত `:` যোগ নকৰিব, `:` pug টেমপ্লেট আৰু js ক'ডত ৰাখক, আৰু ইয়াক স্প্লাইচ কৰক
- সহজে পিছৰ স্থান আঁতৰাবলৈ `yml` ৰ বহু-শাৰী লিখনীৰ বাবে `|-` ব্যৱহাৰ কৰক।
- যদি কোনো আন্তৰ্জাতিক প্ৰৱেশ নাই, `I18N.js` ফাইলটো পৰিষ্কাৰ কৰক (একো লিখিব নালাগে, `export` লিখিব নালাগে)
- প্ৰৱেশসমূহ এটা ভাষাত হ'ব লাগিব, আৰু একাধিক ভাষা মিহলি কৰাটো কঠোৰভাৱে নিষিদ্ধ।

প্ৰক্ৰিয়া:

1. ৰূট পঞ্জিকাত বিন্যাস কৰা উৎস ভাষা ক'ড চাওক `tran.yml` → `tran` → `from`

2. `src/component name` পঞ্জিকা সুমুৱাওক

3. উপাদানসমূহ আন্তৰ্জাতিককৰণ
   - `_.js` দ্বাৰা ব্যৱহাৰ কৰা লিখনী `src/component name/i18n/source language/js.yml` ত এটা প্ৰৱেশ হিচাপে লিখক।
   - `উপাদানৰ নাম/I18N.js` পৰিবৰ্তন কৰক আৰু `yml` প্ৰৱেশ চাবিৰ নামৰ ধ্ৰুৱক বিন্যাস কৰক, যেনে `export const KEY = "ymlKey"`
   - `উপাদানৰ নাম/_.js` ত প্ৰৱৰ্তিত আৰু বান্ধি দিয়া হৈছে (যদি `js.yml` আছে, যদি নাই তেন্তে ইয়াৰ প্ৰয়োজন নাই):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     i18nUrl আমদানি, { KEY } "./I18N.js" ৰ পৰা; // ফ্ৰেমৱৰ্কে স্বয়ংক্ৰিয়ভাৱে i18nUrl ইনজেক্ট কৰিব, অবিকল্পিত ৰপ্তানি অবিকল্পিত, নিজে সৃষ্টি নকৰিব

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    fLang কল হ্ৰাস কৰিবলৈ বেচত বান্ধিবলৈ চেষ্টা কৰক
    উপাদানৰ `_.js` ত লিখনী থকা স্ট্ৰিং থাকিব নালাগে

4. উপস্থাপনৰ আন্তৰ্জাতিককৰণ
   - `demo/_.pug`ত, অনুবাদ কৰিবলগীয়া লেবেল লিখনীটো `c-t চাবিৰ নাম`ৰ আকাৰত দখল কৰা হয় (এয়া এটা স্বনিৰ্বাচিত লেবেল, এটা পৃথক শাৰী), আৰু প্ৰৱেশটো `demo/i18n/source language/pug.yml`ত লিখা হয়
   - `demo/_.js` `demo/i18n/source language/js.yml` ত ব্যৱহৃত মূল লিখনী প্ৰবিষ্ট লিখক আৰু ইয়াক `demo/I18N.js` ত ৰাখক
   - `fetchLang` `demo/_.js` ত প্ৰৱৰ্তন কৰক আৰু `./I18N.js` লোড কৰক (এইটো `demo/I18N.js`), ওপৰৰ উপাদান আন্তৰ্জাতিককৰণ প্ৰক্ৰিয়া চাওক (উদাহৰণস্বৰূপে, `demo/i18n` ৰ তলত `js.yml` নাই, এই পদক্ষেপ এৰি দিয়ক)
   - `demo/I18N.js` `demo/i18n/source language/js.yml` ত কি' নামৰ ধ্ৰুৱকসমূহ বিন্যাস কৰক আৰু ৰপ্তানি কৰক

5. উপাদানৰ বিৱৰণ `উপাদানৰ নাম/i18n/source language/README.md` ত লিখক। প্ৰক্ৰিয়াটো এনেধৰণৰ
  ৫.১ মূল শিৰোনাম লিখা। শিৰোনামটো `#`ৰে আৰম্ভ হ’ব লাগে, এটা বাক্যত উপাদানটোৰ উদ্দেশ্য বৰ্ণনা কৰিব লাগে (উপাদানৰ নাম লিখিব নালাগে), আৰু শেষত বিৰাম চিহ্ন যোগ নকৰিব লাগে।
  5.2 উপাদানৰ মূল বৈশিষ্ট্যসমূহ তালিকাভুক্ত কৰক, মূল শিৰোনামৰ পৰা আৰম্ভ কৰি, আৰু গৌণ শিৰোনাম লিখক। যদি একাধিক বৈশিষ্ট্য আছে, এটা অক্ৰমবদ্ধ তালিকা ব্যৱহাৰ কৰক (যদি অক্ৰমবিহীন তালিকাৰ বিৱৰণ তুলনামূলকভাৱে চুটি হয়, শেষত কোনো সময়সীমা নাথাকিব)।
  5.3 লিখা `## ডেমো` ব্যৱহাৰ কৰি, ডেমো ক'ড ` ```html ` ৰে আৰম্ভ হয় (ডেমো ক'ডত, একক উপাদান টেগ বন্ধ নহয়), যদি `js` আছে, আভ্যন্তৰীণভাৱে `html` লিখক, ব্যৱহাৰ কৰক `<script type="module">` টেগ;
  যদি ই বিশুদ্ধ js হয়, ` ```js ` ৰ পৰা আৰম্ভ কৰক। `js` প্ৰদৰ্শনত, যদি আপুনি `component/_.js` আমদানি কৰিব লাগে, অনুগ্ৰহ কৰি "webc.site/component name.js";` ৰ পৰা `import xxx ব্যৱহাৰ কৰক।
  `js` সংক্ষিপ্ত লিখা ব্যৱহাৰ কৰক, যেনে এটা `const` একাধিক ধ্ৰুৱক ঘোষণা কৰিবলৈ

মূল লিখনীয়ে উপাদান, আন্তঃপৃষ্ঠ প্ৰাচলসমূহ, বিভিন্ন অৱস্থাত `css` আৰু `class`ৰ মাজত চুইচিং ইত্যাদি কেনেকৈ ব্যৱহাৰ কৰিব লাগে তাৰ পৰিচয় দিয়ে।
পৰৱৰ্তী সময়ত প্ৰাচলসমূহ, ঘূৰাই দিয়া মান, শৈলীৰ অৱস্থা, ইত্যাদি (যদি আছে) বৰ্ণনা কৰক, আৰু এটা গৌণ শিৰোনাম যোগ কৰক।

শব্দ সংক্ষিপ্তভাৱে ব্যৱহাৰ কৰক আৰু বিশেষণ পৰিহাৰ কৰক।

কপিৰাইটিঙে আজে বাজে কথা এৰাই চলিব লাগে আৰু অৰ্থহীন শব্দ (যেনে: ইউনিভাৰ্চেল, পাত্ৰ আদি) লিখিব নালাগে।
