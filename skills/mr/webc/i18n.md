# आंतरराष्ट्रीयीकरण

केस: घटक आणि डेमोचे आंतरराष्ट्रीयीकरण लागू करण्यासाठी `src/BoxX` चा संदर्भ घ्या.

नियम:
- `yml` एंट्रीच्या शेवटी `:` जोडू नका, पग टेम्प्लेट आणि js कोडमध्ये `:` टाका आणि त्याचे विभाजन करा
- ट्रेलिंग स्पेस सहज काढण्यासाठी `yml` च्या मल्टी-लाइन मजकूरासाठी `|-` वापरा.
- कोणतीही आंतरराष्ट्रीयीकृत नोंद नसल्यास, `I18N.js` फाइल साफ करा (काहीही लिहू नका, `export` लिहू नका)
- नोंदी एकाच भाषेत असणे आवश्यक आहे आणि अनेक भाषांचे मिश्रण करणे कठोरपणे प्रतिबंधित आहे.

प्रक्रिया:

1. मूळ निर्देशिकेत कॉन्फिगर केलेला स्त्रोत भाषा कोड पहा `tran.yml` → `tran` → `from`

2. `src/component name` निर्देशिका एंटर करा

3. घटकांचे आंतरराष्ट्रीयीकरण करा
   - `src/component name/i18n/source language/js.yml` मध्ये `_.js` द्वारे वापरलेला मजकूर लिहा.
   - `घटक नाव/I18N.js` सुधारा आणि `yml` एंट्री की नावाचा स्थिरांक कॉन्फिगर करा, जसे की `export const KEY = "ymlKey"`
   - `परिचय केलेले आणि घटक नावात बांधलेले/_.js` (जर `js.yml` असेल तर, नसल्यास ते आवश्यक नाही):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     "./I18N.js" वरून i18nUrl, { KEY } आयात करा; // फ्रेमवर्क स्वयंचलितपणे i18nUrl इंजेक्ट करेल, डीफॉल्ट निर्यात डीफॉल्ट, ते स्वतः तयार करू नका

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    फ्लँग कॉल्स कमी करण्यासाठी बॅचमध्ये बांधण्याचा प्रयत्न करा
    घटकाच्या `_.js` मध्ये मजकूर असलेली स्ट्रिंग नसावी

4. सादरीकरणांचे आंतरराष्ट्रीयीकरण
   - `demo/_.pug` मध्ये, भाषांतर करण्याची आवश्यकता असलेला लेबल मजकूर `c-t की नाव` (हे एक सानुकूल लेबल आहे, एक वेगळी ओळ आहे), आणि एंट्री `डेमो/i18n/source language/pug.yml` मध्ये लिहिली आहे.
   - `demo/_.js` `डेमो/i18n/source language/js.yml` मध्ये वापरलेली मूळ मजकूर नोंद लिहा आणि ती `demo/I18N.js` मध्ये ठेवा
   - `fetchLang` चा `demo/_.js` मध्ये परिचय करा आणि `./I18N.js` लोड करा (हे `demo/I18N.js` आहे), वरील घटक आंतरराष्ट्रीयीकरण प्रक्रियेचा संदर्भ घ्या (उदाहरणार्थ, `demo/i18n` खाली `js.yml` नाही, ही पायरी वगळा)
   - `demo/I18N.js` `demo/i18n/source language/js.yml` मध्ये की नेम स्थिरांक कॉन्फिगर करा आणि निर्यात करा

5. घटकाचे वर्णन `घटक नाव/i18n/source language/README.md` मध्ये लिहा. प्रक्रिया खालीलप्रमाणे आहे
  5.1 मुख्य शीर्षक लिहा. शीर्षक `#` ने सुरू झाले पाहिजे, एका वाक्यात घटकाच्या उद्देशाचे वर्णन करा (घटकाचे नाव लिहू नका), आणि शेवटी विरामचिन्हे जोडू नका.
  5.2 मुख्य शीर्षकापासून सुरू होणाऱ्या घटकाच्या मुख्य वैशिष्ट्यांची यादी करा आणि दुय्यम शीर्षके लिहू नका. एकापेक्षा जास्त वैशिष्ट्ये असल्यास, अक्रमित सूची वापरा (जर अक्रमित सूचीचे वर्णन तुलनेने लहान असेल, तर शेवटी कोणताही कालावधी नसेल).
  5.3 `## डेमो वापरून` लिहा, डेमो कोड ` ```html ` ने सुरू होतो (डेमो कोडमध्ये, सिंगल एलिमेंट टॅग बंद नाही), `js` असल्यास, `<b><b><b><b> class="``` class="``` वापरा class="`<b><b><b><b><b><b>`` टॅग
  ते शुद्ध js असल्यास, ` ```js ` ने प्रारंभ करा. `js` प्रात्यक्षिकात, तुम्हाला `component/_.js` आयात करायचे असल्यास, कृपया `"webc.site/component name.js";` वरून xxx आयात करा.
  `js` साध्या लेखी पद्धतीने, उदाहरणार्थ एका `const` मध्ये अनेक स्थिरांक जाहीर करणे

मुख्य मजकूर घटक, इंटरफेस पॅरामीटर्स कसे वापरायचे, `css` आणि `class` मध्ये वेगवेगळ्या राज्यांमध्ये स्विच करणे इ.
त्यानंतर पॅरामीटर्स, रिटर्न व्हॅल्यू, स्टाइल स्टेटस इ.चे वर्णन करा (असल्यास), आणि दुय्यम शीर्षक जोडा.

शब्द संक्षिप्तपणे वापरा आणि विशेषण टाळा.

कॉपीरायटिंगने मूर्खपणा टाळावा आणि निरर्थक शब्द लिहू नये (जसे की: सार्वत्रिक, कंटेनर इ.).
