# інтэрнацыяналізацыя

Справа: звярніцеся да `src/BoxX`, каб рэалізаваць інтэрнацыяналізацыю кампанентаў і дэманстрацыю.

правіла:
- `yml` Не дадавайце `:` у канцы запісу, устаўце `:` у шаблон pug і код js і злучыце яго
- Выкарыстоўвайце `|-` для шматрадковага тэксту `yml`, каб лёгка выдаліць прабелы ў канцы.
- Калі інтэрнацыяналізаванага запісу няма, ачысціце файл `I18N.js` (нічога не пішыце, не пішыце `export`)
- Запісы павінны быць на адной мове, і змешванне некалькіх моў строга забаронена.

працэс:

1. Прагляд зыходнага кода мовы, наладжанага ў каранёвым каталогу `tran.yml` → `tran` → `from`

2. Увядзіце каталог `src/component name`

3. Інтэрнацыяналізацыя кампанентаў
   - Запішыце тэкст, які выкарыстоўваецца `_.js` як запіс у `src/component name/i18n/source language/js.yml`.
   - Змяніце `імя кампанента/I18N.js` і наладзьце канстанту `yml` імя ключа ўваходу, напрыклад `export const KEY = "ymlKey"`
   - `Уводзіцца і звязваецца ў назве кампанента/_.js` (калі ёсць `js.yml`, гэта не абавязкова, калі не):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     імпарт i18nUrl, { КЛЮЧ } з "./I18N.js"; // Фреймворк аўтаматычна ўвядзе i18nUrl, стандартны экспарт па змаўчанні, не стварайце яго самастойна

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Паспрабуйце прывязваць пакетамі, каб паменшыць колькасць выклікаў fLang
    Кампанент `_.js` не павінен мець радкоў з тэкстам

4. Інтэрнацыяналізацыя прэзентацый
   - У `demo/_.pug` тэкст меткі, які трэба перакласці, займае форму `імя ключа c-t` (гэта карыстальніцкая метка, асобны радок), а запіс запісваецца ў `demo/i18n/source language/pug.yml`
   - `demo/_.js` Напішыце арыгінальны тэкставы запіс, які выкарыстоўваецца ў `demo/i18n/source language/js.yml`, і змясціце яго ў `demo/I18N.js`
   - Увядзіце `fetchLang` у `demo/_.js` і загрузіце `./I18N.js` (гэта `demo/I18N.js`), звярніцеся да працэсу інтэрнацыяналізацыі кампанента вышэй (напрыклад, `demo/i18n` не мае `js.yml` ніжэй, прапусціце гэты крок)
   - `demo/I18N.js` Наладзьце канстанты імя ключа ў `demo/i18n/source language/js.yml` і экспартуйце

5. Напішыце апісанне кампанента ў `Назва кампанента/i18n/source language/README.md`. Працэс адбываецца наступным чынам
  5.1 Напішыце асноўную назву. Назва павінна пачынацца з `#`, апісваць прызначэнне кампанента адным сказам (не пісаць назву кампанента) і не ставіць знакі прыпынку ў канцы.
  5.2 Пералічыце асноўныя характарыстыкі кампанента, пачынаючы з асноўнага загалоўка, і не пішыце пабочных загалоўкаў. Калі ёсць некалькі характарыстык, выкарыстоўвайце неўпарадкаваны спіс (калі апісанне неўпарадкаванага спісу адносна кароткае, кропка ў канцы не будзе).
  5.3 Напісанне `## З дапамогай дэманстрацыі`, дэма-код пачынаецца з ` ```html ` (у дэма-кодзе тэг аднаго элемента не закрыты), калі ёсць `js`, напішыце `html` унутрана, выкарыстоўвайце `<script type="module">` тэг;
  Калі гэта чысты js, пачніце з ` ```js `. `js` У дэманстрацыі, калі вам трэба імпартаваць `component/_.js`, выкарыстоўвайце `import xxx з "webc.site/component/_.js";`.
  `js` Каб аб'явіць некалькі канстант, выкарыстоўвайце кароткі запіс, напрыклад `const`

Асноўны тэкст знаёміць з выкарыстаннем кампанента, параметрамі інтэрфейсу, пераключэннем паміж `css` і `class` у розных станах і г.д.
Затым апішыце параметры, вяртанае значэнне, статус стылю і г.д. (калі ёсць) і дадайце другасны загаловак.

Выкарыстоўвайце словы коратка і пазбягайце прыметнікаў.

Капірайтынгу трэба пазбягаць глупстваў і не пісаць бессэнсоўных слоў (тыпу: універсальны, кантэйнер і інш.).

6. Запуск `./sh/tran.js` з каранёвага каталога, пераклад
