# Internationalization

Case: Refer to `src/BoxX` to implement internationalization of components and demo.

rule:
- `yml` Do not add `:` at the end of the entry, put `:` into the pug template and js code, and splice it
- Use `|-` for multi-line text of `yml` to easily remove trailing spaces.
- If there is no internationalized entry, clear the `I18N.js` file (don't write anything, don't write `export`)
- Entries must be in a single language, and mixing multiple languages ​​is strictly prohibited.

process:

1. View the source language code configured in the root directory `tran.yml` → `tran` → `from`

2. Enter the `src/component name` directory

3. Internationalize components
   - Write the text used by `_.js` as an entry in `src/component name/i18n/source language/js.yml`.
   - Modify `Component name/I18N.js` and configure the constant of `yml` entry key name, such as `export const KEY = "ymlKey"`
   - `Introduced and bound in component name/_.js` (if there is `js.yml`, it is not necessary if not):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     import i18nUrl, { KEY } from "./I18N.js"; // The framework will automatically inject i18nUrl, the default export default, do not create it yourself

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Try to bind in batches to reduce fLang calls
    Component's `_.js` should not have strings containing text

4. Internationalization of presentations
   - In `demo/_.pug`, the label text that needs to be translated is occupied in the form of `c-t key name` (this is a custom label, a separate line), and the entry is written into `demo/i18n/source language/pug.yml`
   - `demo/_.js` Write the original text entry used in `demo/i18n/source language/js.yml` and place it in `demo/I18N.js`
   - Introduce `fetchLang` into `demo/_.js` and load `./I18N.js` (this is `demo/I18N.js`), refer to the component internationalization process above (for example, `demo/i18n` does not have `js.yml` below, skip this step)
   - `demo/I18N.js` Configure key name constants in `demo/i18n/source language/js.yml` and export

5. Write the component description in `Component name/i18n/source language/README.md`. The process is as follows
  5.1 Write the main title. The title should start with `#`, describe the purpose of the component in one sentence (do not write the component name), and do not add punctuation at the end.
  5.2 List the main features of the component, starting with the main title, and do not write secondary titles. If there are multiple characteristics, use an unordered list (if the unordered list description is relatively short, there will be no period at the end).
  5.3 Write `## Using the demo`, the demo code starts with ` ```html ` (in the demo code, the single element tag is not closed), if there is `js`, write `html` internally, use `<script type="module">` tag;
  If it is pure js, start with ` ```js `. `js` In the demonstration, if you need to import `component/_.js`, please use `import xxx from "webc.site/component/_.js";`.
  `js` Use concise writing, such as a `const` to declare multiple constants

The main text introduces how to use the component, interface parameters, switching between `css` and `class` in different states, etc.
Subsequently describe the parameters, return value, style status, etc. (if any), and add a secondary title.

Use words concisely and avoid adjectives.

Copywriting should avoid nonsense and do not write meaningless words (such as: universal, container, etc.).

6. Run `./sh/tran.js` of the root directory, translation
