# internasionalisasi

Kasus: Rujuk `src/BoxX` kanggo ngleksanakake internasionalisasi komponen lan demo.

aturan:
- `yml` Aja nambah `:` ing mburi entri, lebokake `:` menyang template pug lan kode js, lan sambungake
- Gunakake `|-` kanggo teks multi-baris `yml` kanggo mbusak spasi mburine kanthi gampang.
- Yen ora ana entri internasional, mbusak file `I18N.js` (aja nulis apa-apa, aja nulis `export`)
- Entri kudu ing basa siji, lan nyampur macem-macem basa dilarang banget.

proses:

1. Deleng kode basa sumber sing dikonfigurasi ing direktori root `tran.yml` ā†’ `tran` ā†’ `from`

2. Ketik direktori `src/component name`

3. Internationalize komponen
   - Tulis teks sing digunakake `_.js` minangka entri ing `src/component name/i18n/source language/js.yml`.
   - Owahi `Jeneng komponen/I18N.js` lan konfigurasi konstanta `yml` jeneng kunci entri, kayata `export const KEY = "ymlKey"`
   - `Diwenehake lan diikat ing jeneng komponen/_.js` (yen ana `js.yml`, ora perlu yen ora):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     ngimpor i18nUrl, {KEY} saka "./I18N.js"; // Framework bakal kanthi otomatis nyuntik i18nUrl, standar ekspor standar, aja nggawe dhewe

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Coba ikatan ing batch kanggo nyuda telpon fLang
    Komponen `_.js` kudune ora duwe string sing ngemot teks

4. Internasionalisasi presentasi
   - Ing `demo/_.pug`, teks label sing kudu diterjemahake dikuwasani ing wangun `c-t key name` (iki label khusus, baris sing kapisah), lan entri ditulis ing `demo/i18n/source language/pug.yml`
   - `demo/_.js` Tulis entri teks asli sing digunakake ing `demo/i18n/source language/js.yml` banjur lebokake ing `demo/I18N.js`
   - Kenalake `fetchLang` menyang `demo/_.js` lan muat `./I18N.js` (iki `demo/I18N.js`), deleng proses internasionalisasi komponen ing ndhuwur (contone, `demo/i18n` ora duwe `js.yml` ing ngisor iki, lewati langkah iki)
   - `demo/I18N.js` Konfigurasi konstanta jeneng tombol ing `demo/i18n/source language/js.yml` lan ekspor
