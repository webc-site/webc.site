# internasionalisasi

Kasus: Rujuk `src/webc/BoxX` kanggo ngleksanakake internasionalisasi komponen lan demo.

aturan:
- `yml` Aja nambah `:` ing mburi entri, lebokake `:` menyang template pug lan kode js, lan sambungake
- Gunakake `|-` kanggo teks multi-baris `yml` kanggo mbusak spasi mburine kanthi gampang.
- Yen ora ana entri internasional, mbusak file `I18N.js` (aja nulis apa-apa, aja nulis `export`)
- Entri kudu ing basa siji, lan nyampur macem-macem basa dilarang banget.

proses:

1. Deleng kode basa sumber sing dikonfigurasi ing direktori root `tran.yml` → `tran` → `from`

2. Ketik direktori `src/webc/component name`

3. Internationalize komponen
   - Tulis teks sing digunakake ing `jeneng komponen.js` minangka entri menyang `src/webc/component name/i18n/source language/js.yml`.
   - Ngowahi `src/webc/component name/I18N.js` lan konfigurasi konstanta `yml` jeneng kunci entri, kayata `export const KEY = "ymlKey"`
   - Pambuka lan ikatan ing `src/webc/component name.js` (yen ana `js.yml`, yen ora, ora perlu):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     ngimpor i18nUrl, {KEY} saka "./jeneng komponen/I18N.js"; // Framework bakal kanthi otomatis nyuntik i18nUrl, standar ekspor standar, aja nggawe dhewe

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Coba ikatan ing batch kanggo nyuda telpon fLang
    `komponen jeneng.js` kudune ora ana string sing ngemot teks

4. Internasionalisasi presentasi
   - Ing `demo/_.pug`, teks label sing kudu diterjemahake dikuwasani ing wangun `c-t key name` (iki label khusus, baris sing kapisah), lan entri ditulis ing `demo/i18n/source language/pug.yml`
   - `demo/_.js` Tulis entri teks asli sing digunakake ing `demo/i18n/source language/js.yml` banjur lebokake ing `demo/I18N.js`
   - Kenalake `fetchLang` menyang `demo/_.js` lan muat `./I18N.js` (iki `demo/I18N.js`), deleng proses internasionalisasi komponen ing ndhuwur (contone, `demo/i18n` ora duwe `js.yml` ing ngisor iki, lewati langkah iki)
   - `demo/I18N.js` Konfigurasi konstanta jeneng tombol ing `demo/i18n/source language/js.yml` lan ekspor

5. Tulis katrangan komponen ing `src/webc/component name/i18n/source language/README.md`. Prosese kaya ing ngisor iki
  5.1 Tulis judhul utama. Judhul kudu diwiwiti kanthi `#`, njlèntrèhaké tujuan komponen ing siji ukara (aja nulis jeneng komponen), lan aja ditambahake tanda baca ing pungkasan.
  5.2 Dhaptar fitur utama komponen, diwiwiti kanthi judhul utama, lan aja nulis judhul sekunder. Yen ana pirang-pirang ciri, gunakake dhaptar sing ora diurutake (yen katrangan dhaptar sing ora diurutake relatif cendhak, ora ana wektu ing pungkasan).
  5.3 Nulis `## Nggunakake demo`, kode demo diwiwiti nganggo ` ```html ` (ing kode demo, tag unsur siji ora ditutup), yen ana `js`, tulis `html<b class="iCODE"> internally,><b class="icode"> class="CODE"> Tag, `js` Ora ana indentasi ing baris pisanan isi.
  Yen iku murni js, miwiti karo ` ```js `. `js` Ing demonstrasi, yen sampeyan kudu ngimpor `komponen/_.js`, gunakake `import xxx saka "webc.site/component name.js";`. Iki minangka `js` sing dikompilasi, sing bakal nyuntikake js lan css kanthi otomatis.
  `js` Gunakake tulisan sing ringkes, kayata `const` kanggo ngumumake pirang-pirang konstanta
