# ဒီမို

`src/BoxX/demo`ကို ကိုးကားပြီး `src/component name/demo/` တွင် အစိတ်အပိုင်း၏သရုပ်ပြကို ပြုစုပါ။

`demo/_.pug`၊ သရုပ်ပြပုံစံခွဲ

သေးငယ်သော အစိတ်အပိုင်း တဂ်များသာ ပါဝင်ပြီး၊ စာမျက်နှာ အပြည့်အစုံ မလိုအပ်ဘဲ၊ ပုံစံများနှင့် ဇာတ်ညွှန်းများ တင်သွင်းခြင်းကို တားမြစ်ထားသည်။ မူဘောင်သည် အလိုအလျောက် ထိုးသွင်းလိမ့်မည် (ဥပမာ `demo/_.styl`၊ `demo/_.js`)။

ခေါင်းစဉ်တစ်ခုမရေးပါနှင့် (ဥပမာ၊ xxx ဒီမိုမရေးပါနှင့်) အစိတ်အပိုင်းအမည်ကို မရေးပါနှင့်၊ အစိတ်အပိုင်းကို မမိတ်ဆက်ပါနှင့်၊ သရုပ်ပြအသုံးပြုမှုအခြေအနေအားသာ ရေးပါ (အခြေအနေမရှိလျှင် ခေါင်းစဉ်မရေးပါနှင့်)၊

`_.pug` ၏ အမြစ်ဒြပ်စင်သည် `main.demo.Lg` ၊ ၎င်းသည် `demo/_.styl` တွင် စတိုင်ကို သတ်မှတ်ပေးထားပြီး `main.demo.Lg` ကိုယ်တိုင်၏ စတိုင်ကို သတ်မှတ်မထားပါ။

ပြသမှုအစိတ်အပိုင်း၏ကွန်တိန်နာကို `main.demo.Lg`နှင့် ချိန်ညှိသင့်ပြီး ၎င်းတို့ကြားတွင် `margin` နှင့် `padding` မရှိသင့်ပါ၊ အဘယ်ကြောင့်ဆိုသော် `main.demo`ကိုယ်တိုင်က `main.demo`ကို သတ်မှတ်ပြီးဖြစ်သောကြောင့်၊ ၎င်းကို အလွတ်သတ်မှတ်ခြင်းကို ထပ်ခါတလဲလဲ ပြုလုပ်ပါမည်။

`demo/_.styl`ပြကွက်တွင် အစိတ်အပိုင်းတစ်ခုသာရှိပါက၊ ၎င်း၏ကွန်တိန်နာတွင် ပြင်ပဘောင်တစ်ခုရှိမည်မဟုတ်ပါ၊ နှင့်သာသတ်မှတ်မည်ဖြစ်သောကြောင့် `max-width` (တင်ဆက်မှုကွန်တိန်နာသည် ပြင်ပဘောင်ကို အလိုအလျောက်ထည့်သွင်းပေးမည်ဖြစ်သောကြောင့်)။ အများအပြားရှိလျှင် ကွန်တိန်နာကို အပြင်ဘောင်ဖြင့် ပိုင်းခြားနိုင်သည်။

`img`ကို `demo/_.pug`တွင် မရေးပါနှင့်၊ နောက်ခံပုံအား သတ်မှတ်ရန် `demo/_.styl`ကို အသုံးပြုပါ။

အစိတ်အပိုင်းတွင် ပုံစံမျိုးစုံရှိပါက၊ ၎င်းတို့ကို အစီအစဥ်အတိုင်း ပြသပါ (ဥပမာ- loading၊ ဗလာဒေတာ၊ ဒေတာ၊ မအောင်မြင်သော၊ စသည်)၊ ဒေါင်လိုက်ပုံစံကို flex

`demo/_.styl`: `stylus` သရုပ်ပြစာမျက်နှာ၏ စတိုင်၊ [./styl.md](./styl.md) ၏ ကုဒ်သတ်မှတ်ချက်များနှင့် ဒီဇိုင်းဗေဒကို လိုက်နာပါ

သရုပ်ပြမည့် အစိတ်အပိုင်းကို တင်သွင်းရန် `@import '../_.styl'`ကို မရေးပါနှင့်၊ ၎င်းသည် အလိုအလျောက် ထိုးသွင်းလိမ့်မည်၊

ကျေးဇူးပြု၍ `@import '../../Btn/_.styl'`ကို `demo/_.styl`တွင် ထည့်သွင်းပြီး `.Btn`ကို အသုံးပြုပါ။

`demo/_.js`- ဇာတ်ညွှန်းကို ခေါ်ဆိုပါ၊ ဖော်မတ် `export default (root) => { ... }`၊ ကန့်သတ်ချက် `root` သည် အိမ်ရှင် (`document` သို့မဟုတ် သဲပုံးကွန်တိန်နာ၏ အရင်းအနှီး အစိတ်အပိုင်း)

`D.createElement` တားမြစ်ထားသည်၊ [./js.md](./js.md) မှ ဖော်ပြထားသော `newEl`ကို အသုံးပြုပါ။

`js`ကို `c-t`နှင့် နိုင်ငံတကာသို့ ပြောင်းလဲခြင်းအတွက် `fLang`ကို အသုံးပြုရန် တားမြစ်ထားသည်။

`../_.js` (`../_.js` အစိတ်အပိုင်းကို ကိုယ်တိုင်ပြသရန်ဖြစ်ပါက၊ အခြားသရုပ်ပြများ လိုအပ်သည့် မှီခိုအစိတ်အပိုင်းများကို တင်သွင်းရန် လိုအပ်ပါသေးသည်) သာလျှင် `import` လိုအပ်ပြီး သီးခြားအကြောင်းအရာကို ထည့်သွင်းခြင်းမရှိပါ၊ ချန်လှပ်ထားနိုင်သည် (မူဘောင်သည် ၎င်းကို အလိုအလျောက် ထိုးသွင်းလိမ့်မည်)

`pug` `js`နှင့် `styl` တင်သွင်းရန် မလိုအပ်ပါ၊ မူဘောင်သည် ၎င်းတို့ကို အလိုအလျောက် ထိုးသွင်းမည်
