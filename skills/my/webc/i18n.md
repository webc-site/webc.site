# နိုင်ငံတကာပြုခြင်း

ဖြစ်ရပ်- အစိတ်အပိုင်းများနှင့် ဒီမိုကို နိုင်ငံတကာသို့ အကောင်အထည်ဖော်ရန် `src/BoxX`ကို ကိုးကားပါ။

စည်းမျဉ်း-
- `yml`ထည့်သွင်းမှု၏အဆုံးတွင် `:`မထည့်ပါနှင့်၊ `:` pug ပုံစံနှင့် js ကုဒ်ထဲသို့ ထည့်ပါ၊ ပေါင်းထည့်ပါ။
- နောက်မှလိုက်နေသောနေရာများကို အလွယ်တကူဖယ်ရှားရန် `|-`ကိုသုံးပါ။
- နိုင်ငံတကာတွင် ထည့်သွင်းမှု မရှိပါက၊ `I18N.js`ဖိုင်ကို ရှင်းလင်းပါ (ဘာမှမရေးပါနှင့်၊ `export`ဟု မရေးပါနှင့်)
- ထည့်သွင်းမှုများကို ဘာသာစကားတစ်ခုတည်းတွင်ရှိရမည်ဖြစ်ပြီး ဘာသာစကားများစွာကို ရောနှောခြင်းကို တင်းကြပ်စွာတားမြစ်ထားသည်။

လုပ်ငန်းစဉ်-

1. root directory `tran.yml` → `tran` → `from`၊

2. `src/component အမည်` လမ်းညွှန်ကို ထည့်ပါ။

3. အစိတ်အပိုင်းများကို နိုင်ငံတကာသို့ ပြောင်းလဲပါ။
   - `src/component name/i18n/source language/js.yml` တွင် ထည့်သွင်းမှုအဖြစ် `_.js`အသုံးပြုသော စာသားကို ရေးပါ။
   - `အစိတ်အပိုင်းအမည်/I18N.js` ကို မွမ်းမံပြင်ဆင်ပြီး `yml` ဝင်ခွင့်သော့အမည်ဖြစ်သည့် `export const KEY = "ymlKey"` ၏ အဆက်မပြတ်ကို စီစဉ်သတ်မှတ်ပါ
   - `မိတ်ဆက်ပြီး အစိတ်အပိုင်းအမည်/_.js` (`js.yml`ရှိပါက၊ မရှိလျှင် မလိုအပ်ပါ)။
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     "./I18N.js" မှ i18nUrl၊ { KEY } ကို တင်သွင်းပါ။ // framework သည် i18nUrl ကို အလိုအလျောက် ထိုးသွင်းမည်ဖြစ်ပြီး၊ ပုံသေ တင်ပို့မှု ပုံသေ၊ ၎င်းကို ကိုယ်တိုင် မဖန်တီးပါ။

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    fLang ခေါ်ဆိုမှုများကို လျှော့ချရန် အစီအစဥ်ဖြင့် စုစည်းကြည့်ပါ။
    အစိတ်အပိုင်း၏ `_.js` စာသားပါရှိသော စာကြောင်းများ မရှိသင့်ပါ။

4. နိုင်ငံတကာသို့ တင်ဆက်မှုများ
   - `demo/_.pug`တွင်၊ ဘာသာပြန်ရန် လိုအပ်သော အညွှန်းစာသားကို `c-t သော့အမည်` ပုံစံဖြင့် သိမ်းပိုက်ထားသည် (၎င်းသည် စိတ်ကြိုက်အညွှန်း၊ သီးခြားစာကြောင်းဖြစ်သည်)၊ ၎င်းကို `demo/i18n/source language/pug.yml` တွင် ရေးထားသည်။
   - `demo/_.js` `demo/i18n/source language/js.yml` တွင် အသုံးပြုထားသော မူရင်းစာသားထည့်သွင်းမှုကို ရေးပြီး `demo/I18N.js`
   - `fetchLang`ကို `demo/_.js`ထဲသို့ မိတ်ဆက်ပြီး `./I18N.js` (၎င်းသည် `demo/I18N.js`)၊ အထက်ဖော်ပြပါ အစိတ်အပိုင်း နိုင်ငံတကာအသွင်ကူးပြောင်းရေး လုပ်ငန်းစဉ်ကို ကိုးကားပါ (ဥပမာ၊ `demo/i18n` အောက်တွင် `js.yml` မပါပါ၊ ဤအဆင့်ကို ကျော်သွားပါ)
   - `demo/I18N.js` `demo/i18n/source language/js.yml` တွင် သော့အမည် ကိန်းသေများကို ပြင်ဆင်သတ်မှတ်ပြီး ထုတ်ယူပါ။

5. အစိတ်အပိုင်းဖော်ပြချက်ကို `အစိတ်အပိုင်းအမည်/i18n/အရင်းအမြစ်ဘာသာစကား/README.md` တွင် ရေးပါ။ လုပ်ငန်းစဉ်မှာ အောက်ပါအတိုင်းဖြစ်သည်။
  5.1 ပင်မခေါင်းစဉ်ကိုရေးပါ။ ခေါင်းစဉ်သည် `#`ဖြင့် စတင်သင့်သည်၊ ဝါကျတစ်ကြောင်းတွင် အစိတ်အပိုင်း၏ ရည်ရွယ်ချက်ကို ဖော်ပြပါ (အစိတ်အပိုင်းအမည်ကို မရေးပါနှင့်) အဆုံးတွင် သတ်ပုံမထည့်ပါနှင့်။
  5.2 ပင်မခေါင်းစဉ်မှစတင်ကာ အစိတ်အပိုင်း၏ အဓိကအင်္ဂါရပ်များကို စာရင်းပြုစုပြီး ဒုတိယခေါင်းစဉ်များကို မရေးပါနှင့်။ လက္ခဏာများစွာရှိနေပါက၊ အစီအစဥ်မထားသောစာရင်းကိုအသုံးပြုပါ (အစီအစဥ်မတင်ရသေးသောစာရင်းဖော်ပြချက်သည် အတော်လေးတိုနေပါက၊ အဆုံးတွင်ကာလရှိမည်မဟုတ်ပါ)။
  5.3 ရေးပါ `## ဒီမို`ကို အသုံးပြုပါ၊ သရုပ်ပြကုဒ်သည် ` ```html ` နှင့် စတင်သည် (သရုပ်ပြကုဒ်တွင်၊ တစ်ခုတည်းသော အစိတ်အပိုင်းတဂ်ကို မပိတ်ပါ)၊ `js` Tag ရှိပါက `html``id; အတွင်းတွင်၊ ရေးရန် `html``id;
  စစ်မှန်သော js ဖြစ်ပါက၊ ` ```js ` ဖြင့် စတင်ပါ။ `js`သရုပ်ပြမှုတွင်၊ သင်သည် `component/_.js`တင်သွင်းရန် လိုအပ်ပါက၊ ကျေးဇူးပြု၍ "webc.site/component name.js";` မှ xxx ကို`တင်သွင်းပါ။
  `js` ကိန်းသေများစွာကို ကြေညာရန် `const` ကဲ့သို့သော တိုတိုတုတ်တုတ် အရေးအသားကို သုံးပါ

ပင်မစာသားသည် အစိတ်အပိုင်း၊ အင်တာဖေ့စ်ဘောင်များကို အသုံးပြုနည်း၊ `css` နှင့် `class` မတူညီသောပြည်နယ်များကြားတွင် ပြောင်းခြင်းစသည်တို့ကို မိတ်ဆက်ပေးသည်။
နောက်ပိုင်းတွင် ကန့်သတ်ချက်များ၊ ပြန်ပေးသည့်တန်ဖိုး၊ ပုံစံအခြေအနေ၊ စသည်တို့ကို ဖော်ပြပါ (ရှိပါက) နှင့် ဒုတိယခေါင်းစဉ်ကို ထည့်ပါ။

စကားလုံးများကို တိုတိုတုတ်တုတ်သုံး၍ နာမဝိသေသနများကို ရှောင်ကြဉ်ပါ။

Copywriting သည် အဓိပ္ပါယ်မဲ့သောစကားလုံးများကို ရှောင်ရှားသင့်ပြီး အဓိပ္ပါယ်မဲ့စကားလုံးများ (ဥပမာ- universal၊ container စသည်တို့) ကို မရေးသင့်ပါ။

6. root directory ၏ `./sh/tran.js`၊ ဘာသာပြန်ခြင်းကို လုပ်ဆောင်ပါ။
