# 국제화

사례: 구성 요소의 국제화 및 데모를 구현하려면 `src/BoxX`을 참조하세요.

규칙:
- `yml` 항목 끝에 `:`를 추가하지 말고 `:`를 pug 템플릿과 js 코드에 넣고 연결하세요.
- 후행 공백을 쉽게 제거하려면 `yml`의 여러 줄 텍스트에 `|-`를 사용하세요.
- 국제화된 항목이 없으면 `I18N.js` 파일을 지우십시오(아무 것도 쓰지 말고 `export`도 쓰지 마십시오).
- 출품작은 단일 언어로 작성되어야 하며, 여러 언어를 혼합하는 것은 엄격히 금지됩니다.

프로세스:

1. 루트 디렉터리 `tran.yml` → `tran` → `from`에 구성된 소스 언어 코드 보기

2. `src/구성요소 이름` 디렉토리를 입력하세요.

3. 구성요소 국제화
   - `_.js`에서 사용하는 텍스트를 `src/컴포넌트 이름/i18n/source 언어/js.yml`의 항목으로 작성하세요.
   - `구성 요소 이름/I18N.js`를 수정하고 `yml` 항목 키 이름의 상수를 구성합니다(예: `export const KEY = "ymlKey"`).
   - `컴포넌트 이름/_.js에 도입 및 바인딩`(`js.yml`이 있으면 필요하지 않음):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     "./I18N.js"에서 i18nUrl, { KEY } 가져오기; // 프레임워크는 기본 내보내기 기본값인 i18nUrl을 자동으로 삽입합니다. 직접 만들지 마세요.

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    fLang 호출을 줄이기 위해 일괄 바인딩을 시도해 보세요.
    구성 요소의 `_.js`에는 텍스트가 포함된 문자열이 있어서는 안 됩니다.

4. 프레젠테이션의 국제화
   - `demo/_.pug`에서 번역해야 할 라벨 텍스트는 `c-t 키 이름`(맞춤 라벨, 별도의 줄) 형식으로 채워지며 해당 항목은 `demo/i18n/source 언어/pug.yml`에 기록됩니다.
   - `demo/_.js` `demo/i18n/source 언어/js.yml`에 사용된 원본 텍스트 항목을 작성하여 `demo/I18N.js`에 넣습니다.
   - `fetchLang`를 `demo/_.js`에 도입하고 `./I18N.js`(`demo/I18N.js`)를 로드합니다. 위의 구성요소 국제화 프로세스를 참조하세요(예를 들어 `demo/i18n`에는 아래 `js.yml`이 없습니다. 이 단계를 건너뛰세요).
   - `demo/I18N.js` `demo/i18n/source 언어/js.yml`에서 키 이름 상수를 구성하고 내보냅니다.

5. `컴포넌트 이름/i18n/소스 언어/README.md`에 컴포넌트 설명을 작성하세요. 과정은 다음과 같습니다
  5.1 주요 제목을 작성합니다. 제목은 `#`로 시작해야 하며, 컴포넌트의 목적을 한 문장으로 설명하고(컴포넌트 이름을 쓰지 않음), 끝에 구두점을 추가하지 않아야 합니다.
  5.2 주요 제목부터 시작하여 구성 요소의 주요 기능을 나열하고 보조 제목을 쓰지 마십시오. 여러 특성이 있는 경우 순서가 지정되지 않은 목록을 사용합니다. 순서가 지정되지 않은 목록 설명이 상대적으로 짧으면 끝에 마침표가 없습니다.
  5.3 `## 데모`를 사용하여 데모 코드는 ` ```html `로 시작합니다(데모 코드에서는 단일 요소 태그가 닫히지 않음). `js`가 있는 경우 내부적으로 `html`를 작성하고 `<script type="module">` 태그를 사용합니다.
  순수 js인 경우 ` ```js `로 시작하세요. `js` 데모에서 `comComponent/_.js`를 가져와야 하는 경우 `import xxx from "webc.site/comComponent/_.js";`를 사용하세요.
  `js` 여러 상수를 선언하려면 `const` 와 같이 간결하게 작성하세요.

본문에서는 구성요소 사용 방법, 인터페이스 매개변수, 다양한 상태에서 `css`과 `class` 간 전환 등을 소개합니다.
이어서 매개변수, 반환 값, 스타일 상태 등(있는 경우)을 설명하고 보조 제목을 추가합니다.

단어를 간결하게 사용하고 형용사는 피하세요.

카피라이팅은 말도 안 되는 말을 피하고 의미 없는 단어(예: 보편적, 컨테이너 등)를 쓰지 않아야 합니다.
