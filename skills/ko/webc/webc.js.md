
  `js` 구성요소의 경우 `src/webc/comComponentname.js`에서 `Light DOM`를 사용하여 구성요소를 구현합니다.

  상대 경로를 사용하여 `import "./comComponent name.js"`와 같은 다른 구성 요소를 가져옵니다. (각 구성 요소의 JS 파일에서 먼저 종속성을 가져온 다음 해당 `_.styl`을 가져옵니다(예: `import "./comComponent name/_.styl";`)).

  `this` 웹 구성요소의 기본 콜백 함수만 정의하고 나머지는 독립 함수로 작성하고 `this` 및 필수 매개변수를 전달합니다.

  외부에 노출된 인터페이스 외에도 트리 쉐이킹 최적화 및 코드 압축을 용이하게 하기 위해 `this`에 사용자 정의 비공개 속성을 바인딩하는 것이 금지되어 있으며(함수 언로드에는 `this.$` 사용) 중간 변수를 보유하기 위한 클로저를 반환하는 데 고차 함수를 사용할 수 있습니다.

  `constructor`을 쓰지 마십시오. 속성은 `connectedCallback`에서 얻습니다.


  `src/js/`의 공개 기능을 재사용하려면 [./js.md](./js.md)를 참조하세요.

  컨테이너의 경우 속성 대신 `slot`을 사용하세요.
  데이터를 로드해야 하는 경우 로딩 인터페이스를 비동기 콜백 함수 형태로 상위 계층에 노출하고 주석을 사용하여 반환 값의 형식을 나타냅니다.
  구성 요소는 메모리 누수를 방지해야 합니다. `disconnectedCallback` 제거 이벤트를 하나의 함수로 병합하고 `this.$` 여러 제거 함수를 작성하지 마십시오.
  인터페이스 반환 값은 개체가 아닌 단일 값, 배열 또는 배열의 배열일 수 있습니다.
  구성 요소에 여러 상태가 있는 경우 여러 줄 주석을 사용하여 각 상태의 데이터 형식을 명확하게 설명합니다. (각 주석 줄 앞에 `*`을 붙이지 마세요)
  숫자 상수(`const.js`로 정의됨)를 사용하여 상태 전환.
  콜백이 로드되면 `const xx = newEl('b');xx.className="Ing"`을 사용하여 애니메이션을 표시하고 구성 요소 JS 파일의 `import "./Ing.js"`를 통해 종속성을 도입합니다.
