# 데모

`src/BoxX/demo`을 참조하고 `src/comComponent name/demo/`에서 구성 요소의 데모를 개발하세요.

`demo/_.pug`, 데모 하위 템플릿

최소한의 구성 요소 태그만 포함되며 전체 페이지가 필요하지 않으며 스타일 및 스크립트 가져오기가 금지됩니다. 프레임워크가 자동으로 주입됩니다(예: `demo/_.styl`, `demo/_.js`).

제목을 쓰지 마세요(예를 들어 xxx 데모를 쓰지 마세요), 컴포넌트 이름을 쓰지 마세요, 컴포넌트 소개를 하지 마세요, 데모 사용 사례의 상태만 쓰세요(상태가 없으면 제목을 쓰지 마세요)

`_.pug`의 루트 요소는 `demo/_.styl`에 스타일을 정의하고 `main.demo.Lg` 자체의 스타일을 정의하지 않는 `main.demo.Lg`를 사용합니다.

디스플레이 구성 요소의 컨테이너는 `main.demo.Lg`에 맞춰 정렬되어야 하며, `main.demo` 자체가 이미 `padding`를 설정하고 공백으로 설정하면 반복되므로 둘 사이에 `margin` 및 `padding`가 없어야 합니다.

`demo/_.styl` 디스플레이 구성요소가 하나만 있는 경우 해당 컨테이너에는 외부 프레임이 없으며 `max-width`만 설정됩니다(프레젠테이션 컨테이너가 자동으로 외부 프레임을 추가하기 때문). 컨테이너가 여러 개인 경우 외부 프레임으로 컨테이너를 분리할 수 있습니다.

`demo/_.pug`에 `img`를 쓰지 말고 `demo/_.styl`을 사용하여 배경 이미지를 설정하세요.

구성 요소에 여러 양식이 있는 경우 순서대로 표시하십시오(예: 로드, 빈 데이터, 데이터, 실패 등), 유연한 수직 레이아웃

`demo/_.styl`: `stylus` 데모 페이지 스타일은 [./styl.md](./styl.md)의 코드 사양과 디자인 미학을 따르세요.

시연할 구성 요소를 가져오려면 `@import '../_.styl'`을 쓰지 마세요. 자동으로 삽입됩니다.

`@import '../../Btn/_.styl'`를 `demo/_.styl`에서 가져오고 `.Btn`을(를) 사용하세요.

`demo/_.js`: 스크립트 호출, 형식 `export default (root) => { ... }`, 매개변수 `root`은 호스트(`document` 또는 샌드박스 컨테이너의 루트 요소)입니다.

`D.createElement`은 금지되어 있습니다. [./js.md](./js.md)에서 언급한 `newEl`을 사용하세요.

`js`은 `c-t` 및 국제화 `fLang`에서 사용이 금지됩니다.

`../_.js`(`../_.js`가 구성 요소 자체를 표시하는 경우, 다른 데모에 필요한 종속 구성 요소를 계속 가져와야 함) `import`만 필요하며 특정 콘텐츠를 가져오지 않으므로 생략할 수 있습니다(프레임워크가 자동으로 삽입합니다).

`pug` `js` 및 `styl` 을 가져올 필요가 없습니다. 프레임워크가 자동으로 삽입합니다.
