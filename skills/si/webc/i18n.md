# ජාත්යන්තරකරණය

අවස්ථාව: සංරචක සහ ආදර්ශන ජාත්‍යන්තරකරණය ක්‍රියාත්මක කිරීමට `src/BoxX` වෙත යොමු වන්න.

රීතිය:
- `yml` ප්‍රවේශය අවසානයේ `:` එකතු නොකරන්න, `:` pug සැකිල්ලට සහ js කේතයට දමා එය කොටස් කරන්න.
- පසුපස ඇති හිස්තැන් පහසුවෙන් ඉවත් කිරීමට `yml` බහු පේළි පෙළ සඳහා `|-` භාවිත කරන්න.
- ජාත්‍යන්තරකරණය වූ ප්‍රවේශයක් නොමැති නම්, `I18N.js` ගොනුව හිස් කරන්න (කිසිවක් ලියන්න එපා, `export` ලියන්න එපා)
- ඇතුළත් කිරීම් තනි භාෂාවකින් විය යුතු අතර බහු භාෂා මිශ්‍ර කිරීම සපුරා තහනම් වේ.

ක්රියාවලිය:

1. මූල නාමාවලියෙහි වින්‍යාස කර ඇති මූලාශ්‍ර භාෂා කේතය බලන්න `tran.yml` → `tran` → `from`

2. `src/component name` බහලුම ඇතුල් කරන්න

3. සංරචක ජාත්‍යන්තරකරණය කරන්න
   - `src/component name/i18n/source language/js.yml` හි ප්‍රවේශයක් ලෙස `_.js` විසින් භාවිතා කරන ලද පෙළ ලියන්න.
   - `සංරචක නාමය/I18N.js` වෙනස් කර `export const KEY = "ymlKey"` වැනි `yml` ඇතුල්වීමේ යතුරු නාමයේ නියතය වින්‍යාස කරන්න
   - `හඳුන්වා දීම සහ සංරචක නාමය/_.js` (`js.yml` තිබේ නම්, එය අවශ්‍ය නොවේ නම්):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     i18nUrl, { KEY } "./I18N.js" වෙතින් ආයාත කරන්න; // රාමුව ස්වයංක්‍රීයව i18nUrl එන්නත් කරයි, පෙරනිමි අපනයන පෙරනිමිය, එය ඔබම සාදන්න එපා

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    FLang ඇමතුම් අඩු කිරීමට කණ්ඩායම් වශයෙන් බැඳීමට උත්සාහ කරන්න
    සංරචකයේ `_.js` පෙළ අඩංගු තන්තු නොතිබිය යුතුය

4. ඉදිරිපත් කිරීම් ජාත්‍යන්තරකරණය
   - `demo/_.pug` හි, පරිවර්තනය කිරීමට අවශ්‍ය ලේබල් පාඨය `c-t යතුරු නාමය` (මෙය අභිරුචි ලේබලයකි, වෙනම පේළියකි) ආකාරයෙන් අල්ලාගෙන ඇත, සහ ප්‍රවේශය `demo/i18n/source language/pug.yml` වෙත ලියා ඇත.
   - `demo/_.js` `demo/i18n/source language/js.yml` හි භාවිතා කරන ලද මුල් පෙළ ප්‍රවේශය ලියා `demo/I18N.js` තුළ තබන්න.
   - `fetchLang` `demo/_.js` වෙත හඳුන්වා දී `./I18N.js` පූරණය කරන්න (මෙය `demo/I18N.js`), ඉහත සංරචක ජාත්‍යන්තරකරණ ක්‍රියාවලිය වෙත යොමු වන්න (උදාහරණයක් ලෙස, `demo/i18n` පහතින් `js.yml` නොමැත, මෙම පියවර මඟ හරින්න)
   - `demo/I18N.js` `demo/i18n/source language/js.yml` හි යතුරු නාම නියතයන් වින්‍යාස කර අපනයනය කරන්න

5. සංරචක විස්තරය `Component name/i18n/source language/README.md` හි ලියන්න. ක්රියාවලිය පහත පරිදි වේ
  5.1 ප්‍රධාන මාතෘකාව ලියන්න. මාතෘකාව `#` වලින් ආරම්භ විය යුතු අතර, සංරචකයේ අරමුණ එක් වාක්‍යයකින් විස්තර කළ යුතුය (සංරචක නාමය ලියන්න එපා), සහ අවසානයේ විරාම ලකුණු එකතු නොකරන්න.
  5.2 ප්‍රධාන මාතෘකාවෙන් ආරම්භ වන සංරචකයේ ප්‍රධාන ලක්ෂණ ලැයිස්තුගත කරන්න, සහ ද්විතියික මාතෘකා ලියන්න එපා. බහුවිධ ලක්ෂණ තිබේ නම්, ඇණවුම් නොකළ ලැයිස්තුවක් භාවිතා කරන්න (ඇණවුම් නොකළ ලැයිස්තු විස්තරය සාපේක්ෂව කෙටි නම්, අවසානයේ කාලසීමාවක් නොමැත).
  5.3 ලියන්න `## demo භාවිතා කරමින්`, demo code එක ` ```html ` වලින් ආරම්භ වේ (ආදර්ශන කේතයේ, තනි මූලද්‍රව්‍ය ටැගය වසා නැත), `js` තිබේ නම්, අභ්‍යන්තරව CO DE, CO DE භාවිතා කරන්න `<script type="module">` ටැගය;
  එය pure js නම්, ` ```js ` වලින් පටන් ගන්න. `js` ප්‍රදර්ශනයේදී, ඔබට `component/_.js` ආයාත කිරීමට අවශ්‍ය නම්, කරුණාකර "webc.site/component/_.js";` වෙතින් xxx ආනයනය කරන්න `භාවිතා කරන්න.
  `js` බහු නියතයන් ප්‍රකාශ කිරීමට `const` වැනි සංක්ෂිප්ත ලිවීම් භාවිතා කරන්න

ප්‍රධාන පෙළ මඟින් සංරචකය, අතුරුමුහුණත් පරාමිති, විවිධ ප්‍රාන්තවල `css` සහ `class` අතර මාරුවීම යනාදිය භාවිතා කරන ආකාරය හඳුන්වා දෙයි.
පසුව පරාමිති, ආපසු ලැබෙන අගය, මෝස්තර තත්ත්වය ආදිය විස්තර කරන්න (ඇත්නම්), සහ ද්විතියික මාතෘකාවක් එක් කරන්න.

වචන සංක්ෂිප්තව භාවිතා කර විශේෂණ පදවලින් වළකින්න.

පිටපත් ලිවීම විකාර වලින් වැළකිය යුතු අතර අර්ථ විරහිත වචන (විශ්වීය, බහාලුම්, ආදිය) නොලිය යුතුය.

6. මූල නාමාවලියෙහි `./sh/tran.js` ධාවනය කරන්න, පරිවර්තනය
