# بین المللی شدن

مورد: برای اجرای بین المللی سازی اجزا و نسخه ی نمایشی به `src/webc/BoxX` مراجعه کنید.

قانون:
- `yml` `:` را در انتهای ورودی اضافه نکنید، `:` را در قالب pug و کد js قرار دهید و آن را به هم متصل کنید.
- از `|-` برای متن چند خطی `yml` استفاده کنید تا به راحتی فضاهای انتهایی را حذف کنید.
- اگر ورودی بین المللی وجود ندارد، فایل `I18N.js` را پاک کنید (چیزی ننویسید، `export` را ننویسید)
- ورودی ها باید به یک زبان باشد و ترکیب چند زبان اکیداً ممنوع است.

فرآیند:

1. مشاهده کد زبان مبدأ پیکربندی شده در فهرست اصلی `tran.yml` → `tran` → `from`

2. دایرکتوری `src/webc/component name` را وارد کنید

3. بین المللی کردن مولفه ها
   - متن استفاده شده در `component name.js` را به عنوان ورودی در `src/webc/component name/i18n/source language/js.yml` بنویسید.
   - `src/webc/component name/I18N.js` را تغییر دهید و ثابت `yml` نام کلید ورودی، مانند `export const KEY = "ymlKey"` را پیکربندی کنید.
   - معرفی و صحافی در `src/webc/component name.js` (اگر `js.yml` وجود داشته باشد، اگر نه، لازم نیست):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     i18nUrl، { KEY } را از "./component name/I18N.js" وارد کنید. // چارچوب به طور خودکار i18nUrl، پیش فرض صادرات پیش فرض را تزریق می کند، خودتان آن را ایجاد نکنید

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    سعی کنید به صورت دسته ای باند شوید تا تماس های fLang را کاهش دهید
    `component name.js` جزء نباید رشته ای حاوی متن داشته باشد

4. بین المللی سازی ارائه ها
   - در `demo/_.pug`، متن برچسبی که باید ترجمه شود به شکل `نام کلید c-t` اشغال می‌شود (این یک برچسب سفارشی است، یک خط جداگانه)، و ورودی در `demo/i18n/source language/pug.yml` نوشته می‌شود.
   - `demo/_.js` ورودی متن اصلی مورد استفاده در `demo/i18n/source language/js.yml` را بنویسید و آن را در `demo/I18N.js` قرار دهید
   - `fetchLang` را در `demo/_.js` معرفی کنید و `./I18N.js` را بارگیری کنید (این `demo/I18N.js` است)، به فرآیند بین المللی سازی مؤلفه در بالا مراجعه کنید (به عنوان مثال، `demo/i18n` `js.yml` در زیر ندارد، از این مرحله بگذرید)
   - `demo/I18N.js` ثابت های نام کلید را در `demo/i18n/source language/js.yml` پیکربندی کنید و صادر کنید

5. توضیحات مؤلفه را در `src/webc/component name/i18n/source language/README.md` بنویسید. روند به شرح زیر است
  5.1 عنوان اصلی را بنویسید. عنوان باید با `#` شروع شود، هدف جزء را در یک جمله توضیح دهد (نام جزء را ننویسید)، و در پایان علائم نگارشی اضافه نکنید.
  5.2 ویژگی های اصلی جزء را با عنوان اصلی شروع کنید و عناوین ثانویه را ننویسید. اگر چندین ویژگی وجود دارد، از یک لیست نامرتب استفاده کنید (اگر شرح لیست نامرتب نسبتا کوتاه باشد، در پایان دوره وجود نخواهد داشت).
  5.3 بنویسید `## با استفاده از نسخه نمایشی`، کد نمایشی با ` ```html ` شروع می شود (در کد آزمایشی، تگ عنصر واحد بسته نیست)، اگر `js` وجود دارد، <b>internal class="CODE"> را بنویسید <b> class="CODE"> `html id="3">` برچسب، `js` در خط اول محتوا دارای تورفتگی نیست.
  اگر js خالص است، با ` ```js ` شروع کنید. `js` در نمایش، اگر نیاز به وارد کردن `component/_.js` دارید، لطفاً از `import xxx from "webc.site/component name.js";` استفاده کنید. این `js` کامپایل شده است که به طور خودکار js و css را تزریق می کند.
  `js` از نوشتن مختصر، مانند `const` برای اعلام چندین ثابت استفاده کنید

متن اصلی نحوه استفاده از کامپوننت، پارامترهای رابط، جابجایی بین `css` و `class` در حالت های مختلف و غیره را معرفی می کند.
سپس پارامترها، مقدار بازگشتی، وضعیت سبک و غیره (در صورت وجود) را شرح دهید و یک عنوان ثانویه اضافه کنید.

کلمات را به طور مختصر به کار ببرید و از صفت ها اجتناب کنید.

کپی رایتینگ باید از بیهوده پرهیز شود و کلمات بی معنی (مانند: جهانی، ظرف و ...) ننویسد.

6. `./sh/tran.js` دایرکتوری ریشه، ترجمه را اجرا کنید
