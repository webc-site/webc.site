# Демо

Обратитесь к `src/webc/BoxX/demo` и разработайте демо-версию компонента в `src/webc/comComponent name/demo/`.

`demo/_.pug`, демонстрационный подшаблон

Включены только минималистичные теги компонентов, полная страница не требуется, а импорт стилей и скриптов запрещен. Платформа автоматически внедрит (например, `demo/_.styl`, `demo/_.js`).

Не пишите заголовок (например, не пишите xxx demo), не пишите имя компонента, не представляйте компонент, пишите только статус варианта использования демо (если нет статуса, не пишите заголовок)

Корневой элемент `_.pug` использует `main.demo.Lg` , который определяет стиль в `demo/_.styl` и не определяет стиль самого `main.demo.Lg`.

Контейнер компонента отображения должен быть выровнен по `main.demo.Lg`, и между ними не должно быть `margin` и `padding`, поскольку сам `main.demo` уже установил `padding`, и установка его пустым будет повторяться.

`demo/_.styl` Если имеется только один компонент отображения, его контейнер не будет иметь внешней рамки, а будет установлен только `max-width` (поскольку контейнер представления автоматически добавит внешнюю рамку). Если их несколько, контейнер можно разделить внешней рамкой.

Не пишите `img` в `demo/_.pug`, используйте `demo/_.styl`, чтобы установить фоновое изображение.

Если компонент имеет несколько форм, отображайте их по порядку (например, загрузка, пустые данные, данные, сбой и т. д.), гибкое вертикальное расположение.

`demo/_.styl`: стиль `stylus` демонстрационной страницы. Следуйте спецификациям кода и эстетике дизайна [./styl.md](./styl.md).

Если вы не напишете `@import '../_.styl'` для импорта демонстрируемого компонента, он будет автоматически внедрен; запрещено использовать `@import` в `demo/_.styl` для импорта стилей других компонентов. Если демо-версия зависит от других компонентов (например, кнопки `.Btn`), импортируйте зависимые компоненты через `import "../../Btn.js"` в `demo/_.js`.

`demo/_.js`: вызов сценария, формат `export default (root) => { ... }`, параметр `root` — это хост (`document` или корневой элемент контейнера песочницы).

`D.createElement` запрещен, используйте `newEl`, упомянутый в [./js.md](./js.md)

`js` запрещено использовать в `c-t` и интернационализировать `fLang`

Если `../comComponent name.js` (`../comComponent name.js` предназначен для отображения самого компонента, а другие зависимые компоненты, необходимые для демонстрации, все равно необходимо импортировать), требуется только `import`, а не импортируется никакой конкретный контент, который можно опустить (фреймворк автоматически внедрит его)

`pug` Нет необходимости импортировать `js` и `styl`, платформа автоматически внедрит их.
