# интернационализация

Случай: см. `src/BoxX`, чтобы реализовать интернационализацию компонентов и демонстрационную версию.

правило:
- `yml` Не добавляйте `:` в конце записи, поместите `:` в шаблон мопса и код js и соедините их.
- Используйте `|-` для многострочного текста `yml`, чтобы легко удалять конечные пробелы.
- Если интернационализированной записи нет, очистите файл `I18N.js` (ничего не записывайте, не записывайте `export`).
- Заявки должны быть на одном языке, смешивание нескольких языков строго запрещено.

процесс:

1. Просмотрите код исходного языка, настроенный в корневом каталоге `tran.yml` → `tran` → `from`

2. Введите каталог `src/имя компонента`.

3. Интернационализация компонентов
   - Запишите текст, используемый `_.js`, в качестве записи в `src/comComponent name/i18n/source Language/js.yml`.
   - Измените `имя компонента/I18N.js` и настройте константу имени ключа входа `yml`, например `export const KEY = "ymlKey"`
   - `Введено и привязано к имени компонента/_.js` (если есть `js.yml`, то нет необходимости):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     импортировать i18nUrl, {KEY} из "./I18N.js"; // Платформа автоматически внедрит i18nUrl, стандартный экспортный стандарт, не создавайте его самостоятельно

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Попробуйте выполнить пакетное связывание, чтобы уменьшить количество вызовов Flang.
    `_.js` компонента не должен содержать строки, содержащие текст.

4. Интернационализация презентаций
   - В `demo/_.pug` текст метки, которую необходимо перевести, занимает вид `имя ключа c-t` (это пользовательская метка, отдельная строка), а запись записывается в `demo/i18n/source Language/pug.yml`
   - `demo/_.js` Запишите исходную текстовую запись, использованную в `demo/i18n/source Language/js.yml`, и поместите ее в `demo/I18N.js`
   - Введите `fetchLang` в `demo/_.js` и загрузите `./I18N.js` (это `demo/I18N.js`), обратитесь к процессу интернационализации компонента выше (например, `demo/i18n` не имеет `js.yml` ниже, пропустите этот шаг).
   - `demo/I18N.js` Настройте константы имен ключей в `demo/i18n/source Language/js.yml` и экспортируйте их.

5. Запишите описание компонента в `Имя компонента/i18n/source Language/README.md`. Процесс заключается в следующем
  5.1 Напишите основной заголовок. Заголовок должен начинаться с `#`, описывать назначение компонента в одном предложении (название компонента не писать) и не ставить знаки препинания в конце.
  5.2 Перечислите основные возможности компонента, начиная с основного названия, и не пишите второстепенные названия. Если характеристик несколько, используйте неупорядоченный список (если описание неупорядоченного списка относительно короткое, в конце точки не будет).
  5.3 Напишите `## Используя демонстрационный пример`, демонстрационный код начинается с ` ```html ` (в демонстрационном коде тег одного элемента не закрывается), если есть `js`, напишите `html` внутри, используйте тег `<script type="module">`;
  Если это чистый js, начните с ` ```js `. `js` В демонстрации, если вам нужно импортировать `компонент/_.js`, используйте `import xxx из "webc.site/comComponent name.js";`.
  `js` Используйте краткое написание, например `const`, чтобы объявить несколько констант.

В основном тексте рассказывается, как использовать компонент, параметры интерфейса, переключение между `css` и `class` в разных состояниях и т. д.
Затем опишите параметры, возвращаемое значение, статус стиля и т. д. (если есть) и добавьте дополнительный заголовок.

Используйте слова кратко и избегайте прилагательных.

Копирайтинг должен избегать ерунды и не писать бессмысленных слов (типа: универсальный, контейнер и т.п.).

6. Запустите `./sh/tran.js` корневого каталога, перевод
