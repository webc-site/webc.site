
  Для компонента `js` используйте `Light DOM` для реализации компонента в `src/comComponent name/_.js`.

  Используйте относительные пути для импорта других компонентов, например: `import "../имя компонента/_.js"` (`js` не импортирует `styl`, платформа внедрит его автоматически).

  `this` Определите только собственную функцию обратного вызова веб-компонента, остальные напишите как независимые функции, передайте `this` и необходимые параметры.

  В дополнение к внешним интерфейсам, чтобы облегчить оптимизацию дерева и сжатие кода, запрещено привязывать пользовательские частные атрибуты к `this` (используйте `this.$` для функций выгрузки), а функции высокого порядка могут использоваться для возврата замыканий для хранения промежуточных переменных.

  Не пишите `constructor`, атрибут получается в `connectedCallback`.


  Обратитесь к [./js.md](./js.md), чтобы повторно использовать общедоступные функции в `src/js/`.

  Для контейнеров используйте `slot` вместо атрибутов.
  Если вам нужно загрузить данные, откройте интерфейс загрузки верхнему уровню в виде асинхронной функции обратного вызова и используйте комментарии для указания формата возвращаемого значения.
  Компоненты должны избегать утечек памяти. `disconnectedCallback` объединяйте события удаления в одну функцию `this.$` и не записывайте несколько функций удаления.
  Возвращаемое значение интерфейса может быть одним значением, массивом или массивом массивов, а не объектом.
  Если компонент имеет несколько состояний, используйте многострочные комментарии, чтобы четко описать формат данных каждого состояния. (Не предваряйте каждую строку комментариев `*`)
  Переключение состояний с использованием числовых констант (определенных для `const.js`).
  Пока загружается обратный вызов, покажите анимацию с помощью `const xx = newEl('b');xx.className="Ing"` и `@import "../Ing/_.styl"` в `_.styl`
