Следите за процессом, составляйте план и развивайтесь шаг за шагом, чтобы избежать упущений.

1. Запустите `./sh/new.js имя компонента типа проекта` и создайте компонент в каталоге `src/webc`.
   Тип проекта может быть `css` (компонент чистого стиля), `js` (компонент веб-страницы).
   Напишите первую букву названия компонента с заглавной буквы.

2. Прочтите спецификации кодирования для `.agents/doc/code` и приступайте к разработке.

   Стиль компонента записан в `_.styl`. Стиль, использованный для демонстрации, записан в `demo/_.styl`. Запрещено писать его в `имя компонента/_.styl`.

  Использование `@import` для импорта стилей из других компонентов запрещено как в `_.styl`, так и в `demo/_.styl` .

  Следуйте стандартам кодирования и эстетике дизайна [./styl.md](./styl.md).

  Зависимые компоненты (и их стили) импортируются через `import "./otherComponents.js"` в `webc/comComponentname.js`; зависимости для демонстрации импортируются через `import "../../otherComponents.js"` в `demo/_.js`.

   Все компоненты (включая компоненты чистого стиля) должны иметь файл сценария `comment name.js`. `имя компонента.js` компонентов чистого стиля импортирует только соответствующий `_.styl`. Для компонентов с логикой JS следуйте процессу разработки [webc.js.md](./webc.js.md).

3. Нажмите [demo.md](./demo.md) демонстрация разработки процесса

4. Запустите `./test.sh` корневого каталога. Исправляйте ошибки и сигналы тревоги, извлекайте функции, оптимизируйте код и избегайте дублирования и избыточности. Спецификации кода см. в `.agents/doc/code/js.md`.

5. Прочитайте [dbg.md](./dbg.md) и отладьте компоненты в соответствии с процессом.

6. Следуйте процессу [i18n.md](./i18n.md) для реализации интернационализации компонентов и демонстраций и написания документов.

7. Протестируйте еще раз и оптимизируйте код

8. Откройте субагент и позвоните `.agents/skills/js_review/SKILL.md`, чтобы просмотреть и оптимизировать код.

9. Отладка снова
