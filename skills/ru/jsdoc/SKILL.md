## Выполнить шаги

1. Запустите `./sh/jsDoc.js`, чтобы просмотреть функции с отсутствующей документацией (если вывода нет, добавлять его не нужно).
2. Согласно списку выходных файлов читайте существующие документы и коды один за другим и дополняйте их (последовательно обрабатывайте и разбивайте один за другим).

## Требования к документации

Если экспорт является функцией, напишите
- Параметры и возвращаемые значения:
  - Пишите соответствующие описания только в том случае, если есть параметры или возвращаемые значения, иначе не пишите их.
  - Параметры вводятся с помощью неупорядоченных списков.
  - Если возвращаемое значение представляет собой массив, используйте неупорядоченный список для его описания по одному (в противном случае напишите напрямую, `Возвращаемое значение: описание`, всего одну строку).
  - Если параметры или возвращаемое значение являются функциями, используйте поднеупорядоченные списки для описания параметров и возвращаемых значений (если их нет, не записывайте их).

Если экспорт является константой, укажите, что это такое, всего одна строка

- Используйте краткий, профессиональный язык и избегайте использования прилагательных.
- Пересмотрите существующий документ, не переписывайте его полностью.

## шаблон формата

Каждое имя пишите только один раз, чтобы избежать дублирования, и старайтесь четко выразить его в одной строке. Формат ссылки следующий

```
# имя файла

## Имя функции (параметр)

Описание использования

параметр:
- `xx` : Цель.
  - параметр
    - `a` : Цель.
  - Возвращаемое значение: описание

Возвращаемое значение: Описание
```
