# بین الاقوامی کاری

کیس: اجزاء اور ڈیمو کو بین الاقوامی بنانے کے لیے `src/webc/BoxX` سے رجوع کریں۔

اصول:
- `yml` انٹری کے آخر میں `:` شامل نہ کریں، پگ ٹیمپلیٹ اور js کوڈ میں `:` ڈالیں، اور اسے الگ کر دیں۔
- پچھلی جگہوں کو آسانی سے ہٹانے کے لیے `yml` کے ملٹی لائن ٹیکسٹ کے لیے `|-` استعمال کریں۔
- اگر کوئی بین الاقوامی اندراج نہیں ہے تو، `I18N.js` فائل کو صاف کریں (کچھ نہ لکھیں، `export` نہ لکھیں)
- اندراجات ایک زبان میں ہونے چاہئیں، اور متعدد زبانوں کو ملانا سختی سے ممنوع ہے۔

عمل:

1. روٹ ڈائرکٹری `tran.yml` → `tran` → `from` میں تشکیل کردہ ماخذ زبان کا کوڈ دیکھیں

2. `src/webc/component name` ڈائریکٹری درج کریں۔

3. اجزاء کو بین الاقوامی بنائیں
   - `component name.js` میں استعمال شدہ متن کو `src/webc/component name/i18n/source language/js.yml` میں بطور اندراج لکھیں۔
   - `src/webc/component name/I18N.js` میں ترمیم کریں اور `yml` اندراج کلیدی نام، جیسے `export const KEY = "ymlKey"` کے مستقل کو ترتیب دیں۔
   - `src/webc/component name.js` میں تعارف اور بائنڈنگ (اگر وہاں `js.yml` ہے، اگر نہیں، تو یہ ضروری نہیں ہے):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     "./component name/I18N.js" سے i18nUrl، { KEY } درآمد کریں؛ // فریم ورک خود بخود i18nUrl انجیکشن کرے گا، ڈیفالٹ ایکسپورٹ ڈیفالٹ، اسے خود نہ بنائیں

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    fLang کالوں کو کم کرنے کے لیے بیچوں میں باندھنے کی کوشش کریں۔
    جزو کے `component name.js` میں متن پر مشتمل سٹرنگ نہیں ہونی چاہیے

4. پیشکشوں کی بین الاقوامی کاری
   - `demo/_.pug` میں، جس لیبل کا متن ترجمہ کرنے کی ضرورت ہے اسے `c-t کلیدی نام` (یہ ایک حسب ضرورت لیبل ہے، ایک الگ لائن) کی شکل میں قبضہ کیا جاتا ہے، اور اندراج `demo/i18n/source language/pug.yml` میں لکھا جاتا ہے۔
   - `demo/_.js` `demo/i18n/source language/js.yml` میں مستعمل اصل متن کا اندراج لکھیں اور اسے `demo/I18N.js` میں رکھیں
   - `fetchLang` کو `demo/_.js` میں متعارف کروائیں اور `./I18N.js` لوڈ کریں (یہ `demo/I18N.js` ہے)، اوپر دیے گئے اجزاء کی بین الاقوامی کاری کے عمل سے رجوع کریں (مثال کے طور پر، `demo/i18n` نیچے `js.yml` نہیں ہے، اس مرحلہ کو چھوڑ دیں)
   - `demo/I18N.js` کلیدی نام کے مستقل کو `demo/i18n/source language/js.yml` میں ترتیب دیں اور برآمد کریں۔

5. اجزاء کی تفصیل `src/webc/component name/i18n/source language/README.md` میں لکھیں۔ عمل درج ذیل ہے۔
  5.1 مرکزی عنوان لکھیں۔ عنوان کا آغاز `#` سے ہونا چاہیے، جز کا مقصد ایک جملے میں بیان کریں (جز کا نام نہ لکھیں)، اور آخر میں اوقاف کا اضافہ نہ کریں۔
  5.2 مرکزی عنوان سے شروع ہونے والے اجزاء کی اہم خصوصیات کی فہرست بنائیں، اور ثانوی عنوانات نہ لکھیں۔ اگر متعدد خصوصیات ہیں تو غیر ترتیب شدہ فہرست استعمال کریں (اگر غیر ترتیب شدہ فہرست کی تفصیل نسبتاً مختصر ہے تو آخر میں کوئی وقفہ نہیں ہوگا)۔
  5.3 `## ڈیمو کا استعمال کرتے ہوئے` لکھیں، ڈیمو کوڈ ` ```html ` سے شروع ہوتا ہے (ڈیمو کوڈ میں، واحد عنصر کا ٹیگ بند نہیں ہوتا ہے)، اگر وہاں `js` ہے، تو لکھیں ``<b><b><b><b><b><b>`````````` <b></b class="/ class="CODE"> ٹیگ، `js` مواد کی پہلی لائن میں انڈینٹیشن نہیں ہے۔
  اگر یہ خالص js ہے تو ` ```js ` سے شروع کریں۔ `js` مظاہرے میں، اگر آپ کو `component/_.js` درآمد کرنے کی ضرورت ہے، تو براہ کرم `"webc.site/component name.js" سے xxx درآمد کریں؛` استعمال کریں۔ یہ مرتب کردہ `js` ہے، جو خود بخود js اور css کو انجیکشن کرے گا۔
  `js` جامع تحریر کا استعمال کریں، جیسے کہ `const` متعدد مستقل کا اعلان کرنے کے لیے

مرکزی متن میں جزو، انٹرفیس کے پیرامیٹرز، مختلف ریاستوں میں `css` اور `class` کے درمیان سوئچنگ وغیرہ کو استعمال کرنے کا طریقہ متعارف کرایا گیا ہے۔
اس کے بعد پیرامیٹرز، واپسی کی قیمت، طرز کی حیثیت، وغیرہ (اگر کوئی ہے) کی وضاحت کریں، اور ایک ثانوی عنوان شامل کریں۔

الفاظ کو اختصار سے استعمال کریں اور صفتوں سے گریز کریں۔

کاپی رائٹنگ کو بکواس سے گریز کرنا چاہیے اور بے معنی الفاظ نہیں لکھنا چاہیے (جیسے: عالمگیر، کنٹینر، وغیرہ)۔

6. روٹ ڈائرکٹری کا `./sh/tran.js` چلائیں، ترجمہ
