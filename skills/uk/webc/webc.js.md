
  Для компонента `js` реалізуйте компонент за допомогою `Light DOM` у `src/webc/componentname.js`.

  Використовуйте відносні шляхи для імпорту інших компонентів, наприклад: `import "./component name.js"` (У файлі JS кожного компонента відповідний `_.styl` має бути імпортований на початку, наприклад `import "./component name/_.styl";`).

  `this` Визначте лише власну функцію зворотного виклику веб-компонента, запишіть інші як незалежні функції, передайте `this` та необхідні параметри.

  На додаток до зовнішніх відкритих інтерфейсів, щоб полегшити оптимізацію іерархії та стиснення коду, заборонено прив’язувати власні приватні атрибути до `this` (використовуйте `this.$` для функцій вивантаження), а функції високого порядку можна використовувати для повернення закриття для зберігання проміжних змінних.

  Не пишіть `constructor`, атрибут отримується в `connectedCallback`.


  Зверніться до [./js.md](./js.md), щоб повторно використовувати загальнодоступні функції під `src/js/`

  Для контейнерів використовуйте `slot` замість атрибутів.
  Якщо вам потрібно завантажити дані, відкрийте інтерфейс завантаження для верхнього рівня у формі функції асинхронного зворотного виклику та використовуйте коментарі, щоб вказати формат значення, що повертається.
  Компоненти повинні уникати витоку пам'яті. `disconnectedCallback` об'єднайте події видалення в одну функцію `this.$` і не створюйте кілька функцій видалення.
  Значення, що повертається інтерфейсом, може бути одним значенням, масивом або масивом масивів, а не об’єктом.
  Якщо компонент має кілька станів, використовуйте багаторядкові коментарі, щоб чітко описати формат даних кожного стану. (Не ставте `*` перед кожним рядком коментарів)
  Перемикання станів за допомогою числових констант (визначених як `const.js`).
  Під час завантаження зворотного виклику використовуйте `const xx = newEl('b');xx.className="Ing"` для відображення анімації та введіть залежності через `import "./Ing.js"` у файлі компонента JS.
