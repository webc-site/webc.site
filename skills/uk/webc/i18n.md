# інтернаціоналізація

Випадок: зверніться до `src/BoxX`, щоб реалізувати інтернаціоналізацію компонентів і демонстрацію.

правило:
- `yml` Не додавайте `:` у кінець запису, помістіть `:` у шаблон pug і js-код і з’єднайте його
- Використовуйте `|-` для багаторядкового тексту `yml`, щоб легко видалити пробіли в кінці.
- Якщо інтернаціоналізованого запису немає, очистіть файл `I18N.js` (нічого не пишіть, не пишіть `export`)
- Записи повинні бути на одній мові, і змішування кількох мов суворо заборонено.

процес:

1. Перегляньте код вихідної мови, налаштований у кореневому каталозі `tran.yml` → `tran` → `from`

2. Введіть каталог `src/component name`

3. Інтернаціоналізація компонентів
   - Напишіть текст, який використовує `_.js` як запис у `src/component name/i18n/source language/js.yml`.
   - Змініть `Component name/I18N.js` і налаштуйте константу `yml` імені ключа входу, наприклад `export const KEY = "ymlKey"`
   - `Введено та зв’язано в назві компонента/_.js` (якщо є `js.yml`, це не потрібно, якщо ні):
     ```js
     import { fetchLang } from "../js/fetchLang.js";
     імпорт i18nUrl, { КЛЮЧ } з "./I18N.js"; // Фреймворк автоматично впровадить i18nUrl, стандартне експортне значення, не створюйте його самостійно

     const fLang = fetchLang(i18nUrl);
     fLang((i18n) => {
       btn.title = i18n[KEY];
     });
     ```
    Спробуйте прив’язувати пакети, щоб зменшити кількість викликів fLang
    `_.js` компонента не має містити рядки, що містять текст

4. Інтернаціоналізація презентацій
   - У `demo/_.pug` текст мітки, який потрібно перекласти, займає форму `назви ключа c-t` (це спеціальна мітка, окремий рядок), а запис записується в `demo/i18n/source language/pug.yml`
   - `demo/_.js` Напишіть оригінальний текстовий запис, який використовується в `demo/i18n/source language/js.yml`, і розмістіть його в `demo/I18N.js`
   - Введіть `fetchLang` в `demo/_.js` і завантажте `./I18N.js` (це `demo/I18N.js`), зверніться до процесу інтернаціоналізації компонента вище (наприклад, `demo/i18n` не має `js.yml` нижче, пропустіть цей крок)
   - `demo/I18N.js` Налаштуйте константи імен ключів у `demo/i18n/source language/js.yml` та експортуйте

5. Напишіть опис компонента в `Назва компонента/i18n/вихідна мова/README.md`. Процес виглядає наступним чином
  5.1 Напишіть основну назву. Заголовок повинен починатися з `#`, описувати призначення компонента одним реченням (не писати назву компонента) і не додавати розділові знаки в кінці.
  5.2 Перелічіть основні характеристики компонента, починаючи з основного заголовка, і не пишіть другорядних заголовків. Якщо є кілька характеристик, використовуйте невпорядкований список (якщо опис невпорядкованого списку відносно короткий, крапки в кінці не буде).
  5.3 Напишіть `## Використовуючи демонстраційний код`, демонстраційний код починається з ` ```html ` (у демонстраційному коді тег одного елемента не закрито), якщо є `js`, запишіть `html` внутрішньо, використовуйте `<script type="module">` тег;
  Якщо це чистий js, починайте з ` ```js `. `js` У демонстрації, якщо вам потрібно імпортувати `component/_.js`, скористайтеся `import xxx із "webc.site/component/_.js";`.
  `js` Використовуйте стислі записи, наприклад `const`, щоб оголосити кілька констант

Основний текст представляє, як використовувати компонент, параметри інтерфейсу, перемикання між `css` та `class` у різних станах тощо.
Згодом опишіть параметри, значення, що повертається, статус стилю тощо (якщо є) і додайте вторинний заголовок.

Використовуйте слова коротко і уникайте прикметників.

У копірайтингу варто уникати дурниць і не писати безглуздих слів (типу: універсальний, контейнер тощо).

6. Запустіть `./sh/tran.js` кореневого каталогу, переклад
