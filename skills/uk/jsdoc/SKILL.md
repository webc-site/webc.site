## Біг кроками

1. Запустіть `./sh/jsDoc.js`, щоб переглянути функції з відсутньою документацією (якщо вихідних даних немає, їх не потрібно додавати).
2. Відповідно до списку вихідних файлів прочитайте по черзі існуючі документи та коди та доповніть їх (обробіть їх по черзі та розбийте по черзі).

## Вимоги до документації

Якщо експорт є функцією, напишіть
- Параметри та значення, що повертаються:
  - Пишіть відповідні описи, лише якщо є параметри або значення, що повертаються, інакше не пишіть їх.
  - Параметри вводяться за допомогою невпорядкованих списків.
  - Якщо значення, що повертається, є масивом, використовуйте невпорядкований список, щоб описати його по одному (інакше напишіть безпосередньо `Повернене значення: опис`, лише один рядок).
  - Якщо параметри або значення, що повертається, є функціями, використовуйте невпорядковані списки, щоб описати параметри та значення, що повертаються (якщо їх немає, не пишіть їх).

Якщо експорт є константою, вкажіть, що це таке, одним рядком

- Використовуйте стислу, професійну мову та уникайте вживання прикметників.
- Перегляньте існуючий документ, не переписуйте його повністю

## шаблон формату

Напишіть кожне ім'я лише один раз, щоб уникнути дублювання, і намагайтеся чітко висловити його одним рядком. Формат посилання такий

```
# ім'я файлу

## Назва функції (параметр)

Опис використання

параметр:
- `xx` : Мета
  - параметр
    - `a` : Мета
  - Повернене значення: опис

Повернене значення: Опис
```
